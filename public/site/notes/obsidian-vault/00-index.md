## 这批笔记真正关心什么

这批笔记表面上很散：机器人调参、Nav2 报告、Claude Code 推广、公司观察、读书分享、旅行计划、音乐和弦。但读下来会发现，它们其实在反复追问同一件事：

**一个人如何在 AI 时代，用工程化的方法，把混乱的想法变成可交付的现实？**

机器人是最残酷的训练场。仿真能跑，不代表真机能跑；代码能编译，不代表电源、传感器、TF、状态机和现场人员能一起工作。于是笔记里反复出现 `costmap`、`frontier`、`map->odom`、`CycloneDDS`、`recovery`、`launch`、`标定` 这些词。它们不是术语堆叠，而是现实在提醒：一个系统能不能落地，取决于最脆弱的连接点。

AI 编程是另一个训练场。Claude Code、OpenClaw、MCP、agent，这些工具看似让人更快，实际也迫使人更清楚地定义任务。AI 不会自动让工程变好，它只会放大已有的组织能力、拆解能力和验证能力。好的 prompt 背后其实是好的工程判断。

公司与职业反思则是第三条线。一个公司为什么能活十几年？为什么工程师会陷入代理人心态？为什么降成本经常等于降品质？为什么领导力不是头衔，而是品格和真实？这些问题看起来离代码很远，但它们最终决定代码是否有人维护、经验是否有人沉淀、产品是否值得被认真打磨。

## 五个主题

### 01. 机器人不是算法

机器人系统不是算法秀场，而是现实世界的压力测试。一个导航失败可能来自 planner，也可能来自 costmap、TF、雷达时间戳、电源、脚本、状态机或测试流程。解决它的方式不是“再加一个聪明模型”，而是建立可以验证的工程链路。

### 02. AI 是工作流，不是捷径

Claude Code 和 OpenClaw 的价值不在于替你写几行代码，而在于把“需求、执行、验证、恢复、复盘”变成可重复流程。AI 越强，人越需要判断什么是正确问题。

### 03. 公司是保存经验的机器

公司最重要的能力不是某个英雄工程师，而是能否把经验保存下来。文档、流程、复盘、接口、测试和交付标准，都是组织记忆的一部分。

### 04. 生活笔记是原材料

音乐、旅行、读书分享和零散想法看似不重要，但它们记录的是人的兴趣和方向。它们暂时不一定能成为项目，但可能在未来成为产品、文章或人生选择的线索。

### 05. 日期笔记是黑匣子

日期笔记像飞行记录仪。它记录当时发生了什么、为什么做这个选择、问题卡在哪里。真正的价值不在当天，而在之后能不能从里面抽出原则。

## 不纳入正文的内容

按你的要求，Copilot 自定义 prompt 模板只保留在原 vault 里，不进入主题文章。Obsidian 配置、图片、HTML、CSS、JS 和其他非 markdown 文件也不进入这组整理文章。
