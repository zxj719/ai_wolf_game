## 二、本轮改动

### 改动：ResultScreen 玩家招式分布 Top3

**问题背景**

Round 74 添加了「对手最常用招式 Top3」，附带「克：X」建议（教玩家如何对付对手）。但这是单向视角——玩家只知道对手用什么，不知道**自己用什么**。这导致两个信息盲区：

1. 玩家无法发现自己出招单调的问题（例如一场重炮打了 12 次）
2. 玩家无法意识到自己惯用招式的弱点（例如重炮被切削 1.5× 克，但玩家完全没察觉）

**实现方式**

**新增 `topPlayerMoves` useMemo**（`ResultScreen.jsx`）

```jsx
const topPlayerMoves = useMemo(() => {
  if (!matchStats?.moveUsage) return [];
  return Object.entries(matchStats.moveUsage)
    .sort((a, b) => b[1] - a[1])
    .slice(0, 3)
    .map(([moveId, count]) => ({
      moveId, count,
      name: MOVES[moveId]?.name ?? moveId,
      icon: MOVE_ICONS[moveId] ?? '🎾',
      beatenBy: COUNTER_FOR[moveId] ? MOVES[COUNTER_FOR[moveId]]?.name : null,
    }));
}, [matchStats]);
```

- 数据来源：`matchStats.moveUsage`（battleReducer 自 Round 初期已追踪，`CHOOSE_MOVE` action 每球写入）
- 零后端改动，零新数据采集
- `beatenBy` 是 `COUNTER_FOR[moveId]`——与对手 chips 的 `counter` 语义完全对称，但方向相反

**新增 `playerSelfNote` useMemo**

```jsx
const playerSelfNote = useMemo(() => {
  if (!topPlayerMoves.length) return null;
  const top1 = topPlayerMoves[0];
  if (top1.beatenBy) {
    return `你最常出「${top1.name}」，对手若出「${top1.beatenBy}」可以克制你，下次注意换招`;
  }
  return null;
}, [topPlayerMoves]);
```

**UI 设计**：「你本场偏爱」section 放在「对手最常用招式」section 之前

| 层级 | 颜色调性 | 语义 |
|------|---------|------|
| 玩家招式（蓝色底） | `rgba(30,58,138,.10)` | 自我画像 |
| 对手招式（绿色底） | `rgba(13,59,46,.10)` | 对手分析（已有） |

- **玩家 chips**：「被克：X」用 `var(--clay)`（橙色），醒目但非报错色——提示风险而非惩罚
- **对手 chips**：「克：X」用 `var(--optic)`（黄绿色），代表破招机会

**信息闭环验证**

| 信息问题 | 本轮答案 |
|---------|---------|
| 我出了什么？ | 你本场偏爱：重炮 ×12、扣杀 ×4、旋转 ×2 |
| 我的惯用招有什么弱点？ | 被克：切削（即重炮被切削克） |
| 对手用什么反制了我？ | 对手最常用：切削 ×8、挑高 ×5... |
| 下次怎么破解对手？ | 💡 下次多出「X」来破解... |
