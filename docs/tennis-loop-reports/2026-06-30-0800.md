## 三、本轮改动

### 功能：「对手面板排序说明」极小标签

**背景与动机**

R104 实现了对手面板按「推荐→中立→劲敌→NEW」智能排序，但面板展开后没有任何提示说明这个排序逻辑。玩家看到铁蛋排第一时，不知道这是「你胜率高的对手被优先推荐」，可能以为是随机顺序或字母顺序，错过了这个功能传递的信息。

**设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 展示时机 | 仅在面板展开（showOppHistory=true）时显示 | 收起时不占空间 |
| 内容 | `按胜率排序：推荐 · 中立 · 劲敌 · NEW` | 与实际标签完全对应，一眼明白 |
| 色彩编码 | 绿色推荐 / 灰色中立 / 红色劲敌 / 橙色 NEW | 复用已有 opp-tag 色系，视觉一致 |
| 布局 | `grid-column: 1 / -1` 跨全行 | grid 容器内无论几列都自动撑满 |
| 字体 | `--mono`，0.58rem | 与对手名字同字族同大小，属于「元信息」层，不喧宾夺主 |
| 位置 | panel 内最顶行，chip 之前 | 先读说明再看内容，信息流自然 |

**视觉效果**

展开对手面板后，顶部显示一行极小的说明文字：
```
按胜率排序：[绿色]推荐 · [灰色]中立 · [红色]劲敌 · [橙色]NEW
```

下方才是 7 个对手卡片，其中已挑战过且胜率≥60% 的对手显示绿色「推荐」标签，胜率<35% 的显示红色「劲敌」标签，未见过的显示橙色「NEW」。说明行与标签颜色完全对应，无需额外学习成本。
