## 三、本轮改动

### 改动：每日排行榜当前选手高亮（2 个文件，纯增量）

**背景**

R86 完成了家庭每日出战排行榜，但所有条目外观相同——选手选好身份后，无法直观识别「自己是否已打完今日一战」。本轮补上这块体验缺失。

**设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 触发条件 | `picked && c.player_name === picked` | 身份未选时不高亮（`picked=''`），避免误判 |
| 高亮色系 | 琥珀色 `rgba(251,191,36,.12)` + outline | 与游戏主色调（amber-600）一致；不遮盖绿/灰的胜负边框 |
| 「你」徽章 | 内联 `dc-me-badge`，深色背景白字 | 辅助色觉困难用户；英文字符短小不占空间 |
| 标题附加 | `· 包括你`（IIFE 内 `meInList` 判断） | 给玩家一个顶部摘要确认；不显示完成时间等细节 |
| 实现方式 | JSX IIFE（局部变量 `sliced`/`meInList`） | 避免提取新组件，保持与现有 `dailyBoard` 渲染逻辑同层 |

**变更详情**

#### `src/modules/tennis/components/SelectScreen.jsx`

将日榜渲染块改为 IIFE，提取 `sliced`（`slice(0,7)` 的结果）和 `meInList`（是否有条目匹配 `picked`），在：
- 列表标题末尾追加 `· 包括你`（当 `meInList` 为 true）
- 每个 `<li>` 追加 `dc-me` class（当 `c.player_name === picked`）
- 匹配条目内插入 `<span className="dc-me-badge">你</span>`

#### `src/modules/tennis/tennis.css`

新增 2 条规则：
```css
.tennis-scope .dc-me { background: rgba(251,191,36,.12); outline: 1px solid rgba(251,191,36,.35); }
.tennis-scope .dc-me-badge { font-size: .6rem; font-weight: 700; background: rgba(251,191,36,.8); color: #1a1a1a; border-radius: 3px; padding: 0 4px; margin-left: 2px; line-height: 1.6; }
```

**用户流程示例**

```
莹打开 /tennis → 选「莹」身份 →
  今日排行：
    今日出战 · 3 位家人 · 包括你          ← "包括你" 标题
    🐯 诚 → 🦊 Elza   ✓ 胜  3分15秒
  ▶ [琥珀高亮] 🦢 莹 [你] → 🦊 Elza  ✗ 败   ← 自己的行高亮
    🐰 菲比 → 🐰 铁蛋  ✓ 胜  2分40秒

  效果：莹一眼看到「我今天已经打过了，输了」，不用逐行找名字
```
