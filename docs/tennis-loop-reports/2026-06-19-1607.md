## 二、本轮改动

### 核心功能：赛后「下次目标」提示卡

**问题定位：**

玩家打完一局后，结算屏有成就解锁庆祝（AchievementUnlockCard）、角色攻克进度（CharWinsProgressCard）、本场亮点（HighlightsCard），但缺乏「下一局我该做什么」的明确方向感。
mode 页已有 `achHint`（随机提示一条未解锁成就），但赛后最佳时机是根据刚结束这场比赛的数据给出有针对性的建议——玩家此时情绪最高，最容易转化为「再来一局」的行动。

**实现方案（2 文件变更）：**

**1. TennisRoute.jsx（+14 行）**

新增 `nextGoal` useMemo，复用 `progress.achievements` + `state.lastMatchStats`，按上下文优先级排序：

```js
const nextGoal = useMemo(() => {
  const locked = ACHIEVEMENTS.filter((a) => !progress.achievements.includes(a.id));
  if (locked.length === 0) return null;
  const stats = state.lastMatchStats;
  const has = (id) => locked.find((a) => a.id === id);
  const candidates = [
    stats?.aces >= 1 && has('aceMaster'),        // 本场有 ACE → 离 ACE 大师差1步
    stats?.aces >= 1 && has('consecAce'),         // 有 ACE → 引导连ACE
    stats?.longestWinStreak >= 3 && has('winStreak5'), // 连胜3+ → 离5连胜近
    stats?.maxMgMult >= 1.2 && has('proTouch'),   // 倍率高 → 差0.2到神之触感
    has('clutchMaster'),                           // 关键先生（通用诉求）
    has('allChars'),                               // 全能选手（收集驱动）
    has('firstWin'),                               // 新手第一目标
  ].filter(Boolean);
  return candidates[0] ?? locked[Math.floor(Math.random() * locked.length)];
}, [progress.achievements.length, state.lastMatchStats]);
```

优先级设计原则：本场差一步即达成 > 通用收集目标 > 随机兜底。

**2. ResultScreen.jsx（+12 行）**

新增 `NextGoalCard` 功能组件，零新 CSS（复用 `.card.flat` + `.cwp-hint`）：

```jsx
function NextGoalCard({ nextGoal }) {
  if (!nextGoal) return null;
  return (
    <div className="card flat">
      <h2>🎯 下次目标</h2>
      <p className="cwp-hint" style={{ fontSize: '1rem', marginTop: 6 }}>
        {nextGoal.icon} <strong>「{nextGoal.name}」</strong> — {nextGoal.desc}
      </p>
    </div>
  );
}
```

在 `HighlightsCard` 之后、`MatchReplayCard` 之前插入：

```jsx
<HighlightsCard matchStats={matchStats} />
<NextGoalCard nextGoal={nextGoal} />
<MatchReplayCard rallyLog={rallyLog} matchStats={matchStats} />
```
