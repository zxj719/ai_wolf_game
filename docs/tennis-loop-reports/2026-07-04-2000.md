## 三、本轮改动

### 功能：家族冲刺榜（Sprint 本机最高分 Top 10）

**背景与动机**

R132 的「复制成绩」按钮解决了「打完想分享」的问题，但仍缺少一个驱动家庭成员反复游玩的长线目标。家庭游戏场景最典型的粘度来源是「我能打过爸爸吗？」——需要有一个榜单让全家人知道谁的 Sprint 历史最高分是多少。纯 localStorage 实现无需后端，对所有用户（游客/登录）均生效。

**架构设计（模块化 + 纯函数）**

新建 `src/modules/tennis/modes/sprintScores.js`，沿用 `localBoard.js` 的 store IIFE 安全降级模式（隐私模式 / SSR 环境自动 fallback 到内存对象）。三个导出函数：
- `loadSprintHiscores()` → 降序榜单数组（最多 10 条）
- `saveSprintHiscore({ totalPts, matchCount, winCount, grade, playerName }, { now? })` → `{ rank, isNew }`，`now` 参数用于测试注入（避免 mock 全局 Date）
- `clearSprintHiscores()` → 测试用清空

排序规则：分高者前，同分时先打完（ts 较小）者前。最多保留 10 条，超出自动淘汰低分项。

**SprintScreen 变更**

1. `computeGrade(pts)` 从组件内联提取为模块级导出函数，消除 render 和 hiscore-save 的双重定义
2. 新增 `hiRank` state + `currentTsRef` + `hiSavedRef`（防止重复写入）
3. `useEffect([phase, results, basePlayer.name])`：`phase === 'done'` 且 `results.length > 0` 时触发一次保存（React 18 批量更新保证 results 为最终值）
4. 结算屏新增两个 UI 区块：
   - **排名 banner**：🥇 家族历史最高分！/ 🏅 第 N 名！/ 📊 榜单第 N 名（按 rank 变色）
   - **家族冲刺榜 Top 5**：每行显示「#N / 角色名 / 分数 / 称号图标 / 日期」，当前局行用黄色背景高亮

**边界处理**

- 0 对局（用户立即点「结束」）：`results.length === 0` 时跳过保存，不写入 0 分记录
- 同分局：较早完成者排名靠前（家庭场景中更公平：先打的不应被后打的同分挤掉）
- localStorage 不可用：store IIFE 降级为内存对象，榜单仅在当前会话内有效（静默降级，不报错）
