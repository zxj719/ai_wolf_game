## 二、本轮改动

### 改动 1：锁定绝技芯片内联拥有者提示

**问题**：`ult-gallery` 锁定芯片（灰显）通过 HTML `title` 属性传递解锁提示「击败 X 解锁」。移动端无 hover 状态，用户无法得知如何解锁特定绝技。这让「击败家人解锁绝技」的核心玩法机制对移动端用户完全不透明。

**方案**：在芯片正文中（图标和名称下方）新增一行 `.ult-chip-hint` 文本，直接显示「击败🐯诚」（emoji + 角色名），利用 `ULTIMATES` 数据中已有的 `face` 和 `owner` 字段，无需额外数据查找。

**`TennisRoute.jsx`（+1 行）**：
```jsx
{!unlocked && <span className="ult-chip-hint">击败{u.face}{u.owner}</span>}
```

**`tennis.css`（+1 行）**：
```css
.tennis-scope .ult-chip-hint { font-size: .58rem; color: rgba(242,238,224,.32); font-family: var(--mono); margin-top: 3px; white-space: nowrap; }
```

设计细节：
- 字号 0.58rem，极小不抢焦点，与灰色整体调性一致
- 透明度 0.32（仅略高于背景），提示存在但不干扰
- `white-space: nowrap` 防止「击败🦢莹」在窄屏折行变丑

### 改动 2：PrepScreen 绝技描述即时预览

**问题**：备战页「出战绝技」`<select>` 下拉框在 iOS/Android 原生实现中，当前选中项通常只显示文字不显示 emoji，且选项描述在 select 关闭状态下不可见。用户换装绝技时无法直观确认所选绝技的效果，需要打开下拉才能看到。

**方案**：在 `<select>` 下方追加一行 `.ult-pick-desc`，即时显示当前选中绝技的描述：

```jsx
{(() => { const sel = ultimateOptions.find((u) => u.name === equippedUltimate); return sel ? <p className="ult-pick-desc">⚡ {sel.face} {sel.name}：{sel.desc}</p> : null; })()}
```

**`tennis.css`（+1 行）**：
```css
.tennis-scope .ult-pick-desc { font-size: .78rem; color: var(--muted); font-family: var(--mono); margin: 6px 0 0; line-height: 1.45; }
```

效果：选择不同绝技时描述行即时更新（React 响应式），玩家无需打开下拉即可确认效果。

### git 历史修复

**发现的问题**：前 51 轮 tennis 循环的所有代码改动，均被 git commit 到了 detached HEAD（而非 `main` 分支），从未推送到 `origin/main`。`origin/main` 停留在第 12 轮左右的某个早期 commit（`1a8e54b docs: 可玩性/平衡性评估报告`）。

**修复过程**：
1. 验证 `56741f0`（detached HEAD 最新状态）是 `1a8e54b`（main 当前 tip）的直接后代（`git merge-base` 确认共同祖先 = `1a8e54b`）
2. `git checkout main`（带冲突解决）
3. `git merge --ff-only 56741f0`（快进合并，无需新合并 commit）
4. 本轮新改动 commit 后推送

**原因推测**：早期会话执行了 `git checkout <hash>` 进入 detached HEAD，后续会话继续在 detached HEAD 提交，但 push 被 git 忽略（detached HEAD 不对应任何远程追踪 ref）或 push 失败未察觉。
