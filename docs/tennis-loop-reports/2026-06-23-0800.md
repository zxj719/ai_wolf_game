## 二、本轮改动

### 改动 A（主项）：SelectScreen 对手历史面板 — 精确胜率数字

**问题背景**

「推荐/劲敌」标签能传达方向，但不传达量。玩家无法从标签本身判断这是「3/3 全胜」还是「7/9 小优势」——对是否要加练前的决策而言，这个数字差异有意义。

**实现方式**

在 `SelectScreen.jsx` 的 `opp-history-chip` 中，seen 分支追加一个 `opp-wr` span：

```jsx
{oppWinRateMap[c.n] && (
  <span className="opp-wr">{oppWinRateMap[c.n].wins}/{oppWinRateMap[c.n].total}</span>
)}
```

`oppWinRateMap` 已在 `useMemo` 中提前算好，零额外计算。`wins/total` 格式（如「3/5」）比百分比更直观——玩家知道这来自多少局历史，而非浮动的百分数。

CSS 新增规则：
```css
.tennis-scope .opp-wr { display: block; font-size: .55rem; font-family: var(--mono); color: var(--muted); margin-top: 2px; text-align: center; }
```

**效果示例（showOppHistory 展开后）**

```
👵 大妈   ✓  劲敌   1/5
🧔 老爸   ✓  推荐   4/5
🧒 弟弟   ✓           2/3
```

### 改动 B（微型打磨）：LadderScreen 按摩推荐高亮

**问题背景**

上轮（第 64 轮）为赛间三选一加了动态提示，但所有提示都用 `--muted` 灰色，「推荐」和「收益低」视觉权重相同，玩家在快速扫视时无法区分。

**实现方式**

`getIntermissionHints` 新增返回字段 `massageRecommended`（条件：`massageDelta > 5 && energyBase < 55`，即体力极低时按摩收益最大）：

```js
const massageRecommended = massageDelta > 5 && energyBase < 55;
return { trainHint, massageHint, shopHint, massageRecommended };
```

按摩按钮的 `.opt-hint` 条件追加 `.opt-hint-hi` 类：
```jsx
<span className={`opt-hint${hints.massageRecommended ? ' opt-hint-hi' : ''}`}>
  {hints.massageHint}
</span>
```

CSS 新增规则（覆盖 `.opt-hint` 的 `--muted` 颜色和 0.75 透明度）：
```css
.tennis-scope .opt .opt-hint-hi { color: var(--clay); opacity: 1; font-weight: 600; }
```

这样当体力危急时，「体力仅 XX%，按摩→YY%，**推荐**」会以橙色加粗渲染，与其他提示形成明显对比。
