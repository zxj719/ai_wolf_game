## 二、本轮改动

### 改动：MovePicker 克制关系速查浮层

**问题背景**

Rounds 73-74 构建了「球后复盘 + tell 预告」两层即时提示，但这两层都是**反应式**的——只在特定事件发生后才显示。玩家仍然缺少一个**主动查阅**克制关系的渠道：

- `OnboardingModal`（新手引导）展示了克制表，但只在首次进入时显示，之后无法再看
- 玩家在出招阶段若想复盘「切削被重炮克了三次，我该怎么对付」，没有任何 UI 入口

**实现方式**

**新增 `CounterChart` 组件**（`BattleScreen.jsx`）

```jsx
function CounterChart({ onClose }) {
  const pairs = Object.keys(COUNTER_QUIPS).map(k => k.split('>'));
  return (
    <div className="bt-chart">
      <div className="bt-chart-hd">
        <span>📋 克制速查表（1.5× 克中 / 0.7× 被克）</span>
        <button type="button" className="bt-chart-close" onClick={onClose}>✕</button>
      </div>
      <div className="bt-chart-grid">
        {pairs.map(([a, b]) => (
          <div key={`${a}>${b}`} className="bt-chart-row">
            <span className="bt-chart-win">{MOVES[a].name}</span>
            <span className="bt-chart-arr">→克→</span>
            <span className="bt-chart-lose">{MOVES[b].name}</span>
          </div>
        ))}
      </div>
    </div>
  );
}
```

- 数据来源：直接遍历已有 `COUNTER_QUIPS` 的键（格式 `"attacker>victim"`），零额外导入
- `COUNTER_QUIPS` 涵盖全部 9 对克制关系，保证覆盖完整
- 胜方（深绿色）→克→ 败方（深红色）：颜色直觉化，一眼识别方向

**修改 `MovePicker` 组件**

增加本地 `useState(false)` 控制浮层显隐：

```jsx
const [showChart, setShowChart] = useState(false);
```

在招式网格上方增加 `bt-moves-bar`：

```jsx
<div className="bt-moves-bar">
  <button className={`bt-chart-btn${showChart ? ' active' : ''}`}
          onClick={() => setShowChart(v => !v)}>
    📋 克制表
  </button>
</div>
{showChart && <CounterChart onClose={() => setShowChart(false)} />}
```

- 按钮 `active` 状态：加深边框+背景，给用户明确的"已展开"反馈
- 浮层在 `showChart` 期间持久显示（不自动关闭），玩家可以边看表边选招
- 点「✕」关闭，下次进入 cards 阶段默认收起（按本地 state 重置）

**四层教学体系完整闭环**

| 层级 | 何时显示 | 内容 | 触发方式 |
|------|---------|------|---------|
| **速查表（本轮）** | cards 阶段，随时 | 全部 9 对克制关系 | 玩家主动点「📋 克制表」 |
| tell 提示（Round 74） | cards 阶段，对手发出 tell 时 | 「可用 X 克制」建议 | 自动（被动） |
| 球后复盘（Round 73） | idle 阶段，丢分后 | 「X 被 Y 克制」学习 | 自动（被动） |
| 关键分警报（Round 69）| cards 阶段，盘点/金球 | 「对手惯用 X，出 Y 克」 | 自动（被动） |

从「主动查 → 被动教 → 球后学 → 紧急警报」四级覆盖，满足不同学习节奏的玩家。

**样式设计**

```css
.tennis-scope .bt-moves-bar { display: flex; justify-content: flex-end; margin-bottom: 6px; }
.tennis-scope .bt-chart-btn { /* 小圆角按钮，静止态低调 */ }
.tennis-scope .bt-chart-btn:hover, .bt-chart-btn.active { /* 展开态加深 */ }
.tennis-scope .bt-chart { /* 白底面板，圆角边框 */ }
.tennis-scope .bt-chart-grid { /* auto-fill 双列，适应宽屏/手机 */ }
.tennis-scope .bt-chart-win { color: #155743; font-weight: 600; }  /* 深绿 = 克方 */
.tennis-scope .bt-chart-lose { color: #7a3020; }                    /* 深红 = 被克方 */
```

- `grid-template-columns: repeat(auto-fill, minmax(175px, 1fr))`：宽屏 2 列，手机 1 列，无需媒体查询
- 按钮静止态（`var(--muted)` 灰色）低调，避免与招式按钮争抢视觉焦点
- 面板 `background: rgba(13,59,46,.06)`：轻微绿色底调，与 `bt-arena-card` 白底区分
