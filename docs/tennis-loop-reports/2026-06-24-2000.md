## 二、本轮改动

### 改动：opponentAI.js `hintMove` + BattleScreen tell 克制提示

**问题背景**

Round 73 实现的 `bt-counter-tip` 在球结束后（丢分时）显示「被 X 克，下次出 Y」。但还存在一个更早的决策时间点：对手的 **tell 阶段**（cards 阶段显示的小动作预读）。

- 当前 tell 只显示「偷瞄了一眼网前的空当……」
- 玩家需要自己记忆「放小球 → 被上旋克」，没有即时决策提示
- 结果：新手在 tell 阶段知道对手要出什么，却不知道该选哪招反制

**数据层修改：`opponentAI.js`**

`makeTell` 新增 `hintMove` 字段，让 UI 层知道告示对应的是哪招（无论真假）：

```js
// 真告示
return { text: TELLS[actualMove], isTrue: true, hintMove: actualMove };
// 假告示
return { text: TELLS[fake], isTrue: false, hintMove: fake };
```

设计考量：
- `hintMove` 存储的是告示所指的招式 ID（真告示=实际招式，假告示=假招式）
- UI 层不关心 isTrue，直接用 `hintMove` 查 `COUNTER_FOR` 即可
- 75% 概率下 hintMove 正确 → 提示准确；25% 概率下 hintMove 错误 → 玩家被假告示骗入错误反制，属游戏设计意图（假告示的博弈性）

**UI 层修改：`BattleScreen.jsx`**

两个分支：

| 情景 | 显示效果 |
|------|---------|
| 普通 tell（非透视） | `{tell.text}…… 💡 出「X招」可克制` |
| 透视卡（reveal 效果） | `出招已被透视：X招！ 💡 出「Y招」克制！` |

实现代码：
```jsx
// 透视分支
{COUNTER_FOR[state.oppMove] && (
  <small className="bt-tell-counter">💡 出「{MOVES[COUNTER_FOR[state.oppMove]].name}」克制！</small>
)}
// 普通 tell 分支
{state.tell.hintMove && COUNTER_FOR[state.tell.hintMove] && (
  <small className="bt-tell-counter">💡 出「{MOVES[COUNTER_FOR[state.tell.hintMove]].name}」可克制</small>
)}
```

- 无克制方的招式（如 flatDrive、smash）：`COUNTER_FOR[moveId]` 为 `undefined`，条件渲染自动跳过
- 透视场景用「克制！」（确定语气），普通 tell 用「可克制」（不确定语气，因可能假告示）

**样式：`tennis.css`**

```css
.tennis-scope .bt-tell .bt-tell-counter {
  display: block; font-size: .70rem; font-style: normal;
  color: #1e4d38; opacity: .90; margin-top: 4px;
}
```

- `display: block`：换行显示，不挤在 tell 文字之后
- `font-style: normal`：bt-tell 默认 `font-style: italic`，克制提示作为操作建议应为正体
- `color: #1e4d38`：深绿色，在告示的黄绿背景（rgba(217,255,61,.25)）上高对比度且不刺眼

**测试：`opponentAI.test.js`**

两处测试更新：
1. 真告示 case：新增 `expect(t.hintMove).toBe('smash')` 断言
2. 假告示 case：新增 `expect(t.hintMove).not.toBe('smash')` + `expect(CHAR_BUILDS['诚'].moves).toContain(t.hintMove)` + `expect(t.text).toBe(TELLS[t.hintMove])` 确保文案与 hintMove 对应

**教学系统完整闭环（三层）**

| 时机 | 提示 | 信息 |
|------|------|------|
| cards 阶段（tell 显示时） | **bt-tell-counter**（本轮新增） | 「建议出 X 反制」（决策前） |
| idle 阶段（球结束丢分时） | bt-counter-tip（Round 73） | 「对手 X 被 Y 克」（结果复盘） |
| cards 阶段（关键分 + 对手惯用招≥2次） | CrisisHint（Round 69） | 「关键分！对手惯用 X，出 Y 可克」（高优先级预警） |

三层从「决策前建议 → 球后复盘 → 关键分警报」形成完整教学体系。
