## 三、本轮改动

### 功能：SprintScreen 结算屏「复制成绩」分享按钮

**背景与动机**

R131 已在 OnboardingModal 加入了「游戏模式」Tab 介绍限时冲刺，让新手了解规则。下一步自然是让打完 Sprint 的玩家能把成绩分享给家庭成员——「晒图对比」是家庭游戏场景最常见的社交动作，一键复制文字是最低摩擦力的实现方式。

**设计原则**

- **纯函数优先**：将文字生成逻辑抽出为 `buildShareText({ totalPts, matchCount, winCount, grade })`，完全可测试，不依赖 React
- **剪贴板双降级**：优先 `navigator.clipboard.writeText`（现代浏览器），fallback `document.execCommand('copy')`（iOS WebView 等旧环境）
- **反馈即时**：点击后按钮文字切换为「✅ 已复制」，2 秒后恢复，避免用户重复点击
- **Hook 位置合规**：`const [copied, setCopied]` 放在组件顶部 state 声明区，避免在条件分支前调用 Hook 的 React 规则违反

**分享文字格式**

```
⏱️ 限时15分钟 · 27分 · 9场 · 7胜 · 进阶冲刺手🥈
```

**实现细节**

1. `SprintScreen.jsx` 导出 `buildShareText` 纯函数
2. 在组件顶部 state 区新增 `const [copied, setCopied] = useState(false)`
3. `phase === 'done'` 分支内计算 `shareText = buildShareText(...)` 并定义 `handleCopy`
4. 结算卡片「返回」按钮上方新增一行 `<button class="btn ghost">` 容纳「📋 复制成绩」
5. 新建 `__tests__/sprint.test.js`：涵盖常量、`formatTime`、`buildShareText` 共 18 个用例（含 `it.each` 遍历 4 种称号）
