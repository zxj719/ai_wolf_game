## 二、本轮改动

### 改动：SelectScreen「对手推荐/劲敌标签」

**问题背景**

玩家在「已挑战对手历史面板」中能看到所有对手的 ✓/NEW 状态，但无法快速判断哪些对手对自己更顺手、哪些是天然克星。每次面对随机抽取的对手，玩家需要靠记忆或重新展开历史面板——信息在那里，但提取成本高。

**实现方式**

在 `SelectScreen.jsx` 添加两个改动：

```js
// 模块级纯函数（组件外）
function getOppTag(name, map) {
  const data = map[name];
  if (!data || data.total === 0) return null;
  const rate = data.wins / data.total;
  if (rate >= 0.6) return { label: '推荐', cls: 'opp-tag-recommend' };
  if (rate < 0.35) return { label: '劲敌', cls: 'opp-tag-rival' };
  return null;
}

// 组件内：单次读取 records，派生两个结构
const records = useMemo(() => loadLocalRecords(), []);
const seenOpps = useMemo(() => new Set(records.map((r) => r.o)), [records]);
const oppWinRateMap = useMemo(() => {
  const map = {};
  for (const r of records) {
    if (!map[r.o]) map[r.o] = { wins: 0, total: 0 };
    map[r.o].total++;
    if (r.sp > r.so) map[r.o].wins++;
  }
  return map;
}, [records]);
```

JSX 中在 `✓` 徽章后追加标签：
```jsx
{CHARS.map((c) => {
  const tag = seenOpps.has(c.n) ? getOppTag(c.n, oppWinRateMap) : null;
  return (
    <div key={c.n} className={`opp-history-chip${...}`}>
      ...
      {seenOpps.has(c.n) ? (
        <>
          <span className="opp-history-badge">✓</span>
          {tag && <span className={`opp-tag ${tag.cls}`}>{tag.label}</span>}
        </>
      ) : (
        <span className="opp-history-badge opp-history-new">NEW</span>
      )}
    </div>
  );
})}
```

CSS 追加三条规则：
```css
.tennis-scope .opp-tag { font-size: .68rem; font-weight: 700; padding: 1px 5px; border-radius: 3px; margin-left: 4px; vertical-align: middle; }
.tennis-scope .opp-tag-recommend { background: rgba(110,231,183,.18); color: #6ee7b7; }
.tennis-scope .opp-tag-rival { background: rgba(248,113,113,.18); color: #f87171; }
```

**决策逻辑**

| 胜率区间 | 标签 | 颜色 | 含义 |
|----------|------|------|------|
| ≥ 60% | 推荐 | 绿（`#6ee7b7`） | 历史占优，对局多半顺手 |
| 35–59% | （无标签） | — | 中性，势均力敌 |
| < 35% | 劲敌 | 红（`#f87171`） | 历史落败多，需要重视 |
| 0 场历史 | （无标签） | — | 数据不足，不予评价 |

**设计要点**

- `getOppTag` 放模块级，纯函数接收 `map` 参数，零闭包、零副作用
- 将原来两次 `loadLocalRecords()` 调用合并为一次 `useMemo`（`records` memo），再从同一引用派生 `seenOpps` 和 `oppWinRateMap`，减少 localStorage 读取次数
- 标签仅在 `seenOpps.has(c.n)` 为真时才计算（未见过的对手无历史，不展示标签）
- 颜色透明度较低（0.18），标签在深色背景上存在但不抢过主要 ✓ 徽章的视觉权重
- 不引入新状态、不新增 import，改动轻量（+18 行 JS + 3 行 CSS）
