## 三、本轮改动

### 改动：Leaderboard.jsx 全网榜角色筛选

**问题背景**

R89 前，全网榜展示所有选手的聚合战绩 + 最近 20 场战报，无任何分维度视角。当玩家想知道「用 Elza 的人都赢了多少场」或「最近有没有人用 菲比 打」时，需要人工浏览整张表。R89 提出加入角色筛选 chip，因为后端已返回 `character`（recent 表）和 `lastCharacter`（players 聚合）字段，纯前端过滤即可。

**设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 筛选维度 | `players.lastCharacter` + `recent.character` | 后端已有字段，零改动 |
| chip 来源 | 固定 `CHARS` 数组（从 `gameData.js` 导入） | 始终展示全部 7 个角色，不依赖当前榜单有没有数据 |
| 切换方式 | 点选同一角色 chip 再次点击 → 取消筛选 | 与「全部」按钮等价，交互自然 |
| 空榜文案 | 个性化：`还没有以 🦊 Elza 上榜的选手 ——` | 鼓励玩家率先用该角色创建战绩 |
| 样式 | `.lb-fc` / `.lb-fc.active`：复用 `.lb-tab` 视觉语言（圆角胶囊，激活态黄绿背景） | 视觉一致，不引入新设计语言 |

**实现路径**

1. `Leaderboard.jsx`：
   - 顶部导入 `CHARS`（`../gameData`）
   - `GlobalBoard` 新增 `const [charFilter, setCharFilter] = useState(null)`
   - 计算 `filteredPlayers` / `filteredRecent` / `activeChar`
   - 渲染 `.lb-filter-chips` chip 行（`全部` + `CHARS.map`）
   - 把原先 `players.map` → `filteredPlayers.map`，`recent` → `filteredRecent`（2 处替换）
   - 空榜提示文案随 `charFilter` 动态切换

2. `tennis.css`：
   - 新增 `.lb-filter-chips`（flex, gap, margin）
   - 新增 `.lb-fc` 基础样式（与 `.lb-tab` 同形，尺寸略小）
   - 新增 `.lb-fc.active`（黄绿背景激活态）

**三种交互路径**

| 操作 | 结果 |
|------|------|
| 点击角色 chip（未激活） | 激活该角色，过滤两张表 |
| 点击同一 chip（已激活） | 取消筛选，恢复全部 |
| 点击「全部」 | 取消筛选，恢复全部 |
