## 三、本轮改动

### 功能：今日效率排名（Sprint 结算屏）

**背景与动机**

R133 的家族冲刺榜按「总分」排名——谁打的场次多、运气好，总分就高。但家庭游戏中还有另一维度的竞争：「谁打得最干净？」（每场平均拿几分）。效率 = 分/场，最高效率 = 3.0（全胜），最低 = 1.0（全负）。当家里多人用同一台设备玩 Sprint，同一天的效率对比能让玩家看到「我虽然总分不如爸爸，但我今天赢的场次比率更高」，提供另一维度的竞争目标，不需要时间投入更多才能领先。

只有当今日记录 ≥ 2 条时才显示此子榜，单独玩的场次不会孤独地「自我效率排名」。

**架构设计（纯函数 + 可测试注入）**

`sprintScores.js` 新增两个函数：
- `computeEff(pts, matchCount)` → `number|null`：matchCount=0 返回 null 防止除零
- `getTodayEffBoard(hiscores, { today? })` → sorted array：过滤今日 + 排序（eff降序 → pts降序 → ts升序），沿用 `isToday` 的 `today` 注入模式

`SprintScreen.jsx` 变更：
- 导入 `getTodayEffBoard`
- `done` 阶段：`allHiscores = loadSprintHiscores()`，从中切片出 `hiscores`（Top 5 用于家族榜），另算 `todayEffBoard = getTodayEffBoard(allHiscores)`
- `isEffChamp = myEffRank === 1 && todayEffBoard.length >= 2`
- 排名 banner 扩展：在历史排名行下新增「⚡ 今日效率冠军！」（仅当 `isEffChamp`）
- 新增 `todayEffBoard.length >= 2` 条件渲染块：显示 Top 3 效率行，高亮当前局（青色背景）

**优先级/边界逻辑**

| 情形 | 表现 |
|------|------|
| 今天只有当前一次记录 | 不显示效率子榜（`length >= 2`） |
| 当前局效率最高（非孤单） | banner 显示「⚡ 今日效率冠军！」 |
| 未完成任何对局（skip save） | `currentTsRef.current === null`，myEffRank=0，不触发 |
| `matchCount === 0` 的异常记录 | `getTodayEffBoard` 过滤掉（`s.matches > 0`） |
