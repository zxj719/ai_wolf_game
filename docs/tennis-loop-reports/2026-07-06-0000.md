## 本轮主要任务：noob 装备效果澄清 + 新手保护期延长

### 问题重现

R133 报告列出「noob 装备无效（裸装 0% → 全史诗 0%，装备对新手无帮助）」。本轮深入分析发现这是**仿真场景错误**，而非代码 bug。

#### 根因

仿真 Section 5 使用 `oppBand: [55, 65]`（类似于「Ross 中档对手」），但 `TennisRoute.jsx:124` 在实际游戏中对新手施加 `-15` 属性惩罚：

```js
const novicePenalty = isNovice() ? NOVICE_STAT_PENALTY : 0;  // 15
oppStats: { sta: Math.max(20, rand(40, 90) - novicePenalty) }  // 实际 25-75 段
```

真实新手对手段应为 **25-75（均值 50）**，不是 55-65。

#### 正确仿真结果（Section 5b，新手保护期内）

| 场景 | 胜率 |
|------|------|
| noob 裸装（保护期内，对手 25-75） | **23.8%** |
| noob 全史诗（保护期内，对手 25-75） | **43.0%** |
| **提升** | **+19.2pp** ✅ |
| noob 裸装（保护期后，全段对手 40-90） | 0.5% |
| noob 全史诗（保护期后，全段对手 40-90） | 3.3% |

**结论**：装备对新手帮助显著（+19.2pp），仿真之前测错了场景。

### 新手保护期延长（已实施）

保护期内新手胜率约 23-43%（无/有装备），保护期结束后骤降至 0.5-3.3%，断崖式难度提升。

通过延长保护期 **3 → 5 场**：
- 玩家多获得 2 场装备掉落机会，进入全段对手时拥有更好装备
- 首次体验更多「能赢」的场合，提升留存
- 不影响已通过保护期的玩家

**文件变更**：

```diff
// src/modules/tennis/meta/noviceTracker.js

+ export const NOVICE_GAME_THRESHOLD = 5;
```
