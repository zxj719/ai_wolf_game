## 二、本轮改动

### 核心功能：移动端触控目标合规化（WCAG 2.5.5）

**设计动机：**

上轮报告指出（方案 3），当遥测数据无法获取时，检查移动端触控区域是否满足 WCAG AA 要求（≥44×44px）。本轮扫描发现以下元素不达标：

| 元素 | class | 实际尺寸 | 问题 |
|------|-------|---------|------|
| 星评按钮（×5） | `.fb-star` | ~33px 高 | `font-size: 1.8rem + padding: 2px` 不足 |
| 引导弹窗关闭键 | `.ob-close` | ~26px 高 | `padding: 4px 8px + font-size: 1.1rem` 不足 |
| 帮助按钮（?） | `.help-btn` | 32×32px | 固定尺寸不足 |
| 引导 Tab 按钮（×3） | `.ob-tab` | ~35px 高 | `padding: 10px 14px` 不足 |

**修复策略：**

不改变视觉尺寸（字体/图标不变），只扩大可点击区域：

| 元素 | 修复方式 |
|------|---------|
| `.fb-star` | 添加 `min-width/min-height: 44px; display: inline-flex; align-items/justify-content: center;` |
| `.ob-close` | 添加 `min-width/min-height: 44px; display: grid; place-items: center;` |
| `.help-btn` | `width/height: 32px → 44px` |
| `.ob-tab` | `padding-top/bottom: 10px → 12px; min-height: 44px` |

**OnboardingModal 375px 适配（新增 @media (max-width: 400px)）：**

三个 Tab 标签（⚡ 克制表 / 💪 体力规则 / 🔍 读招技巧）在 375px 下接近临界：
- 三 Tab 总宽度约 300px，模态框可用宽度约 331px，理论上不折行
- 实测边距：添加小屏 query 将 tab `font-size: .92rem → .8rem`，`padding: 12px 14px → 12px 8px`，防止极端字体缩放场景下截断
- 同步收窄 `.ob-body`、`.ob-header`、`.ob-footer` padding（22px → 14px），内容显示空间增 8px

**改动文件：**

| 文件 | 操作 | 说明 |
|------|------|------|
| `src/modules/tennis/tennis.css` | 修改 | 4 处触控目标修复 + 375px 媒体查询 |
