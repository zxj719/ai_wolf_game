## 三、本轮改动

### 功能：PrepScreen「备战历史对比」（prep-history-hint）

**背景与动机**

PrepScreen 已有两层信息：
- `last-match-hint`（灰色）：上次整体对战结果（赢/输）
- `prep-strategy-hint`（橙色）：今次对手属性建议

但两者都缺少一个关键维度：**「我上次对这个对手备战完是什么状态？」**

玩家在 4 回合备战时面临组合决策（体力/技巧/心态三维加点），每次备战后进入对局，结果会受备战质量影响。然而下次面对同一对手时，玩家需要重新回忆：「上次我练了什么？那个选法效果怎样？」——这个记忆在下次备战开始时最有价值，但现有 UI 完全不提供。

**「备战历史对比」正是填补这一空缺**：玩家在第 1 回合看到青色框，立刻知道「上次备战完我体力 65、技巧 72、心态 58」，可以据此调整本次策略——是继续强化同一维度，还是尝试不同配比。

**视觉设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 颜色 | 青色 `#22d3ee` | 区别于橙色（建议）和灰色（历史战绩）；「记忆/回溯」的冷色调语义 |
| 背景 | `rgba(34,211,238,.07)` + 左边框 `rgba(34,211,238,.4)` | 与 `prep-strategy-hint` 视觉权重相当，但颜色语义不同 |
| 显示条件 | `prepRound === 0` + 存在历史记录 | 只在第 1 回合显示（有意义的决策点），无历史则不占版面 |
| 存储粒度 | 每个 `playerName × oppName` 只保留最新一条 | 备战策略是面对特定对手的选择，保留最近一次即可（不需要长串历史）|
| 与战绩记录解耦 | 独立 key `tennis_prep_history_v1` | 清除战绩不应清除备战记忆；两者有不同的有效期语义 |

**UI 展示示意（PrepScreen 备战回合 1/4）**

```
┌─────────────────────────────────────┐
│ 备战回合 1 / 4                       │
│ 力量特训                             │
│ 今天要认真练体力还是技巧？            │
│                                     │
│ ⚡ 上次你面对 🦊Elza——你输了 😤       │  ← last-match-hint（灰色）
│                                     │
│ 📋 上次备战 🦊Elza：                  │  ← prep-history-hint（青色）← 本轮新增
│    体力 65 · 技巧 72 · 心态 58        │
│                                     │
│ 💡 球探速报：🦊Elza 今天技巧见长，    │  ← prep-strategy-hint（橙色）
│    备战可侧重强化体力或心态           │
│                                     │
│ [A] 重体力  [B] 均衡  [C] 重技巧      │
└─────────────────────────────────────┘
```

玩家看到「上次我技巧 72 还是输了，这次 Elza 技巧见长，是不是应该换策略练体力？」——决策感从「随机选」变为「基于历史的有意识调整」。

## 四、实现路径

### `src/modules/tennis/localBoard.js`

新增常量 `PREP_HISTORY_KEY = 'tennis_prep_history_v1'` + 内部 `loadPrepHistoryStore()`，以及两个导出函数：

```js
export function savePrepHistory(playerName, oppName, { sta, skill, mind }) {
  const h = loadPrepHistoryStore();
  if (!h[playerName]) h[playerName] = {};
  h[playerName][oppName] = { sta, skill, mind };
  store.setItem(PREP_HISTORY_KEY, JSON.stringify(h));
}

export function loadLastPrepStats(playerName, oppName) {
  const h = loadPrepHistoryStore();
  return h[playerName]?.[oppName] ?? null;
}
```

存储格式：`{ [playerName]: { [oppName]: { sta, skill, mind } } }`  
上限：O(角色数 × 对手数) = 最多 7×7=49 条，每条 3 个整数，总体积极小。

### `src/modules/tennis/__tests__/localBoard.test.js`

导入 `savePrepHistory, loadLastPrepStats`；新增 `describe('savePrepHistory / loadLastPrepStats')` 6 个用例：
1. 无记录时返回 null
2. 保存后可读回相同数据
3. 多次保存后只保留最新一条（覆盖写）
4. 不同玩家互不干扰
5. 不同对手互不干扰
6. 未保存的玩家+对手组合返回 null

### `src/modules/tennis/TennisRoute.jsx`

导入 `savePrepHistory`；在 `window.scrollTo` effect 之后新增：

```jsx
// 备战完成→对局开始时，保存本次备战结果（供下次 PrepScreen 展示对比）
useEffect(() => {
  if (state.screen === 'match' && state.player && state.opp) {
    savePrepHistory(state.player.name, state.opp.name, {
      sta: state.player.sta, skill: state.player.skill, mind: state.player.mind,
    });
  }
}, [state.screen]); // eslint-disable-line react-hooks/exhaustive-deps
```

`state.screen` 变化时精准触发一次，无冗余 IO。

### `src/modules/tennis/components/PrepScreen.jsx`

导入 `loadLastPrepStats`；新增 `prepHistoryHint` useMemo（round 0，依赖 player.name + opp.name）：

```jsx
const prepHistoryHint = useMemo(() => {
  if (state.prepRound !== 0 || !state.player || !state.opp) return null;
  const last = loadLastPrepStats(state.player.name, state.opp.name);
  if (!last) return null;
  return `📋 上次备战 ${state.opp.face}${state.opp.name}：体力 ${last.sta} · 技巧 ${last.skill} · 心态 ${last.mind}`;
}, [state.prepRound, state.player?.name, state.opp?.name]);
```

JSX 插入位于 `lastMatchHint` 之后、`oppStrategyHint` 之前（叙事顺序：整体结果 → 备战细节 → 今日建议）。

### `src/modules/tennis/tennis.css`

新增 1 条规则：
```css

```
