## 三、本轮改动

### 改动：每日排行榜最速奖章（2 个文件，纯增量）

**背景**

R87 完成了日榜当前选手高亮，让玩家能一眼找到自己。本轮在此基础上追加竞争维度——对今日胜者里完成最快的一位颁发 `⚡ 最速` 徽章，形成轻度时间竞赛激励（家人们会互相比「谁打得快」）。

**设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 计算位置 | 纯前端（IIFE 内 `Math.min`） | `duration_s` 已在 `handleTennisDailyLeaderboard` 响应中，无需后端改动 |
| 触发条件 | `c.won && c.duration_s > 0` | 只对胜者颁奖；`duration_s = null` 或 0 的记录（旧数据/调试数据）排除 |
| 平局处理 | 取 `find` 第一个最小值 | 两人用时相同时，按 `won DESC, created_at ASC` 排序在前的获奖（先完成有优势，公平） |
| 颜色 | 蓝色 `rgba(59,130,246,.75)` | 与琥珀色「你」徽章明显区分；蓝色在绿色球场背景上可读性好 |
| 徽章位置 | 在「你」徽章之后，`→` 之前 | 紧贴选手名，与「你」并列或单独存在，不干扰对阵和结果字段 |

**变更详情**

#### `src/modules/tennis/components/SelectScreen.jsx`

在 IIFE 日榜渲染块中新增 3 行逻辑：
```javascript
const winners = sliced.filter((c) => c.won && c.duration_s > 0);
const fastestDur = winners.length > 0 ? Math.min(...winners.map((c) => c.duration_s)) : null;
const fastestName = fastestDur !== null ? winners.find((c) => c.duration_s === fastestDur)?.player_name : null;
```

每个 `<li>` 新增：
```javascript
const isFastest = fastestName && c.player_name === fastestName && c.won && c.duration_s === fastestDur;
// ...
{isFastest && <span className="dc-fastest-badge">⚡ 最速</span>}
```

#### `src/modules/tennis/tennis.css`

新增 1 条规则：
```css
.tennis-scope .dc-fastest-badge { font-size: .6rem; font-weight: 700; background: rgba(59,130,246,.75); color: #fff; border-radius: 3px; padding: 0 4px; margin-left: 2px; line-height: 1.6; }
```

**用户流程示例**

```
今日出战 · 3 位家人 · 包括你

🐯 诚 → 🦊 Elza   ✓ 胜  2分10秒   ← 最快胜者
  [绿左边框] 🐯 诚 [⚡ 最速] → 🦊 Elza  ✓ 胜  2m10s
🦢 莹 [你] → 🦊 Elza  ✗ 败         ← 当前选手（败）
🐰 菲比 → 🐰 铁蛋  ✓ 胜  3分40秒

效果：诚一眼看到自己拿了「最速」，莹知道「我比诚慢且输了」，菲比看到诚 2m10s 会想「下次我要更快」
```
