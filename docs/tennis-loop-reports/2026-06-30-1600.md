## 三、本轮改动

### 功能：PrepScreen「球探速报」智能建议加点

**背景与动机**

备战阶段（PrepScreen）共 4 回合，每回合选 A/B/C 分别提升不同属性（体力/技巧/心态）。当前玩家在选项时没有任何关于对手强项的上下文，导致：
- 新手凭感觉点，结果发现对手技巧远高于自己，才意识到本该提升体力反制
- 老玩家通过多次试错才形成策略直觉
- R107 新增的结算屏「对手属性速览」让玩家赛后理解原因，但未在赛前给予方向

「备战信息对称」：玩家在选择如何分配属性点时，应知道对手的大致属性倾向，这样每次备战都是有意识的策略决策，而非随机点击。显示属性类型（体力见长）而非精确数字，保留数值的一定神秘感，同时给予足够的策略方向。

**设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 位置 | PrepScreen 首轮（prepRound === 0）`lastMatchHint` 之后 | 与上场对战历史并列；首轮决定后续策略最关键 |
| 内容 | `💡 球探速报：{face}{name} 今天{X}见长，备战可侧重强化{Y}` | 显示类型不显示精确值；「球探速报」隐喻自然契合赛事设定 |
| 样式 | 琥珀色（var(--clay)）前景 + 8% 琥珀背景 + 左边框 | 与绿色调的 `last-match-hint` 形成视觉区分；琥珀色 = 警示/建议语义 |
| 仅首轮显示 | ✅ prepRound === 0 时才显示 | 后续回合玩家已有方向，无需重复提示 |
| 是否显示精确数字 | ❌ 不显示 | R107 结算屏已显示精确值；备战时显示类型更符合「球探简报」的隐喻 |

**视觉效果（示例）**

```
备战回合 1 / 4
第一回合 · 赛前加练
距离开赛还有一天，练还是不练，这是个问题。

上次你面对 🍳 铁蛋——你赢了 🎉
💡 球探速报：🦊Elza 今天技巧见长，备战可侧重强化体力或心态

[A] 疯狂挥拍 500 下...     体力+20 心态-10
[B] 闭眼冥想...            技巧+20
[C] 给对手狂发搞笑表情包... 心态+20 技巧+5
```

玩家面对 Elza 时，看到「技巧见长」+ 建议「体力或心态」，会更倾向于选 A（体力+20）或 C（心态+20），而非 B（技巧+20）——这才是理性选择。

## 四、实现路径

### `src/modules/tennis/components/PrepScreen.jsx`

在 `lastMatchHint` useMemo 之后新增 `oppStrategyHint` useMemo（11 行），判断 `state.opp` 的最强属性；在 JSX 中 `lastMatchHint` 之后插入条件渲染（4 行）：

```jsx
const oppStrategyHint = useMemo(() => {
  if (state.prepRound !== 0 || !state.opp) return null;
  const { sta, skill, mind, name, face } = state.opp;
  let dominant, advice;
  if (sta >= skill && sta >= mind) { dominant = '体力'; advice = '技巧或心态'; }
  else if (skill >= mind) { dominant = '技巧'; advice = '体力或心态'; }
  else { dominant = '心态'; advice = '体力或技巧'; }
  return `💡 球探速报：${face}${name} 今天${dominant}见长，备战可侧重强化${advice}`;
}, [state.prepRound, state.opp]);
```

### `src/modules/tennis/tennis.css`

在 `.last-match-hint` 规则之后新增 1 条：

