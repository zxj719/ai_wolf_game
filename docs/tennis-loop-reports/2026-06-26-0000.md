## 二、本轮改动

### 改动：AdventureScreen 赛间攻防回顾（pick phase）

**问题背景**

Round 79 在梯子赛 `between` 状态屏加入了攻防回顾一行；奇幻闯关与梯子赛共享 `BattleScreen`，拥有同样的 `rallyLog` / `matchStats` 数据流，但闯关模式在战斗结束后直接进入 `pick`（选路）而非独立 `between` 屏，因此攻防回顾未能同步到位。

**架构差异分析**

| 模式 | 战后流程 | 回顾位置 |
|------|---------|---------|
| 梯子赛 | `fighting` → `between`（独立屏）→ 选三选一 | `between` 屏顶部 |
| 闯关模式 | `node` → `pick`（选路屏，兼充赛间屏） | `pick` 屏内 card，属性行之后 |

`adventureReducer` 没有 `between` 状态，`pick` 屏就是选路+赛间二合一，是插入攻防回顾的天然位置。

**变更清单**

#### `adventureReducer.js`

1. `createAdventure()` 新增字段：
   ```js
   lastMatchStats: null, // { countersWon, counterLost, ... } from most recent battle
   ```
2. `BATTLE_RESULT` 处理器（胜/败/BOSS 三路）统一加入：
   ```js
   const loot = {
     ...,
     lastMatchStats: action.matchStats ?? null,
   };
   ```
   通过 `loot` 展开，三路复用同一写法：失败路直接展开、胜利路通过 `advance()` 的 `patch` 参数携带、BOSS 胜利同样携带。

#### `AdventureScreen.jsx`

1. `handleBattleOver` 的 `dispatchRun` 调用增加 `matchStats` 字段：
   ```js
   dispatchRun({ type: 'BATTLE_RESULT', win, remainingEnergy: pEnergy, drop, coins, matchStats });
   ```
2. pick phase 的 card 内，属性状态行（`<p className="hint">…`）之后插入：
   ```jsx
   {run.lastMatchStats && (() => {
     const { countersWon = 0, counterLost = 0 } = run.lastMatchStats;
     let adviceText;
     if (counterLost > countersWon + 1) adviceText = '⚠ 被读穿较多，下场注意变招';
     else if (countersWon > counterLost + 1) adviceText = '✨ 读招精准，继续保持';
     else adviceText = '攻防相当';
     return (
       <p className="hint between-defense-review">
         🔁 上场攻防：
         {countersWon > 0 && <span className="bdp-won">⚔️ 克中 {countersWon}</span>}
         {counterLost > 0 && <span className={`bdp-lost${counterLost >= 3 ? ' bdp-warn' : ''}`}>🛡 被克 {counterLost}</span>}
         {countersWon === 0 && counterLost === 0 && <span>平手无克制</span>}
         <span className="bdp-advice"> — {adviceText}</span>
       </p>
     );
   })()}
   ```
   复用 Round 79 已存在的 CSS 类 `.between-defense-review / .bdp-won / .bdp-lost / .bdp-warn / .bdp-advice`，**零新增 CSS**。

#### `adventureReducer.test.js`

新增 3 个测试用例（覆盖胜/败/无 matchStats 三路）：
- `BATTLE_RESULT 胜利时保存 lastMatchStats，供攻防回顾`
- `BATTLE_RESULT 失败时也保存 lastMatchStats`
- `无 matchStats 时 lastMatchStats 为 null`

**三模式覆盖完整性（最终状态）**

| 场景 | 攻防回顾 |
|------|---------|
| 梯子赛（between 屏） | ✅ Round 79 |
| 闯关模式（pick 屏） | ✅ **本轮** |
| 单场赛 | 无赛间屏（单场，不需要） |
