## 二、本轮改动

### 核心功能：PrepScreen 克制建议条

**问题定位：**

OppIntelCard 已展示"对手招式 + 被谁克制"（轮 35），SelectScreen 有全局克制速查表（轮 37），但两者都是"查信息"而非"给建议"——玩家仍需自己推断"我选的角色哪招能克对手"。这一认知步骤对新手而言是额外负担，容易跳过。

**实现方案（2 文件 / 25 行净增）：**

**1. components/PrepScreen.jsx（+14 行）**

- `OppIntelCard` 接收新 prop `playerName`（从 PrepScreen 传入 `state.player?.name`）
- 新增 `counterHints` 计算：
  ```js
  const pBuild = playerName ? CHAR_BUILDS[playerName] : null;
  const counterHints = pBuild
    ? pBuild.moves.flatMap((pm) => {
        const { beats } = getMoveCounters(pm);
        return build.moves.filter((om) => beats.includes(om)).map((om) => ({ pm, om }));
      })
    : [];
  ```
- 渲染 `counter-advice` 区域（`counterHints.length > 0` 守卫，0 克制时不渲染）
- 复用已有 `.counter-winner` / `.counter-arrow` / `.counter-loser` 类（与 CounterDrawer 视觉一致）

**2. tennis.css（+11 行）**

- `.counter-advice`：绿色半透明背景 + 绿边框 + 圆角，视觉上区别于情报区其他部分
- `.counter-advice-label`：`--optic-deep` 色小标题
- `.counter-advice-chips` + `.ca-chip`：flex wrap 排列，每条克制对一个 chip

**设计决策：**

| 决策 | 选择 | 理由 |
|------|------|------|
| 放在 OppIntelCard 底部 vs 单独卡片 | OppIntelCard 底部 | 与对手情报语义相连；备战屏已有 3 个 card，避免增加滚动量 |
| 无克制时 | 完全隐藏 | 避免"无克制关系，靠属性硬拼"等安慰提示带来负面心理 |
| 每对单独一个 chip vs 全放一行 | chip 列表（flex-wrap） | 条目数 0-4 个不等，flex-wrap 自动适配 |
| 复用 counter-winner/loser vs 新颜色 | 复用 | 视觉语言一致；减少 CSS 维护量 |

**覆盖率示例（诚 vs Elza）：**
- 诚（重炮流：flatDrive/smash/topspin/passingShot）
- Elza（控制流：slice/dropShot/lob/volley）
- 克制对：重炮平击→克→切削放缓 / 上旋抽击→克→放小球 / 穿越球→克→网前截击
- 建议条显示 3 个 chip，策略指引清晰
