## 二、本轮改动

### 1. `tennisReducer` 新增 REMATCH 动作（`src/modules/tennis/useTennisGame.js` +12 行）

**问题：** `REPLAY` 返回 `initialState`（`screen: 'select'`），用户须重走：选角色→选模式→反应测试→备战（4 轮）→比赛，共 5 个步骤。对想"和同一个对手再打一局"的用户摩擦过大。

**方案：** 新增 `REMATCH` case——保留 `player.name`、`opp`（含新手保护期调整后的属性）、`mode`，重置玩家属性为 0，回到 `screen: 'react'`，只需 3 步（反应测试→备战→比赛）。

```javascript
case 'REMATCH': {
  const me = CHARS.find((c) => c.n === state.player.name);
  return {
    ...initialState,
    screen: 'react',
    mode: state.mode,
    player: { name: me.n, face: me.f, talent: 0, sta: 0, skill: 0, mind: 0, ms: null, grade: '' },
    opp: state.opp,
  };
}
```

### 2. ResultScreen 双按钮布局（`src/modules/tennis/components/ResultScreen.jsx` +3 行）

原单个"🔁 不服！再来一局"替换为并排两按钮（`flexWrap` 自适应移动端）：

- **主按钮**：`⚡ 再战 {opp.face} {opp.name}`（`btn`，dispatch REMATCH）
- **次按钮**：`🔁 换对手`（`btn ghost`，dispatch REPLAY，完整重置）

用户体验前后对比：

| 场景 | 改动前 | 改动后 |
|------|--------|--------|
| 想对同一对手再来一局 | 须重走选角→选模式→反应→备战（4 步） | ⚡ 再战按钮，直接跳到反应测试（3 步）|
| 想换个对手 | 点"再来一局"重走全流程 | 🔁 换对手，与之前 REPLAY 行为相同 |

### 3. AdventureScreen FeedbackWidget（`src/modules/tennis/modes/adventure/AdventureScreen.jsx` +8 行）

在 `phase === 'victory' || 'failed'` 终局屏的"返回报名处"按钮后插入 FeedbackWidget，完成三模式（single/ladder/adventure）评价收集闭环：

```jsx
<FeedbackWidget
  mode="adventure"
  character={basePlayer.name}
  result={won ? 'win' : 'loss'}
/>
```

### 4. REMATCH 测试（`src/modules/tennis/__tests__/tennisReducer.test.js` +12 行）

验证：`screen === 'react'`、`mode` 保留、`player.name` 保留、`player.sta === 0`、`player.ms === null`、`opp` 完整保留。
