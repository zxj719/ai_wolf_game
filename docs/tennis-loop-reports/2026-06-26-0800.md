## 二、本轮改动

### 改动：SelectScreen 角色出战统计 + 「✦ 最佳」徽章

**背景**

历次评估循环发现循环一直运行在 detached HEAD（与 origin/main 无共同祖先），导致所有改动从未真正进入生产代码。本轮首次将改动直接应用到 main 分支，从 origin/main 的实际代码状态出发。

**核心设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 最小出战场次门槛 | 2 场 | 1 场 1 胜 = 100% 胜率，样本不可信 |
| 最小胜场要求 | 1 胜 | 全败角色展示「✦ 最佳」语义奇怪 |
| 胜率相同平局规则 | 出战更多者优先 | 更大样本更可信 |
| 视觉层级 | `.nm` 下方、`.char-stat` 上方 | 评价优先于数量 |
| 颜色 | `#fbbf24`（amber-400） | 金色/成就感；区别于主题色 `var(--clay)` |

**变更清单**

#### `localBoard.js`

新增两个纯函数：

```js
export function computeCharStats(records) {
  const map = {};
  for (const r of records) {
    if (!r.p) continue;
    if (!map[r.p]) map[r.p] = { played: 0, won: 0 };
    map[r.p].played++;
    if (r.sp > r.so) map[r.p].won++;
  }
  return map;
}

export function findBestChar(charStatsMap, minPlayed = 2) {
  let bestName = null, bestRate = -1, bestPlayed = 0;
  for (const [name, { played, won }] of Object.entries(charStatsMap)) {
    if (played < minPlayed || won === 0) continue;
    const rate = won / played;
    if (rate > bestRate || (rate === bestRate && played > bestPlayed)) {
      bestRate = rate; bestPlayed = played; bestName = name;
    }
  }
  return bestName;
}
```

从本轮起，改动直接应用到 `main` 分支并推送，以确保变更实际进入生产。
