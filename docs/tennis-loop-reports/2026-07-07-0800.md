## 三、本轮改动

### 功能：克制对决视觉条（`CounterDuelBar`）

**背景与动机**

至 R140，结算屏「本局统计」中对克制数据的展示方式：
- `ms-item` 格：「克制得分 N」和「被克失分 N」各占一格（已有）
- 文字 1：`countersWon ≥ 2` 时显示「🎯 本场克制得分 N 次！」（绿色）
- 文字 2：`counterLost ≥ 2 && counterLost > countersWon` 时显示「🛡 被克 N 次…」（灰色斜体）

这两条文字是分离的、单向的——玩家要自己用数字比大小来理解「到底谁在克谁」。人类对比例/面积的感知比数字运算快得多：一条双色比例条能在不到 100ms 内传达「本局克制平衡偏向哪一侧」。

**组件设计**

```jsx
function CounterDuelBar({ countersWon, counterLost }) {
  const total = countersWon + counterLost;
  if (total < 2) return null;   // < 2 次克制事件 → 随机性太强，不展示
  // 三档裁决
  const ratio = countersWon / total;
  if (ratio >= 0.6) verdict = '读招占优！克制贡献了本局关键分';      // 绿
  else if (ratio >= 0.4) verdict = '势均力敌，继续保持读招意识';     // 黄
  else verdict = '被对手读穿，下局试试变换出招节奏';                 // 红
  // 条形：用 flex: N 而非 width: N% → 零值自然折叠，无需特殊处理边缘
  return (
    <div className="counter-duel-bar">
      <div className="cdb-layout">
        <div className="cdb-side cdb-won">N / 克制得分</div>
        <div className="cdb-bar">
          <div className="cdb-fill-won" style={{ flex: countersWon }} />
          <div className="cdb-fill-lost" style={{ flex: counterLost }} />
        </div>
        <div className="cdb-side cdb-lost">被克失分 / N</div>
      </div>
      <div className="cdb-verdict" style={{ color: verdictColor }}>{verdict}</div>
    </div>
  );
}
```

**触发阈值的调整**

旧逻辑两条文字触发条件不一致（一条是 `≥2`，另一条是 `≥2 && >countersWon`），容易出现「赢了但没有正面反馈」的死角。新的统一条件：`total ≥ 2`，无论哪侧多都展示，逻辑更对称。

**CSS 要点**

- `flex: N` 驱动比例（非 width%），让 0-only 场景自然折叠
- 绿 `#4ade80` / 红 `#f87171` 与 `fx-greenflash` / `fx-redflash` 同色系，强化语义一致性
- `prefers-reduced-motion` 补充（目前动效本身是 CSS gradient 无 transition，规则预留给未来迭代）
