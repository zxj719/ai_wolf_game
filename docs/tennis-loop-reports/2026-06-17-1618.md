## 二、本轮改动

### 核心功能：SelectScreen 对手预告条

**问题定位：**

报名处已经提示「系统会从剩下 6 位家人里随机抽出你的宿敌」，但玩家并不知道那 6 位具体是谁。PersonalBestCard（轮 30）把玩家的注意力锚定在报名屏底部后，这里正好可以承接一块轻量的悬念信息——"我可能遇到 🐯🦊🐰🦖🍳🐱 中的哪一位？"——激发进入下一步的动力。

**实现方案：**

**1. OpponentPreviewStrip 组件（SelectScreen.jsx）**

```jsx
function OpponentPreviewStrip({ picked }) {
  if (!picked) return null;
  const opponents = CHARS.filter((c) => c.n !== picked);
  return (
    <div className="opp-preview-strip" role="region" aria-label="可能的对手预告">
      <span className="opp-preview-label">本轮可能遇到</span>
      <div className="opp-preview-faces">
        {opponents.map((c) => (
          <span key={c.n} className="opp-preview-chip" title={c.n}>
            <span className="opp-preview-face">{c.f}</span>
            <span className="opp-preview-name">{c.n}</span>
          </span>
        ))}
      </div>
      <span className="opp-preview-hint">随机 1 位</span>
    </div>
  );
}
```

**显示逻辑：**
- `picked` 为空（用户未选角色）→ 返回 null，完全不渲染
- `picked` 非空 → 过滤掉自己，展示剩余 6 位家人的 emoji + 名字
- 位置：PersonalBestCard 下方，卡片底部；报名屏的「最后一行」

**2. 样式（tennis.css，+30 行）**

延续 `select-pb-card` 的横幅语言，但用琥珀色（`rgba(255,180,0,…)`）区分：  
- 背景：`rgba(255,180,0,.05)` + `rgba(255,180,0,.2)` 边框（区别于黄绿色历史最佳卡）  
- 每位对手：小圆角卡片，emoji 1.1rem + 名字 .62rem 竖排，hover 时浅琥珀底色  
- 「随机 1 位」右对齐斜体浅灰，说明随机性不影响悬念感

**改动文件：**
