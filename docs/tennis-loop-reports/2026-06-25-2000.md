## 二、本轮改动

### 改动：BattleScreen 新增 `RallyTrend` 攻防趋势点阵

**问题背景**

Round 78 实现了 DefensePressure 实时计数（「被克 N · 克中 N」芯片），Round 79 实现了梯子赛赛间回顾。两者提供了**数量**信息，但没有**时序**信息：玩家无法看出"我前面 4 球都正常，最近 3 球连续被克"这种趋势。趋势信息对「要不要换招」的决策价值高于总计数——看到最近 3 球红点连续，立刻意识到需要换招，而不需要等 DefensePressure 数字升到警戒档才反应。

**设计决策**

- **位置**：DefensePressure 芯片下方（能量条区域尾部），不进入信息已密集的赛场卡
- **数据源**：`state.rallyLog`（battleReducer 已有，存最近 10 球）
- **展示范围**：最近 7 球（`rallyLog.slice(-7)`），更多历史靠 DefensePressure 总计数体现
- **颜色分类**（与 FxOverlay「效果拔群/被克制」特效一致）：
  - 🟢 绿 (`won`)：`r.counterMul > 1 && r.win && !r.hawkeyeSaved` ← 克中且赢点
  - 🔴 红 (`lost`)：`r.counterMul < 1 && !r.win` ← 被克且失点
  - ⚪ 灰 (`neutral`)：其余（含鹰眼平局重打、无克制关系的普通球）
- **隐藏策略**：`rallyLog.length === 0` 时返回 `null`，首球前不占位
- **可访问性**：每个 `<span>` 附 `title` 属性（「克中/被克/普通」），`aria-label` 在容器

**实现**

`BattleScreen.jsx` 新增 `RallyTrend` 函数组件（16 行）：

```jsx
function RallyTrend({ rallyLog }) {
  if (!rallyLog.length) return null;
  const dots = rallyLog.slice(-7);
  return (
    <div className="bt-rally-trend" aria-label="近球攻防趋势">
      {dots.map((r, i) => {
        const type = r.counterMul > 1 && r.win && !r.hawkeyeSaved ? 'won'
          : r.counterMul < 1 && !r.win ? 'lost' : 'neutral';
        return <span key={i} className={`bt-rt-dot ${type}`} title={
          type === 'won' ? '克中' : type === 'lost' ? '被克' : '普通'
        } />;
      })}
    </div>
  );
}
```

插入点（DefensePressure 下方 1 行）：

```jsx
<DefensePressure ... />
<RallyTrend rallyLog={state.rallyLog} />
```

`tennis.css` 新增 4 行样式：

```css

.tennis-scope .bt-rt-dot { width: 9px; height: 9px; border-radius: 50%; flex-shrink: 0; }
.tennis-scope .bt-rt-dot.won { background: #4caf76; box-shadow: 0 0 4px rgba(76,175,118,.5); }
.tennis-scope .bt-rt-dot.lost { background: var(--clay, #cc4444); box-shadow: 0 0 4px rgba(194,94,29,.4); }
```
