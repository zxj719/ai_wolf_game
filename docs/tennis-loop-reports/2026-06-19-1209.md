## 二、本轮改动

### 核心功能：mode 页绝技图鉴条（ult-strip）

**问题定位：**

mode 页（赛事报名屏）的状态摘要行已有 `📖 绝技图鉴 X/7`，但只是数字。`char-wins-bar`（角色攻克行，绿色 chip）已提供角色维度的收集进度可视化，但绝技维度缺乏等价的视觉进度行，导致「收集所有绝技」的驱动力较弱。

**实现方案（2 文件变更）：**

**1. TennisRoute.jsx（+18 行）**

在 `char-wins-bar` 结束后、`ach-gallery` 开始前插入 `ult-strip`：

```jsx
<div className="ult-strip">
  {Object.entries(ULTIMATES).map(([ultName, u]) => {
    const isOwn = u.owner === state.player.name;
    const isUnlocked = ultimateOptions.some((opt) => opt.name === ultName);
    return (
      <span
        key={ultName}
        className={`ult-chip${isUnlocked ? ' ult-chip--on' : ''}`}
        title={isUnlocked ? `「${ultName}」${u.desc}` : `${u.owner}的绝技 · 家族挑战连胜可解锁`}
      >
        {u.face}
        {isOwn && <span className="ult-chip-badge ult-chip-badge--own">⭐</span>}
        {isUnlocked && !isOwn && <span className="ult-chip-badge">✓</span>}
      </span>
    );
  })}
  <span className="char-wins-hint">家族挑战连胜依次解锁⚡</span>
</div>
```

关键设计：
- **零新逻辑**：复用 `ultimateOptions`（已在组件计算）和 `ULTIMATES`（已 import）
- **isOwn 徽章**：自有绝技显示紫色 ⭐，区别于解锁他人的橙色 ✓
- **tooltip 双模式**：已解锁显示「绝技名 + 效果描述」，未解锁显示「谁的绝技 · 家族挑战连胜可解锁」
- **复用 `.char-wins-hint`** 样式（已有，零额外 CSS）

**2. tennis.css（+25 行）**

新增 `ult-strip / ult-chip / ult-chip--on / ult-chip-badge / ult-chip-badge--own` 类。

色彩设计：
- 已解锁：amber（#fbbf24）— 与现有 `ult-desc-tip` 颜色统一，形成绝技专属色调
- 本命徽章：紫色（#7c3aed）— 区分角色行的绿色（#4ade80）和解锁徽章的橙色（#d97706）

**玩家体验前后对比：**

| 场景 | 改动前 | 改动后 |
|------|--------|--------|
| 进入 mode 页 | 只看到「绝技图鉴 1/7」 | 直观看到 7 个 chip，哪些金色=已解锁 |
| 悬停未解锁 chip | 无信息 | tooltip：「XX的绝技 · 家族挑战连胜可解锁」 |
| 悬停已解锁 chip | 无信息 | tooltip：「「绝技名」效果描述」 |
| 自有绝技 | 无区分 | 紫色⭐标记 |
