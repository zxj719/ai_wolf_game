## 二、本轮改动

### 设计原理

**为什么 move 层扰动无效（调参第一步的教训）：**

初版实现在 `pickOpponentMove` 内随机化出招，但 `tellReader` 会读取 *实际出招* 的 tell 并反制——信息优势未被破坏，反而让对手有时出了非优势招型，胜率 81%→84%（反增）。

**正确机制：tell 层强制假提示**

- 当 `oppAttrAvg ≥ 75` 且 `disruptRoll < DISRUPT_RATE (0.12)` 时，`battleReducer` 强制将 `truthRoll` 提高至 1.0，使 `makeTell` 必然返回假提示
- 玩家在 12% 的球上看到的 tell 是错的，即使正确"读招"也选出了错误的反制招
- 被克中 + 失去克制优势的双重惩罚，使每次假提示对胜率影响显著（37pp/28%，约 1.3pp 每 1% rate）

### ✅ opponentAI.js — 导出 `DISRUPT_RATE`，更新注释

```js
// 扰动实现在 battleReducer.BEGIN_RALLY（tell 层）
export const DISRUPT_RATE = 0.12;
```

### ✅ battleReducer.js — BEGIN_RALLY tell 层扰动

```js
const oppAttrAvg = ((state.opponent.sta ?? 0) + (state.opponent.skill ?? 0) + (state.opponent.mind ?? 0)) / 3;
// ...pickOpponentMove 不变...
const disruptFires = oppAttrAvg >= 75 && action.disruptRoll !== undefined && action.disruptRoll < DISRUPT_RATE;
const tell = makeTell({
  ...
  // predictable 扭曲（BOT-3000）永真优先；扰动不覆盖 twist
  truthRoll: state.twists.predictable ? 0 : (disruptFires ? 1.0 : action.truthRoll),
  ...
});
```

**向后兼容：** `disruptRoll: undefined`（旧测试和未升级的 BEGIN_RALLY dispatch）不触发扰动。

### ✅ BattleScreen.jsx — BEGIN_RALLY 新增 disruptRoll

```js
dispatch({
  type: 'BEGIN_RALLY', rng: Math.random, moveRoll: Math.random(),
  truthRoll: noviceTell ? 0 : Math.random(),
  fakeRoll: Math.random(),
  disruptRoll: Math.random(),  // ← 新增
});
```

### ✅ balance-sim.mjs — simMatch 支持 disruptRollFn + §9 对比节

- `disruptRollFn = Math.random`（默认）：正常使用，activates disruption for 75+ opponents
- `disruptRollFn = () => 1.0`：强制不触发扰动（baseline 对比）

### ✅ 测试（battleReducer.test.js + opponentAI.test.js）

新增 6 个测试用例，覆盖：
- 中档对手 + disruptRoll<0.28 → tell 不变
- 高档对手 + disruptRoll<0.12 → 强制假提示（isTrue=false）
- 高档对手 + disruptRoll≥0.12 → 不触发（isTrue=true）
- 高档对手 + disruptRoll 未提供 → 不触发（向后兼容）
- twists.predictable 时扰动不覆盖永真
