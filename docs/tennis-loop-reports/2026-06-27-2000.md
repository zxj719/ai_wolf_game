## 三、本轮改动

### 改动：每周家族最强徽章

**问题背景**

家庭游戏场景中，「家里这周谁赢得最多」是天然高频的竞争话题。R91 的本地榜角色筛选让「我用铁蛋赢了几场」一键可查；本轮进一步提炼：在榜单顶端自动结算并展示「本周家族最强者」，并在赢球结算屏提供即时荣誉反馈，为日常比赛提供一个轻量级但有说服力的目标。

**设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 时间窗口 | 过去 7 天（`Date.now() - 7×24h`） | 滚动窗口比「周一重置」对家庭随机玩法更友好 |
| 最低门槛 | `minWins = 2` | 防止一场游戏就夺冠；家庭场景每天可能只玩 1-2 场 |
| 向后兼容 | 旧记录无 `ts` 字段，被 `computeWeeklyChamp` 自动忽略 | 不需要任何数据迁移，零破坏性 |
| 时间注入 | `now = Date.now()` 默认参数可由测试注入 | 纯函数，无副作用，9 个测试用例均可控 |
| 结算屏展示条件 | 仅赢球时显示（`playerWon && weeklyChampName === p.name`） | 避免输球时叠加负面情绪 |
| 榜单展示位置 | LocalBoard 顶部、filter chips 上方 | 第一眼可见，不影响筛选操作 |

**实现路径**

#### `src/modules/tennis/localBoard.js`

1. `saveLocalRecord` 记录中追加 `ts: Date.now()`（单行改动）
2. 新增 `computeWeeklyChamp(records, {minWins=2, now=Date.now()} = {})` 函数（25 行）：
   - 过滤 `r.ts && r.ts > now - 7×24h` 的记录（无 ts 字段的旧记录自动被排除）
   - 按 `r.p` 聚合 `wins` 和 `played`
   - 返回 wins 最多（同 wins 时 played 更多）且满足 `minWins` 的角色，无满足者返回 `null`

#### `src/modules/tennis/components/Leaderboard.jsx`

- `LocalBoard` 组件：import `computeWeeklyChamp`，计算 `weeklyChamp = computeWeeklyChamp(records)`
- filter chips 上方条件渲染 `.weekly-champ-banner`：`🏅 本周家族最强：{face} {name} · 周内 {wins} 胜`

#### `src/modules/tennis/components/ResultScreen.jsx`

- import `computeWeeklyChamp`
- 新增 `weeklyChampName` useMemo：将本局虚拟注入历史（含 ts=Date.now()）后调用 `computeWeeklyChamp`（解决 save effect 与 useMemo 的执行顺序问题）
- 结算卡片中，赢球且为本周最强时展示 `.weekly-champ-badge`：`🏅 本周家族最强`

#### `src/modules/tennis/tennis.css`

- 新增 `.weekly-champ-banner`（黄金色背景＋边框，圆角 8px，居中文字）
- 新增 `.weekly-champ-badge`（金色胶囊，与 `ladder-streak` 风格一致）

#### `src/modules/tennis/__tests__/localBoard.test.js`

新增 `describe('computeWeeklyChamp')` 块，9 个测试用例覆盖：
- 空记录返回 null
- 无 ts 字段旧记录被忽略
- 7 天外记录被忽略
- 胜场不足 minWins 时返回 null
- 单角色满足条件时正确返回
- 多角色时返回胜场最多者
- 胜场相同时出战场次更多者优先
- 边界时间（截止时间边界内 +1ms）
- 自定义 minWins=1
