## 二、本轮改动

### 核心功能：MatchReplayCard 三模式统一

**问题定位：**

上轮实现 `MatchReplayCard` 时将其定义为 `ResultScreen.jsx` 内的局部组件，而 `BattleScreen.onMatchOver` 实际上已在三个调用点（`TennisRoute`、`LadderScreen`、`AdventureScreen`）都传出了 `rallyLog`，但后两者的处理函数签名中没有接收 `rallyLog`，导致数据静默丢弃，终局屏无法展示回顾卡。

**修复路径：**

1. **提取共享组件** `src/modules/tennis/components/MatchReplayCard.jsx`  
   - 从 `ResultScreen.jsx` 中原样抽出，只改 import 路径（`MOVES` 从 `../battle/moves`）  
   - `ResultScreen.jsx` 改为 `import { MatchReplayCard } from './MatchReplayCard'`，移除内联定义

2. **LadderScreen 修复**  
   - 新增本地状态 `lastRallyLog` / `lastMatchStats`  
   - `handleMatchOver` 解构 `rallyLog`，调用 `setLastRallyLog` / `setLastMatchStats`  
   - 终局屏（`ladder.status === 'won'/'lost'`）插入 `<MatchReplayCard .../>`

3. **AdventureScreen 修复**  
   - 同上模式，`handleBattleOver` 解构 `rallyLog`  
   - `victory/failed` 终局屏插入 `<MatchReplayCard .../>`

**为什么不需要扩展 BattleScreen：**  
`BattleScreen` 在第 17 轮已被修改为透传 `rallyLog`（第 293 行），本轮仅在消费端补全了接收逻辑，无需改动战斗层。

**改动文件：**

| 文件 | 操作 | 说明 |
|------|------|------|
| `src/modules/tennis/components/MatchReplayCard.jsx` | **新建** | 从 ResultScreen 抽出的共享组件 |
| `src/modules/tennis/components/ResultScreen.jsx` | 修改 | 改为 import 共享组件，移除内联定义，移除 `MOVES` import（已不需要） |
| `src/modules/tennis/modes/LadderScreen.jsx` | 修改 | 捕获 rallyLog/matchStats，终局屏加 MatchReplayCard |
| `src/modules/tennis/modes/adventure/AdventureScreen.jsx` | 修改 | 同上 |
