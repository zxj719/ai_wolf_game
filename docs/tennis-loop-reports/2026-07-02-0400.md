## 三、本轮改动

### 功能：SelectScreen「克制路径可视化」（matchup-wr-sub）

**背景与动机**

R113–R116 建立了从选对手到赛后复盘的完整五节点新手引导链：

| 阶段 | 信息层 |
|------|--------|
| 选对手 | 🎯 matchup-hint（克制招名） |
| 备战加点 | ⚔️ matchup-advice-hint（克制招+属性方向） |
| 对局第 1 球 | 🆕 first-match-hint（入场前绿色提示） |
| 对局持续 | OppCoachHint（对手惯用招 ×2 后） |
| 赛后复盘 | prep-growth-hint（备战成长对比） |

遗留的一段信息断层：**克制建议长期以来只有「方向」，没有「结果反馈」**。玩家多次按建议出招后，没有任何界面告诉他"这个策略实际有效率是多少"，导致玩家无法校准"到底该信这个建议还是自己凭感觉走"。

本轮填补这个断层：

**设计思路**

| 维度 | 设计选择 | 理由 |
|------|---------|------|
| 显示位置 | 在 matchup-hint 内部，第二行小字 | 空间紧凑，视觉上紧挨克制招名，语义关联清晰 |
| 显示条件 | `playerOppWrMap[c.n]?.total >= 2` | 1 场数据（0/1 或 1/1）方差过大无参考价值；2 场起才有相对意义 |
| 数据来源 | `computePlayerOppWinRates(records, picked)` | 双维度过滤（玩家身份 + 对手），不混入其他家庭成员的记录 |
| 文案 | `以此 W/N` | 极简；「以此」暗示"用这个克制策略时" |
| 颜色 | `rgba(251,191,36,.48)` | 与 matchup-hint 同色系（琥珀黄）但亮度更低，视觉层级降低不喧宾夺主 |
| 字号 | `.46rem` | 比 matchup-hint(.52rem) 更小，实现信息主次分层 |

**与既有 oppWinRateMap 的区别**

| 字段 | 过滤维度 | 用途 |
|------|---------|------|
| `oppWinRateMap[c.n]` | 只按对手，跨所有玩家 | 对手难度标签（推荐/劲敌）、排序 |
| `playerOppWrMap[c.n]` | 玩家 + 对手双维度 | 「以此 W/N」克制策略回顾（本轮新增）|

## 四、实现路径

### `src/modules/tennis/localBoard.js`

新增纯工具函数：

```js
export function computePlayerOppWinRates(records, playerName) {
  const map = {};
  for (const r of records) {
    if (r.p !== playerName) continue;
    if (!map[r.o]) map[r.o] = { wins: 0, total: 0 };
    map[r.o].total++;
    if (r.sp > r.so) map[r.o].wins++;
  }
  return map;
}
```

### `src/modules/tennis/__tests__/localBoard.test.js`

新增 `describe('computePlayerOppWinRates', ...)` 4 个测试：
- 空记录返回空对象
- 玩家无记录时返回空对象
- 正确按对手分组统计胜负（包含跨玩家隔离验证）
- 全败时 wins 为 0

### `src/modules/tennis/components/SelectScreen.jsx`

1. Import 新增 `computePlayerOppWinRates`
2. 新增 `playerOppWrMap` useMemo（`picked` + `records` 依赖）
3. 修改 matchup-hint JSX：内嵌 `<span className="matchup-wr-sub">以此 W/N</span>`（total >= 2 时）

### `src/modules/tennis/tennis.css`
