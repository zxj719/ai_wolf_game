## 二、本轮改动

### 角色赛后评语（ResultScreen Character Quotes）

**问题背景**

当前结算页的 `.comment` 文本来自 `ENDINGS` 对象，按盘分结果（2-0/2-1/1-2/0-2）固定四条通用文案。玩家与不同家庭成员对阵却看到相同文本，缺乏人物个性和情感张力，削弱了「家庭网球公开赛」的叙事核心卖点。

**方案**

在 `gameData.js` 中新增 `CHAR_QUOTES` 数据，为 7 位角色各写胜负两套评语（`victory` 和 `defeated` 各 3 条），结算页随机展示一条。角色台词风格紧贴各自人设：

| 角色 | 人设风格 | 败后台词示例 |
|------|---------|-------------|
| 诚 🐯 | 傲骄强壮 | 「你…你运气好！改天我一定赢回来！」 |
| Elza 🦊 | 狡黠冷静 | 「承认你今天打得很好。（翻了个白眼）」 |
| 菲比 🐰 | 活泼少女 | 「啊啊啊居然输了！！算了，我去找妈妈哭鼻子！」 |
| Ross 🦖 | 学术理性 | 「数据分析显示：我今天的失误率出现了统计异常。」 |
| 铁蛋 🍳 | 市井热情 | 「什么！？我连早饭都没吃完就输了！！」 |
| 丫 🐱 | 慵懒冷漠 | 「喵……（蜷起来，假装在睡觉）」 |
| 莹 🦢 | 优雅知性 | 「优雅地输球也是一种风度～（但心里不服！）」 |

**实现细节**

**`src/modules/tennis/gameData.js`（+50 行）**：新增 `CHAR_QUOTES` 导出对象，7 个角色 × 2 结果 × 3 条台词。

**`src/modules/tennis/components/ResultScreen.jsx`（+7 行）**：
```jsx
// 导入
import { ENDINGS, CHAR_QUOTES, rand } from '../gameData';

// 组件顶部（在 ending 之后）
const playerWon = state.setsP > state.setsO;
const oppQuotePool = CHAR_QUOTES[o.name]?.[playerWon ? 'defeated' : 'victory'];
const oppQuote = oppQuotePool ? oppQuotePool[rand(0, oppQuotePool.length - 1)] : null;

// JSX（ending-hero 与战报卡之间）
{oppQuote && (
  <div className="card flat char-quote-card">
    <span className="char-quote-who">{o.face} {o.name} 赛后说：</span>
    <p className="char-quote-text">「{oppQuote}」</p>
  </div>
)}
```

**`src/modules/tennis/tennis.css`（+3 行）**：
```css
.tennis-scope .char-quote-card { text-align: center; padding: 18px 24px; }
.tennis-scope .char-quote-who { display: block; font-size: .78rem; color: var(--muted); font-family: var(--mono); letter-spacing: .04em; margin-bottom: 10px; }
.tennis-scope .char-quote-text { font-size: 1.05rem; color: var(--card-ink); line-height: 1.65; margin: 0; font-style: italic; }
```

**设计决策**：
- 台词在每次结算页 render 时通过 `rand()` 产生，不存入 state——每次重新进入页面会随机另一条，增加「惊喜感」
- `CHAR_QUOTES[o.name]?.` 可选链确保未来新增角色时不崩溃
- 台词放在 `gameData.js` 而非内联，便于未来扩充文案池
