## 二、本轮改动

### 改动 1：SelectScreen 角色收集进度条（SelectScreen.jsx + tennis.css）

**问题背景**

游戏有 7 位可对战家人，但选择页没有任何「你还没和谁打过」的提示，玩家可能一直只和同一个角色对战，错过了探索所有家人风格的乐趣。

**方案**

在 `.roster` 角色列表下方，插入一个从 localStorage 实时读取历史对手数据的进度条：

```jsx
// SelectScreen.jsx 新增
const seenOpps = useMemo(() => new Set(loadLocalRecords().map((r) => r.o)), []);
const seenCount = seenOpps.size;
const totalOpps = CHARS.length; // 7，对未来扩充健壮

// JSX（roster 后 equip-mini-bar 前）
{seenCount > 0 && (
  <div className="opp-progress">
    <span className="opp-progress-label">
      已挑战 {seenCount}/{totalOpps} 位家人
      {seenCount === totalOpps && '　🏅 全家大挑战完成！'}
    </span>
    <div className="opp-progress-bar">
      <div className="opp-progress-fill" style={{ width: `${(seenCount / totalOpps) * 100}%` }} />
    </div>
  </div>
)}
```

**CSS 新增（4 行）：**
```css
.tennis-scope .opp-progress { margin-top: 14px; }
.tennis-scope .opp-progress-label { display: block; font-size: .75rem; color: var(--muted); font-family: var(--mono); margin-bottom: 6px; }
.tennis-scope .opp-progress-bar { height: 5px; background: rgba(13,59,46,.15); border-radius: 4px; overflow: hidden; }
.tennis-scope .opp-progress-fill { height: 100%; background: linear-gradient(90deg, var(--clay), var(--optic)); border-radius: 4px; transition: width .35s ease; }
```

**设计决策**：
- `useMemo(fn, [])` 在组件挂载时读一次 localStorage，避免每次 re-render 重新解析 JSON
- `seenCount > 0` 条件渲染：新玩家首次进入不显示（无历史数据），不打扰首次体验
- `CHARS.length` 而非硬编码 7，对未来扩充角色自动适应
- 进度条用 `clay→optic`（橙→荧光绿）渐变，与网球场配色主题一致
- 集满触发「🏅 全家大挑战完成！」行内提示，无需弹窗干扰

### 改动 2：ResultScreen 连胜记录徽章（ResultScreen.jsx）

**问题背景**

玩家可能连续赢多场，但目前结算页只显示本场大比分，没有任何「连胜感」的累计反馈，错失强化「我在进步！」情绪的机会。

**方案**

利用 effect 挂载晚于首次渲染的时序特性，在 `useMemo([])` 中读取历史记录（此时当前游戏尚未写入），叠加当前局结果计算真实连胜数：

```jsx
// ResultScreen.jsx 新增
import { useEffect, useMemo, useRef } from 'react';
import { saveLocalRecord, loadLocalRecords } from '../localBoard';

const winStreak = useMemo(() => {
  const mine = loadLocalRecords().filter((r) => r.p === p.name);
  let streak = playerWon ? 1 : 0;
  for (let i = mine.length - 1; i >= 0 && mine[i].sp > mine[i].so; i--) streak++;
  return streak;
}, []); // 挂载时读一次，计算包含本局的连胜

// JSX（大比分后、comment 前）
{playerWon && winStreak >= 2 && (
  <div style={{ textAlign: 'center', marginTop: 10 }}>
    <span className="ladder-streak">🔥 本次连胜 {winStreak} 场</span>
  </div>
)}
```

**设计决策**：
- 时序保证：React effect 在 paint 后运行，`useMemo([])` 在首次渲染中执行，此时 `saveLocalRecord` 的 effect 尚未运行，历史记录不含当前局，手动 +1 即可
- 复用 `.ladder-streak`（LadderScreen 已有的火焰徽章样式），零 CSS 新增
- 只在 `playerWon && winStreak >= 2` 时显示：败场不显示（不打击玩家），首胜（streak=1）不显示（只有连续才有意义）
- 倒序遍历 `mine` 遇到首条败绩即 break，O(streak) 而非 O(N)
