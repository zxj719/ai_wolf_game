## 二、本轮改动

### 新端点：`GET /api/tennis/feedback/summary`

**文件**：`workers/auth/tennis.js`（新增 `handleTennisFeedbackSummary`）、`workers/auth/index.js`（新增导入 + 路由）

**查询结构**（单次 D1 batch，5 个并发语句）：

| 查询 | 说明 |
|------|------|
| `byRating` | 按 1-5 星分桶，返回每桶数量 |
| `byMode` | 按模式聚合，返回数量 + 平均星级 |
| `byCharacter` | 按角色聚合，返回数量 + 平均星级 |
| `recentComments` | 最新 20 条有内容评论（rating, comment, mode, character, result, created_at） |
| `total` | 总评价数（用于加权平均星级计算） |

**响应体**：
```json
{
  "total": 42,
  "avgRating": 4.12,
  "byRating": [{ "rating": 5, "count": 18 }, ...],
  "byMode": [{ "mode": "adventure", "count": 28, "avg_rating": 4.36 }, ...],
  "byCharacter": [{ "character": "Elza", "count": 12, "avg_rating": 3.92 }, ...],
  "recentComments": [{ "rating": 4, "comment": "太好玩了", "mode": "adventure", ... }, ...]
}
```

**设计说明**：
- 单次 batch 调用，避免串行请求延迟
- `avgRating` 在 JS 层加权计算（`Σ(rating × count) / total`），精确且无 SQL 浮点陷阱
- 访问不受权限限制（公开只读），与 telemetry/summary 保持一致
- `node --check` 通过，无运行时依赖变更
