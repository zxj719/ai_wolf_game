## 三、本轮改动

### 改动：今日最速排名定位强化

**问题背景**

R95 下轮计划第 1 项：「今日最速排名定位强化：在日榜完成名单中，在「我的今日得分」旁加上「我的名次」——遍历日榜按 duration_s 排序找到当前角色的排名（「第 2 快」），零后端，纯前端计算。」

**设计原则**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 函数提取 | `computeDailyRank(completions, playerName)` 到 `dailyChallenge.js` | 纯函数便于单元测试，无 DOM/localStorage 依赖 |
| 排名对象 | 仅 `won && duration_s > 0` 的胜者 | 与现有 ⚡ 最速 badge 使用完全相同的过滤逻辑，保持一致 |
| 排名算法 | 按 `duration_s` 升序 `sort → findIndex` | O(n log n)，n=日参与人数（极小），足够 |
| 展示条件 | `myDailyStats.won && myDailyRank !== null` | 负者无需排名；无日榜数据时不展示避免空状态 |
| 视觉区分 | `.my-daily-rank-chip` 金色（`#fbbf24`）+ 加粗 | 与普通指标 chip 区分，强调荣誉感 |

**实现路径**

#### `src/modules/tennis/meta/dailyChallenge.js`
- 新增 `computeDailyRank(completions, playerName)` 导出函数（19 行）

#### `src/modules/tennis/components/SelectScreen.jsx`
- 导入 `computeDailyRank`
- 新增 `myDailyRank` useMemo（依赖 `dailyBoard`, `picked`）
- 在 `myDailyStats` 卡片末尾追加 `⚡ 第 {rank} 快` chip（条件渲染）

#### `src/modules/tennis/tennis.css`
- 新增 `.tennis-scope .my-daily-rank-chip` 金色样式规则

#### `src/modules/tennis/__tests__/dailyRank.test.js`（新建）

13 个测试用例覆盖：
- null/undefined completions → null
- 空/null/undefined playerName → null
- 空数组 → null
- 玩家不在列表 → null
- 玩家负场（losers 排除）→ null
- 玩家 duration_s=0（零时长排除）→ null
- 排名 1（最快胜者）
- 排名 2（第二快）
- 排名 3（最慢胜者）
- 负场者不影响计数
- 零时长胜者不影响计数
- 唯一胜者排名 1
