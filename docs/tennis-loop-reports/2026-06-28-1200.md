## 三、本轮改动

### 改动：今日最速排名定位强化

**问题背景**

R95 下轮计划第 1 项：「在日榜完成名单中，在「我的今日得分」旁加上「我的名次」——遍历日榜按 duration_s 排序找到当前角色的排名（「第 2 快」），零后端，纯前端计算。」

**设计原则**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 数据来源 | `dailyBoard.completions`（云端） | 排名需要全量数据，localStorage 只有当前设备，用云端 completions 才准确 |
| 排名逻辑 | 仅统计 `won && duration_s > 0` 的胜者 | 败者不计入速度排名；duration=0 为无效记录 |
| 排序方式 | `duration_s` 升序（越短越快） | 符合「最速」语义 |
| 显示格式 | 第 1 名：`⚡ 最速`（黄色加粗）；第 N 名：`第 N 快`（黄色） | 第 1 名区分视觉层次，对齐 dc-fastest-badge 语言风格 |
| 不显示条件 | `myDailyRank === null`（未参赛或未胜） | 仅胜者有速度排名，避免败者看到无意义信息 |

**实现路径**

#### `src/modules/tennis/meta/dailyChallenge.js`
- 新增 `computeDailyRank(playerName, completions)` 导出函数
- 参数校验：playerName 空值 / completions 非数组 / 空数组均返回 null
- 胜者排序 + findIndex 返回 1-indexed 名次

#### `src/modules/tennis/components/SelectScreen.jsx`
- 导入 `computeDailyRank`
- 新增 `myDailyRank` useMemo（依赖 `picked` 和 `dailyBoard`）
- `myDailyStats` 卡片中插入 rank chip（位置：赛果 score 之后，其他 chip 之前，优先级最高）

#### `src/modules/tennis/tennis.css`
- 新增 `.my-daily-rank-chip`：黄色边框 + 字色（`#fbbf24`）
- 新增 `.my-daily-rank-first`：黄色背景底色加深 + 字色 `#fde047` + 加粗，区分第 1 名

#### `src/modules/tennis/__tests__/dailyRank.test.js`（新建）

11 个测试用例覆盖：
- 空/null playerName → null
- null/非数组 completions → null
- 空数组 completions → null
- 玩家在败者列表中 → null
- 玩家胜但 duration_s=0 → null
- 单一胜者 → 1
- 多胜者中最快 → 1
- 第 2、3 名正确
- 败者被排除在排名外
- 玩家不在 completions 中 → null
- 仅败者时胜者排名为 1
