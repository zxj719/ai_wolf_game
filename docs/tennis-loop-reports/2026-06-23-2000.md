## 二、本轮改动

### 改动：ResultScreen「下次应对」个性化建议

**问题背景**

上上轮（第 66 轮）在赛后结算屏加入了「对手最常用招式 Top3」，每个招式卡片已标注「克：XXX」。但这是数据展示，玩家还需要自行推断「那我下次应该多练 XXX」。这一步从数据到建议的转化，可以用一句话自动完成，让信息直接变成行动指令。

**实现方式**

**ResultScreen.jsx（逻辑层）**

在已有 `topOppMoves` useMemo（第 50 行）之后，新增级联 `nextAdvice` useMemo：

```js
const nextAdvice = useMemo(() => {
  if (!topOppMoves.length) return null;
  const top1 = topOppMoves[0];
  if (top1.counter) {
    return `下次多出「${top1.counter}」来破解 ${o.name} 惯用的${top1.name}`;
  }
  return `${o.name} 本局偏爱${top1.name}，提前读招可增加克制得分`;
}, [topOppMoves, o.name]);
```

依赖链：`matchStats` → `topOppMoves` → `nextAdvice`，React 每层按引用比较，只在真正有新数据时重算。

JSX 中在 `opp-moves-row` 之后、`opp-moves-section` 内追加：

```jsx
{nextAdvice && <p className="next-advice">💡 {nextAdvice}</p>}
```

**tennis.css（样式层）**

在 `.omc-counter` 规则后新增 1 条：

```css
.tennis-scope .next-advice {
  margin: 12px 0 0;
  font-size: .78rem;
  color: var(--optic);   /* lime #d9ff3d，与 .omc-counter 色调一致 */
  font-style: italic;
  text-align: center;
  opacity: .88;
}
```

**效果示例（对战 Elza 后，top1 = 切削放缓，克制 = 上旋抽击）**

```
🔍 对手最常用招式
[✂️ 切削放缓 ×6  克：上旋抽击] [🏹 挑高球 ×4  克：高压扣杀] [🎯 放小球 ×3  克：上旋抽击]

💡 下次多出「上旋抽击」来破解 Elza 惯用的切削放缓
```

**效果示例（top1 = 高压扣杀，暂无克制关系时）**

```
💡 Elza 本局偏爱高压扣杀，提前读招可增加克制得分
```

**设计决策**

- `nextAdvice` 只依赖 `topOppMoves[0]` 和 `o.name`，不引入新数据源
- 建议使用书名号「」包裹招式名，视觉上与普通文本区分
- 无克制关系时退化为读招提示（不让空建议出现），保持信息密度
- 样式选 `var(--optic)` + `italic`：与现有克制标注色一致，区别于数据统计的橙色
