## 二、本轮改动

### 核心功能：SelectScreen 角色展开详情面板

**问题定位：**

报名处（SelectScreen）目前提供了角色名、emoji 表情和打法流派标签，但新玩家无法在选角前了解：
- 该角色具体有哪四招？
- 各招式侧重体力/技巧/心态哪个维度？
- 角色的整体属性倾向（是「力量型」还是「心态型」）？
- 该角色的专属绝技效果是什么？

已有的对手情报卡（PrepScreen OppIntelCard）在对阵阶段展示了这些信息，但当时玩家已经无法更换角色。**信息到位太晚**，导致选了不熟悉角色后才发现不适合自己的游戏风格。

对比的参照物：Adventure 模式和 Ladder 模式均有针对「对手」的情报呈现，而「自己角色」的招式信息在选角阶段却缺失。

**实现方案：**

**1. 交互模式**

点击 roster 中的任意角色 chip，在 roster 下方弹出该角色的展开详情面板（`.char-detail-panel`）。再次点击同一 chip 收起。点击不同 chip 切换。下拉框选角色时自动展开对应角色详情。

优先选用 chip 点击（而非 hover）触发，原因：
- 触屏设备无 hover 状态
- 点击有明确意图，不会在浏览时意外弹出
- 与下拉框保持双向同步，方便键鼠用户用下拉框预览

**2. 详情面板内容（三层结构）**

**层一：招式列表（char-moves）**

遍历该角色 `CHAR_BUILDS[name].moves` 的四个招式，每行展示：
- 属性徽章（体力/技巧/心态，三色标：橙/绿/蓝）
- 招式中文名
- 招式简短描述（来自 `MOVES[id].desc`）

**层二：属性倾向条（char-stat-bars）**

计算函数 `calcStatWeights(moves)`：
- 对四招求 `powerFactor` 加权和（按 stat 分 sta/skill/mind 三组）
- 归一化为百分比

举例：
```
诚（重炮流）flatDrive(sta,×1.30) + smash(sta,×1.22) + topspin(skill,×1.10) + passingShot(mind,×1.15)
→ sta: 2.52/4.77 = 53% | skill: 1.10/4.77 = 23% | mind: 1.15/4.77 = 24%
→ 视觉上明显「体力偏重」，与「重炮流」名称吻合
```

为何用 powerFactor 而非简单计招数：

仅按招数统计时，诚和 Elza 都是 sta:2 / skill:1 / mind:1 或 skill:2 / mind:2，无法区分 flatDrive（高耗高输出 ×1.30）和 slice（低耗低输出 ×0.85）的实际权重差异。powerFactor 加权后，高耗/高输出招式在条形图中占比更高，更如实反映该角色「实际战斗重心」。

**层三：绝技行（char-ult-row）**

从 `ULT_BY_OWNER`（按所有者名字索引的预构建映射）取出绝技名和描述，展示在一条带有金色「绝技」徽章的横行。

**3. 语义化升级**

原来的 chip 是 `<div>`，无法通过键盘访问。本轮改为 `<button>`，增加：
- `aria-expanded` 属性反映展开状态
- `aria-label="查看 ${name} 角色详情"`
- `:focus-visible` 轮廓（复用主题色 `--optic`）
- CSS button reset（`appearance: none`、`font-family: inherit`、`cursor: pointer`）
