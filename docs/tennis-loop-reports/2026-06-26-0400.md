## 二、本轮改动

### 改动：SelectScreen 角色出战统计小字

**背景**

`decisions-pending.md`「SelectScreen 角色解锁」条目三个方案中，方案 C（可视化出战进度，不锁定角色）属于纯增量功能（只加显示，不改游戏规则），且 `localBoard.js` 记录中已含 `r.p`（玩家身份）和 `r.sp > r.so`（胜负）字段，无需新 schema，直接实现。

**核心设计决策**

| 方案 | 选择？ | 理由 |
|------|--------|------|
| 修改 `progressStore.js` 添加 `charStats` 字段 | ❌ | 不必要：`localBoard.js` 已有完整数据，引入 schema 变更徒增合并复杂度 |
| 从 `localBoard.js` 派生（本次） | ✅ | 零新增存储，老用户历史记录无损自动显示，老记录也含 `p` 字段 |

**变更清单**

#### `localBoard.js`

新增纯函数 `computeCharStats(records)`:
```js
export function computeCharStats(records) {
  const map = {};
  for (const r of records) {
    if (!r.p) continue;
    if (!map[r.p]) map[r.p] = { played: 0, won: 0 };
    map[r.p].played++;
    if (r.sp > r.so) map[r.p].won++;
  }
  return map;
}
```
- 按玩家身份（`r.p`）聚合出战场次与胜场
- 跳过缺失 `p` 字段的历史记录（防御性处理旧数据格式）

#### `SelectScreen.jsx`

1. `import` 增加 `computeCharStats`
2. 新增 `charStatsMap` useMemo（依赖 `records`）：
   ```js
   const charStatsMap = useMemo(() => computeCharStats(records), [records]);
   ```
3. roster chip 渲染：有出战记录时在 `.nm` 下方插入：
   ```jsx
   {cs && <span className="char-stat">出战 {cs.played} · 赢 {cs.won}</span>}
   ```
   初次使用时不显示（`cs` 为 `undefined`），避免全 0 的无意义信息

#### `tennis.css`

新增一条规则（`.char-stat`）：
```css
.tennis-scope .roster .chip .char-stat {
  display: block; font-size: .62rem; color: var(--muted);
  font-family: var(--mono); margin-top: 3px; opacity: .8;
}
```
视觉语言与 `.opp-history-chip` 内小字保持一致（同 `.62rem` / `var(--muted)` / `var(--mono)`）。

#### `src/modules/tennis/__tests__/localBoard.test.js`（新建）

4 个测试用例：
- 未出战角色不出现在 map 中（空数组）
- 正确统计出战场次与胜场（3 场 2 胜）
- 平局（sp=0, so=0）不计入胜场
- 缺失 `p` 字段记录被跳过

**功能效果**

| 场景 | 显示 |
|------|------|
| 首次进入，未玩过 | 所有角色格无小字（干净） |
| 以「诚」出战过 3 场赢 2 场 | 诚格下方显示「出战 3 · 赢 2」 |
| 以某角色出战但从未赢过 | 显示「出战 N · 赢 0」 |
| 以多个角色出战 | 各角色格分别独立显示 |
