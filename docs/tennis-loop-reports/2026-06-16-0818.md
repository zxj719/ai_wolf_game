## 二、本轮改动

### 核心功能：PrepScreen 对手情报卡

**问题定位：**

PrepScreen 的「4 轮备战加点」是玩家主动决策的核心环节：每轮从三个选项选一个，各自加减 sta/skill/mind。但现有界面只展示「当前状态面板」（我方属性），玩家看不到对手的打法特征，无法做有依据的判断——比如对 Elza（控制流：切削/放小球/挑高球/截击，全走技巧+心态系），玩家若一味把点数加在 sta 上，反而会被大量打 volley 的对手频繁让体力回复从而长期吃亏。

**实现方案：**

新增 `OppIntelCard` 子组件，仅在以下条件全满足时渲染：
- `state.mode === 'single'`（单局快打：对手固定为 `state.opp`，信息准确）
- `state.opp` 非 null
- `CHAR_BUILDS[state.opp.name]` 存在（某些 Adventure 离谱对手不在配招表里，不强行渲染）

卡片内容：
- 头像 + 姓名 + 打法风格标签（复用 SelectScreen 已有的 `chip-style` 视觉语言）
- 四招列表：每招显示名称 + 属性系（`💪 体力系` / `🎯 技巧系` / `🧘 心态系`）
- 提示文案：「了解对手惯用四招，针对性强化弱项属性」

**信息价值：**

以 Elza 为例：
| 招式 | 属性系 | 加点建议 |
|------|--------|---------|
| 切削放缓 | 🎯 技巧系 | skill ↑ 才能应对 |
| 放小球 | 🧘 心态系 | mind ↑ 才能应对 |
| 挑高球 | 🧘 心态系 | mind ↑ 才能应对 |
| 网前截击 | 🎯 技巧系 | skill ↑ 才能应对 |

玩家看到这张卡，立刻明白「对 Elza 要堆技巧+心态，别浪费点在体力」。

**ladder/adventure 不展示的理由：**
- ladder 模式：`state.opp` 是 SelectScreen 随机分配的「宿敌」，但 LadderScreen 自管一套 5 连战对手队列，PrepScreen 里的 `state.opp` 不是第一个 ladder 对手
- adventure 模式：同理，实际对手是 AdventureScreen 内部按章节路径选的离谱对手，`state.opp` 没有指向意义

**CSS（26 行，tennis.css 新增）：**
- `.opp-intel-card`：间距
- `.opp-intel-header`：flex + display font
- `.opp-intel-style`：仿 chip-style 的小标签（optic 色调）
- `.opp-move-list`：auto-fit 网格
- `.opp-move-row`：每招小块（轻灰底圆角）
- `.opp-move-name`/`.opp-move-stat`：字体层次
