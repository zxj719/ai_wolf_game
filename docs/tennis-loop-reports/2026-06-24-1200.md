## 二、本轮改动

### 改动：LadderScreen 第 1 站赛前预告

**问题背景**

Round 70 在 between（赛间）状态加入了对手配招风格预告，覆盖了第 2-6 站。但第 1 站（`ladder.stage === 0`）直接从 SelectScreen 进入 fighting 状态，没有 between 中间屏，玩家在进场前无法提前了解第一位对手的招式风格。

这造成了信息不对等：第 2-6 站玩家都有「⚔️ 控制流：切削放缓 / ...」预告，唯独最关键的第 1 站（初来乍到，最需要引导的时刻）没有这条信息。

**实现方式**

**LadderScreen.jsx（逻辑层）**

在 `fighting` 状态的 `ladder-bar` 闭合标签之后、`BattleScreen` 之前，插入一个条件渲染 `<p>`：

```jsx
{ladder.stage === 0 && CHAR_BUILDS[opponent.name] && (
  <p className="hint opp-style-preview">
    ⚔️ {CHAR_BUILDS[opponent.name].style}：{CHAR_BUILDS[opponent.name].moves.map(m => MOVES[m]?.name ?? m).join(' / ')}
  </p>
)}
```

- `ladder.stage === 0`：仅在第 1 站显示（第 2-6 站由 between 屏的 `opp-style-preview` 覆盖，不重复）
- `CHAR_BUILDS[opponent.name] &&`：防御性 guard，避免未来添加未注册对手时报错
- 逻辑数据全部来自已有 `CHAR_BUILDS`（`LadderScreen.jsx` 顶部已导入），零额外 import

**tennis.css（样式层）**

新增 1 条 CSS 相邻兄弟选择器规则，覆盖 between 屏语境的负 margin：

```css
.tennis-scope .ladder-bar + .opp-style-preview { margin: 4px 0 2px; text-align: center; }
```

- `.ladder-bar + .opp-style-preview`：精确匹配「紧跟在 ladder-bar 之后的 opp-style-preview」——即第 1 站 fighting 状态的情形
- between 屏中 `.opp-style-preview` 位于 `.card > p` 结构里，不触发此规则，原有 `margin: -8px 0 10px` 不受影响
- `text-align: center`：在横向布局的 fighting 屏中对齐正文中心

**效果示例（第 1 站，对手为诚）**

```
[○  ●  ○  ○  ○  ○]  第 1 / 6 站
⚔️ 重炮流：重炮平击 / 高压扣杀 / 上旋抽击 / 穿越球
[BattleScreen 正文...]
```

**效果示例（第 2 站，赛间 between 屏）**

```
🔮 下一站：🦊 Elza，历史 5 战 2 胜 3 负
⚔️ 控制流：切削放缓 / 放小球 / 挑高球 / 网前截击  （原有，between 屏）
💡 胜率 40%，特训或逛店补装备皆可
```

两处预告格式完全一致，信息结构统一。

**覆盖范围完成情况**

| 站次 | 预告来源 | 状态 |
|------|---------|------|
| 第 1 站（stage=0） | fighting 状态 `opp-style-preview`（本轮新增） | ✅ |
| 第 2-6 站 | between 状态 `opp-style-preview`（Round 70） | ✅ |
