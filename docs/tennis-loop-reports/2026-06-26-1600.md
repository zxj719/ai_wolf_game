## 二、本轮改动

### 改动：每日挑战 UI 集成（SelectScreen + TennisRoute）

**背景**

`meta/dailyChallenge.js` 于更早轮次实现（含 `getDailyChallenge` / `isDailyChallengeCompleted` / `markDailyChallengeCompleted` / `DAILY_BONUS_COINS=30`），并在 `meta/__tests__/dailyChallenge.test.js` 中有完整的 4 个测试用例。但该模块从未被 SelectScreen 或 TennisRoute 引入——日常挑战功能对用户完全不可见。

**核心设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 触发时机 | 选手身份选定后展示 | `getDailyChallenge(playerName)` 需要 playerName 参数（用于排除自身） |
| 对手信息 | 面部 emoji + 姓名 + 三维属性 | 让用户做赛前心理预期 |
| 已完成状态 | 变灰 + 提示「明天再来」 | 不隐藏，保留今日战报上下文 |
| 奖励方案 | 仅胜利后 +DAILY_BONUS_COINS 金币 | 失败不奖励，保持挑战感 |
| 后端依赖 | 无（localStorage） | 快速实现，不阻塞部署 |
| isDaily 跟踪 | `isDailyRef` (useRef) | 只需在事件处理函数中读取，不驱动 UI，无需 useState |

**变更详情**

#### `SelectScreen.jsx`

```jsx
// 新增 import
import { getDailyChallenge, isDailyChallengeCompleted, DAILY_BONUS_COINS } from '../meta/dailyChallenge';

// 新增 props
export function SelectScreen({ onStart, onStartDaily, toast, boardProps, equipment = {} })

// 新增 useMemo / computed
const dailyChallenge = useMemo(() => (picked ? getDailyChallenge(picked) : null), [picked]);
const dailyDone = picked ? isDailyChallengeCompleted() : false;
```

新增「今日一战」横幅（在 roster 下方、`opp-progress` 上方）：

| 状态 | 显示 |
|------|------|
| 未选身份 | 不显示（`dailyChallenge` 为 null） |
| 选了身份 + 未完成 | 对手 emoji + 姓名 + 属性 + 「今日一战 +30💰」按钮 |
| 选了身份 + 已完成 | 同上但变灰，按钮替换为「明天再来挑战！🎾」文字 |

#### `TennisRoute.jsx`

```jsx
// 新增 import
import { getDailyChallenge, markDailyChallengeCompleted, DAILY_BONUS_COINS } from './meta/dailyChallenge';

// 新增 ref
const isDailyRef = useRef(false);

// 新增 callback
const onStartDaily = useCallback((playerName) => {
  const dc = getDailyChallenge(playerName);
  isDailyRef.current = true;
  dispatch({ type: 'START', playerName, oppName: dc.foe.n, oppStats: {...} });
  toast(`⚡ 今日一战！宿敌：${dc.foe.f} ${dc.foe.n}　胜利可得 +${DAILY_BONUS_COINS}💰！`);
}, [dispatch, toast]);
```

在 `onSingleMatchOver` 结算处新增日常奖励逻辑：

```js
const dailyBonus = (isDailyRef.current && win) ? DAILY_BONUS_COINS : 0;
if (isDailyRef.current && win) {
  markDailyChallengeCompleted();
  isDailyRef.current = false;
}
updateProgress({ ...progress, coins: progress.coins + coins + soldFor + dailyBonus, ... });
const dailyTag = dailyBonus > 0 ? `　⚡今日一战 +${dailyBonus}💰` : '';
toast(`🎁 掉落：...${dailyTag}`);
```

还在普通 `onStart` 中加 `isDailyRef.current = false` 重置，防止状态污染。

#### `tennis.css`

新增 14 行规则（`.daily-banner` / `.daily-header` / `.daily-face` / `.daily-body` / `.btn-daily` 等），复用 `--optic` / `--clay` / `--muted` 等 CSS 变量，风格与现有卡片一致。

**功能效果**

```
用户流程：
  选身份「诚」→ 横幅出现：「⚡ 今日一战 · 2026-06-26」
                          🏆 莹 · 体力65 · 技巧58 · 心理72
                          [今日一战 +30💰 →]
  点击 → toast：「⚡ 今日一战！宿敌：🎀 莹　胜利可得 +30💰！」
  进入正常游戏流程（模式选择 → 反应测试 → 备战 → 对战）
  赢了 → toast：「🎁 掉落：稀有拍框 +35💰　⚡今日一战 +30💰」
  今日完成 → 横幅变灰，按钮替换为「明天再来挑战！🎾」
```
