## 三、本轮改动

### Bug：OppCoachHint / CrisisHint「幽灵克制建议」

**根本原因**

`OppCoachHint` 和 `CrisisHint` 的克制提示逻辑：
```
对手频繁使用 X → 查 COUNTER_FOR[X] → 建议出 Y
```
但 COUNTER_FOR 是全局克制表（基于完整 COUNTER_PAIRS），不知道玩家当前角色的配招集。在均势对阵（莹 vs Ross）中：

| 对手（Ross）惯用招 | 全局克制招 | 莹的配招 | 结论 |
|------|------|------|------|
| flatDrive | dropShot | topspin/slice/lob/passingShot | ❌ 莹没有 dropShot |
| slice | flatDrive | topspin/slice/lob/passingShot | ❌ 莹没有 flatDrive |
| smash | 无 | — | null（正确）|
| passingShot | 无 | — | null（正确）|

当 Ross 频繁使用 flatDrive 时，原代码会说「出「放小球」可克制」——这个招式玩家根本打不出来，会让玩家困惑地在手牌里寻找一张不存在的招。

**修复方案**

```jsx
// OppCoachHint 新增 playerMoves prop
const canCounter = playerMoves?.includes(counter);
if (!canCounter) {
  // 均势提示（绿色⚡）
  return <div className="bt-coach-hint bt-coach-balanced"> ... 均势——读招节省体力 </div>;
}
// 原有克制提示（琥珀🎯）
return <div className="bt-coach-hint"> ... 出「X」可克制 </div>;
```

```jsx
// CrisisHint 更新：counter && playerMoves?.includes(counter)
const canCounter = counter && playerMoves?.includes(counter);
{canCounter ? <>，出「{X}」可克制</> : '，注意读招节省体力'}
```

**颜色系统一致性**

本轮延续 R117-R118 建立的视觉语义：
- 🎯 琥珀黄 = 有直接克制，有具体行动
- ⚡ 绿色 = 均势/无直接克制，提示宏观策略（读招节省体力）

这套颜色系统现在贯穿整个 matchup 信息链：
```
SelectScreen （R117 克制黄 / R118 均势绿）
    ↓
PrepScreen   （R115 克制黄框 / R118 均势绿框）
    ↓
BattleScreen rally 1    （R116 FirstMatchHint 克制绿框）
    ↓
BattleScreen 非关键分   （本轮 OppCoachHint 克制黄 / 均势绿）  ← 修复
    ↓
BattleScreen 关键分     （本轮 CrisisHint 克制黄 / 均势文本）   ← 修复
    ↓
ResultScreen 复盘       （R85 已有）
```
