## 二、本轮改动

### 改动 1：绝技图鉴全收集提示（TennisRoute.jsx）

**问题背景**

当玩家集齐所有 7 枚绝技时，界面上没有任何庆祝反馈，`📖 绝技图鉴 7/7` 这个数字显得很静默，与「收集驱动」的玩法设计不符。

**方案**

在 `ult-chips` div 之后、`ult-gallery` 关闭之前，条件性地插入一行 `.rule-strip` 横幅（该样式已全局定义，此前用于成就徽章行）：

```jsx
{unlockedUltNames.size === Object.keys(ULTIMATES).length && (
  <div className="rule-strip" style={{ marginTop: 12, marginBottom: 0 }}>
    🏆 绝技图鉴已完成收集！七大绝技尽在掌握！
  </div>
)}
```

**设计决策**：
- 用 `Object.keys(ULTIMATES).length` 而非硬编码 7，对未来扩充角色时更健壮
- 复用 `.rule-strip`（成就徽章行样式），零 CSS 新增，视觉一致性好
- 仅在 mode 页面可见（`state.screen === 'mode'` 时渲染），属于「里程碑时刻」出现的庆祝横幅

### 改动 2：PrepScreen 赛前威胁台词（PrepScreen.jsx）

**问题背景**

备战的最后一回合（第 4 回合）是进入正式对决前的最后准备，目前只有静态的「临赛压力」文案。对手角色是已知的，但在备战阶段完全「缺席」，少了家庭对抗的喜剧感和叙事张力。

**方案**

在第 4 备战回合的 `.opts` 选项列表上方，展示对手从 `CHAR_QUOTES.victory` 池中随机取出的「赛前嘚瑟」台词（反向用 victory 台词——对手赛前就嘚瑟了，增加喜剧效果）：

**`PrepScreen.jsx`（+14 行）**：
```jsx
// 导入变更
import { useMemo, useRef, useState } from 'react';
import { CHAR_QUOTES, FXNAME, PREP, rand } from '../gameData';

// 组件内（round 声明后）
const preMatchQuote = useMemo(() => {
  if (state.prepRound !== PREP.length - 1 || !state.opp) return null;
  const pool = CHAR_QUOTES[state.opp.name]?.victory;
  return pool ? pool[rand(0, pool.length - 1)] : null;
}, [state.prepRound, state.opp?.name]);

// JSX（opts div 之前）
{preMatchQuote && (
  <div className="char-quote-card">
    <span className="char-quote-who">{state.opp.face} {state.opp.name} 赛前嘚瑟：</span>
    <p className="char-quote-text">「{preMatchQuote}」</p>
  </div>
)}
```

**设计决策**：
- `useMemo` 依赖 `[state.prepRound, state.opp?.name]`——当 prepRound 切换到 3（第 4 回合）时锁定一条随机台词，后续 re-render 不重新随机，避免「抖动」
- 直接复用 `tennis.css` 已有的 `.char-quote-card` / `.char-quote-who` / `.char-quote-text` 样式（第 53 轮 ResultScreen 评语功能已定义），零 CSS 新增
- `CHAR_QUOTES[opp.name]?.` 可选链对未知角色名安全降级
- 不用 toast：toast 在 2200ms 后消失，而选项卡上方的内联显示在整个第 4 回合期间持续可见，更有利于玩家反应并产生「我要在 ta 嘚瑟前赢回来！」的情绪驱动
