## 三、本轮改动

### 功能：「复仇提示」横幅

**背景与动机**

SelectScreen 已经有丰富的对手面板信息（胜率、走势点、连胜徽章、最高连胜、上次对战日期）。但这些信息都是被动展示——玩家需要主动展开面板才能看到，而且没有任何信息在「当下」主动推动玩家做出选择。

在游戏化设计中，**「未了的心结」**（unfinished business）是驱动回流的强力动机之一。当玩家在 7 天内败给某对手且还没有复仇时，心理上存在一个待解决的张力。「复仇提示横幅」将这个张力外显化，把它变成一个看得见的行动号召。

**设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 触发条件 | 最近 7 天内有败绩 + 此后未复仇 | 7 天内的败绩玩家还有印象；太久前的视为「已翻篇」 |
| 数据来源 | `localBoard.js`（localStorage，带 ts 字段）| 无需后端；R92+ 记录均有 ts |
| 返回策略 | 最近一条未复仇败绩（不是"最多"而是"最新"）| 一次只给一个目标，避免信息过载 |
| 与日榜去重 | 若复仇目标 = 今日一战对手，不显示横幅 | 避免日榜横幅和复仇横幅同时针对同一对手 |
| 依赖 `picked` | 玩家未选身份时不显示 | 复仇是「角色视角」的记忆，需要先确定身份 |
| 视觉风格 | 红色调（`rgb(252,165,165)`）+ 呼吸光晕 | 与日榜（amber/橙）、全家制霸（金色）视觉区分；红色→危机感/未了心结 |
| 措辞 | 「😤 [对手] 还欠你一局！X天前输的，该复仇了！」| 拟人化、家庭游戏语气；「欠你」强化主动权感 |

**展示场景**

| 场景 | 结果 |
|------|------|
| 未选身份 | 不显示 |
| 没有 7 天内的败绩 | 不显示 |
| 7 天内败给铁蛋，此后未赢过铁蛋 | 显示「😤 🤖 铁蛋 还欠你一局！3天前输的，该复仇了！」|
| 7 天内败给铁蛋，但之后赢过一次 | 不显示（已复仇）|
| 复仇目标恰好是今日日榜对手 | 不显示（避免重复） |
| 今天输的 | 「😤 ... 今天输的，该复仇了！」|
| 昨天输的 | 「😤 ... 昨天输的，该复仇了！」|

## 四、实现路径

### `src/modules/tennis/localBoard.js`

新增 `findRevengeOpportunity(records, playerName, maxDays = 7)` 纯函数：

```js
export function findRevengeOpportunity(records, playerName, maxDays = 7) {
  const mine = records.filter((r) => r.p === playerName);
  const cutoff = Date.now() - maxDays * 86400000;
  for (let i = mine.length - 1; i >= 0; i--) {
    const r = mine[i];
    if (!r.ts || r.ts < cutoff) break;
    if (r.sp > r.so) continue;
    const oppName = r.o;
    const beatenSince = mine.slice(i + 1).some(
      (lr) => lr.o === oppName && lr.sp > lr.so,
    );
    if (!beatenSince) {
      return { name: oppName, face: r.of ?? '', daysAgo: Math.floor((Date.now() - r.ts) / 86400000) };
    }
  }
  return null;
}
```

### `src/modules/tennis/__tests__/localBoard.test.js`

- 导入 `findRevengeOpportunity`
- 新增 7 个测试用例（`describe('findRevengeOpportunity')`）：
  - 空记录 → null
  - 只有胜场 → null
  - 7天内败给对手且未复仇 → 返回该对手（含 daysAgo 验证）
  - 败后已复仇 → null
  - 超过 maxDays 的败绩忽略 → null
  - 无 ts 字段的记录忽略 → null
  - 有两条未复仇败绩，返回最新一条
  - 其他玩家记录不干扰

### `src/modules/tennis/components/SelectScreen.jsx`

- 导入 `findRevengeOpportunity`
- 新增 `revengeOpp` useMemo（依赖 `records`、`picked`、`dailyChallenge`）
- 在日榜横幅之后、`opp-progress` 之前，条件渲染 `.revenge-banner`：

```jsx

    <span className="revenge-text">
      {revengeOpp.face} <strong>{revengeOpp.name}</strong> 还欠你一局！

```
