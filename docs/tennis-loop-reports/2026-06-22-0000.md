## 二、本轮改动

### 改动 A：ResultScreen 连胜火焰分级

**问题背景**

上轮的连胜徽章显示 `🔥 本次连胜 N 场`，无论连胜 2 场还是 10 场，都是同一个火焰。玩家无法从视觉上感受到连胜递进带来的奖励感爬升。

**方案**

用三元表达式在同一个 `ladder-streak` 徽章内切换图标数量：

```jsx
// Before
<span className="ladder-streak">🔥 本次连胜 {winStreak} 场</span>

// After
<span className="ladder-streak">
  {winStreak >= 5 ? '🔥🔥🔥' : winStreak >= 3 ? '🔥🔥' : '🔥'} 本次连胜 {winStreak} 场
</span>
```

分级规则：
- 连胜 2 场：🔥（入门激励）
- 连胜 3-4 场：🔥🔥（明显升级感）
- 连胜 ≥ 5 场：🔥🔥🔥（顶级连胜）

---

### 改动 B：SelectScreen NEW 徽章 + unseen 芯片重设计

**问题背景**

上轮加入对手历史面板后，未挑战角色被 `opacity: .3 + filter: grayscale(1)` 处理成灰色，传达的是"缺失/不可用"感。但 CSS `opacity` 向子元素继承，任何子元素 badge 也会被等比灰化，无法在其中放置清晰可见的"NEW"标记。

**方案**

将 `.opp-history-chip.unseen` 从"灰化"改为"轻橙调背景"，去掉 opacity 级联问题：

```css
/* Before */
.tennis-scope .opp-history-chip.unseen { opacity: .3; filter: grayscale(1); }

/* After */
.tennis-scope .opp-history-chip.unseen { background: rgba(226,129,79,.07); }
.tennis-scope .opp-history-new { color: var(--clay); }
```

JSX 层：将原来"只给 seen 显示 ✓"改为"seen 显示 ✓、unseen 显示橙色 NEW"：

```jsx
{seenOpps.has(c.n) ? (
  <span className="opp-history-badge">✓</span>
) : (
  <span className="opp-history-badge opp-history-new">NEW</span>
)}
```

**设计决策**：
- 移除 grayscale/opacity 后，所有 7 位角色芯片都完整可见，用 seen（绿调+✓）vs unseen（橙调+NEW）区分状态
- 这将"集邮缺口"从 negativity bias（灰暗=少了什么）翻转为 approach motivation（NEW=有机会）
- `var(--clay)` = `#e2814f`，与整体赛场橙调一致，不引入新色彩
