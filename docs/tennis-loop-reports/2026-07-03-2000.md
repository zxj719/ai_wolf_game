## 三、本轮改动

### 功能：冒险模式第四章「传说殿堂」

**背景与动机**

R126 完成了成就解锁即时通知，奖励反馈维度已达满分（10.0）。下一个高价值改进方向是**内容层扩充**——增加 Chapter 4，让通关 3 章的玩家有新挑战可以追求，直接提升趣味性和长线粘度。

家庭网球公开赛场景中，「第一次打通 4 章」是一个天然的家庭分享节点：玩家会告诉其他家人「我打到传说殿堂了！」这比 UI 层改进更能驱动口碑传播。

**设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 章节主题 | 传说殿堂（legend） | Ch1 市井、Ch2 仙界、Ch3 科幻，Ch4 用「传说」收束，给人终章感 |
| 常规对手 | 传说幽灵（timeScale:0.6） | timeScale 比外卖小哥（0.7）更极端，小游戏时限 −40%；无需新 twist 代码 |
| BOSS | 永恒球王（forcedMindDuel:2 + bossPhaseSwap） | 每两球心态对决（比修仙童子的 3 更频繁）+ 三套配招轮换；双 twist 叠加制造传说级压迫感 |
| 章节上限修复 | `Math.min(chapterIdx, run.map.chapters.length - 1)` | 消除硬编码 `2`，未来再加章节无需重改 |
| 事件池大小 | 6 条（2 小游戏 + 4 剧情） | 每步事件池不少于 3 条即可避免重复；Ch4 池不低于 Ch1/2/3 单类事件数 |
| boss 判定 | `ci === CHAPTERS.length - 1` | 表驱动，新增章节自动把 boss 移到最后一章 |

**对现有章节的影响**

- `网球之神`（Chapter 3 精英）：从 `boss:true`（地图节点）降为普通精英节点，但其对手对象本身不改变，`bossPhaseSwap` twist 仍生效；
- 之前通关 3 章的玩家存档（sessionStorage）不受影响，因为存档里的 `run.map` 已包含 3 章节的完整地图；
- 新开局才会生成 4 章地图。
