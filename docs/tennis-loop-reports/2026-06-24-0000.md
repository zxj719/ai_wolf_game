## 二、本轮改动

### 改动：BattleScreen 关键时刻读招提示（`CrisisHint`）

**问题背景**

从第 66 轮起，`battleReducer` 在每球 RESOLVE 时累计 `oppMoveUsage`（对手各招式使用次数），这份数据在赛后结算屏（第 66 轮 Top3 可视化）和个性化建议（第 68 轮「下次应对」）中已被利用。但 `oppMoveUsage` 最高价值的利用时机其实是**对局内部**：金球/盘点等高压节点，玩家需要在最短时间内做出出招决策。这正是读招数据应该出现的地方。

**实现方式**

**BattleScreen.jsx（逻辑层）**

模块顶层新增 `COUNTER_FOR` 映射（仅在模块加载时运算一次）：

```js
const COUNTER_FOR = {};
Object.keys(COUNTER_QUIPS).forEach(key => {
  const [a, b] = key.split('>');
  if (!COUNTER_FOR[b]) COUNTER_FOR[b] = a;
});
```

新增 `CrisisHint` 纯展示组件（在 `BattleScreen` 函数之前）：

```jsx
function CrisisHint({ matchStats, score, phase }) {
  if (phase !== 'cards' || !isKeyPoint(score)) return null;
  const usage = matchStats?.oppMoveUsage ?? {};
  const top = Object.entries(usage).sort((a, b) => b[1] - a[1])[0];
  if (!top || top[1] < 2) return null;
  const [moveId, count] = top;
  const counter = COUNTER_FOR[moveId];
  return (
    <div className="bt-crisis-hint">
      <span className="bt-crisis-icon">⚡</span>
      <span className="bt-crisis-text">
        关键分！对手惯用「{MOVES[moveId]?.name}」(×{count})
        {counter ? <>，出「{MOVES[counter]?.name}」可克制</> : '，注意读招'}
      </span>
    </div>
  );
}
```

`cards` 阶段 JSX 中，在绝技横幅（`bt-ult-armed`）之后、手牌（`HandCards`）之前插入：

```jsx
<CrisisHint matchStats={state.matchStats} score={state.score} phase={state.phase} />
```

**显示条件设计（三层过滤）**

1. `phase !== 'cards'`：不在出招阶段不显示（防止遮挡小游戏/日志）
2. `!isKeyPoint(score)`：非关键分不显示（普通局点信息密度已够，不要干扰节奏）
3. `top[1] < 2`：对手该招式少于 2 次不显示（避免基于单次出招的「假模式」误导）

**效果示例（对战 Elza，她已用切削放缓 3 次，此球为盘点）**

```
⚡ 关键分！对手惯用「切削放缓」(×3)，出「重炮平击」可克制
```

**效果示例（top1 为高压扣杀，无直接克制对应出招时）**

```
⚡ 关键分！对手惯用「高压扣杀」(×2)，注意读招
```

**tennis.css（样式层）**

在 `.bt-ult-armed` 之后新增 3 条规则：

```css
.tennis-scope .bt-crisis-hint {
  display: flex; align-items: center; gap: 8px;
  padding: 7px 10px; margin-bottom: 10px; border-radius: 8px;
  background: rgba(217,255,61,.08); border: 1px solid rgba(217,255,61,.28);
}
.tennis-scope .bt-crisis-icon { font-size: 1rem; flex-shrink: 0; }
.tennis-scope .bt-crisis-text { font-size: .72rem; color: var(--optic); font-weight: 600; line-height: 1.4; }
```

颜色使用 `var(--optic)`（lime #d9ff3d）与结算屏的「下次应对」建议颜色一致，形成视觉语言的统一——绿色系 = 战略建议/读招提示，橙色系 = 警告/消耗提示。
