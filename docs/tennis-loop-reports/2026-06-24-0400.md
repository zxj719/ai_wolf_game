## 二、本轮改动

### 改动：LadderScreen 赛间「对手配招风格」一行预告

**问题背景**

家族挑战模式（Ladder）每站连战，第 1 站没有「赛前」屏直接进入战斗，第 2–6 站在赛间休息屏有一定决策时间。赛间屏（`between` 状态）已有：
- 下一站对手名字 + 历史胜率
- AI 建议（特训/逛店/按摩的推荐文案）
- 三选一按钮动态提示

但玩家在进场前不知道下一位对手惯用什么招式，只能等第一球出现后开始读招。对「读招」是关键机制的游戏来说，进场前建立心理模型可以显著提升头几球的决策质量。

**实现方式**

**LadderScreen.jsx（逻辑层）**

新增 `MOVES` 到已有 import（原来只有 `CHAR_BUILDS, ULTIMATES`）：

```js
import { CHAR_BUILDS, ULTIMATES, MOVES } from '../battle/moves';
```

在 between 状态的 `nextOppForBetween` 块中，于历史胜率段落之后、`between-suggest` 建议之前插入：

```jsx
{CHAR_BUILDS[nextOppForBetween.name] && (
  <p className="hint opp-style-preview">
    ⚔️ {CHAR_BUILDS[nextOppForBetween.name].style}：{CHAR_BUILDS[nextOppForBetween.name].moves.map(m => MOVES[m]?.name ?? m).join(' / ')}
  </p>
)}
```

`CHAR_BUILDS[name].moves` 是 4 个招式 ID 数组，`.map(m => MOVES[m]?.name ?? m)` 做安全查找并 fallback 到 ID，`.join(' / ')` 生成可读字符串。

**效果示例（第 2 站赛间，下一站为 Elza）**

```
🔮 下一站：🦊 Elza，历史 5 战 2 胜 3 负
⚔️ 控制流：切削放缓 / 放小球 / 挑高球 / 网前截击
💡 胜率 40%，特训或逛店补装备皆可
```

**效果示例（下一站为 诚，首次对阵）**

```
🔮 下一站：🐯 诚，首次对阵
⚔️ 重炮流：重炮平击 / 高压扣杀 / 上旋抽击 / 穿越球
💡 首次对阵，建议加练摸清套路
```

**tennis.css（样式层）**

在 `.between-suggest` 之前新增 1 条规则：

```css
.tennis-scope .opp-style-preview { font-size: .78rem; font-family: var(--mono); color: var(--muted); margin: -8px 0 10px; opacity: .9; }
```

选 `var(--mono)` 字体：招式名称是技术术语，mono 字体在视觉上与普通提示文本区分，也与 BattleScreen 的 `bt-crisis-text` 形成整体风格统一。负 margin-top `-8px` 贴近上方段落，不显得孤立。

**设计决策**

- **不加克制提示**：只列配招，不在这里加「X 克制 Y」—— 那个工作已由 CrisisHint（关键时刻）和 nextAdvice（赛后建议）完成，赛间屏堆砌信息反而降低阅读效率
- **覆盖范围**：第 2–6 站（5/6 站）。第 1 站直接进入战斗，无 between 状态；如需覆盖第 1 站，需在 `fighting` 状态加「pre-match 子状态」，改动量更大，留作后续评估
- **防御性 guard**：`CHAR_BUILDS[nextOppForBetween.name] &&` 防止未来新增对手但未加入 CHAR_BUILDS 时报错
