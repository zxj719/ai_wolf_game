## 二、本轮改动

### 核心功能：Adventure 最深章节纪录

**问题定位：**

奇幻闯关（Adventure）已有 `adventureClears` 追踪通关次数，但没有追踪"历史最深到达第几章"——失败玩家（占大多数）无法看到自己的历史最佳进度，失去了"再突破一关"的具体目标。这与上轮为 Ladder 实现的 `ladderBestStreak` 是对称的痛点。

**实现方案：**

**1. 数据层（progressStore.js）**

新增 `adventureBestFloor: 0` 字段到 `EMPTY_PROGRESS`，mergeProgress 按「取大」策略合并：
```javascript
adventureBestFloor: Math.max(a.adventureBestFloor ?? 0, b.adventureBestFloor ?? 0)
```
语义：0 = 从未到达，1/2 = 止步第 1/2 章，3 = 全关通关。
`chapterIdx`（0-indexed）+1 = 章节数（1-indexed），因此 `chapterIdx + 1` 即为到达的章节数。

**2. 纪录更新时机（AdventureScreen.jsx）**

两条互斥终态路径各守一个 settled ref：

```javascript
// 通关路径（原有 settledRef）
adventureBestFloor: Math.max(progress.adventureBestFloor ?? 0, 3)

// 失败路径（新增 failedSettledRef）
useEffect(() => {
  if (run.phase !== 'failed' || failedSettledRef.current) return;
  failedSettledRef.current = true;
  onUpdateProgress({
    ...progress,
    adventureBestFloor: Math.max(progress.adventureBestFloor ?? 0, run.chapterIdx + 1),
  });
}, [run.phase]);
```

`settledRef` / `failedSettledRef` 防 React StrictMode 双调用，保证进度只写入一次。

**3. 展示（AdventureScreen.jsx + tennis.css）**

*选路（`pick`）阶段横幅*（仅当有历史记录时）：
```
📊 历史最深  第 N 章  · 通关 X 次  ‥‥‥ 超越自己！
```
- 位置：地图步骤可视化（`adv-map`）之下，Boss 预警之上
- 颜色：冷光绿（与 Ladder 横幅一致的 `rgba(217,255,61,.08)` 背景）
- 「超越自己！」斜体浅灰，不抢焦点

*结算屏*（`won`/`failed` 状态，新增 `adv-run-record` 行）：
- 通关：「🏆 第 N 次通关」+ 若首次则「🎉 历史新纪录！」
- 失败：「本轮止步第 N 章」+ 「🎉 历史新纪录！」或「历史最深第 N 章」

**4. 新纪录判断（`preRunFloorRef`）**

在组件挂载时捕获赛前纪录：
```javascript
const preRunFloorRef = useRef(progress.adventureBestFloor ?? 0);
```
由于 progress 在 effects 异步更新后才变化，而 ref 保持 mount 时的值，因此结算屏渲染时 `preRunFloorRef.current < currentFloor` 的判断始终准确。

**5. 环境预存问题修复**

`postcss-selector-parser@6.1.4` 中 `dist/util/maxNestingDepth.js` 在此远程执行环境下缺失（npm 缓存局部损坏），导致构建失败。修复方式：从 tarball 手动提取并放入 node_modules：
```bash
cd /tmp && npm pack postcss-selector-parser@6.1.4
tar -xzf postcss-selector-parser-6.1.4.tgz package/dist/util/maxNestingDepth.js
cp /tmp/package/dist/util/maxNestingDepth.js node_modules/postcss-selector-parser/dist/util/
```
此为环境问题，非代码问题，后续轮次若重新 `npm install` 可能会重现。
