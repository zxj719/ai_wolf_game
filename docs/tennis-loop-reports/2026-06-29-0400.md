## 三、本轮改动

### 改动：对手走势点（opp-history-chip）

**问题背景**

R98 下轮计划第 1 项：「在「已挑战 N/7 位家人」展开面板中，每个对手 chip 上也加类似的走势点，但从玩家视角展示「我对 ta 最近几场的结果」」。当前 opp-chip 已展示整体胜率（wins/total）和推荐/劲敌标签，但无法反映「最近手感」——玩家可能总胜率 80%，但最近 3 场连输给 ta，就需要走势点来快速感知并激发复仇动力。

**设计原则**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 过滤维度 | `p === playerName && o === oppName` 双重过滤 | 只统计「当前选定身份」对「该对手」的记录，而非所有玩家总和 |
| 走势窗口 | 最近 3 场（默认 n=3） | 与 roster chip 一致，3 点紧凑、足够反映短期手感 |
| 显示时机 | `picked` 已选时才计算 `oppTrendMap` | 未选身份时对手趋势无意义（不知道「我」是谁） |
| 点大小 | 5px（vs roster 6px） | opp chip 比 roster chip 更窄，5px 更协调 |
| 与 roster 命名隔离 | `.opp-trend` / `.ot-dot` / `.ot-w` / `.ot-l` | 避免复用 `.char-trend` 类时产生 CSS scope 冲突 |
| n=0 边界 | 继承 `computeOppRecentResults` 的 `if (n <= 0) return []` | 与 `computeRecentResults` 同源保护 |

**实现路径**

#### `src/modules/tennis/localBoard.js`
- 新增 `computeOppRecentResults(records, playerName, oppName, n=3)` 导出函数
- 双维度过滤：`r.p === playerName && r.o === oppName`
- `.slice(-n)` → `.map(r => r.sp > r.so)` 返回 boolean[]，旧→新顺序
- 边界：`n <= 0` 立即返回 `[]`

#### `src/modules/tennis/components/SelectScreen.jsx`
- 导入 `computeOppRecentResults`
- 新增 `oppTrendMap` useMemo（依赖 `records` + `picked`）：当 `picked` 为空时返回 `{}`；否则遍历 CHARS，为每个对手存最近 3 局结果
- opp-history-chip 内 `opp-wr` span 之后插入 `.opp-trend` 行（条件渲染：有记录才显示）

#### `src/modules/tennis/tennis.css`
- 新增 `.opp-trend`：`display: flex; justify-content: center; gap: 3px; margin-top: 4px`
- 新增 `.ot-dot`：5×5px 圆形 `flex-shrink: 0`
- 新增 `.ot-w`：绿色 `#4caf76`
- 新增 `.ot-l`：半透明红 `rgba(248,113,113,.55)`

#### `src/modules/tennis/__tests__/localBoard.test.js`
- 导入 `computeOppRecentResults`
- 新增 9 个测试用例（`describe('computeOppRecentResults')`）：
  - 空记录 → []
  - 玩家名不匹配 → []
  - 对手名不匹配 → []
  - 单场胜利 → [true]
  - 单场失败 → [false]
  - 3 场（旧→新顺序）
  - 超 3 场只取最近 3 条
  - 混杂其他玩家/对手时只统计指定组合
  - n=0 → []
  - n=1 → 仅最后 1 条

**展示路径**

| 场景 | 结果 |
|------|------|
| `picked` 未选 | `oppTrendMap = {}`，不渲染任何对手走势点 |
| `picked` 已选，该对手有 ≥1 条对战记录 | 显示最近 min(played_vs_opp, 3) 枚点 |
| `picked` 已选，该对手无记录 | 不显示 `.opp-trend`（oppTrendMap 无该键） |
