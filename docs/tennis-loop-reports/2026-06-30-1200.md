## 三、本轮改动

### 功能：ResultScreen「对手属性速览」

**背景与动机**

结算屏目前展示顺序为：
1. ending-hero 卡（比分 + 连胜/失败徽章 + 赛后评语 + 按钮）
2. oppQuote 卡（对手台词）
3. matchStats 卡（总球数/克制/关键分/招式分析）
4. 赛后战报 卡（双方体力/技巧/心态详细对比）
5. FeedbackWidget
6. Leaderboard

玩家赛后第一眼看到的是「大比分 X:Y」，但无法立即知道「我输了是因为对手体力差距？还是技巧？」，需要滚动经过 matchStats 才到赛后战报。

「就近反馈原则」（proximate feedback）：反馈信息应出现在玩家最需要它的时机和位置。将对手属性摘要提升到比分行正下方，让玩家在看到「0:2」的瞬间就能看到「对手 体力 82 · 技巧 75 · 心态 69」，立即形成因果归因，无需滚动。

**设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 位置 | `.score-line` 正下方（ending-hero 内） | 与比分在同一视觉组，关联强；赛后战报仍保留，不删 |
| 内容 | `{face} {name} 属性：体力 N · 技巧 N · 心态 N` | 最少信息量，一行完整表达；不重复「宿敌」等已有标注 |
| 样式 | monospace 字体，0.68rem，--muted 色，0.82 透明度 | 属于「上下文元信息」层，不抢主标题（比分、连胜）视线 |
| 赛后战报保留 | ✅ 原有详细对比不删除 | 速览 ≠ 替代；详细对比仍有价值（含赛后状态文案） |
| 是否标注「最强属性」 | ❌ 本轮不做 | 保持极简；属性数字已足够自明；避免「标注逻辑」引入新 useMemo |

**视觉效果（示例）**

```
                    🏆
                  完美发挥！
        大比分 2 : 0（2:0 / 2:1）
        🤖 铁蛋 属性：体力 72 · 技巧 65 · 心态 55
  ...（连胜徽章、结局评语、按钮）
```
