## 三、本轮改动

### 功能：SelectScreen「对阵相性速查」（matchup-hint）

**背景与动机**

R113 报告的下轮计划：
> SelectScreen「对阵相性速查」：在 opp-history-chip 右侧增加一个极小「克」徽章，鼠标悬停/点击时展开"你最佳克制招：{招式}"提示（基于玩家当前角色的配招 vs 对手配招交集计算）

现有信息路径：
| 时机 | 读招信息来源 |
|------|------------|
| 对局中输球时 | RallyLog「💡 对手招被克制招克制」 |
| 对局非关键分 cards 阶段 | OppCoachHint「🎯 对手惯用X，出Y可克制」（R113）|
| 对局关键分 cards 阶段 | CrisisHint 告警（R113 之前）|
| **选对手时** | ❌ **无任何信息**（本轮填补此空白）|

「选对手」是最早的决策点——玩家在此时知道应对思路，能带着策略进场，而不是在对局中出错后才看到提示。这与 OppCoachHint 的「对局中实时教练」形成时间轴互补：

```
选对手时：「出重炮平击克 ta 的切削放缓」  ← matchup-hint（本轮）
进入对局：「对手惯用切削（×3），出重炮可克制」← OppCoachHint（R113）
输球后：  「切削放缓被重炮平击克制」        ← RallyLog（R86 基础）
```

**算法设计**

每个角色有 4 招配招集（`CHAR_BUILDS`），克制关系由 9 个克制对（`COUNTER_PAIRS`）定义。

对于玩家角色 P（4 招）vs 对手角色 O（4 招）：
- 遍历 COUNTER_PAIRS（9 对），找第一个满足「P 有 counter 且 O 有 victim」的对
- 存储 counter 的 moveId，即「P 应该出的招」

复杂度：O(4 × 9) = O(36) per opponent，7 个对手总计 O(252)，全为静态常量查表，useMemo 缓存后仅在 `picked` 变化时重算。

**覆盖率分析（7 角色 × 6 对手）**

| 玩家 | vs 对手 | 推荐克制招 |
|------|---------|-----------|
| 诚（重炮平击、高压扣杀、上旋抽击、穿越球） | Elza（切削放缓）| 重炮平击（克切削）|
| 诚 | 菲比（网前截击）| 上旋抽击（... 依克制对顺序）|
| Elza（切削放缓、放小球、挑高球、网前截击）| 诚（重炮平击）| 放小球（克重炮）|
| ... | ... | ... |

由于 COUNTER_PAIRS 覆盖 8 种招式中的 9 个克制对，绝大多数配招组合至少有一个克制交集。极少数情况无交集时，matchupMap[oppName] 为 undefined，不渲染任何标签（不显示误导信息）。

**设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 显示范围 | 已挑战 + 未挑战对手均显示 | 未挑战对手更需要备战信息 |
| 图标 | 🎯（与 OppCoachHint 一致）| 「瞄准」语义；色系统一 |
| 颜色 | 琥珀金 `rgba(251,191,36,.78)` | 项目主色（amber-600）；区别于淡青色建议（opp-best-streak/OppCoachHint）|
| 字号 | `.52rem` | 最小信息层（比 opp-last-date 0.48rem 稍大，是有意义的策略提示而非纯元信息）|
| 触发条件 | `picked && matchupMap[c.n]` | 未选身份时不显示（无玩家角色无法计算）|
| 显示位置 | 在 seen/unseen 分支之后 | 同等对待两类对手，避免代码重复 |

### `src/modules/tennis/components/SelectScreen.jsx`

**新增 import**：
```js
import { CHAR_BUILDS, COUNTER_PAIRS, MOVES } from '../battle/moves';
```

**新增 `matchupMap` useMemo**（插在 `sortedOppChars` useMemo 之前）：
```js
const matchupMap = useMemo(() => {
  if (!picked) return {};
  const playerMoves = CHAR_BUILDS[picked]?.moves ?? [];
  const map = {};
  for (const c of CHARS) {
    if (c.n === picked) continue;
    const oppMoves = CHAR_BUILDS[c.n]?.moves ?? [];
    for (const [counter, victim] of COUNTER_PAIRS) {
      if (playerMoves.includes(counter) && oppMoves.includes(victim)) {
        map[c.n] = counter;
        break;
      }
    }
  }
  return map;
}, [picked]);
```

**新增 matchup-hint 渲染**（在 seen/unseen 分支之后、chip `</div>` 之前）：
```jsx
{picked && matchupMap[c.n] && (
  <span className="matchup-hint" aria-label={`出${MOVES[matchupMap[c.n]].name}可克制对方`}>
    🎯 {MOVES[matchupMap[c.n]].name}
  </span>
)}
```
