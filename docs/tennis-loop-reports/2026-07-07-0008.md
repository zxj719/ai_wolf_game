## 四、本轮改动

### 功能：盘胜庆祝横幅（`SetWinBanner`）

**背景与动机**

对战界面现有的关键时刻专属反馈：

| 时刻 | 反馈 |
|------|------|
| ACE | 金色 goldflash + 「ACE！🎾」 |
| 鹰眼改判 | 「🦅 鹰眼改判！」burst |
| 克制得分 | 绿色 greenflash + 「效果拔群！🎯」 |
| 被克制失分 | 红色 redflash + 「被克制……」 |
| CLUTCH | 金色 burst + 「⚡ CLUTCH！⚡」 |
| 绝技发动 | 金色 burst + 技能名 |
| **赢盘** | ~~无~~（本轮补全）|

赢盘是比赛进程中最重要的里程碑之一，但在此前版本中仅通过计分板的 `SETS 1-0` 数字反映。对家庭玩家来说，这个时刻缺少庆祝感，显得平淡。

**实现细节**

- **触发条件**：`state.score.sets[0]`（玩家盘分）增加且 `!state.score.matchOver`
  - 在 best-of-3 规则下，此条件仅在玩家赢下第一盘（非赛点）时触发
  - 赢下第二盘时 `matchOver=true`，进入 ResultScreen 处理，不触发此横幅
- **时间同步**：横幅显示 1400ms，与 idle → BEGIN_RALLY 的延迟相同，确保横幅消失时发球阶段恰好开始，不会污染 serve/cards 阶段 UI
- **ref 去重**：使用 `prevPlayerSets.current` ref 追踪上次盘分，避免组件重渲染时重复触发

```jsx
const prevPlayerSets = useRef(0);
const [setWinBanner, setSetWinBanner] = useState(null);
useEffect(() => {
  const cur = state.score.sets[0];
  if (cur > prevPlayerSets.current && !state.score.matchOver) {
    prevPlayerSets.current = cur;
    setSetWinBanner({ num: cur, key: Date.now() });
    const t = setTimeout(() => setSetWinBanner(null), 1400);
    return () => clearTimeout(t);
  }
  prevPlayerSets.current = cur;
  return undefined;
}, [state.score.sets[0], state.score.matchOver]);
```

**渲染与样式**

```jsx
{setWinBanner && (
  <div className="bt-set-win-overlay" key={setWinBanner.key} aria-live="assertive">
    <div className="bt-set-win-text">
      🎾 先下一城！
      <span className="bt-set-win-sub">第一盘胜</span>
    </div>
  </div>
)}
```

- `z-index: 86`（比 fx-greenflash/redflash 的 79 高，比 streak-burst-overlay 的 9998 低）
- `@keyframes bt-set-win-pop`：淡入 → 弹出 → 消散，与现有 fx 动效视觉语言一致
- 绿色 `#4ade80` 配色（与 counter-win / fx-greenflash 同系，表示正面结果）
- `aria-live="assertive"` 无障碍支持
- `prefers-reduced-motion`：压制动效，保留可见性
