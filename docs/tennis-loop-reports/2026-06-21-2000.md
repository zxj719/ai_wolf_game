## 二、本轮改动

### 改动：SelectScreen 对手历史展开面板（SelectScreen.jsx + tennis.css）

**问题背景**

上轮的角色收集进度条（`已挑战 N/7 位家人`）只显示数字，玩家无法直观知道「还差哪几位」——需要靠记忆或逐一对战才能知道。这减少了「集邮型」收集动机的有效性。

**方案**

将进度条的标签文字升级为可点击的 `button`（`aria-expanded` 语义），点击展开一个 7 格角色网格：

```jsx
// SelectScreen.jsx 新增 state
const [showOppHistory, setShowOppHistory] = useState(false);

// 进度条 + 展开面板结构
<button
  type="button"
  className="opp-progress-toggle"
  onClick={() => setShowOppHistory((v) => !v)}
  aria-expanded={showOppHistory}
>
  <span>已挑战 {seenCount}/{totalOpps} 位家人 …</span>
  <span className="opp-progress-chevron">{showOppHistory ? '▲' : '▼'}</span>
</button>
<div className="opp-progress-bar">…</div>
{showOppHistory && (
  <div className="opp-history-panel">
    {CHARS.map((c) => (
      <div key={c.n} className={`opp-history-chip${seenOpps.has(c.n) ? ' seen' : ' unseen'}`}>
        <span className="opp-history-face">{c.f}</span>
        <span className="opp-history-name">{c.n}</span>
        {seenOpps.has(c.n) && <span className="opp-history-badge">✓</span>}
      </div>
    ))}
  </div>
)}
```

**CSS 新增（9 行）：**
```css
.opp-progress-toggle    // flex 行，width:100%，button reset
.opp-progress-chevron   // 右侧折叠指示器 ▲▼
.opp-history-panel      // grid repeat(7,1fr)，7 格均分
.opp-history-chip       // 基础芯片（flex 纵向居中）
.opp-history-chip.seen  // 已挑战：背景高亮
.opp-history-chip.unseen // 未挑战：opacity 0.3 + grayscale
.opp-history-face       // 大号表情 1.25rem
.opp-history-name       // 等宽小字 0.58rem，溢出 ellipsis
.opp-history-badge      // 右上角 ✓ 绿色徽章
```

**设计决策**：
- `seenOpps`（Set）基于 localStorage 记录的 `.o`（对手名字字段），O(1) 查找
- `seenCount > 0` 条件渲染保持不变：新玩家不展示（无历史无意义）
- `CHARS.map` 遍历保持角色顺序稳定（不受玩家对战顺序影响）
- 默认收起（`showOppHistory = false`）：不干扰新老玩家初始视线
- 已挑战者 `opacity 1 + 荧光绿背景`，未挑战者 `opacity 0.3 + grayscale(1)`，对比鲜明
- 保留旧 `.opp-progress-label` CSS（死规则无害，避免影响其他可能引用处）
