## 二、本轮改动

### 核心功能：成就补全提示行

**问题定位：**

模式选择页的成就图鉴已有 16 个 chip（已解锁亮色/未解锁 🔒 灰显），但玩家看到灰色后无法快速判断"该做什么来解锁它"——tooltip 只在 PC hover 可见，移动端完全无效。缺乏主动引导会导致"目标迷失"：玩家知道有成就可刷，却不知道当前可以优先追哪个。

**实现方案（2 文件 / ~15 行）：**

**1. TennisRoute.jsx（+11 行）**
- 新增 `useMemo` import
- 添加 `achHint = useMemo(...)` 计算 hook：
  - 过滤出 `ACHIEVEMENTS` 中未在 `progress.achievements` 的条目
  - 随机取一条（`Math.floor(Math.random() * locked.length)`）
  - 全部解锁时返回 `null`（提示行自动隐藏）
  - dependency: `progress.achievements.length`（只在成就数量变化时重新抽签，避免每次渲染都换）
- 在 `ach-gallery` 之后渲染 `{achHint && <p className="ach-hint-unlock">...}`

**2. tennis.css（+8 行）**
- `.ach-hint-unlock`：flex wrap 行，`.78rem`，muted 色，`-6px 0 14px` margin 紧贴 gallery
- `.ach-hint-tip`：`#60a5fa` 蓝色 + 斜体（与 OppIntelCard 克制提示视觉语言一致）

**显示效果示例：**
```
还差 11 项成就未解锁  💡 试试：🌊「势如破竹」—— 单场连续赢球 5 次以上
```

**设计决策：**

| 决策 | 选择 | 理由 |
|------|------|------|
| 每次渲染随机 vs useMemo | useMemo（成就变化时重抽） | 稳定提示，让玩家能"认准目标"，避免刷屏感 |
| 显示全部未解锁 vs 随机一条 | 随机一条 | 避免信息过载；每局后重进可能看到新提示，增加探索感 |
| 全部解锁时 | 完全隐藏（`achHint === null`） | 奖励满贯玩家：图鉴区域干净，无冗余"还差 0 项"文字 |
