## 二、本轮改动详情

### 发现问题

检查代码时发现优先级(1)(2)(3b)的实现文件早已存在于磁盘，但均处于"孤儿"状态：
- `OnboardingModal.jsx` — 无任何父组件 import
- `FeedbackWidget.jsx` — 无任何父组件 import，且 `sendMatchFeedback` 在 tennisService.js 中不存在
- `noviceTracker.js` — 无任何调用方

**影响：用户根本看不到新手引导、赛后评价和新手保护。**

### 修复：7 个文件变更

#### 1. workers/auth/tennis.js（+48 行）

新增两个 handler：

**`handleTennisFeedback`**（POST /api/tennis/feedback）：
- 严格白名单校验：rating 1-5、comment ≤200字、mode/character/result 枚举限制
- 写入已存在的 `tennis_feedback` 表（migration 009 已上线）

**`handleTennisFeedbackSummary`**（GET /api/tennis/feedback/summary）：
- 聚合：总条数、平均分、好评数、差评数
- 最近 10 条有文字评价（供评估循环读取）

#### 2. workers/auth/index.js（+8 行）

添加两个路由：`POST /api/tennis/feedback` 和 `GET /api/tennis/feedback/summary`

#### 3. src/services/tennisService.js（+12 行）

新增 `sendMatchFeedback`，fire-and-forget 模式（与 `sendMatchTelemetry` 相同设计）

#### 4. src/modules/tennis/TennisRoute.jsx（+20 行）

- import `OnboardingModal` + `checkOnboardingSeen`
- import `isNovice` + `incrementNoviceGames` + `NOVICE_STAT_PENALTY`
- 新增 state：`showOnboarding`（首次进入自动弹出）
- 修改 `onStart`：新手前 3 场对手属性 -15（Math.max(20, ...)防止下溢），并显示「新手保护中」提示
- 修改 `onSingleMatchOver`：每场结束后 `incrementNoviceGames()`
- mode 页标题区新增 "?" 帮助按钮，可随时重打开引导
- render 末尾渲染 `<OnboardingModal />`

#### 5. src/modules/tennis/components/ResultScreen.jsx（+4 行）

在战报和排行榜之间插入 `<FeedbackWidget mode="single" character={} result={} />`

#### 6. src/modules/tennis/modes/LadderScreen.jsx（+8 行）

- import `FeedbackWidget` + `incrementNoviceGames`
- `handleMatchOver` 中调用 `incrementNoviceGames()`
- won/lost 结算卡后插入 `<FeedbackWidget mode="ladder" />`

#### 7. src/modules/tennis/tennis.css（+120 行）

新增 `ob-*`（引导浮层）和 `fb-*`（评价控件）全套样式，全部在 `.tennis-scope` 命名空间内，包含：
- 浮层蒙层 + 模态框（flex 布局，max-height 90vh 滚动）
- 三 tab 切换（克制表 / 体力规则 / 读招技巧）
- 移动端友好：响应式排布
- 五星评价按钮动效（hover scale 1.15）
- 文字评价输入框（focus 高亮边框）
