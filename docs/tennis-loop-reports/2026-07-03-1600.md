## 三、本轮改动

### 功能：结算屏「成就解锁通知」横幅（achievement-unlock-banner）

**背景与动机**

`meta/achievements.js` 定义了 12 个里程碑成就（首胜、ACE 大师、关键先生、S 级天赋、传说装备、球王……）。`TennisRoute.jsx` 的 `onSingleMatchOver` 在每局结束时会自动把新解锁成就写入 `progress.achievements`，但玩家完全感知不到：
- 一局打完，金币/装备 toast 出现了
- 成就悄悄进了 mode 页的小芯片列表
- **没有任何即时的「你做到了」时刻**

这是体验链里最后一段无反馈空白：玩家可能玩了十场才偶然注意到 mode 页多了一枚成就徽章。对家庭游戏场景来说，这种里程碑时刻本该是呼朋引伴的社交节点。

**设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 位置 | 结算屏最顶部（StreakBurst 之后，ending-hero 之前） | 第一眼看到，未被内容淹没 |
| 外观 | 金色 `rgba(251,191,36)` 边框/背景/文字 | 与 mode 页成就芯片色彩系统一致；金色=稀有感 |
| 动效 | `ach-pop-in` scale 0.88→1 + opacity 0→1，0.45s cubic-bezier(.17,.67,.35,1.3) | 有弹性感，活泼但不打扰；与 StreakBurst 大动效层级区分 |
| 内容 | 成就图标（大号） + 名称（金色加粗） + 描述（小字淡色） | 三层信息：感官冲击→命名→理解条件 |
| 多成就 | 同局可能解锁多个（如首局赢 + 3 ACE）→ 列表展开，每项独立行 | 每个成就都值得被看见；不合并 |
| 无新成就 | 组件 return null，零渲染开销 | 不影响老玩家的默认结算屏 |
| 重置时机 | `setNewAchievements(newlyUnlocked)` 每次 `onSingleMatchOver` 覆盖写 | 新一局赢了无成就 → 自然变空，不会展示上局的 |

**新增 `AchievementUnlockBanner` 组件（约 20 行）：**
