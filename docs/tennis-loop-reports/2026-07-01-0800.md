## 三、本轮改动

### 功能：ResultScreen「备战成长对比」（prep-growth-hint）

**背景与动机**

R111 在 PrepScreen 首轮显示「上次备战 Elza：体力 65 · 技巧 72 · 心态 58」，让玩家在备战前有历史参考。但这个信息只出现在开始备战时——对局结束后，玩家回到 ResultScreen 时并不知道**本次备战相比上次有多少提升或退步**。

「备战成长对比」填补这一空缺：玩家在 ResultScreen 立刻看到：

```
对手属性：🦊Elza  体力 72 · 技巧 78 · 心态 65
较上次备战 Elza：体力 +5  ·  技巧 +3  ·  心态 +7
```

绿色正号代表进步，红色负号代表退步，`0` 显示为灰色。这建立了「备战→对局→复盘→再备战」的完整认知循环：

```
PrepScreen：「上次我练到了 65/72/58」（起点参考）
玩家重新备战，得到 70/75/65
ResultScreen：「这次比上次好 +5/+3/+7」（成长反馈）
```

**显示条件**：仅在玩家对同一对手至少经历过两次完整备战周期时出现（即 `loadPrevPrepStats` 返回非 null 时），新玩家或首次对战某对手不显示，避免"没有基准时的无效信息"。

**设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 位置 | `opp-stats-hint` 正下方，`ending-hero` card 内 | 紧接在「对手属性」后，玩家已知道对手强度，再看「我的准备提升了多少」信息流自然 |
| 字号 | `.66rem`（略小于 `opp-stats-hint` 的 `.68rem`）| 属于"辅助元信息"层，不与主标题竞争视觉权重 |
| 颜色编码 | 绿色 `#86efac` / 红色 `#f87171` / 灰色 muted | 复用 opp-tag 色系（推荐绿/劲敌红），色彩语义一致 |
| 中性值（0）| 灰色 `pg-eq`，无 + 号 | `+0` 视觉上显眼但语义空洞，灰色 `0` 更符合「持平无变化」|
| 存储粒度 | 保留最近两条（current + previous）| 一层 previous 足以"上次 vs 这次"对比；不需要完整历史序列，节省 localStorage |
| 向后兼容 | `existing.current ?? existing` | R111 存的旧格式 `{ sta, skill, mind }` 在首次新保存时自动迁移为 previous，无数据断层 |

**视觉效果（ResultScreen 顶部 ending-hero 卡片）**

```
🏆 完美制霸
大比分 2 : 0（6-3 / 6-2）
🦊 Elza 属性：体力 72 · 技巧 78 · 心态 65
较上次备战 Elza：体力 [绿+5] · 技巧 [绿+3] · 心态 [绿+7]   ← 本轮新增
🔥 本次连胜 3 场
...
```

若本次备战退步（如玩家选了更省力的方案）：

```
较上次备战 Ross：体力 [红-8] · 技巧 [灰 0] · 心态 [绿+2]
```

## 四、实现路径

### `src/modules/tennis/localBoard.js`

**`savePrepHistory` 升级**（双条格式）：保存前先读旧值，将其存为 `previous`，新值存为 `current`。兼容旧格式判断：`existing.current ?? existing`（旧格式无 `.current` 字段，直接作为 previous 使用）。

```js
export function savePrepHistory(playerName, oppName, { sta, skill, mind }) {
  const h = loadPrepHistoryStore();
  if (!h[playerName]) h[playerName] = {};
  const existing = h[playerName][oppName];
  const prevCurrent = existing ? (existing.current ?? existing) : null;
  h[playerName][oppName] = { current: { sta, skill, mind }, previous: prevCurrent };
  try { store.setItem(PREP_HISTORY_KEY, JSON.stringify(h)); } catch { /* 隐私模式 */ }
}
```

**`loadLastPrepStats` 更新**（向后兼容）：
```js
export function loadLastPrepStats(playerName, oppName) {
  const entry = h[playerName]?.[oppName] ?? null;
  if (!entry) return null;
  return entry.current ?? entry; // 新格式取 current；旧格式直接返回
}
```

**新增 `loadPrevPrepStats`**：
```js
export function loadPrevPrepStats(playerName, oppName) {
  const entry = h[playerName]?.[oppName] ?? null;
  return entry?.previous ?? null;
}
```

### `src/modules/tennis/__tests__/localBoard.test.js`

- 导入新增的 `loadPrevPrepStats`
- 更新 `'多次保存后只保留最新一条（覆盖写）'` 描述为 `'多次保存后 loadLastPrepStats 返回最新一条'`（行为不变，描述更准确）
- 新增 `describe('loadPrevPrepStats')` 包含 6 个用例：
  1. 无记录 → null
  2. 仅一次保存 → null（无 previous）
  3. 两次保存后返回第一次数据
  4. 三次保存后 previous 只保留第二次（not 第一次）
  5. 不同玩家互不干扰
  6. 未保存的组合返回 null

### `src/modules/tennis/components/ResultScreen.jsx`

- 导入 `loadPrevPrepStats`
- 新增 `prepGrowthData` useMemo（依赖 `[]`）：调用 `loadPrevPrepStats(p.name, o.name)`，若无数据返回 null，否则返回 `{ dSta, dSkill, dMind }`
- 在 `opp-stats-hint` div 之后、`winStreak` 徽章之前，插入条件渲染 JSX

### `src/modules/tennis/tennis.css`

在 `.opp-stats-hint` 之后新增 4 条规则：

```css
.tennis-scope .ending-hero .prep-growth-hint { font-family: var(--mono); font-size: .66rem; color: var(--muted); margin-top: 4px; text-align: center; letter-spacing: .02em; }
.tennis-scope .pg-up { color: #86efac; font-weight: 700; }
.tennis-scope .pg-down { color: #f87171; font-weight: 700; }
.tennis-scope .pg-eq { color: var(--muted); }
```
