## 本轮主要任务：读招提示 UI 增强

### 问题分析

R134 发现 `energySaver` vs `tellReader` 胜率差距巨大，是最高杠杆改进方向：

| 水平 | energySaver | tellReader | 差距 |
|------|-------------|------------|------|
| noob | 4.3% | 77.8% | +73.5pp |
| avg | 53.8% | 100.0% | +46.2pp |
| pro | 94.8% | 100.0% | +5.2pp |

玩家如果完全忽视读招提示，就只能靠运气（random 9%）或节能策略（4.3%），而使用读招策略的 noob 胜率直接达到 77.8%。

**核心问题**：原 `bt-tell-counter` 是 `#1e4d38`（深绿色）0.70rem 小字，在黄绿色背景上几乎不可见，玩家容易忽略。

### 修复：三处代码改动

#### 1. `tennis.css` — `bt-tell-counter` 样式升级

**之前**：
```css
.tennis-scope .bt-tell .bt-tell-counter {
  display: block; font-size: .70rem; font-style: normal;
  color: #1e4d38; opacity: .90; margin-top: 4px;
}
```

**之后**：
```css
.tennis-scope .bt-tell .bt-tell-counter {
  display: flex; align-items: center; gap: 4px;
  font-size: .78rem; font-style: normal; font-weight: 700;
  color: #7c1d0c; margin-top: 8px; padding: 4px 10px;
  background: rgba(255, 255, 255, 0.60); border-radius: 6px;
  border-left: 3px solid #c0510a;
}
```

视觉变化：
- 字号：0.70rem → 0.78rem（更易读）
- 颜色：深绿 `#1e4d38` → 橙红 `#7c1d0c`（高对比度，与黄绿色背景对比鲜明）
- 加粗：正常 → font-weight: 700
- 背景：无 → 60% 不透明白色（图底分离）
- 左边框：无 → 橙色 3px（行动指示器模式）

#### 2. `BattleScreen.jsx` — 前缀从 `💡` 改为 `↳`

**之前**：`💡 出「X」可克制`
**之后**：`↳ 出「X」克制！`

`↳` 是「从上方信息推导出的行动」的通用 UI 符号，比 💡（灯泡=提示）更直接传达「按这个做」语义。感叹号强调紧迫性（透视出招时尤为关键）。

#### 3. `OnboardingModal.jsx` — 默认标签页改为「读招技巧」

**之前**：`useState('counter')`
**之后**：`useState('tell')`

原「克制表」虽是好参考资料，但对新玩家来说最高杠杆行动是：**看到提示就选克制招**。将「🔍 读招技巧」设为首显标签：
- 最大化新玩家首次打开弹窗就看到最重要信息的概率
- 「克制表」仍可通过标签访问，不移除内容

### 力竭球占比分析

**关键发现**：力竭球占比高 ≠ 体验差

| 策略 | 水平 | 力竭占比 | 胜率 | 解读 |
|------|------|---------|------|------|
| tellReader | avg | 80.1% | 100% | 高力竭下仍全胜 → 系统正常 |
| tellReader | pro | 72.6% | 100% | pro 用能力更快赢，力竭更少 |
| energySaver | avg | 86.4% | 53.8% | 越省能量 → 拖得越久 → 力竭反而更多 |
| powerSpam | avg | 91.0% | 56.5% | 高耗体招 → 最快耗尽 |

**结论**：力竭比例高是局时长度的函数，不是平衡问题。energySaver 反而比 tellReader 力竭更多，因为它拖长了对局时间。切削（负耗体）是对抗力竭的正确工具，游戏已在 OnboardingModal 体力规则 tab 说明。
