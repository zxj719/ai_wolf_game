## 二、本轮改动

### 核心功能：PrepScreen 绝技效果提示行

**问题定位：**

备战阶段绝技 `<select>` 中每条 option 的文字格式为 `{face} {name} — {desc}`，例如 `🐯 虎啸正手 — 本球自动克中对手，威力 1.5×`。当玩家解锁多张绝技时需要在此切换出战绝技。

移动端原生 picker（iOS/Android）在折叠态只显示当前选中项的文字，且会截断长 option text；在展开 picker 时，系统原生列表里的每条 option 文字虽完整，但字体小、无强调，认知负担高。PC 浏览器的折叠 `<select>` 同样在 input 框内截断长文字。

**实现方案（2 文件 / 5 行净增）：**

**1. PrepScreen.jsx（+4 行）**

在 `ultimateOptions.length > 1` 块的 `</select>` 后、`</div>` 前增加 IIFE 渲染：

```jsx
{(() => {
  const ult = ultimateOptions.find((u) => u.name === equippedUltimate);
  return ult ? <p className="ult-desc-tip">{ult.face} {ult.desc}</p> : null;
})()}
```

- `equippedUltimate` 随 select onChange 更新，提示行实时刷新
- 当 `equippedUltimate` 为 null 时 `.find()` 返回 undefined，条件渲染安全兜底

**2. tennis.css（+1 行）**

```css
.tennis-scope .ult-desc-tip { width: 100%; margin: 4px 0 0; font-size: .8rem; color: #fbbf24; font-style: italic; }
```

- `width: 100%` 利用父容器 `flex-wrap: wrap` 自然换行，不需要改布局结构
- `#fbbf24`（amber-400）呼应绝技的 ⚡ 图标与战斗感，与成就提示行的蓝色（`#93c5fd`）形成色彩区分

**效果示例（玩家已解锁多张绝技时）：**

```
⚡ 出战绝技  [🐯 虎啸正手 — 本球自动克中对手，威力 1.5×  ▼]
           🐯 本球自动克中对手，威力 1.5×
```

切换 select 后立即刷新为新选中绝技的描述。

**设计决策：**

| 决策 | 选择 | 理由 |
|------|------|------|
| IIFE vs 抽取变量 | IIFE 在 JSX 内 | 逻辑仅此一处，无需提取组件/变量 |
| 颜色选择 | amber-400 (`#fbbf24`) | ⚡ 绝技主题色，与蓝色成就提示形成区分 |
| 是否对 `length===1` 也显示 | 否 | `length===1` 时 select 不显示，说明玩家只有本命绝技，无选择成本，提示行无必要 |
| 是否去掉 option 内的 desc | 否 | option 内描述在 PC 展开 dropdown 时有用，保留无副作用 |
