## 二、本轮改动

### 核心功能：Ladder 连胜纪录横幅

**问题定位：**

家族挑战（Ladder）已有 `championships` 字段记录球王加冕次数，但缺少「历史最佳单轮战绩」的可见化。玩家无法知道自己历史最多连赢过几站，多周目动力不足——尤其是未完全通关的玩家，他们努力达到的「止步第 4 站」是一个值得记住并超越的成就。

**实现方案：**

**1. 数据层（progressStore.js）**

新增 `ladderBestStreak: 0` 字段到 `EMPTY_PROGRESS`，mergeProgress 按「取大」策略合并：
```javascript
ladderBestStreak: Math.max(a.ladderBestStreak ?? 0, b.ladderBestStreak ?? 0)
```
与既有 `championships`、`adventureClears` 等标量字段保持一致的合并语义。

**2. 纪录更新时机（LadderScreen.jsx）**

两条互斥路径分别写入：
- **失败路径**（`handleMatchOver` 中 `!win`）：`ladderBestStreak = max(current, ladder.stage)`
  - `ladder.stage` 此时 = 失败站编号 = 已赢得站数
- **球王路径**（`won` useEffect）：`ladderBestStreak = max(current, STAGE_COUNT)` = 6
  - 避免在 `handleMatchOver` 胜利路径重复写入（6 站通关必经 won useEffect）

**3. 展示（LadderScreen.jsx + tennis.css）**

*`between` 赛间屏*（仅当有历史记录时）：
```
📊 历史最佳 · {N} 连站 · 加冕 X 次  ‥‥‥ 超越自己！
```
- 颜色：冷光绿（`rgba(217,255,61,.08)` 背景，纪录数字 `rgba(217,255,61,.9)`），与赛程进度条主题一致
- 「超越自己！」斜体浅灰，不抢视觉焦点

*`won`/`lost` 结算屏*（新增 `ladder-run-record` 行）：
- 球王：「🏆 第 N 次加冕」+ 若首次称王则「✨ 首次称王！」
- 止步（赢过 ≥1 站）：「本轮连赢 X 站」+ 「🎉 个人新纪录！」或「历史最佳 Y 站」

**4. 新纪录判断（`preRunStreakRef`）**

在组件挂载时捕获赛前纪录：
```javascript
const preRunStreakRef = useRef(progress.ladderBestStreak ?? 0);
```
由于 `progress` 在 effects 后更新（parent state），结算屏渲染时 `progress.ladderBestStreak` 已是新值，而 `preRunStreakRef.current` 始终保留赛前基准，因此对比正确。
