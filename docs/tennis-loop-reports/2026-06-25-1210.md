## 二、本轮改动

### 改动：BattleScreen 新增 `DefensePressure` 实时攻防芯片

**问题背景**

Round 77 在 ResultScreen 加入了「被克失分」计数，但这是**赛后**统计——玩家只有在整场结束后才看到自己被克了几次。而真正影响当局决策的窗口是**出招前**：如果玩家知道自己已经被克了 3 次，他更可能主动换招。这是「被克失分」数据的最高价值使用点，却一直没有展示。

**设计决策**

- 位置：能量条（`bt-energies`）与赛场主卡（`bt-arena-card`）之间，不进入已信息密集的赛场卡，也不打扰发球/出招阶段的交互
- 隐藏策略：`countersWon === 0 && counterLost === 0` 时返回 `null`，首球前不占位，视觉零侵入
- 颜色分档（`counterLost`）：
  - 1 次：低对比度灰白（`rgba(242,238,224,.5)`）—— 一次可能是意外
  - 2-3 次（`.warn`）：橙色（`#d9814f`）—— 提醒换招
  - 4+ 次（`.danger`）：深红（`var(--clay)`）—— 防守崩溃警告
- 「克中」永远绿色（`#4caf76`）—— 正向激励，不分档

**实现**

`BattleScreen.jsx` 新增 `DefensePressure` 函数组件（14 行）：
```jsx
function DefensePressure({ countersWon, counterLost }) {
  if (!countersWon && !counterLost) return null;
  const lostTier = counterLost >= 4 ? 'danger' : counterLost >= 2 ? 'warn' : '';
  return (
    <div className="bt-defense-pressure">
      {counterLost > 0 && (
        <span className={`bt-dp-chip lost ${lostTier}`}>🛡 被克 {counterLost}</span>
      )}
      {countersWon > 0 && (
        <span className="bt-dp-chip won">⚔️ 克中 {countersWon}</span>
      )}
    </div>
  );
}
```

插入点（能量条下方 6 行）：
```jsx
<DefensePressure
  countersWon={state.matchStats.countersWon}
  counterLost={state.matchStats.counterLost ?? 0}
/>
```

`tennis.css` 新增 7 行样式（`.bt-defense-pressure` + `.bt-dp-chip` 四态）。

**三模式覆盖**

因为 `BattleScreen` 是共享组件，梯子赛、冒险模式、单场赛三种模式均自动受益，无需额外接入。这也解决了 Round 77 遗留验证项「AdventureScreen matchStats 传递」——冒险模式的 BattleScreen 实例同样可以实时显示攻防压力（即使冒险模式没有独立的 ResultScreen）。
