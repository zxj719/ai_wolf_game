## 二、本轮改动

### ✅ battleReducer.js — matchStats 扩展

新增 5 个追踪字段：

| 字段 | 类型 | 含义 |
|------|------|------|
| `curConsecAces` | number | 当前连续 ACE 数（非 ACE 发球时清零） |
| `maxConsecAces` | number | 本场最大连续 ACE 数 |
| `curWinStreak` | number | 当前连胜球数 |
| `longestWinStreak` | number | 本场最长连胜球数 |
| `maxMgMult` | number | 本场最佳小游戏倍率 |

更新逻辑：
- `SERVE_DONE` ACE：`curConsecAces + 1`，更新 `maxConsecAces`；win streak + 1
- `SERVE_DONE` 非 ACE：重置 `curConsecAces = 0`
- `MINIGAME_DONE`：`maxMgMult = Math.max(maxMgMult, m)`
- `RESOLVE`：win 时 `curWinStreak + 1`，失分时重置为 0，同步更新 `longestWinStreak`

### ✅ BattleScreen.jsx — FxOverlay 连续 ACE 特效

```
当 curConsecAces ≥ 3 且比上帧增加：
  → 显示 "🔥 连ACE ×N！" 橙焰特效（比普通 ACE 更大更长）
  → 同时更新 prevAces.current，防止同帧再触发普通 ACE 动画
```

依赖数组新增 `state.matchStats.curConsecAces`。

### ✅ ResultScreen.jsx — 本场亮点卡

新增 `HighlightsCard` 子组件：
- ACE 数（≥3 显示 "ACE Master" 徽章）
- 最长连ACE（≥3 显示 "发球炮台" 徽章）
- CLUTCH 成功（≥2 显示 "压哨王" 徽章）
- 最长连胜球
- 最佳操作倍率（≥1.4 显示 "Pro 操作" 徽章）
- 平均操作倍率

位置：战报卡片 → **亮点卡** → 历史战绩榜 → 评价区

### ✅ useTennisGame.js
- `initialState` 新增 `lastMatchStats: null`
- `MATCH_OVER` 保存 `action.matchStats ?? null`

### ✅ TennisRoute.jsx
- `onSingleMatchOver` 的 dispatch 新增 `matchStats` 字段
- `ResultScreen` 传入 `matchStats={state.lastMatchStats}`

### ✅ tennis.css
- `.fx-burst.consecace`：橙色（`#ff6b35`），字体更大（`clamp(2.4rem, 9vw, 4.2rem)`），持续 1.5s，带发光效果
- `.hl-card / .hl-grid / .hl-item / .hl-value / .hl-badge`：亮点卡响应式网格布局
