## 二、本轮改动

### 核心功能：charWins 追踪 + allChars 成就

**设计目标：** 激励玩家体验所有 7 种角色流派，间接缓解"只用 Elza"的选择收敛问题，提升重玩价值。

**技术实现：**

| 层 | 文件 | 变更 |
|----|------|------|
| D1 迁移 | `migrations/010_tennis_charwins.sql` | `ALTER TABLE tennis_progress ADD COLUMN char_wins TEXT DEFAULT '{}'` |
| 后端校验 | `workers/auth/tennisProgressLib.js` | 新增 `validCharWins()`（枚举白名单 + 增量 ≤ 1）；DEFAULT_PROGRESS 加 `charWins: {}`；ACHIEVEMENT_IDS 加 `allChars`；`validateProgressUpdate` 输出 merge 后的 charWins |
| 后端 D1 | `workers/auth/tennis.js` | `rowToProgress` 解析 `char_wins`；INSERT/UPDATE SQL 写 `char_wins` 列 |
| 前端成就 | `src/modules/tennis/meta/achievements.js` | 第 16 条：`{ id: 'allChars', icon: '🌈', name: '全能选手', desc: '用所有 7 位家人各赢一场' }` |
| 前端进度 | `src/modules/tennis/meta/progressStore.js` | EMPTY_PROGRESS 加 `charWins: {}`；`mergeProgress` 做 max 合并 |
| 前端逻辑 | `src/modules/tennis/TennisRoute.jsx` | `onSingleMatchOver` 在胜局时 `charWins[player.name]++`，检测所有 7 角色均 ≥ 1 胜则 `achievements.add('allChars')` |
| 前端视图 | `src/modules/tennis/TennisRoute.jsx` | 模式选择屏新增角色图鉴行（7 个 emoji 圆圈，赢过显绿色 + ✓ 徽章）+ hint 文案 |
| 前端样式 | `src/modules/tennis/tennis.css` | `.char-wins-bar`、`.char-chip`、`.char-chip--won`、`.char-badge`、`.char-wins-hint` |
| 测试 | `workers/auth/__tests__/tennisProgressLib.test.js` | 新增 3 个 charWins 专项测试：合法增量、新角色初胜、omit 向后兼容 |

**merge 策略：**

```
客户端 charWins: { 诚: 2, Elza: 1 }
D1     charWins: { 诚: 1, 菲比: 1 }
→合并: { 诚: 2, Elza: 1, 菲比: 1 }  （每角色取 max）
```

此策略与 `coins`（取 max）保持一致，离线多端场景安全合并，不会丢数据。

**向后兼容：**
- D1 旧行没有 `char_wins` 列（migration 前）→ `rowToProgress` 中 `JSON.parse(row.char_wins || '{}')` 安全回退为空对象
- 旧版前端不发送 `charWins` → `body.charWins ?? null` → `validCharWins(null, ...)` 直接返回 `true`
- 新列有 `DEFAULT '{}'`，D1 层零风险
