## 二、本轮改动（优先级 2：赛后评价收集）

### ✅ 已实现：结算屏 1-5 星评价

**新建文件：** `migrations/009_tennis_feedback.sql`
- `tennis_feedback` 表：`id, rating, comment, mode, character, result, created_at`
- CHECK 约束 `rating BETWEEN 1 AND 5`
- 索引 `idx_tennis_feedback_created`

**修改文件：** `workers/auth/tennis.js`
- 新增 `handleTennisFeedback` handler（30 行）
- 严格白名单校验：rating(1-5)、mode/character 白名单、comment ≤ 200 字符
- 公开端点（含游客），无需 JWT

**修改文件：** `workers/auth/index.js`
- 导入 `handleTennisFeedback`
- 注册 `POST /api/tennis/feedback` 路由

**修改文件：** `src/services/tennisService.js`
- 新增 `sendMatchFeedback` 函数（fire-and-forget，静默失败）

**修改文件：** `src/modules/tennis/components/ResultScreen.jsx`
- 新增 `recordingDone` state（战绩上传完成后设 true，触发评价区显示）
- 新增 `fbRating`、`fbComment`、`fbDone` state 控制评价 UI
- 战绩记录完成后展示：五星点评按钮 + 可选文本框 + 提交/跳过按钮
- 提交后显示感谢语，跳过直接关闭评价区

**修改文件：** `src/modules/tennis/tennis.css`
- 追加约 35 行 `.fb-*` 命名空间样式（评价区、星级按钮、文本框、按钮组）
