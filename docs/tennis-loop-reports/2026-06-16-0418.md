## 二、本轮改动

### 核心功能：Adventure 选路阶段 MatchReplayCard 折叠展开

**问题定位：**

奇幻闯关模式在对战结束后，`adventureReducer` 调用 `advance()` 直接将 `phase` 切回 `pick`（选路屏），玩家面对的是「选择去向：对战 / 奇遇 / 商店 / 休息」的决策。这些决策与上一场的对战结果高度相关：
- 体力消耗严重（被高频暴扣）→ 优先选休息（+50）而非立刻再打
- 被某个招式反复克制 → 商店买针对性卡牌
- 整体状态良好 → 大胆选精英战

然而上一场的 `lastRallyLog` / `lastMatchStats` 数据完整存在于 `AdventureScreen` 的 state 中，却完全没有在 `pick` 阶段被渲染。本轮填补了这个信息盲区。

**与 Ladder 的对称性：**

| 场景 | 触发条件 | 实现轮次 |
|------|---------|---------|
| Ladder `between` 屏折叠回顾 | `ladder.status === 'between'` | 轮 20 |
| **Adventure `pick` 屏折叠回顾** | `run.phase === 'pick' && lastRallyLog.length > 0` | **本轮** |

两处实现逻辑完全对称：state 初始化 → useEffect 重置 → 条件渲染折叠按钮 + MatchReplayCard。

**CSS 复用：**

第 20 轮已将 `.replay-toggle-wrap`/`.replay-toggle` 写入 `tennis.css`。本轮直接复用，零新增样式，仅改 `AdventureScreen.jsx`。

**关键新增代码（仅 3 处）：**

```jsx
// 1. state 声明（AdventureScreen 顶层）
const [showReplay, setShowReplay] = useState(false);

// 2. 每次进入 pick 阶段时收起（防止旧展开状态渗漏）
useEffect(() => {
  if (run.phase === 'pick') setShowReplay(false);
}, [run.phase]);

// 3. 选路屏 .card 之后（有上一场数据时才显示）
{lastRallyLog.length > 0 && (
  <div className="replay-toggle-wrap">
    <button type="button" className="replay-toggle"
      onClick={() => setShowReplay((s) => !s)}>
      {showReplay ? '▲ 收起回顾' : '📊 查看上一场回顾 · 找找弱点再选方向'}
    </button>
    {showReplay && <MatchReplayCard rallyLog={lastRallyLog} matchStats={lastMatchStats} />}
  </div>
)}
```

**边界条件：**
- 首次进入 Adventure（尚未打过一场）：`lastRallyLog` 为空数组，按钮不显示 ✅
- 非战斗节点（奇遇/商店/休息）完成后进入 pick：仍显示最近一次对战的回顾（语义正确，因为 MatchReplayCard 本身就是"对战"回顾）✅
- 进入新的 pick 阶段：`useEffect` 自动收起，防止上次展开状态渗漏 ✅
- 通关/失败终局：直接渲染 `victory`/`failed` 屏，pick 渲染不执行 ✅

| `src/modules/tennis/modes/adventure/AdventureScreen.jsx` | 修改 | 新增 showReplay state + pick-phase reset effect + 选路屏折叠渲染 |
