## 二、本轮改动

### 改动：ResultScreen「本局统计摘要」卡片

**问题背景**

当前结算屏只显示大比分（X:Y）和角色属性战报，缺乏对本局过程的量化反馈。玩家打完一场后无法直观感知自己的操作质量（ACE 了几次？克制了几次？关键分赢了几个？），错失了复盘和成就感。

**数据流设计**

`battleReducer` 已持续追踪 `matchStats: { aces, countersWon, clutchWins, moveUsage, mgSum, mgCount }` 和 `rallyCount`，但 `BattleScreen.onMatchOver` 回调原本不包含 `rallyCount`，且 `useTennisGame` 的 `MATCH_OVER` action 也不存储这些数据。

为最小侵入地将数据传到结算屏，选择**不修改游戏 reducer**（避免污染状态机），而是：

1. `BattleScreen.jsx`：`onMatchOver` 回调新增 `rallyCount: state.rallyCount`
2. `TennisRoute.jsx`：新增 `const [lastMatchStats, setLastMatchStats] = useState(null)`，在 `onSingleMatchOver` 存储 `{ ...matchStats, rallyCount, durationS }`
3. `ResultScreen.jsx`：接受 `matchStats` prop，条件渲染统计卡片
4. `tennis.css`：新增 `.match-stat-grid`、`.ms-item`、`.ms-val`、`.ms-label` 样式（4 行）

**展示内容**（6 项，响应式 `auto-fill minmax(90px,1fr)` 网格）：

| 指标 | 来源 | 说明 |
|------|------|------|
| 总球数 | `rallyCount` | 含 ACE，即本局总回合数 |
| ACE | `matchStats.aces` | 发球直接得分次数 |
| 克制得分 | `matchStats.countersWon` | 打出克制招式且得分的次数 |
| 关键分胜 | `matchStats.clutchWins` | 关键分以 ≥1.5× 小游戏倍率赢下 |
| 平均操作倍率 | `mgSum/mgCount` | 衡量小游戏平均表现（仅有小游戏时显示） |
| 局时 | `durationS` | 格式化为「X 分 Y 秒」 |

**设计决策**：
- `matchStats` 为 `null` 时（阶梯/冒险模式结算，或游戏刚进入结算但未打完单局）卡片完全不渲染，零干扰
- 不在 REMATCH/REPLAY 时主动清空 `lastMatchStats`——下次单局完成时会自动覆盖，不引入额外副作用
- `fmtDuration` 辅助函数本地定义（3 行），不引入新依赖
