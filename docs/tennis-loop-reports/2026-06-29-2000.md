## 三、本轮改动

### 功能：「全家制霸」成就持久化大徽章

**背景与动机**

SelectScreen 的挑战进度栏（opp-progress）在 `seenCount === totalOpps` 时，只在 toggle 按钮里插入「🏅 全家大挑战完成！」这几个字——与未完成时的文字混在一起，视觉权重极低，刷新后消失，没有任何持久感。

玩家打通全 7 位家人是一个需要投入多局比赛的里程碑，值得有独立的、持久的、有日期的成就标识。参考游戏化设计中的「里程碑感」原则：重要成就需要在视觉上「停下来被看见」，而不是混进正文。

**设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 存储方式 | `progressStore.allFamilyChampAt`（ms 时间戳） | 与其他成就同走 D1 双写，登录可跨设备持久化 |
| 合并策略 | 双方均有时取 `Math.min`（早者）；一方 null 取非 null | 保留「首次达成」的真实日期，防跨设备同步时被更晚的时间覆盖 |
| 触发时机 | SelectScreen `useEffect`，依赖 `seenCount / totalOpps / familyChampAt` | 进入页面即自动检测；once 语义（`!familyChampAt`）保证只写一次 |
| 徽章位置 | 进度条之后、历史面板之前 | 视觉焦点在完成状态的即时反馈，不被面板内容掩盖 |
| 视觉风格 | 金色（`rgb(251,191,36)`）+ 呼吸光晕动画 | 与现有暗色主题形成对比；仪式感 > 信息量 |
| 日期格式 | `X月X日`（不含年份） | 家庭游戏场景，精确到日即可；避免泄露具体年份 |
| 徽章持久性 | 离开页面后再进仍显示 | 从 progressStore 读取，不依赖 session state |

**展示场景**

| 场景 | 结果 |
|------|------|
| `seenCount < totalOpps`（未打遍所有家人） | 不显示徽章 |
| `seenCount === totalOpps` 且 `allFamilyChampAt === null`（首次打通） | `useEffect` 写入 `Date.now()` → 触发 re-render → 显示徽章 |
| `seenCount === totalOpps` 且 `allFamilyChampAt` 已有值（再次进入） | 直接从 progressStore 读取显示 |
| 两设备登录，较早设备已达成，较晚设备也达成 | merge 时取 `Math.min`，保留最早记录 |

## 四、实现路径

### `src/modules/tennis/meta/progressStore.js`

新增 `allFamilyChampAt` 字段：

```js
export const EMPTY_PROGRESS = {
  // ...
  allFamilyChampAt: null,  // 首次「全家制霸」达成时间戳（ms），取早者保留
};
```

`mergeProgress` 合并策略（取早者）：

```js
allFamilyChampAt: (a.allFamilyChampAt != null && b.allFamilyChampAt != null)
  ? Math.min(a.allFamilyChampAt, b.allFamilyChampAt)
  : (a.allFamilyChampAt ?? b.allFamilyChampAt ?? null),
```

### `src/modules/tennis/meta/__tests__/progressStore.test.js`

新增 3 个测试用例（`describe('mergeProgress')`）：
- 双方均有 `allFamilyChampAt` → 取早者（Math.min）
- 一方 null → 取非 null 值
- 双方均为 null → 结果也是 null

### `src/modules/tennis/TennisRoute.jsx`

新增 `onFamilyChamp` useCallback 及传递给 SelectScreen：

```jsx
const onFamilyChamp = useCallback((ts) => {
  updateProgress({ ...progress, allFamilyChampAt: ts });
}, [progress, updateProgress]);

// 在 SelectScreen 使用处：
<SelectScreen
  ...
  familyChampAt={progress.allFamilyChampAt ?? null}
  onFamilyChamp={onFamilyChamp}
/>
```

### `src/modules/tennis/components/SelectScreen.jsx`

- 导入 `useEffect`
- 新增 `fmtChampDate(ts)` 辅助函数（`X月X日`）
- 函数签名新增 `familyChampAt = null`、`onFamilyChamp`
- 新增 `useEffect`（once 语义，依赖 `seenCount / totalOpps / familyChampAt`）
- 进度条后新增条件渲染 `.family-champ-badge`

```jsx
{familyChampAt && seenCount === totalOpps && (
  <div className="family-champ-badge" role="status" aria-label="全家制霸成就">
    <span className="fcb-icon">🏆</span>
    <span className="fcb-title">全家制霸！</span>
    <span className="fcb-date">首次达成：{fmtChampDate(familyChampAt)}</span>
  </div>
)}
```

### `src/modules/tennis/tennis.css`

新增 `.family-champ-badge` 及子元素样式：金色渐变背景、1.5px 金边框、3s 呼吸光晕动画（`@keyframes fcb-glow`）。
