## 三、本轮改动

### 功能：BattleScreen「首局迎新辅助」（first-match-hint）

**背景与动机**

R113 引入 OppCoachHint（对手已用某招 ≥2 次后提示克制招），R114 在 SelectScreen 加了 matchup-hint（选对手时预告克制招），R115 在 PrepScreen 加了 matchup-advice-hint（备战时说明需要强化哪个属性）。

遗留的最后一段信息空白：**对局已经开始，但对手还没出过 2 次招（OppCoachHint 触发条件未满足）**。在这第 1 球，新玩家茫然出招，往往在 2-3 次被克制后才看到 OppCoachHint。

首局迎新辅助填补这个关键窗口：

| 阶段 | 信息层 | 出现时机 |
|------|--------|---------|
| 选对手 | 🎯 matchup-hint（克制招名） | 始终显示（R114）|
| 备战加点 | ⚔️ matchup-advice-hint（克制招+属性）| 第 1 轮备战（R115）|
| **对局第 1 球** | 🆕 **first-match-hint（绿色提示条）** | **首次对战此对手（本轮）**|
| 对局持续 | 🎯 OppCoachHint（对手惯用招 ×2 后） | 对手出 ≥2 次同招后 |
| 输球时 | 💡 RallyLog 克制提示 | 每球结算后 |
| 赛后复盘 | 📊 ResultScreen 备战成长对比 | 结算屏（R112）|

五个节点覆盖完整的「首次接触 → 建立认知 → 实战感受 → 复盘成长」信息路径。

**与 OppCoachHint 的区别**

| 提示 | 触发条件 | 设计意图 |
|------|---------|---------|
| OppCoachHint | 对手已用某招 ≥2 次 | 实时教练：从本局观察中归纳 |
| FirstMatchHint | `isFirstMatch && rallyCount === 1` | 预告式：入场前预建策略框架 |

FirstMatchHint 使用与 PrepScreen 相同的静态查表逻辑（`CHAR_BUILDS` + `COUNTER_PAIRS`），不依赖对局进行中的实时观察。

**设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 显示时机 | `rallyCount === 1 && phase === 'cards'` | 第 1 球、对手最高 1 次出招，OppCoachHint 必然静默；此时信息价值最高 |
| 可关闭 | ✅ × 按钮 | 熟悉该对手的老玩家可能已看过多次（每次重启浏览器 localRecords 也可能清空）；不强迫阅读 |
| 颜色 | 绿色 `rgba(74,222,128, .07/.28/.85)` | 区别于 OppCoachHint 蓝色 / CrisisHint 橙色；绿色语义：引导/新手友好 |
| 触发条件计算位置 | TennisRoute（`localRecords.some()`） | 纯 JS 表达式，无函数封装，零运行时开销；比在 BattleScreen 内 import localBoard 职责更清晰 |
| 自然消失 | 玩家选完招式后 phase 离开 'cards'，hint 随整个 cards 区块卸载 | 无需复杂生命周期，React 卸载即关闭 |
| 无匹配克制时 | return null（不渲染） | 与 PrepScreen matchupAdviceHint 一致；极少发生 |

### `src/modules/tennis/battle/BattleScreen.jsx`

**新增 import**：

```js
import { MOVES, ULTIMATES, COUNTER_QUIPS, CHAR_BUILDS, COUNTER_PAIRS } from './moves';
```

**新增 `FirstMatchHint` 组件**（在 `BattleScreen` export 函数之前）：

```jsx
function FirstMatchHint({ opponent, playerMoves }) {
  const [dismissed, setDismissed] = useState(false);
  if (dismissed) return null;
  const oppMoves = CHAR_BUILDS[opponent.name]?.moves ?? [];
  let counter = null, victim = null;
  for (const [c, v] of COUNTER_PAIRS) {
    if (playerMoves.includes(c) && oppMoves.includes(v)) { counter = c; victim = v; break; }
  }
  if (!counter) return null;
  const statLabel = { sta: '体力 💪', skill: '技巧 🎯', mind: '心态 🧘' };
  const attr = statLabel[MOVES[counter].stat] ?? MOVES[counter].stat;
  return (
    <div className="bt-first-match-hint">
      <span className="bt-fmh-badge">🆕 初次对战</span>
      <span className="bt-fmh-text">
        出「{MOVES[counter].name}」可克制 {opponent.face}{opponent.name} 惯用的
        「{MOVES[victim].name}」——{attr}越强克制效果越佳
      </span>
      <button type="button" className="bt-fmh-close"
        onClick={() => setDismissed(true)} aria-label="关闭">×</button>
    </div>
  );
}
```

**BattleScreen props 新增 `isFirstMatch`，渲染：**

```jsx
{isFirstMatch && state.rallyCount === 1 && (
  <FirstMatchHint opponent={opponent} playerMoves={playerMoves} />
)}
```
