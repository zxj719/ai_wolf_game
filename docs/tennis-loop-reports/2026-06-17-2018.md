## 二、本轮改动

### 核心功能：ReactTest S 级天赋成就徽章

**问题定位：**

S 级天赋成就（`sGrade`）已在 `TennisRoute.jsx` 的 `useEffect` 中静默写入 progress，但玩家完全感知不到「我刚刚解锁了一个成就」这件事——唯一的视觉就是一个大字母「S」和「进入备战」按钮。反应测试是每局的必经路径，高技术玩家（<250ms）应该在这个峰值时刻获得正向反馈。

**实现方案：**

**1. ReactTest.jsx 改动（+18 行）**

- 新增 `showSAch` state（boolean）
- 新增 `sGradeAlreadyUnlocked = false` prop（默认值确保向后兼容）
- 新增 `sGradeAlreadyUnlockedRef`（用 ref 读取 prop 值，避免 `settle` useCallback 依赖变化导致 arena 事件绑定重建）
- `settle()` 中：检测 `grade === 'S' && !sGradeAlreadyUnlockedRef.current`，命中则 `setShowSAch(true)` + 2800ms 后 `setShowSAch(false)`
- 在 `grade-show` 区域的 `grade-meta` 与「进入备战」按钮之间注入徽章：
  ```jsx
  {showSAch && (
    <div className="sgrade-ach-badge" role="status" aria-live="polite">
      <span className="sgrade-ach-icon">⚡</span>
      <span>成就解锁：<strong>S 级天赋</strong></span>
    </div>
  )}
  ```
- 引入 `achievementById` helper 确保名称/图标与成就表一致

**2. TennisRoute.jsx 改动（+3 行）**

将单行 `<ReactTest ... />` 展开为多行，传入：
```jsx
sGradeAlreadyUnlocked={progress.achievements.includes('sGrade')}
```

**3. tennis.css 改动（+23 行）**

```css
@keyframes sgradeAchPop {
  0%   { opacity: 0; transform: scale(.82) translateY(8px); }
  18%  { opacity: 1; transform: scale(1.07) translateY(0); }
  32%  { transform: scale(1); }
  78%  { opacity: 1; }
  100% { opacity: 0; transform: scale(.95) translateY(-5px); }
}
.tennis-scope .sgrade-ach-badge { /* 琥珀绿光晕 + 背景光 + 2.8s animation forwards */ }
```

动效节奏：
- 0–0.5s：弹出（scale 0.82→1.07→1，opacity 0→1）
- 0.5–2.18s：保持满透明度
- 2.18–2.8s：淡出（opacity→0，轻微上飘）

**改动文件：**

|------|------|------|
| `src/modules/tennis/components/ReactTest.jsx` | 修改 | `sGradeAlreadyUnlocked` prop + `showSAch` state + 徽章 JSX（+18 行） |
| `src/modules/tennis/TennisRoute.jsx` | 修改 | 传入 `sGradeAlreadyUnlocked` prop（+3 行） |
