## 二、本轮改动

### 改动：每日一战全局排行榜（全栈，7 个文件）

**背景**

R85 已将每日挑战 UI 连入 SelectScreen（每天固定对手、完成标记、+30💰奖励），但完成记录只存在 localStorage，不同家庭成员无法互相看到对方是否参与了今日一战。本轮补上这块短板，把完成情况持久化到 D1。

---

**核心设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 认证要求 | 公开（无 JWT） | 家庭场景，未注册成员也能参与和上榜 |
| 去重方式 | `UNIQUE(date, player_name)` + `INSERT OR IGNORE` | 原子操作，防重复，天然幂等 |
| 记录时机 | 每日对战结束（win 或 loss 均记录） | 败者也值得留名；只记胜会鼓励放弃 |
| 排序 | `won DESC, created_at ASC` | 胜者优先展示，同胜负按时间先后 |
| 显示上限 | 7 条（`slice(0, 7)`） | 家庭成员总数 = 7，自然上限 |
| 刷新时机 | mount + 每日对战结束后 800ms delay | delay 留给 POST 写入完成的时间窗口 |
| 失败处理 | fire-and-forget，catch 静默 | 网络问题不阻塞游戏，一致性降级为本地 localStorage |

---

**变更详情**

#### `migrations/011_tennis_daily.sql`（新建）

```sql
CREATE TABLE IF NOT EXISTS tennis_daily_completions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  date TEXT NOT NULL,
  player_name TEXT NOT NULL,
  foe_name TEXT NOT NULL,
  won INTEGER NOT NULL DEFAULT 0,
  duration_s INTEGER,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(date, player_name)
);
CREATE INDEX IF NOT EXISTS idx_daily_date ON tennis_daily_completions(date, created_at);
```

#### `workers/auth/tennis.js`（新增 2 个 handler）

- `handleTennisDailyRecord`：POST 端点，验证 `playerName/foeName`（复用 `TELEMETRY_CHARS` 白名单）、`won`（boolean）、`durationS`（0–7200 整数或 null），`INSERT OR IGNORE` 防重。
- `handleTennisDailyLeaderboard`：GET 端点，查当天记录，胜者优先返回最多 50 条。

#### `workers/auth/index.js`

新增两条路由：
```
POST /api/tennis/daily/record       → handleTennisDailyRecord
GET  /api/tennis/daily/leaderboard  → handleTennisDailyLeaderboard
```

#### `src/services/tennisService.js`

新增两个函数：
- `recordDailyCompletion({ playerName, foeName, won, durationS })` — fire-and-forget POST
- `getDailyLeaderboard()` — async GET，失败返回 null

#### `src/modules/tennis/TennisRoute.jsx`

- 新增 `dailyBoard` state 和 `refreshDailyBoard` callback
- mount 时同时拉取全局排行和每日排行
- `onSingleMatchOver` 中判断 `isDailyRef.current`：无论胜负都调用 `recordDailyCompletion`，仅胜利调用 `markDailyChallengeCompleted`，800ms 后刷新 `dailyBoard`
- SelectScreen 增加 `dailyBoard` prop

#### `src/modules/tennis/components/SelectScreen.jsx`

- 新增辅助函数：`CHAR_FACE` map（角色名→emoji）、`fmtDuration(s)`（秒→分秒字符串）
- `SelectScreen` 接受 `dailyBoard` prop（默认 null）
- 在 `.daily-banner` 后插入 `.daily-completions` 区块，仅当 `dailyBoard.completions.length > 0` 时显示

```
今日出战 · 2 位家人
🐯 诚 → 🦊 Elza   ✓ 胜  3分15秒
🦢 莹 → 🦊 Elza   ✗ 败
```

#### `src/modules/tennis/tennis.css`

新增 14 行 CSS（`.daily-completions` / `.dc-entry` / `.dc-won` / `.dc-lost` / `.dc-player` / `.dc-result` / `.dc-dur` 等），胜者绿色左边框，败者低饱和度，风格与现有 `.daily-banner` 一致。

**用户流程示例**

```
家庭成员诚打开 /tennis →
  报名处加载：今日排行尚空，daily-completions 不显示

诚选「今日一战」→ 对战 Elza（今日固定对手）→ 赢了
  → recordDailyCompletion({ playerName:'诚', foeName:'Elza', won:true, durationS:195 }) POST
  → 800ms 后 refreshDailyBoard
  → SelectScreen 更新：「今日出战 · 1 位家人」
                         🐯 诚 → 🦊 Elza  ✓ 胜  3分15秒

家庭成员莹打开 /tennis →
  mount 拉取 GET /api/tennis/daily/leaderboard
  → SelectScreen 显示：「今日出战 · 1 位家人」，看到诚已完成
  → 受激励，开始自己的今日一战
```
