## 二、本轮改动

### 核心功能：Ladder 赛间屏 MatchReplayCard 折叠展开

**问题定位：**

家族挑战模式的赛间三选一屏（`ladder.status === 'between'`）提供了三个策略选项，但玩家没有任何依据去判断选哪个更合适：
- 加练特训（随机属性 +8）：如果刚才频繁被扣杀打爆，加练 sta 更针对性；但加练的属性是随机的
- 全身按摩（+30 体力）：如果刚才中后段体力不支导致能量惩罚，按摩更合理
- 逛用品店：如果牌库构成有问题，买卡是根治方案

之前 `lastRallyLog` 和 `lastMatchStats` 已经在 LadderScreen 的 state 里保存了完整的上一场数据，只是在 `between` 状态下完全不渲染，白白浪费了。

**实现方案：**

1. **折叠设计**：用一个展开按钮而非默认展开，保持"决策优先，回顾按需"的视觉层次——三个选项始终是第一视觉焦点，回顾卡是辅助信息
2. **自动收起**：`useEffect([ladder.status])` 在每次进入新的 `between` 状态时将 `showReplay` 重置为 `false`，避免上一场展开状态渗漏到下一场
3. **条件渲染**：只有 `lastRallyLog.length > 0` 时才显示按钮（第一场结束前 between 不会出现，但防御性检查保证健壮性）

**改动文件：**

| 文件 | 操作 | 说明 |
|------|------|------|
| `src/modules/tennis/modes/LadderScreen.jsx` | 修改 | 增加 `showReplay` state、reset effect、between 块 toggle + 条件渲染 MatchReplayCard |
| `src/modules/tennis/tennis.css` | 修改 | 新增 `.replay-toggle-wrap` 和 `.replay-toggle` 样式（10 行 CSS） |

**关键代码：**

```jsx
// state 声明（LadderScreen 顶层）
const [showReplay, setShowReplay] = useState(false);

// 每次进入新 between 状态时收起
useEffect(() => {
  if (ladder.status === 'between') setShowReplay(false);
}, [ladder.status]);

// between 渲染块（opts 之后）
{lastRallyLog.length > 0 && (
  <div className="replay-toggle-wrap">
    <button type="button" className="replay-toggle"
      onClick={() => setShowReplay((s) => !s)}>
      {showReplay ? '▲ 收起回顾' : '📊 查看上一场回顾 · 找找弱点再选方向'}
    </button>
    {showReplay && <MatchReplayCard rallyLog={lastRallyLog} matchStats={lastMatchStats} />}
  </div>
)}
```
