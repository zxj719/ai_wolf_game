## 三、本轮改动

### 功能：冲刺榜「今日最高」视觉区分

**背景与动机**

R133 的家族冲刺榜已能持久化 Top 10 历史分数。但榜单按总分降序排列，家庭成员看到一页榜单时无法快速判断「今天谁的分最高」—— 这恰好是家庭游戏场景最常见的日常问题（「今天我们谁打得最好？」）。

今日视觉区分使榜单变成两个维度并存：全历史排名（主排序）+ 今日高亮（辅助线索）。简单、无侵入、不改变排序逻辑。

**架构设计（可测试纯函数）**

在 `sprintScores.js` 中新增 `isToday(ts, { today } = {})` 纯函数，沿用该模块的测试注入模式：
- `ts`：待判断的 Unix ms 时间戳
- `today`（可选）：注入的参考时间戳（测试用），未传时使用 `new Date()`
- 返回：布尔值，本地时区的年/月/日三维度匹配

这个设计保证了 `isToday` 完全脱离副作用（`Date.now()`），与 `saveSprintHiscore` 的 `now` 注入模式一致。

**SprintScreen 变更**

1. 从 `sprintScores` 中导入 `isToday`
2. 在家族冲刺榜的 `.map()` 中，对每条记录计算：
   - `isCurrent = s.ts === currentTsRef.current`（当前局 → 黄色，已有）
   - `todayEntry = !isCurrent && isToday(s.ts)`（今日非当前局 → 新增样式）
3. 新增视觉效果：
   - **背景**：`rgba(34,211,238,.09)`（青色，轻于黄色）
   - **☀️今日 徽章**：`border: 1px solid rgba(34,211,238,.4)`，字体 0.65rem，浮在分数右侧

**优先级规则：**
- 当前局（黄色）> 今日非当前（青色）> 历史（透明）
- 无重叠歧义（当前局一定是今日，但不显示"今日"徽章，减少冗余信息）
