## 三、本轮改动

### 改动：连败激励徽章（4 个文件，纯增量）

**背景**

R88 的日榜最速奖章完善了胜者竞争激励，但败者体验缺乏"再战"动力。ResultScreen 已有连胜 🔥 徽章，对称的连败 💪 徽章能形成闭环：胜了看到连胜进度，败了看到连败挑战，都有继续打的理由。

**设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 连败阈值 | ≥ 2 场 | 与连胜对称；1 败不算"连"，不宜提示 |
| 颜色 | 紫色 `rgba(139,92,246,.75)` | 与橙色连胜徽章明显区分；紫色暗示"战意"而非"失败" |
| 文案分档 | 2场→换策略、3场→复仇时刻、4+场→不倒翁 | 随连数递进，4+ 加脉冲动画，传达"越败越战"精神 |
| 计算位置 | useMemo（saveLocalRecord 之前） | 与 winStreak 同样时机；本局结果已知但尚未存入历史 |
| 纯函数提取 | `computeStreakCount` in localBoard.js | 解耦 UI 与逻辑，便于单测；win/loss 共用同一函数 |

**文案示例**

```
连败 2 场：💪 连败 2 场，换个策略再战！
连败 3 场：💪 连败 3 场，复仇时刻到了！
连败 4 场：💪💪 连败 4 场，不倒翁精神！（+ 脉冲动画）
```

**变更详情**

#### `src/modules/tennis/localBoard.js`

新增纯函数（11 行）：
```javascript
export function computeStreakCount(records, playerName, isWin) {
  const mine = records.filter((r) => r.p === playerName);
  const pred = isWin ? (r) => r.sp > r.so : (r) => r.sp <= r.so;
  let streak = 1;
  for (let i = mine.length - 1; i >= 0 && pred(mine[i]); i--) streak++;
  return streak;
}
```

#### `src/modules/tennis/components/ResultScreen.jsx`

- 导入 `computeStreakCount`
- `winStreak` useMemo 改用 `computeStreakCount`（逻辑等价）
- 新增 `lossStreak` useMemo（playerWon 时直接返回 0）
- JSX：`!playerWon && lossStreak >= 2` 时渲染 `.loss-streak-badge`

#### `src/modules/tennis/tennis.css`

新增 3 条规则（loss-streak-badge、loss-streak-fire、lossStreakPulse keyframe）

#### `src/modules/tennis/__tests__/localBoard.test.js`

新增 `describe('computeStreakCount')` 6 个用例：
- 无历史 → streak = 1
- 3 连胜历史 + 当前胜 → 4
- 2 连败历史 + 当前败 → 3
- 连胜被败局截断
- 多玩家互不干扰
- 平局（sp === so）视为败局纳入连败计数
