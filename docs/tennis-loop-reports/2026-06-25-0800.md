## 二、本轮改动

### 改动：battleReducer `counterLost` + ResultScreen 防守统计

**问题背景**

Round 76 添加了「你本场偏爱」招式 Top3，玩家可以看到自己最常用招式及其弱点。但这是静态分析——玩家仍然不知道对手**实际上**利用了这个弱点多少次。如果对手恰好全程出克制玩家的招式，玩家看到「被克：切削」的标签，却不知道自己因此失了多少分，缺乏量化感知。

**实现方式**

**battleReducer.js：新增 `counterLost` 字段**

初始值：
```javascript
matchStats: { aces: 0, countersWon: 0, counterLost: 0, clutchWins: 0, ... }
```

RESOLVE case 更新（与 `countersWon` 完全对称）：
```javascript
counterLost: state.matchStats.counterLost + (!win && oCounter > 1 ? 1 : 0),
```

条件：`oCounter > 1`（对手招式克制玩家招式）且 `!win`（玩家失分）。  
注：hawkeye 翻判路径不影响——hawkeye 将失分变为不计分的重打，不进入 `counterLost` 累计。

**ResultScreen.jsx：显示「被克失分」**

统计格新增（与「克制得分」相邻）：
```jsx
<div className="ms-item">
  <span className="ms-val">{matchStats.counterLost ?? 0}</span>
  <span className="ms-label">被克失分</span>
</div>
```

防守预警条件：`counterLost >= 2 && counterLost > countersWon`（被克次数多于克中次数，说明对手读招比玩家准）：
```jsx
{(matchStats.counterLost ?? 0) >= 2 && (matchStats.counterLost ?? 0) > matchStats.countersWon && (
  <p className="counter-lost-warn">🛡 被克 {matchStats.counterLost} 次，对手读招很准，下次注意变换出招节奏</p>
)}
```

**信息闭环验证**

| 信息问题 | 答案 |
|---------|------|
| 我出了什么？ | 你本场偏爱：重炮 ×12、扣杀 ×4（Round 76） |
| 我的惯用招有什么弱点？ | 被克：切削（Round 76） |
| 对手实际克中我几次？ | 被克失分 **5 次**（本轮新增） |
| 我克中对手几次？ | 克制得分 **2 次**（已有） |
| 攻防对比 | 被克 5 > 克中 2 → 触发「对手读招很准」预警 |
