## 三、本轮改动

### 功能：PrepScreen「对阵克制备战建议」（matchup-advice-hint）

**背景与动机**

R114 在 SelectScreen 的对手面板中加入了「🎯 {克制招}」金色小标，让玩家在**选对手时**知道"出什么招克 ta"。但进入 PrepScreen（备战加点）后，这个信息不再可见——玩家拿着"出重炮平击"的建议，却不知道重炮平击需要的是体力属性，应该优先把点加在体力上。

本轮填补这个策略传导断层：

| 屏幕 | 已有信息 | 缺失 |
|------|---------|------|
| SelectScreen | 🎯 重炮平击（克制招名）| — |
| PrepScreen | 💡 球探速报：对手体力见长（对手侧） | ⚔️ 你的克制招需要练哪个属性（玩家侧）|
| BattleScreen | 🎯 对手惯用切削（×3），出重炮可克制 | — |

`matchupAdviceHint` 填补中间行：告诉玩家"**克制招对应的属性是什么，应该重点练**"。

**信息流可视化**

```
SelectScreen  : 「🎯 重炮平击」（出什么克制招）
    ↓
PrepScreen    : 「⚔️ 克制参考：出重炮可克制切削——备战重点强化体力 💪」 ← 本轮
    ↓
BattleScreen  : 「🎯 对手惯用切削（×3），出重炮可克制」（实战中持续提醒）
    ↓
ResultScreen  : 「较上次备战：体力 +5」（成长复盘）
```

四个屏幕覆盖了「选→练→战→复盘」的完整策略闭环，每个决策点都有信息支撑。

**与球探速报的区别**

| 提示 | 视角 | 建议内容 | 颜色 |
|------|------|---------|------|
| `oppStrategyHint`（球探速报） | 对手侧 | 对手体力见长 → 加技巧/心态以抗衡 | 琥珀橙 |
| `matchupAdviceHint`（克制参考） | 玩家侧 | 你的克制招是重炮 → 加体力以发挥克制 | 金黄色 |

两者给出的建议可能相同（都指向同一属性）也可能不同（一个说抗衡对手，一个说发挥自身克制），互补而不重复。

**设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 显示时机 | `prepRound === 0`（仅第 1 轮显示） | 与球探速报同步，在最关键的备战开始时刻给全量信息；后续轮加点时无需重复看建议 |
| 颜色 | 金黄 `rgba(251,191,36,.88)` | 与 SelectScreen 的 matchup-hint（R114）一致，形成跨屏色彩记忆 |
| 属性标签 | `体力 💪 / 技巧 🎯 / 心态 🧘` | 与 StatsPanel 的图标一致，直观对应 |
| 位置 | `oppStrategyHint` 之后 | 先看对手强点（球探），再看自己应对（克制）；信息从外部到内部，层次清晰 |
| 无克制招时 | 不渲染（return null） | 理论上极少发生（9 个克制对覆盖所有 7×6 组合中绝大多数）；静默 fallback 比显示误导信息好 |

## 四、实现路径

### `src/modules/tennis/components/PrepScreen.jsx`

**新增 import**：
```js
import { CHAR_BUILDS, COUNTER_PAIRS, MOVES } from '../battle/moves';
```

**新增 `matchupAdviceHint` useMemo**（插在 `prepHistoryHint` 之后、`onPick` 之前）：

```js
const matchupAdviceHint = useMemo(() => {
  if (state.prepRound !== 0 || !state.player || !state.opp) return null;
  const playerMoves = CHAR_BUILDS[state.player.name]?.moves ?? [];
  const oppMoves = CHAR_BUILDS[state.opp.name]?.moves ?? [];
  let counter = null, victim = null;
  for (const [c, v] of COUNTER_PAIRS) {
    if (playerMoves.includes(c) && oppMoves.includes(v)) { counter = c; victim = v; break; }
  }
  if (!counter) return null;
  const statLabel = { sta: '体力 💪', skill: '技巧 🎯', mind: '心态 🧘' };
  const attr = statLabel[MOVES[counter].stat] ?? MOVES[counter].stat;
  return `⚔️ 克制参考：出「${MOVES[counter].name}」可克制 ${state.opp.face}${state.opp.name} 惯用的「${MOVES[victim].name}」——备战重点强化${attr}`;
}, [state.prepRound, state.player?.name, state.opp?.name]); // eslint-disable-line react-hooks/exhaustive-deps
```

**新增渲染**（在 `oppStrategyHint` 渲染之后）：

```jsx
{matchupAdviceHint && (
  <p className="matchup-advice-hint">{matchupAdviceHint}</p>
)}
```

### `src/modules/tennis/tennis.css`

在 `.prep-strategy-hint` 之后插入 1 行（同时将 `.prep-strategy-hint` 的 margin-bottom 从 14px 改为 10px，为新提示让出间距）：

```css
.tennis-scope .matchup-advice-hint { font-size: .78rem; font-family: var(--mono); color: rgba(251,191,36,.88); margin: 0 0 14px; padding: 6px 12px; background: rgba(251,191,36,.07); border-radius: 8px; border-left: 3px solid rgba(251,191,36,.45); }
```
