## 二、本轮改动

### 改动：BattleScreen RallyLog 克制提示

**问题背景**

现有教学系统有一个时间盲区：
- `CrisisHint`（关键分读招提示）仅在 `isKeyPoint(score) === true`（盘点/金球）时触发
- `FxOverlay`（`被克制……`）只显示「被克制」的结果反馈，不告诉玩家「用什么招可以克制对手」
- 玩家丢分时，屏幕只会说「😤 对手拿下此分。」或「被克制了……节奏完全被打乱。」——但缺少「下次该怎么应对」的即时提示

结果：玩家需要靠记忆或反复失败才能摸索出克制关系，与克制系统设计价值（读招→反制→逆转）不符。

**实现方式**

**BattleScreen.jsx（逻辑层）**

利用已有的 `COUNTER_FOR` 对象（文件顶部 21-25 行由 `COUNTER_QUIPS` 逆向推导），在 `RallyLog` 组件中添加条件渲染：

```jsx
{!r.win && counterForOpp && (
  <p className="bt-counter-tip">
    💡 「{MOVES[r.oppMove].name}」被「{MOVES[counterForOpp].name}」克制
  </p>
)}
```

- `!r.win`：仅在玩家丢分时显示（赢球时无需提示，避免噪音）
- `counterForOpp = COUNTER_FOR[r.oppMove]`：查对手招式的克方
- `COUNTER_FOR` 覆盖全部 9 对克制关系，部分招式（如 `flatDrive`）在克制表中无对应克方，此时 `counterForOpp` 为 `undefined`，条件渲染自动跳过

**tennis.css（样式层）**

```css
.tennis-scope .bt-rallylog .bt-counter-tip {
  font-size: .72rem; color: var(--optic); opacity: .75; margin-top: 5px;
  animation: bt-tip-in .35s ease both;
}
@keyframes bt-tip-in { from { opacity: 0; transform: translateY(-3px); } to { opacity: .75; transform: none; } }
```

- `.bt-counter-tip` 选择器比 `.bt-rallylog p` 更具体，无需 `!important` 即可覆盖 `font-size` 和 `margin`
- 渐入动画 0.35s，保持与 RallyLog 出现时间同步
- 颜色 `var(--optic)`（黄绿色）：与 `CrisisHint` 颜色一致，形成视觉语言统一
- `opacity: .75`：比正文稍淡，突出「辅助提示」层级

**与现有教学系统的关系**

| 提示 | 触发条件 | 内容 | 持续时长 |
|------|---------|------|---------|
| `FxOverlay`「被克制……」 | `r.counterMul < 1 && !r.win` | 结果反馈（没有方向） | ~1.2s（fadeOut） |
| `bt-counter-tip` | `!r.win && COUNTER_FOR[oppMove]` | **反制方向**（告诉玩家用什么招） | 随 RallyLog 存活（~1.4s） |
| `CrisisHint` | 关键分 + 对手某招 ≥2 次 | 预测 + 反制建议 | cards 阶段全程可见 |

三层互补：FxOverlay（情绪反馈）→ bt-counter-tip（学习方向）→ CrisisHint（关键时刻主动建议）。

**覆盖的克制对**

`COUNTER_FOR` 中当前包含的被克方：
- `slice` → 被 `flatDrive` 克
- `topspin` → 被 `slice` 克
- `dropShot` → 被 `topspin` / `volley` 克（取首个：`topspin`）
- `flatDrive` → 被 `dropShot` 克
- `volley` → 被 `passingShot` / `lob` 克（取首个：`passingShot`）
- `lob` → 被 `smash` 克

`flatDrive`、`smash`、`passingShot` 在被克方中未出现（它们不被任何招式直接克制），这些对手招式丢分时不显示提示（`counterForOpp === undefined`，自动跳过）。
