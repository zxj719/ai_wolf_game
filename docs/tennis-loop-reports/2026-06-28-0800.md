## 三、本轮改动

### 改动：今日一战个人得分统计

**问题背景**

R94 下轮计划第 1 项：「每日挑战完成后记录本次得分（胜局数、操作倍率、ACE 次数，纯 localStorage），在日榜下方展示「我的今日得分」，与全网日榜形成呼应。零后端，实现成本低。」

**设计原则**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 存储粒度 | `tennis_daily_stats_YYYY-MM-DD` → `{ [playerName]: {...} }` | 日期作为 key 天然刷新，不需清理旧数据；角色名分组支持家庭多人 |
| 记录时机 | `isDailyRef.current` 分支内，`saveDailyStats()` 在 `recordDailyCompletion()` 后立即调用 | 无论胜负都记录（输了也要让玩家看到自己的 ACE 数） |
| 数据字段 | won / setsP / setsO / aces / avgMultiplier / clutchWins / countersWon | 覆盖「赛果」+「亮点」，avgMultiplier 来自 mgSum/mgCount，缺失时为 null |
| 展示位置 | SelectScreen 日榜完成名单下方，仅在选了角色且有当天记录时显示 | 避免空状态，首次进入不显示；选角色后才有上下文 |
| chip 过滤 | 只展示非零（非 null）指标 | 避免一排零信息 chip 降低信噪比 |

**实现路径**

#### `src/modules/tennis/meta/dailyChallenge.js`
- 新增 `saveDailyStats({ playerName, won, setsP, setsO, aces, avgMultiplier, clutchWins, countersWon })`
- 新增 `loadDailyStats(playerName)` → 返回当天记录或 null

#### `src/modules/tennis/TennisRoute.jsx`
- 导入 `saveDailyStats`
- 在 `isDailyRef.current` 分支内（`recordDailyCompletion` 后）调用 `saveDailyStats`
- `avgMultiplier` 由 `matchStats.mgSum / matchStats.mgCount` 计算，保留 2 位小数

#### `src/modules/tennis/components/SelectScreen.jsx`
- 导入 `loadDailyStats`
- 新增 `myDailyStats` useMemo（依赖 `picked`）
- 在日榜完成名单后渲染 `.my-daily-stats` 卡片

#### `src/modules/tennis/tennis.css`
- 新增 `.my-daily-stats`、`.my-daily-stats-win`、`.my-daily-stats-loss`、`.my-daily-result`、`.my-daily-chip` 样式规则

#### `src/modules/tennis/__tests__/dailyStats.test.js`（新建）

8 个测试用例覆盖：
- 无记录时返回 null
- null/undefined/空字符串 playerName 返回 null
- 胜场数据完整存储/读取
- 败场数据（含 avgMultiplier=null）
- 家庭多人：同一天多个角色各自独立存取
- 覆写：同角色第二次调用覆盖前一次
- 缺失可选字段时 defaults 为 0/null
- 已保存一人后查询另一人返回 null
