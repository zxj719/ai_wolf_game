## 二、本轮改动

### 核心功能：Adventure Boss 战前特殊提示

**问题定位：**

Adventure 模式的 Boss 节点（第三章末步「网球之神」）目前在选路屏仅作为普通对战按钮展示，文案为 `👑 网球之神`，没有任何视觉预警。玩家常在体力不足时冲进 Boss 战（如 carryEnergy=30），导致挫败感高。

Boss 节点是 mapGen.js 的确定性标记（`ci === 2 → boss: true`），且末步始终是单节点（无二选一）——可靠检测无误报风险。

**实现方案：**

在选路屏（`// ===== 选路（pick） =====`）中：

1. **Boss 检测**：`const isBossStep = step.some((n) => n.boss)`，复用已有属性。

2. **动效警告卡 `adv-boss-warning`**（条件渲染，`isBossStep` 时展示）：
   - 皇冠大图标：`filter: drop-shadow` 金色光晕
   - 标题「最终决战：网球之神」（`var(--display)` 字体，`#ff8060` 暖红色）
   - 体力状态评估（三档文案，基于 `energyPct = carryEnergy / energyMax`）：
     - `≥ 70%`：「状态饱满，放手一搏！」
     - `40%–70%`：「体力尚可，发挥稳定发挥即可」
     - `< 40%`：「⚠️ 体力偏低，优先用卡回血，节奏放慢」
   - 提示行：「📌 Boss 使用离谱 TWIST 规则·拿满加点收益后再点击进入」
   - 脉冲边框动画 `boss-pulse`（2.4s 循环，红色 glow）

3. **Boss 按钮 `.opt--boss`**：
   - 红色半透明底 + 红色边框（默认态）
   - 悬停：加深边框 + 红色 box-shadow + 横向位移（复用 `.opt:hover` 模式）
   - 图标键 `.key` 着暖红色 `#ff7050`

**体力文案设计原则：**
- Adventure 是线性前进，无法回头，因此不建议玩家「回头找休息节点」
- 改为告知当前状态对应的**战中策略**（用卡回血 vs 放手攻击），让玩家有准备地进入

**CSS（48 行，tennis.css 末尾新增）：**
- `.adv-boss-warning`：红色渐变底 + 脉冲边框动画
- `@keyframes boss-pulse`：0→50→100% 红色边框 glow
- `.adv-boss-crown / -title / -sub / -hint`：字体层次
- `.opt--boss`：红色主题对战按钮
- `.opt--boss:hover / .opt--boss .key`：悬停 + 图标色
