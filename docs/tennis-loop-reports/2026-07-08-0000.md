## 三、本轮改动

### 功能：`MatchReplayCard` 激活 + `topRally` 追踪

**背景与动机**

`MatchReplayCard` 组件（`src/modules/tennis/components/MatchReplayCard.jsx`）在几十轮前创建，定义了「本场最佳一击」「惯用招式」「最后 N 球时间轴」三个展示段，但从未在生产路径中渲染过：

- `battleReducer` 的 `matchStats` 初始化中没有 `topRally` 字段（一直为 undefined）
- `BattleScreen.onMatchOver` 从未透传 `rallyLog`（只传了 score/matchStats/rallyCount/durationS）
- `ResultScreen` 从未 import 或使用 `MatchReplayCard`

本轮打通整条数据流，让「本场最佳一击」真正出现在每局结算屏。

**数据流三层**

```
battleReducer.js
  matchStats.topRally  ← 本轮新增：最高 pMultiplier 赢球时更新
  state.rallyLog       (最后 9 球，已有)
      ↓ (BattleScreen.onMatchOver 透传，本轮新增 rallyLog 字段)
TennisRoute.jsx
  lastMatchStats = { ...matchStats, rallyCount, durationS }  ← topRally 自动包含
  lastRallyLog   ← 本轮新增 useState
      ↓ (ResultScreen props，本轮新增 rallyLog={lastRallyLog})
ResultScreen.jsx
  <MatchReplayCard rallyLog={rallyLog} matchStats={matchStats} />  ← 本轮新增渲染
```

**`battleReducer.js` 变更**

1. 初始 `matchStats` 新增 `topRally: null`
2. RESOLVE action 中，计算 `lastRally` 之后：
   ```js
   const prevTopRally = state.matchStats.topRally;
   const topRally = win && state.pMultiplier > (prevTopRally?.pMultiplier ?? 0)
     ? lastRally
     : prevTopRally;
   ```
   仅赢球且本球倍率更高时替换——确保「最佳一击」永远是正向高光，输球不更新

**`BattleScreen.jsx` 变更**

```js
// 修改前
onMatchOver({ score, matchStats, rallyCount, pEnergy, durationS });
// 修改后
onMatchOver({ score, matchStats, rallyCount, rallyLog: state.rallyLog, pEnergy, durationS });
```

**`TennisRoute.jsx` 变更**

```js
const onSingleMatchOver = useCallback(({ score, matchStats, rallyCount, rallyLog, durationS }) => {
  setLastMatchStats({ ...matchStats, rallyCount, durationS });
  setLastRallyLog(rallyLog ?? []);          // ← 新增
  // ...
}, [...]);

const [lastMatchStats, setLastMatchStats] = useState(null);
const [lastRallyLog, setLastRallyLog] = useState([]);    // ← 新增
```

ResultScreen 渲染调用：
```jsx
<ResultScreen ... rallyLog={lastRallyLog} ... />
```

**`ResultScreen.jsx` 变更**

- 新增 import `MatchReplayCard`
- 函数签名加 `rallyLog` prop
- 在 `CounterDuelBar` 后插入：`<MatchReplayCard rallyLog={rallyLog} matchStats={matchStats} />`

**`tennis.css` 变更（约 50 行）**

新增 `.replay-card`、`.replay-section`、`.replay-label`、`.replay-best-rally`、`.rp-*`、`.replay-usage-pill`、`.replay-timeline`、`.rp-rally` 等样式，遵循现有暗色主题语义：
- 最佳一击：金色边框背景（amber 色系），高亮倍率数字
- 克制标记：绿色 pill
- 时间轴：每球一个小卡片，赢球绿色/输球红色，CLUTCH 球加金色外边框

**MatchReplayCard 展示内容（无修改，直接复用）**

1. **本场最佳一击**（仅当 `topRally` 存在）：招式名 + 克制标记 + vs 对手招 + 倍率数字
2. **惯用招式 pills**（Top-3，已在 ResultScreen 有更详细版本，pills 提供紧凑概览）
3. **最后 N 球时间轴**（最多 9 球，win/lose 色彩编码，CLUTCH 球金色轮廓）
