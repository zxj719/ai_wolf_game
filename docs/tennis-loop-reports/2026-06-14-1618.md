## 二、本轮改动

### 核心功能：FeedbackWidget 组件 + 评价三模式覆盖

**问题定位：**

上轮 feedback 端点和 summary 端点已建成，但审查代码后发现一个系统性盲区：
- `ResultScreen.jsx`：有评价控件 ✅（单局快打用）
- `LadderScreen.jsx`：结算屏无评价 ❌
- `AdventureScreen.jsx`：结算屏无评价 ❌

家族挑战和奇幻闯关因为不走 ResultScreen 路径（直接 `onExit() → REPLAY`），用户玩完整场后进入结算屏，只有"返回报名处"按钮，没有任何反馈渠道。这意味着真实用户的大部分游玩感受无法被采集到 `tennis_feedback` 表。

**解决方案：**

| 步骤 | 内容 |
|------|------|
| 提取组件 | `src/modules/tennis/components/FeedbackWidget.jsx` — 将 ResultScreen 内联 60 行评价 UI 提取为独立组件，接受 `{ mode, character, result }` 三个 props |
| 重构调用方 | `ResultScreen.jsx` — 删除 fbRating/fbComment/fbDone 三个 state 和 submitFeedback 函数，替换为 `<FeedbackWidget mode={state.mode} character={p.name} result={...} />` |
| 覆盖家族挑战 | `LadderScreen.jsx` — 在 won/lost 结算 JSX 中加入 `<FeedbackWidget mode="ladder" character={basePlayer.name} result={won ? 'win' : 'loss'} />` |
| 覆盖奇幻闯关 | `AdventureScreen.jsx` — 在 victory/failed 结算 JSX 中加入 `<FeedbackWidget mode="adventure" character={player.name} result={won ? 'win' : 'loss'} />` |

**设计细节：**

- `FeedbackWidget` 内部完整持有 rating/comment/done 状态，调用方零状态负担
- 样式完全复用现有 `.fb-*` CSS 类，无新样式
- `sendMatchFeedback` 已支持所有三种 mode 字符串（'single'/'ladder'/'adventure'），后端无需变更
- 每个调用点传入 character 名称（用于 byCharacter 聚合分析），result 传 'win'/'loss'（与现有 ResultScreen 一致）

**组件接口：**

```jsx
<FeedbackWidget
  mode="ladder"        // 'single' | 'ladder' | 'adventure'
  character="Elza"     // 角色名（用于 feedback.byCharacter 聚合）
  result="win"         // 'win' | 'loss'
/>
```
