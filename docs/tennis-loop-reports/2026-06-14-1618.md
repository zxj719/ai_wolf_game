## 二、本轮改动

### 核心功能：NextTargetCard 组件

**设计目标：** 将结算屏从"死路"变为"引路"——玩家结束一场比赛后，立即看到下一个最近可达的成就目标和具体行动提示，而不是面对一张榜单后不知道要不要再开一局。

**技术实现：**

| 层 | 文件 | 变更 |
|----|------|------|
| 前端组件 | `ResultScreen.jsx` | 新增 `NextTargetCard({ progress, matchStats })` 组件；`ResultScreen` 签名加 `progress` prop |
| 前端调用 | `TennisRoute.jsx` | ResultScreen 调用加 `progress={progress}` |
| 样式 | `tennis.css` | 新增 `.next-target-card` / `.next-target-body` / `.next-target-icon` / `.next-target-text` / `.next-target-name` / `.next-target-hint` / `.next-target-footer` |

**Closeness Score 计算逻辑（核心算法）：**

每个锁定成就计算一个 `score ∈ [0, 1)` 表示"当前局距离解锁有多近"：

```
aceMaster    → aces / 3         （本场 ACE 数实时反馈）
consecAce    → maxConsecAces / 3
winStreak5   → longestWinStreak / 5
proTouch     → maxMgMult / 1.4
allChars     → charCount / 7    （角色图鉴进度）
allUltimates → unlockedMoves.length / 7
firstWin     → 0.95             （如果未解锁说明一直在输，最高优先级提示）
clutchMaster → 0.5              （中等，有具体操作提示）
sGrade       → 0.35
familyKing   → 0.4
...
```

取最高 score 的成就展示，hint 为个性化提示文案（如 "本场 2/3 ACE，发球连打 🎾 可提高 ACE 概率"）。

**位置：** 成就解锁卡 → 本场亮点卡 → **🎯 下一个目标** → 历史战绩榜 → 赛后评价

### 顺带修复：fb-star 移动端触控目标

```css
/* 修复前 */
.tennis-scope .fb-star { padding: 2px 4px; }
/* 实际点击区 ≈ 33px × 37px，低于 44px 最低标准 */

/* 修复后 */
.tennis-scope .fb-star {
  padding: 8px 6px;
  min-width: 44px; min-height: 44px;
  display: inline-flex; align-items: center; justify-content: center;
}
```

效果：移动端用户不再需要精确点击小星星，提交评价的成功率提升，feedback 数据质量改善。

## 三、会话历史问题说明

前 13 轮评估循环的 commit 均在 **detached HEAD** 状态下提交，且从未推送到任何命名分支（`git push origin main` 推送的是 `main` 分支，不是 detached HEAD 的 commit）。这意味着：

- 所有 13 轮的改动代码存在于本地 git 对象库（孤立 commit），但不在 `main` 上
- `origin/main` 仍指向第 0 轮前的代码（`1a8e54b docs: 可玩性/平衡性评估报告`）
- 孤立 commit 链从 `31e4533`（第 13 轮）向前可追溯到第 1 轮

**本轮解决方案：**
- `git checkout -b loop-recovery 31e4533` 从孤立链末端创建命名分支
- 在 `loop-recovery` 分支上提交本轮改动
- `git push -u origin loop-recovery` 将全部 14 轮的改动首次推送到 GitHub

**下轮建议：**
- 用户需决定是否将 `loop-recovery` 合并入 `main`，或 reset main 到 loop-recovery
- 未来循环应在 `loop-recovery`（或 `main` 指向后的分支）上继续，避免再次 detach
