## 三、本轮改动

### 功能：BattleScreen「非关键分读招教练」（OppCoachHint）

**背景与动机**

R112 报告提出的候选方案：「BattleScreen 新增 oppCoachHint 计算，每球 opponentMoveId 记录后在下一球提示」。

现有对局读招信息分布：

| 时机 | 已有内容 |
|------|---------|
| 每球 idle/resolve 后（输球时） | RallyLog「💡 「对手招」被「克制招」克制」（只在输球时出现）|
| 非关键分 cards 阶段 | ❌ **无任何提示**（本轮填补此空白）|
| 关键分 cards 阶段 | CrisisHint 告警（对手出招 ≥2 次）|
| 对手 tell 读招时 | bt-tell 提示克制招 |

现有 `CrisisHint` 的局限：
- 仅在 `isKeyPoint(score)` 时触发（set point / break point / 金球），普通分没有任何引导
- 新手在前半局可能频繁被克制，却看不到任何"下次该怎么应对"的提示
- 关键分才出现的告警会让玩家感觉"教练只在最紧张的时候才出来"

`OppCoachHint` 填补了**全局常态教练**的空白：
- 一旦对手某招用了 ≥2 次，每个普通出招阶段都能看到提示
- 视觉设计用低调的淡青色（`rgba(103,232,249,.06)` 背景 + `.80` opacity 文字），明显低于 CrisisHint 的霓虹黄（`var(--optic)`），体现信息优先级分层

**设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 互斥条件 | `isKeyPoint(score)` 为 false 时显示 | 避免与 CrisisHint 叠加；关键分已有更高权重的警报 |
| 触发阈值 | 对手出招 ≥2 次 | 1 次属于样本噪音；2 次开始呈现规律 |
| 无克制招时 | return null | 若对手最常出的招恰好没有直接克制招（理论上不会发生，但防御性 return） |
| 颜色 | 淡青色 `rgba(103,232,249,...)` | 与 opp-best-streak（R102）色系一致，全局"建议/信息"色调统一 |
| 字号 | `.68rem`（低于 CrisisHint 的 `.72rem`）| 教练提示是辅助信息，不与主流程竞争 |
| 图标 | 🎯（瞄准）vs ⚡（关键分） | 语义区分：「瞄准对手弱点」vs「紧急关键时刻」|

**与 CrisisHint 视觉对比**

| 组件 | 背景 | 文字颜色 | 字号 | 图标 | 触发条件 |
|------|------|---------|------|------|---------|
| CrisisHint（关键分） | `rgba(217,255,61,.08)` | `var(--optic)` 霓虹黄 600 | `.72rem` | ⚡ | isKeyPoint + ≥2 次 |
| OppCoachHint（普通分） | `rgba(103,232,249,.06)` | `rgba(103,232,249,.80)` 淡青 | `.68rem` | 🎯 | !isKeyPoint + ≥2 次 |

## 四、实现路径

### `src/modules/tennis/battle/BattleScreen.jsx`

在 `CrisisHint` 之前新增 `OppCoachHint` 函数组件：

```jsx
/** 非关键分读招教练：对手已用某招 ≥2 次时低调提示克制招。关键分由 CrisisHint 接管，两者互斥。 */
function OppCoachHint({ matchStats, score, phase }) {
  if (phase !== 'cards' || isKeyPoint(score)) return null;
  const usage = matchStats?.oppMoveUsage ?? {};
  const top = Object.entries(usage).sort((a, b) => b[1] - a[1])[0];
  if (!top || top[1] < 2) return null;
  const [moveId, count] = top;
  const counter = COUNTER_FOR[moveId];
  if (!counter) return null;
  return (
    <div className="bt-coach-hint">
      <span className="bt-coach-icon">🎯</span>
      <span className="bt-coach-text">
        对手惯用「{MOVES[moveId]?.name}」(×{count})，出「{MOVES[counter]?.name}」可克制
      </span>
    </div>
  );
}
```

在 `cards` 阶段 JSX 中，`CrisisHint` 之前插入：

```jsx
<OppCoachHint matchStats={state.matchStats} score={state.score} phase={state.phase} />
<CrisisHint matchStats={state.matchStats} score={state.score} phase={state.phase} />
```

### `src/modules/tennis/tennis.css`

在 `.bt-crisis-text` 之后新增 3 条规则：

```css
.tennis-scope .bt-coach-hint {
  display: flex; align-items: center; gap: 6px;
  padding: 5px 9px; margin-bottom: 8px; border-radius: 6px;
  background: rgba(103,232,249,.06); border: 1px solid rgba(103,232,249,.18);
}
.tennis-scope .bt-coach-icon { font-size: .85rem; flex-shrink: 0; }
.tennis-scope .bt-coach-text { font-size: .68rem; color: rgba(103,232,249,.80); line-height: 1.4; }
```
