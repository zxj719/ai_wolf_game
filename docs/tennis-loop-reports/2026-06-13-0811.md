## 二、本轮改动

### ✅ achievements.js — 3 个新成就

| ID | 图标 | 名称 | 触发条件 |
|----|------|------|---------|
| `consecAce` | 🔥 | 发球炮台 | `maxConsecAces >= 3`（消费 Round 5 追踪字段） |
| `winStreak5` | 🌊 | 势如破竹 | `longestWinStreak >= 5`（消费 Round 5 追踪字段） |
| `proTouch` | ✨ | 神之触感 | `maxMgMult >= 1.4`（消费 Round 5 追踪字段） |

**设计理由：** 门槛不平凡但可达——`maxConsecAces >= 3` 需要连续 3 发不被接回；`longestWinStreak >= 5` 中等以上玩家有机会；`maxMgMult >= 1.4` 要求操作达到 Pro 级别。

### ✅ workers/auth/tennisProgressLib.js — 白名单扩充

ACHIEVEMENT_IDS 新增 `'consecAce', 'winStreak5', 'proTouch'`，确保 D1 云端 PUT 不因枚举外 ID 被拒。

### ✅ useTennisGame.js — state 扩展

- `initialState.lastNewAchievements: []`  
- `MATCH_OVER` case 保存 `action.newAchievements ?? []`

### ✅ TennisRoute.jsx — onSingleMatchOver 成就触发

```js
if ((matchStats.maxConsecAces ?? 0) >= 3) achievements.add('consecAce');
if ((matchStats.longestWinStreak ?? 0) >= 5) achievements.add('winStreak5');
if ((matchStats.maxMgMult ?? 0) >= 1.4) achievements.add('proTouch');
const newAchievements = [...achievements].filter((id) => !progress.achievements.includes(id));
```

- 有新成就时 toast 优先展示成就名（`🏆 新成就解锁：发球炮台 · 神之触感`），无新成就则展示掉落（原逻辑）
- dispatch `MATCH_OVER` 时附带 `newAchievements`
- ResultScreen 接收 `newAchievements={state.lastNewAchievements}`

### ✅ ResultScreen.jsx — AchievementUnlockCard 组件

新增 `AchievementUnlockCard` 子组件：
- 位置：ending-hero 卡之后（第一眼看到），war-report 之前
- 渲染每个新解锁的成就：icon + 名称 + 描述文字
- 仅当 `newAchievements.length > 0` 时渲染

### ✅ tennis.css — 成就解锁动画

- `@keyframes achieve-pop`：0.5s cubic-bezier 弹出动画（scale+translateY）
- `@keyframes achieve-shimmer`：标题金色渐变流动（2.5s loop）
- `.achieve-unlock-card`：金色边框 + 入场动画
- `.achieve-unlock-grid / .achieve-unlock-item / .achieve-icon / .achieve-name / .achieve-desc`：卡片内布局
