## 二、本轮改动

### 改动 A：ResultScreen「连招高光」

**问题背景**

上轮「本局统计」卡片已展示 `countersWon` 数字，但数字本身缺乏成就感——只有克制流玩家才会注意到这项指标，且没有任何"好棒！"的视觉奖励。如果玩家打出 2+ 次克制得分，值得一个额外的高光行来强化技巧感。

**实现方式**

在 `ResultScreen.jsx` 的本局统计卡片中，`match-stat-grid` div 末尾添加：
```jsx
{matchStats.countersWon >= 2 && (
  <p className="counter-hl">🎯 本场克制得分 {matchStats.countersWon} 次！</p>
)}
```

在 `tennis.css` 末尾添加（1 行）：
```css
.tennis-scope .counter-hl { text-align: center; margin: .6rem 0 0; font-size: .85rem; color: var(--clay); font-weight: 700; }
```

**设计决策**：
- 阈值 `>= 2` 而非 `>= 1`：单次克制可能是偶然，两次以上说明玩家在有意识地使用克制策略
- 颜色使用 `--clay`（`#e2814f`，橙色）：与 `ms-val` 数字、`ladder-streak` 徽章同色系，"成就感"视觉语言统一
- 位于 `match-stat-grid` 之后（而非内部），视觉上是对格状数据的总结性升华
- 不需要新数据——`matchStats.countersWon` 已在 battleReducer 追踪，本次只是展示

### 改动 B：LadderScreen「下一站对手历史胜率」

**问题背景**

家族挑战赛间屏（`between` 状态）目前只说"下一站对手更强——赛间做点什么？"，但不告诉玩家下一站是谁。玩家需要在特训/按摩/逛店中做出理性选择，如果知道下一站对手自己过去赢过还是输过，可以更好地规划（如：对大妈历史 0 胜，特训比按摩更有价值）。

**数据来源**：`loadLocalRecords()`——从 `localStorage` 读取，无后端依赖，无延迟。

**实现方式**

在 `LadderScreen.jsx` 中：

1. 新增 import：
   ```js
   import { loadLocalRecords } from '../localBoard';
   ```

2. 在组件顶层（hook 调用之后，早于 early return）计算：
   ```js
   const nextOppForBetween = ladder.status === 'between' ? ladder.opponents[ladder.stage + 1] : null;
   const vsNextHistory = nextOppForBetween
     ? loadLocalRecords().filter((r) => r.p === basePlayer.name && r.o === nextOppForBetween.name)
     : [];
   const vsNextWins = vsNextHistory.filter((r) => r.sp > r.so).length;
   ```

3. 在 `between` 状态的现有 hint 段落之后、opts 之前，插入预览行：
   ```jsx
   {nextOppForBetween && (
     <p className="hint" style={{ marginBottom: 14 }}>
       🔮 下一站：{nextOppForBetween.face} {nextOppForBetween.name}
       {vsNextHistory.length > 0
         ? `，历史 ${vsNextHistory.length} 战 ${vsNextWins} 胜 ${vsNextHistory.length - vsNextWins} 负`
         : '，首次对阵'}
     </p>
   )}
   ```

**设计决策**：
- `between` 状态下 `ladder.stage` 尚未递增（只有 INTERMISSION/SHOP_DONE 才触发 `advanceStage()`），故 `ladder.opponents[ladder.stage + 1]` 即下一站索引，不会越界（`between` 仅在 `stage < STAGE_COUNT - 1` 时出现）
- 首次对阵显示「首次对阵」而非「0 战 0 胜 0 负」，语义更友好
- 历史数据 filter 条件：`r.p === basePlayer.name && r.o === nextOpponent.name`——确保只计算当前选择角色对阵该对手的历史，不混入其他角色的战绩
- `loadLocalRecords()` 返回最多 50 条记录，filter 操作 O(50) 可忽略不计，每次 render 执行一次无性能顾虑
- 使用已有 `.hint` CSS 类，零新增样式
