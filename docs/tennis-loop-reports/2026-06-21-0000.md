## 二、本轮改动

### 问题背景

商店 `ShopPanel` 内容较多（盲盒 3 个 + 战术卡购买 + 卡牌管理 + 装备架 + 装备升级），在移动端需要滚动约 3-4 屏。当前唯一的退出入口是顶部 `.shop-head` 里的「离开商店」按钮：

1. **滚动摩擦**：用户翻完商店后必须**滚回顶部**才能退出，增加了不必要的导航成本。
2. **拇指可达区缺失**：移动端底部是最易触达的区域（拇指自然落点），顶部按钮在单手操作时需要大范围移动。
3. **商店体验连贯性受损**：用户在底部浏览完商品后，无法就地结束流程。

### 修复方案

**双出口设计**：保留顶部快速 `×` 图标（让不需要滚动就确认退出的用户快速离开），在所有商店内容之后新增一个全宽底部退出按钮，两条路径各服务不同使用习惯。

#### 视觉设计

- **顶部 `×`**：原按钮文字改为 `×`，加 `aria-label="关闭商店"`，低不透明度（0.6）让视觉不抢焦点，hover 时恢复并显示边框
- **底部「退出商店」**：全宽（max-width 320px 居中），`btn ghost` 风格与商店整体色系一致，上方用虚线分隔条与商品区分离

#### 实现细节

**`src/modules/tennis/meta/ShopPanel.jsx`（+5 行）**

顶部改动（1 行）：
```jsx
// 前：<button className="btn ghost" onClick={onClose}>离开商店</button>
// 后：
<button type="button" className="btn ghost shop-close-x" onClick={onClose} aria-label="关闭商店">×</button>
```

底部新增（4 行 JSX）：
```jsx
<div className="shop-exit-bar">
  <button type="button" className="btn ghost shop-exit-btn" onClick={onClose}>退出商店</button>
</div>
```

**`src/modules/tennis/tennis.css`（+4 行）**

```css
.tennis-scope .shop-close-x { padding: 4px 10px; border-radius: 50%; font-size: 1.2rem; opacity: .6; border-color: transparent; }
.tennis-scope .shop-close-x:hover { opacity: 1; background: rgba(13,59,46,.1); border-color: var(--court); }
.tennis-scope .shop-exit-bar { margin-top: 20px; padding-top: 16px; border-top: 2px dashed rgba(13,59,46,.18); display: flex; justify-content: center; }
.tennis-scope .shop-exit-btn { width: 100%; max-width: 320px; font-size: 1rem; border-radius: 12px; padding: 14px; }
```

### 用户体验前后对比

| 场景 | 改动前 | 改动后 |
|------|--------|--------|
| 移动端浏览到商店底部 | 必须滚回顶部才能退出 | 底部「退出商店」按钮就在眼前 |
| 快速扫一眼就不买 | 点「离开商店」文字按钮 | 点顶部 `×` 图标，同等便捷 |
| 无障碍访问 | 按钮无语义标签 | `aria-label="关闭商店"` 补全 |
| 单手操作移动端 | 拇指需要大范围滑动 | 底部触达区一键退出 |
