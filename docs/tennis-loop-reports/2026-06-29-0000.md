## 三、本轮改动

### 改动：角色近 3 局走势点（SelectScreen roster chip）

**问题背景**

R97 下轮计划第 1 项：「角色对战历史的「最近 3 局走势」小图标（W/L序列），帮助玩家选角前回顾状态」。当前 roster chip 已展示 `出战 N · 赢 N` 总体统计，但总体胜率无法反映"最近手感"——玩家可能对某角色历史胜率 80%，但连输 3 场，就需要走势点来快速感知。

**设计原则**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 走势窗口 | 最近 3 场（默认 n=3） | 3 个点视觉紧凑，足够反映短期手感；5 个点在 96px 格子里偏拥挤 |
| 显示时机 | 始终显示（有历史即显示） | 无需先选择，所有角色的近期状态一目了然；选角更主动 |
| 排列顺序 | 旧→新（左→右） | 符合时间轴直觉，"最后一场"在右边 |
| 颜色 | 绿（#4caf76）胜 / 红（rgba 半透明）败 | 绿色与球场主色协调；败局用半透明红避免过于刺眼 |
| 点大小 | 6px 圆 | 在 96px 最小宽度 chip 中，3 × 6px + 2 × 3px gap = 24px，绰绰有余 |
| 无记录时 | 不渲染（recentResultsMap 不含该角色） | 新玩家的芯片保持简洁，不出现空白点位 |
| n=0 边界 | `if (n <= 0) return []` | `Array.slice(-0)` = `slice(0)` = 全量，需显式保护 |

**实现路径**

#### `src/modules/tennis/localBoard.js`
- 新增 `computeRecentResults(records, playerName, n=3)` 导出函数
- 过滤指定玩家记录 → `.slice(-n)` → `.map(r => r.sp > r.so)` 返回 boolean[]
- 边界：`n <= 0` 立即返回 `[]`（修复 JS `slice(-0)` 语义歧义）

#### `src/modules/tennis/components/SelectScreen.jsx`
- 导入 `computeRecentResults`
- 新增 `recentResultsMap` useMemo（依赖 `records`）：遍历 CHARS，为每个有记录的角色存最近 3 局结果
- roster chip 内 `char-stat` 之后插入 `.char-trend` 行，包含 `n` 枚 `.ct-dot`（胜+`.ct-w` / 败+`.ct-l`）

#### `src/modules/tennis/tennis.css`
- 新增 `.char-trend`：`display: flex; justify-content: center; gap: 3px; margin-top: 5px`
- 新增 `.ct-dot`：6×6px 圆形 `flex-shrink: 0`
- 新增 `.ct-w`：绿色背景
- 新增 `.ct-l`：半透明红背景

#### `src/modules/tennis/__tests__/localBoard.test.js`
- 新增 9 个 `computeRecentResults` 测试用例：
  - 空记录/无该玩家记录 → []
  - 单场胜/单场败 → [true]/[false]
  - 3 场记录 → 正确序列
  - 超 3 场（5 条）→ 最近 3 条（含正确旧→新顺序）
  - n=1 → 仅最后 1 条
  - n=0 → []（边界）
  - 多玩家混合 → 只统计指定玩家
  - sp===so 平局 → false

**三种展示路径**

| 场景 | 结果 |
|------|------|
| 该角色有 ≥1 条历史记录 | 显示最近 min(played, 3) 枚点 |
| 该角色无历史记录 | 不显示 `.char-trend`（recentResultsMap 无该键） |
| 历史记录仅 1-2 条 | 显示 1-2 枚点（无"占位空白点"） |
