## 三、本轮改动

### 功能：「均势对阵」双路提示（matchup-balanced）

**背景与动机**

R113–R117 建立了从选对手到赛后复盘的完整五节点信息链。其中 SelectScreen 的 `matchupMap` 和 PrepScreen 的 `matchupAdviceHint` 都只在存在直接 COUNTER_PAIR 时才渲染内容。通过人工分析 7 人 × 6 组合，发现至少存在结构性空白：

**莹（topspin/slice/lob/passingShot）vs Ross（flatDrive/smash/passingShot/slice）**：
- 9 个 COUNTER_PAIRS 逐一检查，没有任何一对满足「莹的招 → Ross 的招」方向 → `matchupMap['Ross']` 为 `undefined`，两个屏幕都静默渲染空白。

这种情况下玩家看到的是信息真空，无法判断"这是引导系统没有加载出来还是真的没有策略建议"。

**本轮设计**

| 对阵情况 | SelectScreen | PrepScreen |
|---------|-------------|-----------|
| 有直接克制 | 🎯 {克制招} + W/N（琥珀黄） | ⚔️ 克制参考：出{克制招}…（琥珀黄框）|
| 无直接克制 | ⚡ 均势（绿色，本轮新增）| ⚡ 均势对阵：读招节省体力…（绿色框，本轮新增）|

**颜色选择**：
- 琥珀黄（amber）= 有具体优势、明确行动方向
- 绿色（rgba 134,239,172）= 中立、无直接优劣，提示玩家换策略（均衡加点/读招优先）

**PrepScreen 实现细节**

新增 `balancedMatchupHint` useMemo，依赖 `matchupAdviceHint`：
- 仅当 `matchupAdviceHint === null` 且 `prepRound === 0` 且有玩家/对手时触发
- 即「有克制时不显示均势提示」，两者互斥，逻辑清晰

**SelectScreen 实现细节**

在 `opp-history-panel` 的每个 chip 末尾，新增 `picked && c.n !== picked && !matchupMap[c.n]` 条件渲染：
- `c.n !== picked` 防止玩家自身的 chip 误显示（`matchupMap` 已 skip self，但 `!matchupMap[c.n]` 对 picked 本身也为 true）

## 四、实现路径

### `src/modules/tennis/components/PrepScreen.jsx`

在 `matchupAdviceHint` useMemo 之后新增：
```js
const balancedMatchupHint = useMemo(() => {
  if (state.prepRound !== 0 || !state.player || !state.opp || matchupAdviceHint) return null;
  return `⚡ 均势对阵：${state.opp.face}${state.opp.name} 与你无直接克制——读招节省体力，三维均衡加点是关键`;
}, [state.prepRound, state.player?.name, state.opp?.name, matchupAdviceHint]); // eslint-disable-line react-hooks/exhaustive-deps
```

在 `matchupAdviceHint` 渲染之后新增：
```jsx
{balancedMatchupHint && (
  <p className="matchup-advice-hint matchup-advice-balanced">{balancedMatchupHint}</p>
)}
```

### `src/modules/tennis/components/SelectScreen.jsx`

在 `matchupMap[c.n]` 渲染块之后新增：
```jsx
{picked && c.n !== picked && !matchupMap[c.n] && (
  <span className="matchup-hint matchup-balanced" aria-label="均势对阵，无直接克制">
    ⚡ 均势
  </span>
)}
```

### `src/modules/tennis/tennis.css`

在 `.matchup-hint` 和 `.matchup-wr-sub` 之后新增 2 条规则：
```css
.tennis-scope .matchup-hint.matchup-balanced { color: rgba(134,239,172,.65); border-color: rgba(134,239,172,.2); background: rgba(134,239,172,.04); }
.tennis-scope .matchup-advice-hint.matchup-advice-balanced { color: rgba(134,239,172,.82); background: rgba(134,239,172,.07); border-left-color: rgba(134,239,172,.38); }
```
