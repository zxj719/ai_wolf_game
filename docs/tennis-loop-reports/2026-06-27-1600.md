## 三、本轮改动

### 改动：LocalBoard 本地家族榜角色筛选

**问题背景**

R90 为全网榜加入了角色筛选 chip，让玩家能按角色视角审视他人战绩。本地家族榜（家人们的共享战绩列表）存在同样的需求——尤其在多人共用同一设备时，「看看我用🐯诚打了多少场」是天然高频问题。本地记录的 `r.p` 字段已存储角色名（如 `"诚"`、`"Elza"`），与 `CHARS[i].n` 完全匹配，纯前端过滤即可，无需 schema 变更。

**设计决策**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 筛选字段 | `r.p`（玩家角色名） | 已有字段，直接匹配 CHARS；`r.pf` 是 emoji，展示用非过滤用 |
| chip 来源 | 固定 `CHARS` 数组（与 GlobalBoard 相同） | 始终展示 7 个角色，不依赖当前本地是否有数据 |
| 空态文案 | `还没有以 🐯 诚 出战的记录 —— 快来挑战吧！` | 鼓励玩家选该角色出战 |
| 样式 | 复用 R90 的 `.lb-filter-chips / .lb-fc / .lb-fc.active` | 无新 CSS；双榜视觉完全统一 |
| 早返回移除 | 原 `if (!records.length) return` 前置检查已移除 | 过滤态下 `sorted.length === 0` 但 `records` 非空时需渲染 chip 行和空态提示，不能提前退出 |

**实现路径**

#### `src/modules/tennis/components/Leaderboard.jsx`

`LocalBoard` 函数改动（+25 行）：

1. 新增 `const [charFilter, setCharFilter] = useState(null)`
2. 计算 `filteredRecords`（charFilter 为 null 时退化为全量）
3. `sorted = sortLocalRecords(filteredRecords)`（原来直接传 `records`）
4. 计算 `activeChar`（用于空态文案）
5. 渲染 `.lb-filter-chips` chip 行（复用 GlobalBoard 同款 JSX 结构）
6. 原 `if (!records.length)` 提前返回 → 改为 `sorted.length === 0` 条件渲染空态段落
7. 原 `<table>` 包进 `else` 分支（已有内容不变）

**三种交互路径**

| 操作 | 结果 |
|------|------|
| 点击角色 chip（未激活） | 激活该角色，只展示该角色出战的记录 |
| 点击同一 chip（已激活） | 取消筛选，恢复全部记录 |
| 点击「全部」 | 取消筛选，恢复全部记录 |

**零 CSS 改动**

R90 已在 `tennis.css` 中添加 `.lb-filter-chips`、`.lb-fc`、`.lb-fc.active` 三条规则。本轮 LocalBoard 直接复用，两个榜单视觉完全对称。
