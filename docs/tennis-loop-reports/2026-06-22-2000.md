## 二、本轮改动

### 改动：LadderScreen「赛间 AI 建议」

**问题背景**

上轮（第 61 轮）已在 `between` 屏添加「🔮 下一站」历史胜率展示，但信息的展示是中性的——玩家知道了「历史 3 战 1 胜 2 负」，却不一定能快速得出「那我应该选特训」的结论。在三选一决策点（特训/按摩/逛店），玩家等待时间短，需要即时的策略锚点。

**实现方式**

在 `LadderScreen.jsx` 中添加纯函数 `getBetweenAdvice(winRate, historyCount)` 和一个额外的 `vsNextWinRate` 计算值：

```js
// 模块级纯函数（组件外）
function getBetweenAdvice(winRate, historyCount) {
  if (historyCount === 0) return '💡 首次对阵，建议加练摸清套路';
  const pct = Math.round(winRate * 100);
  if (winRate < 0.3) return `💡 胜率仅 ${pct}%，强烈建议特训冲刺！`;
  if (winRate >= 0.7) return `💡 胜率 ${pct}%，状态不错，随便选都行`;
  return `💡 胜率 ${pct}%，特训或逛店补装备皆可`;
}

// 组件内
const vsNextWinRate = vsNextHistory.length > 0 ? vsNextWins / vsNextHistory.length : null;
```

JSX 中将原来单个 `<p>` 改为 `<>` Fragment，下方追加建议行：
```jsx
{nextOppForBetween && (
  <>
    <p className="hint">
      🔮 下一站：{face} {name}，历史 N 战 W 胜 L 负
    </p>
    <p className="hint between-suggest" style={{ marginBottom: 14 }}>
      {getBetweenAdvice(vsNextWinRate, vsNextHistory.length)}
    </p>
  </>
)}
```

在 `tennis.css` 末尾追加：
```css
.tennis-scope .between-suggest { font-size: .82rem; color: var(--clay); font-weight: 600; opacity: .88; }
```

**决策逻辑**

| 情景 | 建议文本 | 意图 |
|------|---------|------|
| 首次对阵（0 场历史） | 「首次对阵，建议加练摸清套路」 | 信息量不足，保守建议特训 |
| 胜率 < 30% | 「胜率仅 X%，强烈建议特训冲刺！」 | 明确指向特训，减少犹豫 |
| 胜率 30-69% | 「胜率 X%，特训或逛店补装备皆可」 | 都是合理选择，中性建议 |
| 胜率 ≥ 70% | 「胜率 X%，状态不错，随便选都行」 | 轻松氛围，玩家可自由发挥 |

**设计要点**

- `getBetweenAdvice` 放在模块级（组件外）：纯函数无闭包，无需每次 render 重建
- `vsNextWinRate = null` 当 `vsNextHistory.length === 0`，函数内用 `historyCount === 0` 分支处理，不产生 NaN
- 使用 `<>` Fragment 包裹，避免多余 DOM 节点
- CSS 使用 `opacity: .88` 而非纯白，让建议行在视觉上略低于历史数据行的权重，"建议"本该是参考，不是命令
- 不引入新状态，不新增 import，改动轻量（+14 行）
