## 二、本轮改动

### 核心功能：SelectScreen 成就图鉴总览卡

**问题定位：**

玩家首次进入游戏看到的是 SelectScreen（选角色页）。目前该页面缺乏成就目标引导——玩家需要进入 mode 页（赛事报名）才能看到成就图鉴。这导致：
- 新手不知道应该选哪个角色、有什么长期目标
- 打完一局回到 SelectScreen 时，无法立即对比进度决定下一局选谁

**实现方案（3 文件变更）：**

**1. SelectScreen.jsx（+47 行）**

新增 `AchGalleryCard` 子组件，位于 `PersonalBestCard` 下方、`OpponentPreviewStrip` 上方：

```jsx
function AchGalleryCard({ progress }) {
  const [open, setOpen] = useState(false);
  const [expandedAch, setExpandedAch] = useState(null);
  const unlockedCount = ACHIEVEMENTS.filter((a) => progress.achievements.includes(a.id)).length;
  // toggle + ach-gallery + ach-chip-tip 逻辑（同 mode 页，~35 行）
}
```

关键设计：
- **默认折叠**：避免信息过载，符合渐进披露原则
- **toggle 显示进度**：`🏅 成就图鉴 3/16 ▼`，进度比激发好奇心
- **复用 `.counter-toggle`** 样式（与招式克制速查按钮视觉统一）
- **复用 `.ach-chip*`** 图鉴类（mode 页已有全套，零额外 CSS 成本）
- 点击折叠时重置 `expandedAch`，避免重展开后出现孤立 tip 行

**2. tennis.css（+1 行）**

```css
.tennis-scope .select-ach-wrap { margin-top: 10px; }
```

仅需 1 行 wrapper margin，其余全部复用既有类。

**3. meta/achievements.js** — 无改动（直接 import）

**玩家体验前后对比：**

| 场景 | 改动前 | 改动后 |
|------|--------|--------|
| 首次进入 SelectScreen | 不知道有成就系统 | 底部可折叠「成就图鉴 0/16」提示存在 |
| 选角色时 | 无目标参考 | 展开后可看全部目标 + 锁定成就点击说明 |
| 打完一局回 SelectScreen | 需要进入 mode 页才能看进度 | 直接在 SelectScreen 折叠条看 X/16 |

**设计决策：**

| 决策 | 选择 | 理由 |
|------|------|------|
| 默认折叠 vs 展开 | 折叠 | SelectScreen 已经信息较密，默认展开会分散注意力 |
| 位置 | PersonalBestCard 后、OpponentPreviewStrip 前 | 时序：先看最佳纪录，再看成就目标，最后看对手预告 |
| 样式复用 | counter-toggle + ach-chip* | 零新增 CSS，视觉体系统一 |
| 折叠时重置 expandedAch | 是 | 避免重展开后展示已不可见的悬空 tip |
