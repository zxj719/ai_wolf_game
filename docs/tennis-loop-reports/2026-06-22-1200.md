## 二、本轮改动

### 改动 A：ResultScreen「本场王牌招」高亮

**问题背景**

上轮加入的「本局统计」卡片展示了总球数、ACE、克制得分等聚合指标，但 `matchStats.moveUsage`（记录每个招式使用次数的对象）没有被消化——它在 `battleReducer` 里一直被追踪，却从未呈现给玩家。每位角色的风格由配招集决定（如诚的「重炮平击」是主打），显示「本场最常用的招式」能帮助玩家认识自己的打法偏好并与角色风格产生共鸣。

**实现方式**

在 `ResultScreen.jsx` 中：

1. `import { MOVES } from '../battle/moves';`（多 MOVES 是 named export，路径 `../battle/moves` 正确）
2. 用 `useMemo` 从 `matchStats.moveUsage` 中提取最高频招式：
   ```js
   const topMoveEntry = useMemo(() => {
     if (!matchStats?.moveUsage) return null;
     const entries = Object.entries(matchStats.moveUsage);
     return entries.length ? entries.reduce((a, b) => (b[1] > a[1] ? b : a)) : null;
   }, [matchStats]);
   ```
3. 在 `match-stat-grid` 末尾条件渲染：
   ```jsx
   {topMoveEntry && (
     <div className="ms-item">
       <span className="ms-val">×{topMoveEntry[1]}</span>
       <span className="ms-label">王牌：{MOVES[topMoveEntry[0]]?.name ?? topMoveEntry[0]}</span>
     </div>
   )}
   ```

**设计决策**：
- `ms-val` 显示 `×{count}`（次数作为主指标），`ms-label` 显示招式名（复用 .65rem 小字），与现有 5 个统计项完全一致
- `MOVES[key]?.name ?? key` 安全降级：若招式 ID 不在 MOVES 表中（理论上不会），直接显示 ID
- 无需新增 CSS（复用 `ms-item` / `ms-val` / `ms-label`）
- `topMoveEntry` 为 `null`（无 moveUsage 或 matchStats 未传入）时整个 ms-item 不渲染，不影响其他模式结算屏

### 改动 B：AdventureScreen「章节/步骤进度提示」

**问题背景**

奇幻闯关模式地图有 3 章、每章若干步。玩家在选路时只能看到当前章节名称和迷你地图，没有明确的「还剩几步」文字，中途体力紧张时难以规划营地节点的使用时机。

**实现方式**

在 `AdventureScreen.jsx` 的选路（pick）屏和对战（battle）屏的 `adv-banner` 内各添加一行 `<small>`：

```jsx
<div className="adv-banner">
  {chapter.title}
  <small>第 {run.stepIdx + 1} / {chapter.steps.length} 步 · 第 {run.chapterIdx + 1} / {run.map.chapters.length} 章</small>
</div>
```

**设计决策**：
- `.adv-banner small` 的 CSS 规则（`display: block; font-size: .7rem; color: rgba(242,238,224,.6)`）早已存在但从未被使用，本次是激活而非新增
- `run.map.chapters.length` 取动态值（而非硬编码 3），符合「不写魔数」规范
- battle 节点屏同步添加，与 pick 屏保持信息一致性，玩家在对战中也能随时感知当前进度
- 完全不需要新增 CSS 行
