## 三、本轮改动

### 改动：连胜奖励提示横幅（SelectScreen）

**问题背景**

R96 下轮计划第 1 项：「连胜奖励提示：连胜达到 3/5/10 场时，在 SelectScreen 展示「连胜 N 场！」横幅（纯 localStorage 读取 progressStore，零后端）。这是对现有连败激励徽章（R89）的对称设计——败者有激励，胜者也应有庆祝。」

**设计原则**

| 设计点 | 选择 | 理由 |
|--------|------|------|
| 触发时机 | 选角后（picked 变更时）| 每次换角色即时更新，无延迟 |
| 数据来源 | `loadLocalRecords()` 纯 localStorage | 零后端，instant display |
| 阈值 | 3/5/10 场连胜 | 与 ResultScreen 连胜 badge 阈值对齐（2/3/5） |
| 视觉层次 | clay(3) / gold(5) / legend(10+) | 越稀有越震撼，gold/legend 有发光效果 |
| 动画 | `.ssbFadeIn`（0.45s，向下淡入） | 与 `.screen` 动画同风格，不突兀 |
| CSS 复用 | `.ladder-streak` + `.streak-pulse`/`.streak-gold` | 保持全站连胜视觉一致性 |
| 不显示条件 | streak < 3 或未选角 | 避免 0 连胜时出现空横幅 |

**新增函数：`computeCurrentWinStreak(records, playerName)`**

与现有 `computeStreakCount` 的区别：

| 函数 | 使用场景 | 起点 | 返回值 |
|------|---------|------|--------|
| `computeStreakCount` | ResultScreen（赛后） | 从 1 开始（含当前局） | 1-N |
| `computeCurrentWinStreak` | SelectScreen（赛前） | 从 0 开始（只看历史） | 0-N |

**实现路径**

#### `src/modules/tennis/localBoard.js`
- 新增 `computeCurrentWinStreak(records, playerName)` 导出函数
- 从 records 末尾向前遍历，遇到败局即 break，返回连续胜场数

#### `src/modules/tennis/components/SelectScreen.jsx`
- 导入 `computeCurrentWinStreak`
- 新增 `currentWinStreak` useMemo（依赖 `picked` 和 `records`）
- 在 roster chips 之后、dailyChallenge 横幅之前插入 streak 横幅

#### `src/modules/tennis/tennis.css`
- 新增 `.select-streak-banner`：clay 橙底边框，flex 布局
- 新增 `.ssb-gold`：gold 底色（5 连胜）
- 新增 `.ssb-legend`：更亮 gold + 发光阴影（10 连胜）
- 新增 `@keyframes ssbFadeIn`：向下淡入动画
- 新增 `.ssb-fire`、`.ssb-text`：横幅内容排版

#### `src/modules/tennis/__tests__/localBoard.test.js`
- 新增 9 个 `computeCurrentWinStreak` 测试用例：
  - 空记录/无历史/最近一败 → 0
  - 单场胜/3连胜/5连胜/10连胜 → 正确计数
  - 中间有败局 → 只计末尾连胜
  - 多玩家混合记录 → 只统计指定玩家

**横幅文案设计**

| 连胜数 | 横幅文案 | 视觉 |
|--------|---------|------|
| 3–4 场 | 「🔥 连胜 N 场！状态正热，继续！」 | clay 底色 + streak-pulse |
| 5–9 场 | 「🔥🔥 连胜 N 场！势不可挡，挑战新对手！」 | gold 底色 + streak-pulse |
| ≥10 场 | 「🔥🔥🔥 连胜 N 场！家族传奇！再战证明霸主地位！」 | legend 底色 + streak-gold |
