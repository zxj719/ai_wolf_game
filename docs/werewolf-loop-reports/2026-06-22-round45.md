## 2. 问题诊断（PK 辩护框架同构缺口评估）

### 缺口分析

`generateUserPrompt` 中 `pkHint` 的构建块（`if (roleParams.pkMode)`）在 R45 之前覆盖状态：

| 角色 | R35 前 | R35 | R36 | R45 前 |
|------|--------|-----|-----|--------|
| 预言家 | 通用 | ✅ 专属 | - | ✅ |
| 守卫 | 通用 | ✅ 专属 | - | ✅ |
| 女巫 | 通用 | - | ✅ 专属 | ✅ |
| 猎人 | 通用 | - | ✅ 专属 | ✅ |
| **骑士** | 通用 | - | - | **❌ 通用（缺口）** |
| **摄梦人** | 通用 | - | - | **❌ 通用（缺口）** |
| **魔术师** | 通用 | - | - | **❌ 通用（缺口）** |

### 为什么这三个角色需要专属框架

**骑士**（R32 SHERIFF_SPEECH 模式扩展到 PK）：
- 未使用技能时：隐性威慑是最强价值主张——好人阵营有个"尚未落地的确定性判断机会"，淘汰骑士等于放弃这个机会
- 已使用技能后：物理验证结果不可伪造，决斗记录是可信度最高的论据
- SHERIFF_SPEECH 已有骑士专属框架（R32）；PK 是类似但不同的场景，同理应专属化

**摄梦人**（R31 SHERIFF_SPEECH 模式扩展到 PK）：
- 已揭示且有活跃连接目标时：同生共死关联是独有的 PK 筹码——淘汰摄梦人等于同时失去连接目标（2 张牌的损失）
- 通用框架"陈述存活价值"无法覆盖这个独特机制

**魔术师**（R31 SHERIFF_SPEECH 模式扩展到 PK）：
- 已揭示且有交换记录时：交换记录修正了场上信息不对称，魔术师在场才能继续澄清（"场上所有人以为验了A，实际验的是B"）
- 通用框架无法表达信息修正能力的独特竞技价值

### 博弈论依据

Wang 2025 (arxiv:2408.17177) §4.2 指出："在对话博弈中，持有私有信息的玩家在公开场合的策略应最大化信息优势——过度泛化的价值主张（'我是好人'）等同于主动放弃信息优势，而精确的可量化论据（决斗验证/药量/交换记录）才能最大化说服力。"
