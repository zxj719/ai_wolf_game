**LEARNINGS 误记**：

- LEARNINGS 标注"已完成"实为对历史计划的标注，非对现有代码的验证

### 2. 修复内容：knight.js `getKnightDaySpeechPrompt`

**修改前（第 71 行）**：
```js
const { hasUsedDuel } = params;
```

**修改后**：
```js
const { hasUsedDuel, aliveCount: aliveNow = 8 } = params;

// R51: endgame dynamic thresholds
const isEndgame = aliveNow <= 5;
const thresholdA = isEndgame ? 50 : 70;
const thresholdB = isEndgame ? 40 : 60;
const endgameNote = isEndgame
  ? `...残局模式（存活${aliveNow}人）...`
  : '';
```

**Prompt 内三处硬编码替换**：

| 位置 | 修改前 | 修改后 |
|------|--------|--------|
| 阶段2决斗期 | `把握度≥70%` | `把握度≥${thresholdA}%` |
| 优先级A说明 | `确定度 ≥ 70%` | `确定度 ≥ ${thresholdA}%` |
| 优先级B说明 | `确定度 ≥ 60%` | `确定度 ≥ ${thresholdB}%` |
| Step3 决斗条件 | `（≥60%）` | `（≥${thresholdB}%）` |

### 3. 行为变化

| 场景 | 修改前 | 修改后 |
|------|--------|--------|
| 正常局（6-9人存活） | A=70%, B=60%（硬编码） | A=70%, B=60%（无残局注记） |
| 残局（≤5人存活） | A=70%, B=60%（仍硬编码） | A=50%, B=40%（+残局模式注记） |
| 极残局（3-4人存活） | 骑士因阈值过高不决斗 | 骑士以更低把握度可以果断决斗 |

**理论依据**：Wang 2025（arxiv:2408.17177）残局博弈：玩家数减少→信息收集机会减少→等待代价增加→最优策略应降低行动门槛。
