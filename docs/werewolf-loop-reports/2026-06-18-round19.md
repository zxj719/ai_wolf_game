### 优化 1：村民 identity_table 追加格式示例

**位置**：`ROLE_DAY_SPEECH_PROMPTS['村民']` 第 4 条指导（"每轮更新"）之后

**变更前**：
```
- 每轮更新：在上轮 reason 基础上**追加**本轮新观察，不要覆盖历史——模式是多轮积累出来的
```

**变更后**：
```
- 每轮更新：在上轮 reason 基础上**追加**本轮新观察，不要覆盖历史——模式是多轮积累出来的
  【追加示例】上轮 reason="N1发言带节奏" → 本轮末尾补充变为："N1发言带节奏；N2投好人——证实可疑，本轮投票对象"
```

**设计逻辑**：
- 示例展示了分号（；）拼接格式，让 LLM 精确知道怎么"追加"
- `→` 箭头形成"before → after"对比，LLM 在 in-context learning 中对这种对比格式极为敏感
- 示例包含轮次标记（N1/N2）和行为分类（带节奏/投好人/证实可疑），提供了完整的填写范式

---

### 优化 2：猎人 identity_table 追加格式示例

**位置**：`ROLE_DAY_SPEECH_PROMPTS['猎人']` "开枪前回顾"指令之后

**变更前**：
```
- 开枪前回顾上轮 identity_table：若某人 reason 已记"开枪优先级：高"，优先锁定
```

**变更后**：
```
- 开枪前回顾上轮 identity_table：若某人 reason 已记"开枪优先级：高"，优先锁定
  【追加示例】3号 N1 reason="开枪备选：发言可疑" → N2 末尾追加为"开枪备选：发言可疑；N2再攻真预（证实危险）——升为开枪优先"
```

**设计逻辑**：
- 示例展示了优先级"升级"路径（备选 → 优先），让 LLM 知道跨轮积累不只是文字拼接，还应反映优先级变化
- "3号"具体化使示例更真实，避免 LLM 只把它当作抽象指令
- "证实危险"关键词是从 N1 备选到 N2 升级的触发条件，给出了条件性追加的范式
