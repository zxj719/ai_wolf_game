## 2. 问题诊断（LEARNINGS.md 第 92 条审计）

### 审计范围
LEARNINGS 第 92 条（R42 新增）：骑士/摄梦人/魔术师的委托模块 DAY_SPEECH identity_table 写指导是否也存在 DAY→NIGHT 关键词不一致问题。

### 审计结论

| 角色 | 结论 |
|------|------|
| 骑士（knight.js） | ✅ 不适用——骑士无夜间行动（决斗是白天技能），无 NIGHT_* Step 0 需要对齐 |
| 摄梦人（dreamweaver.js） | ❌ 同构缺口——DAY_SPEECH 无写指导；NIGHT_DREAMWEAVER（内联于 aiPrompts.js）无 Step 0；本轮修复 |
| 魔术师（magician.js） | ❌ 同构缺口——DAY_SPEECH 无写指导；getMagicianNightActionPrompt 无 Step 0；本轮修复 |

### 根本原因
R38-R41 修复了 wolf/witch/guard/seer 四个主要角色的读写闭环，但这四个角色均以内联方式在 `aiPrompts.js` 中实现 NIGHT_* case。摄梦人和魔术师的夜间行动混合使用了两种模式（NIGHT_DREAMWEAVER 是内联的；NIGHT_MAGICIAN 委托给 `getMagicianNightActionPrompt`），审计时都没有被 R38-R41 的系统性检查覆盖到。这与 R21 教训（同构 bug 需要主动迁移）完全符合。
