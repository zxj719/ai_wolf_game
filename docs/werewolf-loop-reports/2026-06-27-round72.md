## 本轮优化：摄梦人/魔术师 DAY_VOTE 读写闭环补完

### 问题诊断

R72 审计发现：
- 摄梦人 DAY_SPEECH 写 `"连梦候选"` 到 identity_table（R60 新增），NIGHT_DREAMWEAVER Step 0 读取（R43）——但 **DAY_VOTE 从未读取**
- 魔术师 DAY_SPEECH 写 `"换刀候选"` 到 identity_table（R60 新增），NIGHT_MAGICIAN Step 0 读取（R43）——但 **DAY_VOTE 从未读取**

这是与猎人(R62)、骑士(R63)完全同构的"identity_table 候选写入→DAY_VOTE 不读取"缺口。摄梦人多轮积累的进攻候选、魔术师多轮积累的换刀候选，在投票阶段完全被浪费。

### 修复内容（`src/services/aiPrompts.js`，DAY_VOTE case）

#### 摄梦人投票策略（梦票对齐原则）

```javascript
// 梦票对齐：进攻模式连梦候选 → 投票优先淘汰（节省入梦能力）；防御候选不投
const dreamweaverVoteStrategy = `2. 【摄梦人投票—梦票对齐框架】
   a) 先查【身份推理表】identity_table 中 reason 含"连梦候选"的存活玩家——投票首选与入梦方向对齐（梦票对齐：投票出局 > 夜间入梦连梦，前者更省资源）
   b) 场景评估：今天能推票出局 → 全力推票（直接淘汰，不需消耗入梦能力）；今天无法翻转票型 → 仍然投他，积累好人阵营公共认知，夜间再入梦强化追杀
   c) 防御保护候选（reason含"防御入梦候选"）的玩家方向相反——他们是核心好人，请投向进攻方向候选或查杀目标
   d) 无"连梦候选"标注时：跟投预言家查杀 > 发言逻辑最崩塌的玩家 > 历史热力最高的目标`;
```

**设计要点**：
- 梦票对齐：连梦候选（进攻目标）应优先通过投票出局而非消耗入梦能力
- 防御反向：防御入梦候选是需要保护的核心好人，投票方向相反
- 白熊效应合规：所有分支正向描述 ✅

#### 魔术师投票策略（换票对齐原则）

```javascript
// 换票对齐：换刀候选是强狼嫌疑 → 投票出局比夜间换刀更直接可靠
const magicianVoteStrategy = `2. 【魔术师投票—换票对齐框架】
   a) 先查【身份推理表】identity_table 中 reason 含"换刀候选"的存活玩家——投票首选与换刀方向对齐（换票对齐：投票直接淘汰 > 夜间换刀风险转移）
   b) 场景评估：今天能推票出局 → 全力推票（直接淘汰比夜间交换成功更可靠稳定）；今天无法翻转票型 → 仍然投他，为好人阵营建立公共嫌疑认知，夜间换刀策略作为补充保障
   c) 无"换刀候选"标注时：跟投预言家查杀 > 发言逻辑最崩塌的玩家 > 历史热力最高的目标`;
```
