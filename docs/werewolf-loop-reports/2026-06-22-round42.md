## 2. 问题诊断

### 发现方式
人工审计（R41 建议项 4：identity_table 全量人工审计）：逐一比对所有角色 DAY_SPEECH identity_table 写指导与对应 NIGHT_* Step 0 读取关键词。

### 根本原因：DAY→NIGHT 与 NIGHT→NIGHT 是两条独立的关键词流

```
信息流图（修复前）：

白天 DAY_SPEECH 写入：
  狼人: "刀口候选：X"     ← ❌ NIGHT_WOLF Step 0 读 "高优先刀口" → 永不读到
  预言家: "下次查验候选"  ← ❌ NIGHT_SEER Step 0 读 "排队查验优先级" → 永不读到
  女巫: "毒药备选"        ← ❌ NIGHT_WITCH Step 0 读 "毒药优先候选" → 永不读到
  守卫: "守护优先级高"    ← ❌ NIGHT_GUARD Step 0 读 "守护优先级：高" → 永不读到

夜晚 NIGHT_* 写入 → 下轮 NIGHT_* 读取（N→N 流）：
  NIGHT_WOLF 写 "高优先刀口"   → NIGHT_WOLF Step 0 读 "高优先刀口"   ✅
  NIGHT_SEER 写 "排队查验优先级" → NIGHT_SEER Step 0 读 "排队查验优先级" ✅
  NIGHT_WITCH 写 "毒药优先候选"  → NIGHT_WITCH Step 0 读 "毒药优先候选"  ✅
  NIGHT_GUARD 写 "守护优先级：高" → NIGHT_GUARD Step 0 读 "守护优先级：高" ✅
```

R38-R41 修复的是 NIGHT→NIGHT 闭环，但从未检查 DAY→NIGHT 的关键词是否对齐。白天积累的信息（白天观察到谁可疑、谁需要守护、谁是毒药候选）在 41 轮内从未能流入夜间 Step 0——这些数据只在当轮 DAY_SPEECH 时存在，次日夜间 Step 0 完全看不到它们。

### 影响分析

| 角色 | 未对齐前的效果 | 修复后预期效果 |
|------|--------------|--------------|
| 狼人 | 白天标注的"刀口候选"夜间 Step 0 找不到，每晚从头评估所有目标 | 白天标注的"高优先刀口"夜间 Step 0 直接召回，刀口决策有跨日连续性 |
| 预言家 | 白天发言时标注的"查验候选"夜间 Step 0 找不到，只能利用上轮夜间的标注 | 白天也能记录"排队查验优先级"，夜间 Step 0 召回时包含白天新观察 |
| 女巫 | 白天标注的"毒药备选"夜间 Step 0 找不到，历史毒药候选只包含夜间标注 | 白天"毒药优先候选"标注可被夜间 Step 0 读取，跨白天的连续性修复 |
| 守卫 | 白天标注的"守护优先级高"（无冒号）夜间 Step 0 读不到 | 白天"守护优先级：高"（含冒号）可被夜间 Step 0 读取 |

猎人（R41 修复的唯一例外）：DAY_SPEECH 写"开枪优先级：高" → HUNTER_SHOOT 读"开枪优先级：高" ✅ 之前就已对齐。
