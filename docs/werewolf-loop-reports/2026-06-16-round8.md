## 2. 本轮优化详情

### 2a. 骑士 DAY_SPEECH 接通（关键 Bug 修复）

**文件**：`src/services/aiPrompts.js`（`ROLE_DAY_SPEECH_PROMPTS` 对象，约 line 1116）

**问题**：骑士白天发言一直 fallthrough 到村民模板，输出 JSON 格式为：
```json
{"thought":"...","speech":"...","voteIntention":...,"identity_table":{...}}
```
而 `useSpeechFlow.js:265` 检查的是 `res.shouldDuel`：
```javascript
if (currentSpeaker.role === ROLE_DEFINITIONS.KNIGHT && res.shouldDuel) {
    await handleKnightDuel(currentSpeaker, res.duelTarget, res.duelReason, res.confidence);
```
村民模板从不输出 `shouldDuel: true`，所以骑士 AI 整局无法决斗——决斗机制完全失效（静默 bug）。

**修复**：在 ROLE_DAY_SPEECH_PROMPTS 中添加骑士条目：
```javascript
'骑士': (ctx, params) => getRoleModule('骑士').daySpeech(ctx, params),
```
骑士模块的 `daySpeech`（`getKnightDaySpeechPrompt`）输出包含：
```json
{"thought":"...","speech":"...","shouldDuel":true/false,"duelTarget":数字或null,"duelReason":"...","confidence":0-100,"voteIntention":...,"identity_table":{...}}
```

**影响**：AI 骑士现在可以在发言中声明决斗、翻牌亮身份，完整激活决斗机制。

### 2b. 摄梦人/魔术师 DAY_SPEECH 接通

**文件**：`src/services/aiPrompts.js`（`ROLE_DAY_SPEECH_PROMPTS`）

**问题**：摄梦人和魔术师同样 fallthrough 到村民模板，丧失：
- 摄梦人：入梦历史分析、连梦击杀决策、防御/进攻/殉情三模式逻辑
- 魔术师：逻辑镜像表（交换重定向后的真实技能目标）、是否跳身份判断

**修复**：
```javascript
'摄梦人': (ctx, params) => getRoleModule('摄梦人').daySpeech(ctx, params),
'魔术师': (ctx, params) => getRoleModule('魔术师').daySpeech(ctx, params),
```

**同步的 roleParams 扩展**（在 DAY_SPEECH case 的 roleParams 构建中）：
```javascript
// 特殊神职私有状态
hasUsedDuel: currentPlayer?.hasUsedDuel,
dreamHistory: gameState.dreamweaverHistory || { dreamedPlayers: [], lastDreamTarget: null },
lastDreamTarget: gameState.dreamweaverHistory?.lastDreamTarget ?? null,
swappedPlayers: gameState.magicianHistory?.swappedPlayers || [],
lastSwap: gameState.magicianHistory?.lastSwap || null,
```

这些字段不影响现有的 6 个角色（额外字段被忽略），但让摄梦人和魔术师的提示词函数能读取私有状态。

### 2c. LAST_WORDS 骑士/摄梦人/魔术师 专属分支

**文件**：`src/services/aiPrompts.js`（`PROMPT_ACTIONS.LAST_WORDS` case）

**问题**：3 个特殊神职遗言走通用"好人"分支，丧失各自独有的死亡时刻信息。

**修复**：在守卫分支后添加 3 个分支：

**骑士**：
- 读取 `currentPlayer.hasUsedDuel` 区分"已决斗/未决斗"
- 已决斗：传递最终身份判断
- 未决斗：告知好人最高嫌疑目标（接替决斗机会）

**摄梦人**：
- 读取 `gameState.dreamweaverHistory`（入梦历史 + 活跃连接）
- 如果 `currentDreamTarget ?? lastDreamTarget != null`，触发"⚠️ 同生共死"警告
- 公开完整入梦名单帮助好人还原死讯逻辑

**魔术师**：
- 读取 `gameState.magicianHistory`（交换历史）
- 公开最近一次交换（含"技能目标已重定向"的详细说明）
- 这是修正场上逻辑偏差的唯一途径（否则好人可能一直基于错误的验人信息）
