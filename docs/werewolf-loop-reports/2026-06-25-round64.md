## 问题识别与分析

### SHERIFF_BADGE_PASS identity_table 读写闭环缺口（R63 建议候选）

**问题描述**：
- SHERIFF_BADGE_PASS 是好人警长死亡时的最终动作（每局最多触发一次）
- 好人警长通过 D2+ 的每轮 DAY_SPEECH 和每晚 NIGHT_* 积累了丰富的 identity_table（confidence 分值 + reason 字段）
- 原来的 `bpHint` = "金水>真预言家>发言可信者" 是完全凭空判断，未利用积累的历史推理
- 这是读写闭环的"终局点"缺口：写了 D1→D(n-1) 的身份推理，但在最关键的 D(n) 传徽时完全不读

**与 R38-R54 系列的同构性**：
| 场景 | 写侧 | 读侧（修复前） | 读侧（修复后） |
|------|------|-------------|-------------|
| NIGHT_WOLF | DAY_SPEECH 写"高优先刀口" | 从头分析 | Step0 读取 R38 |
| NIGHT_WITCH | DAY_SPEECH 写"毒药优先候选" | 从头分析 | Step0 读取 R39 |
| SHERIFF_BADGE_PASS | 所有 DAY_SPEECH+NIGHT 写 confidence | 凭空判断 | Step0 读取 R64 ✅ |

**为什么对好人阵营特别重要**：
- 传徽是不可逆决策，传给狼人 = 送 1.5 票 + 未来发言权威
- 好人警长经过多轮发言观察，identity_table 里 confidence 最高（≥ 70）且非狼嫌疑的玩家，就是最可靠的传徽候选
- 预言家金水是外部确定事实（100%），identity_table 是内部积累推断（70%+），两者互补而非替代

**博弈论依据**：
- 单次高价值不可逆决策（irreversible high-value action）的博弈最优策略 = 综合利用所有可用信息（Wang 2025 §2.1 信息利用率）
- identity_table 是 AI 在当前 context 中最接近"完整信息"的私有历史记录，不利用它相当于主动降低信息利用率
- 传徽给高 confidence 好人候选而非"随机发言可信者"，使好人阵营 1.5 票优势的利用率最大化

## 修复实现

**修改文件**：`src/services/aiPrompts.js`，`SHERIFF_BADGE_PASS` case 块

**修改内容**：

1. 新增 `bpIdentityStep` 变量（好人警长专属，狼人警长为空字符串）：
```javascript
const bpIdentityStep = playerRole !== '狼人'
    ? 'Step0: 【读取历史身份推理（传徽决策依据）】查看系统提示中【你之前的身份推理表】：哪些存活候选人的 confidence ≥ 70 且 suspect 不含"狼人"？将其列为传徽优先候选（若与预言家金水⚡一致则更确信；有冲突时以金水为准）。\nStep1: 传徽优先级 → ⚡预言家金水 > identity_table confidence ≥ 70 非狼嫌疑 > 发言可信者 > -1撕毁。'
    : '';
```

2. 简化 `bpHint` 好人分支（优先级已移至 Step1）：
```javascript
'你是好人警长：把警徽传给你最确信的好人。完全无法判断时撕掉警徽（-1）——错传给狼等于送1.5票。'
```

3. 在 template 中将 `bpIdentityStep` 注入到 `bpHint` 之前：
```javascript
${bpIdentityStep ? bpIdentityStep + '\n' : ''}${bpHint}${seerHint}
```

**传徽优先级（好人警长）**：
```
⚡预言家金水（外部确定事实）
↓ （无金水时）
identity_table confidence ≥ 70 + suspect 不含"狼人"（积累推断）
↓ （无高置信候选时）
发言可信者（当轮临时判断）
↓ （完全不确定时）
-1 撕毁警徽
```
