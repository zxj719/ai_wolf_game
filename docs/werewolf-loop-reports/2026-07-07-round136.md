## 问题分析

### 悍跳骑士 claims 闭环断裂（R135 之前 + R136 修复）

R135 实现了骑士对跳检测（`ssKnightCounterClaimants`），但依赖 `claimHistory` 中存在 `jump_knight` 记录。记录写入依赖 R134 的 `recordClaim` 调用，该调用从 `res.claims` 解析。

问题：wolf ssHint 只有 ①悍跳预言家（告知格式）和 ②好人竞选（claims 留 []），没有悍跳骑士路径——狼人不知道悍跳骑士时 claims 应填什么格式，导致：
1. `res.claims` 不含 `jump_knight` 记录
2. `claimHistory` 中 `jump_knight` 为空
3. `ssKnightCounterClaimants` 检测不到任何悍跳
4. 真骑士发言时 `knightSsCounterHint` 为空字符串
5. R135 精心构建的骑士反制机制完全失效

```
R136 修复后：
Wolf → SHERIFF_SPEECH →  ② 悍跳骑士路径 → claims:{type:'jump_knight', duel:{targetId:X}}
          ↓
  R134 recordClaim → claimHistory 写入 jump_knight ✅
          ↓
  R135 ssKnightCounterClaimants 检测到 jump_knight ✅
          ↓
  真骑士获得 knightSsCounterHint（底气框架 + 锚点提示）✅
```
