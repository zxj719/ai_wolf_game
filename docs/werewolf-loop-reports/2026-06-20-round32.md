## 2. 本轮优化详情

### 问题发现

LEARNINGS.md item 64（Round 31 下轮建议）：
> "骑士 SHERIFF_SPEECH/SHERIFF_RUN 评估：骑士目前走好人 fallback（与摄梦人/魔术师修复前状态相同），骑士有'决斗威慑'竞选价值，值得补专属分支"

验证骑士在 SHERIFF_SPEECH ssHint 链中确实走 fallback：
```
预言家 → 狼人 → 猎人 → 摄梦人 → 魔术师 → 好人（${playerRole}）  ← 骑士走这里
```

### 博弈论依据

**骑士警长竞选的独特价值（Wang 2025 信息博弈视角）：**

1. **双重不确定性威慑（未决斗）**：
   - 1.5票权重是"可见威胁"
   - 未用决斗能力是"隐藏威胁"——狼方无法确认警长是否是骑士
   - 攻击未决斗骑士警长的代价是 1.5票消失 + 可能触发决斗反杀，高于攻击普通好人警长
   - 类比R30猎人框架：不确定性本身是战略资产

2. **骑士 vs 猎人威慑框架的差异**：
   - 猎人："倒下了好人不亏"（被动防御，死亡触发）
   - 骑士："判断了就会直接落实"（主动进攻，主动触发）
   - 两者都适合隐性威慑，但措辞需体现"被动 vs 主动"的语义差异

3. **已决斗状态（身份已公开）**：
   - 决斗结果提供了物理验证，是比任何分析都更有说服力的公信力来源
   - 如果决斗对象是狼人 → 判断力已验证（可直接利用）
   - 1.5票 × 已验证判断力 = 最高精准度的票型引导

### 技术实现

**SHERIFF_RUN（骑士分支）：**

在守卫分支之后、摄梦人分支之前插入：
```javascript
} else if (playerRole === '骑士') {
    const knightDueled = currentPlayer?.hasUsedDuel ?? false;
    srHint = knightDueled
        ? `你是骑士（决斗已使用，身份已公开）：强烈建议上警...公信力...`
        : `你是骑士（决斗未使用）：倾向上警...双重不确定性...`;
```

**SHERIFF_SPEECH（骑士分支）：**

在 `magicianSsHint` 定义之后、`ssHint` 三元链之前，添加：
```javascript
const knightHasDueled = currentPlayer?.hasUsedDuel ?? false;
const knightSsHint = knightHasDueled ? `...已验证框架...` : `...隐性主动框架...`;
```

在 ssHint 三元链中 `magicianSsHint` 之后、好人 fallback 之前插入：
```javascript
: playerRole === '骑士'
? knightSsHint
```

**隐性主动框架关键词设计**（遵循R30白熊效应原则）：
- ✅ 使用："主动魄力""判断落实""最直接的方式去落实""绝不透露任何身份词汇"
- ❌ 避免："骑士""决斗"（在框架文本中不出现具体能力词汇）
