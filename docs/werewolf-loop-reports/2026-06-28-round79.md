### 新增 NIGHT_WOLF Step 4: 次日刀后叙事预案

**文件**：`src/services/aiPrompts.js`（NIGHT_WOLF case，Step 3 之后、identity_table 写指导之前）

**新增逻辑**（~480 chars）：

```
4. 【次日刀后叙事预案】刀口确定后，规划明天的公开应对以保持言行一致，避免反应矛盾暴露身份：
   - 回顾今日对该目标的公开立场（投票意向/发言态度）：
     · 今日推过/质疑过TA → 明天可"顺势"（"在我预期中，TA确实有问题"），无需额外解释
     · 今日中立或曾站TA → 明天需"补叙细节"（"其实TA有个发言细节让我后来起疑"），提前准备合理化理由
     · 今日明确保过TA → 风险最高，需引导讨论到第三方嫌疑人，转移好人关注焦点
   - 转移目标预判：若该目标死得出乎好人意料，预选一个好人Y作为"引导嫌疑"方向，在 identity_table 标注备用
   - 记录预案：在今晚刀口目标的 reason 末尾追加"次日叙事：[顺势/补叙细节/引导→Y号]"（下一个白天 Step 0 将读取并执行）
```

**Identity_table 写指导更新**（刀口行）：

## 竞技依据

Wang 2025（arxiv:2408.17177）指出，LLM 狼人最大弱点之一是"跨轮行为一致性"——AI 可以在某轮发言中"保护"某好人，次日却对其死亡表现出不自然的平静或怀疑，暴露决策模式。本轮优化的"次日叙事预案"机制：

1. **在刀口决策阶段（NIGHT）**提前考虑明天的应对策略，将决策影响注记到 identity_table
2. **在发言阶段（DAY）**通过 Step 0 读取前一夜的预案，实现"有准备的应对"
3. **三种预案分支**（顺势/补叙细节/引导）覆盖了所有今日立场与刀口决定之间的不一致程度
