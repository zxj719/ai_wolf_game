## 实施内容

### 新增变量（在 guardNightStyle 之后、return 之前）

```javascript
// R85: NIGHT_GUARD 平安夜守护来源推断框架
const isNightPeacefulGuard = ctx.dayCount > 1 && ctx.lastNightInfo?.includes('平安夜');
const guardNightPrevDay = ctx.dayCount > 1 ? ctx.dayCount - 1 : 0;
const guardNightPeaceStep = isNightPeacefulGuard
    ? `⭕【守卫平安夜守护来源推断（thought 中完成）】
  ① 查 identity_table 中含"N${guardNightPrevDay}夜守护"的玩家，确认昨夜守护目标ID
  ② 查系统提示游戏时间线 D${guardNightPrevDay} 投票记录，分析守护目标昨日票压：
     路径A：守护目标昨日票压高（≥2票指向TA）→ 守卫判断与全场一致，成功拦截狼刀可能性高 → 今晚连守；identity_table 将其 confidence 升15-25
     路径B：守护目标昨日票压低或无票压 → 狼人刀口在别处（女巫可能救了真正的刀口目标）→ 今晚切换至 identity_table 次高优先候选；女巫救药资源可能已消耗
  ③ 更新 identity_table：在守护目标 reason 末尾追加"N${guardNightPrevDay}平安夜：[A连守/B换守]"`
    : '';
```

## 两路径设计依据

| 路径 | 条件 | 推断逻辑 | 决策 |
|------|------|---------|------|
| A（守住了）| 守护目标昨日票压高（≥2票）| 守卫判断与全场一致，成功拦截狼刀 | 连守（值得继续保护）；confidence 升15-25 |
| B（没守住）| 守护目标昨日票压低/无 | 狼刀在别处，女巫可能救了真正的刀口 | 切换至次高优先候选；女巫救药可能已消耗 |

**对称关系（Wang 2025，arxiv:2408.17177）**：
- 狼人（R84）：知道刀口目标 → 用票压判断"守卫守住 vs 女巫救了"→ 换刀/维持
- 守卫（R85）：知道守护目标 → 用票压判断"拦截了狼刀 vs 守错了方向" → 连守/切换

两者互为镜像，利用相同的公开信息（投票记录票压），从各自的私有信息（刀口/守护目标）出发，得出对称的推断结论。
