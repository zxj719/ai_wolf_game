## 2. 本轮优化详情

### 优化 A：守卫 identity_table 追加格式（items 43）

**问题根因**：R19 给村民/猎人添加了"追加不覆盖历史"指令 + 【追加示例】，但守卫 DAY_SPEECH 和 NIGHT_GUARD 均未同步，导致守卫的跨轮记忆可能在每轮被 LLM 覆盖。

**博弈论依据（Wang 2025 arxiv:2408.17177 参考）**：守卫的核心价值在于"根据狼人行为规律调整守护目标"。若每轮 identity_table 被重写，守卫无法跨轮积累"狼人偏向哪类目标"的证据，等于每夜重新推理，浪费已有信息。

**改动路径（1 文件 2 处）**：

| 位置 | 改动 | 说明 |
|------|------|------|
| `aiPrompts.js` 守卫 DAY_SPEECH identity_table | 添加"在上轮 reason 基础上追加，不覆盖历史" + 【追加示例】 | 与村民/猎人格式对齐 |
| `aiPrompts.js` NIGHT_GUARD case（输出行前） | 新增完整 identity_table 填写指导段落 + 追加示例 | 夜间行动也有跨轮记忆指导 |

**守卫 DAY_SPEECH 新增**：
```
- 守护历史跨轮追加：在上轮 reason 基础上追加本轮观察（不覆盖历史）
  【追加示例】上轮 reason="N1守5号→平安夜存活" → 本轮末尾追加为
  "N1守5号→平安夜存活；N2换守3号(发言疑预)→3号次日存活，换守正确"
```

**NIGHT_GUARD 新增完整 identity_table 指导段落**：
```
【identity_table 填写指导（守卫夜间：守护历史跨轮追加，辅助连贯策略）】
- 守护候选/已守护者：confidence 50-80，reason 追加写...【追加示例】...
- 疑似神职（但未守护过）：reason 写"行为疑似[预言家/守卫]，守护优先级：高/中"
- 明确狼嫌疑：reason 写"发言矛盾/带节奏，不守对象"
```

### 优化 B：DAY_VOTE 热力盲从防护（item 42，源自 R11 item 30）

**问题根因**：自 R11 添加热力提示至今，警告文案一直是"可能是狼人刷票的靶子——须结合本轮发言交叉验证"。但这个描述太模糊——LLM 不知道如何判断"是否是靶子"，实际操作中仍可能盲从热力。

**博弈论依据**：真实的竞技狼人杀中，刷票靶子有三种典型识别信号：
1. **热力高但发言质量高**：若该玩家本轮发言逻辑清晰有实质内容，其热力更可能来自狼人刷票（真正危险的好人会成为刷票对象）
2. **多个狼嫌疑一致力推**：多个被认为可疑的玩家同时投同一人，协调性过强，提示刷票合谋
3. **正向信号**：热力高的人总是踩明显狼嫌疑——这实际上是好人在正确分析，应该跟投而非降权

**改动路径（1 文件 1 处）**：

将 `voteMomentumHint` 从模糊警告升级为：
```
【跨轮投票热力】历史投票中：...号被投...次。热力仅作参考——警惕刷票陷阱：
  ① 热力高但本轮发言逻辑清晰/有实质分析 → 可能是狼刷票的靶子，不要盲跟
  ② 多个狼嫌疑玩家一致力推同一目标 → 高度警惕合谋，热力降权
  ③ 热力高的人总是踩明显狼嫌疑 → 可能是好人在正确分析（非靶子，可跟投）
  决策原则：先独立评估本轮发言质量，再参考热力——热力仅作辅助，切勿"高热力就跟投"。
```
