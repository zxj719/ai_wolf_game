## 2. 本轮优化详情

### 背景（LEARNINGS.md items 28 & 29）

**Item 28**（骑士 shouldDuel 回顾）：审计结果为**正常**。
- `knight.js` 输出 JSON 含 `shouldDuel: true/false` + `duelTarget: 数字或null`
- `useSpeechFlow.js:265` 消费 `res.shouldDuel` 并调用 `handleKnightDuel`
- `applyDuelResult` 设置 `hasUsedDuel: true` 在玩家对象上
- `aiPrompts.js` 将 `hasUsedDuel: currentPlayer?.hasUsedDuel` 传入 roleParams
- `knight.js` 使用 `hasUsedDuel` 切换阶段（隐藏期/决斗期/领袖期）
- **全链路完整，无 bug**

**Item 29**（魔术师 shouldReveal 回顾）：发现**死字段 bug**。
- `magician.js` 输出 `shouldReveal: true/false` 但 **useSpeechFlow.js 从未消费此字段**
- 副作用：魔术师跳身份后，下一天 `hasRevealed` 仍为 `false` → 提示词显示"阶段1 伪装成平民" → LLM 可能再次决定不用宣告，造成身份混淆
- 与骑士对比：`shouldDuel` 有机械效果（杀人），必须消费；`shouldReveal` 是信息效果，但仍需持久化以影响后续提示

---

### 修复：4 文件同步

#### 1. `useWerewolfGame.js` — 初始化 hasRevealed
```javascript
hasUsedDuel: false,  // 骑士决斗技能（整局一次）
hasRevealed: false,  // 魔术师跳身份标记（影响白天发言阶段）
```

#### 2. `useSpeechFlow.js` — 消费 shouldReveal
在骑士决斗处理块之后，添加：
```javascript
// 魔术师跳身份处理：更新 hasRevealed 使后续发言进入阶段3
if (currentSpeaker.role === ROLE_DEFINITIONS.MAGICIAN && res.shouldReveal && !currentSpeaker.hasRevealed) {
  logger.debug(`[魔术师暴露] ${currentSpeaker.id}号魔术师决定跳身份`);
  setPlayers(prev => prev.map(p =>
    p.id === currentSpeaker.id ? { ...p, hasRevealed: true } : p
  ));
}
```
注意：与骑士不同，这里**不 `return`** — 魔术师跳身份不结束当天，只是状态更新。

#### 3. `aiPrompts.js` — roleParams 传 hasRevealed
```javascript
hasUsedDuel: currentPlayer?.hasUsedDuel,
hasRevealed: currentPlayer?.hasRevealed,  // 魔术师是否已跳身份
```

#### 4. `magician.js` — 条件化三阶段策略
- `hasRevealed` 时：显示"【⚠ 身份已公开】当前阶段3，无需再次宣告"
- `hasRevealed` 时：省略阶段1/2 说明，跳到阶段3
- `hasRevealed` 时：省略"是否跳身份判断"章节
- 阶段3 增加："狼人已知你是魔术师，考虑在发言中暗示下晚换守"
