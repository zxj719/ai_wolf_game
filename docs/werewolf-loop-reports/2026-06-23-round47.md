## 实施内容

**优化：NIGHT_WOLF N2+ 夜间加入上轮夜间结果核查**

### 问题
- NIGHT_WOLF 的 `wolfHistoryStep` 引导狼人读取 identity_table 的"高优先刀口"历史
- 但没有提供显式的"刀口是否成功"判断框架
- 若女巫救了刀口目标（平安夜），狼人需要通过"目标还活着 + identity_table 有行刀标记"才能间接推断出女巫用了救药
- `ctx.lastNightInfo` 已在 `prepareGameContext()` 中构建，但从未注入 NIGHT_WOLF user prompt

### 修复（`src/services/aiPrompts.js`）

**新增 `wolfLastNightBlock`**（N2+ 注入上轮夜间结果）：
```javascript
const wolfLastNightBlock = ctx.dayCount > 1 ? `\n【上轮夜间结果】${ctx.lastNightInfo}` : '';
```

**扩展 `wolfHistoryStep`** N2+分支：
- 旧版：仅"读取历史刀口"
- 新版：读取历史刀口 + 交叉比对上轮结果
  - 刀口目标死亡 → 刀成功，延续高优先目标逻辑
  - 刀口目标存活 → 女巫救了（女巫只剩毒药），上调女巫优先级
  - 平安夜 → 守卫守住/女巫救了，重新评估

### 数据流验证
- `ctx.lastNightInfo` 来自 `prepareGameContext(gameState)` → 从 `gameState.deathHistory` 过滤上轮夜间死亡
- `gameState.deathHistory` 已在 `useAI.js` gameState 和 `WerewolfModule.jsx` 调用端完整传递（R46 验证）
- 无新的三层传递链风险
