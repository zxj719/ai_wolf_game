## 实施内容

### wolfPeaceNightStep 变量声明

**条件**：`ctx.dayCount > 1 && ctx.lastNightInfo?.includes('平安夜')`

**新增代码**（在 `wolfSpeechLen` 赋值后、`return` 之前）：

```javascript
// R83：狼人平安夜战术推断（DAY_SPEECH 侧，补完平安夜感知好/狼两侧对称）
const isPeacefulNightWolf = ctx.dayCount > 1 && ctx.lastNightInfo?.includes('平安夜');
let wolfPeaceNightStep = '';
if (isPeacefulNightWolf) {
    const prevDay = ctx.dayCount - 1;
    wolfPeaceNightStep = `⭕【狼人平安夜战术推断（thought 中完成；speech 参照 Step0 次日叙事预案自然应对）】
- 昨夜平安夜 → 你的刀被守/救了：刀口目标存活，今天是重新评估的起点
- 守护来源推断：查 D${prevDay} 投票记录票压最高的存活玩家（最可能被刀但仍活=最可能被守护）
  · 高票存活者 = 昨夜刀口目标 → 守卫与狼判断一致（守了高价值好人），连守概率高 → 下夜强烈建议换刀（连守目标刀成功率极低）
  · 高票存活者 ≠ 昨夜刀口目标 → 守卫守了第三人，女巫可能救了刀口目标 → 女巫救药资源已消耗；刀口目标下夜维持高优先（无守护覆盖）
- 资源模型更新：若能定位守卫/女巫玩家，在 identity_table 其 reason 中追加"D${prevDay}平安夜：守护推断"或"D${prevDay}平安夜：救药推断"
- speech 层面：Step0 已读取次日叙事预案，今天 speech 按预案自然应对，无需额外规划\n`;
}
```

**模板注入**：在 Step0 文本末尾与 `Step1 局势评估` 之间注入 `${wolfPeaceNightStep}`，与 R80-R82 好人侧完全对称

### 两路径设计依据

| 路径 | 条件 | 推断逻辑 | 行动 |
|------|------|---------|------|
| A（守卫守住）| 高票存活者 = 刀口目标 | 守卫在保护高价值好人 → 连守概率高 | 换刀（成功率极低） |
| B（女巫救了）| 高票存活者 ≠ 刀口目标 | 守卫守了第三人 → 女巫消耗救药 | 维持高优先（无守覆盖） |

**竞技依据（Wang 2025, arxiv:2408.17177）**：LLM 狼人的弱点之一是跨轮资源推断——无法利用平安夜信息更新守卫/女巫剩余能力估算，导致换刀决策系统性偏差。wolfPeaceNightStep 直接提供守护来源推断框架，修复此弱点。

## 窗口截断修复（R83-A 教训）

**触发文件**：`round57SeerDaySpeechStep0.test.js` T23  
**原因**：R83 在 `wolfSpeechLen` 赋值后新增约 760 chars 的 `wolfPeaceNightStep` 变量块，使 wolf Step0 内容从约 4169 移至 4911 chars 处，超出原来的 4500 窗口  
**修复**：将窗口从 `wolfFnStart + 4500` 升至 `wolfFnStart + 6000`  
**教训（R83-A）**：修改 wolf DAY_SPEECH 变量声明区（`wolfSpeechLen` 之后、`return` 之前）时，必须检查所有以固定偏移搜索 wolf 模板内容（Step0、高优先刀口等）的已有测试文件，确保窗口足够。  
**检测命令**：`grep -rn "wolfFnStart\|wolf.*slice" src/services/__tests__/*.test.js` 列出所有引用 wolf 函数偏移的测试。
