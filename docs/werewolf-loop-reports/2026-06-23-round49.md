## 本轮优化内容

### 1. 发现 magician.js 的 3 个死解构（Item 99 审计产出）

**背景**：通过手动代码审查（为 Item 99 测试做准备），发现 `magician.js` 存在未被 R48 `delegateParams.test.js` 覆盖的死解构：

#### getMagicianNightActionPrompt（第 84-95 行）
```js
// 修改前：包含从未引用的 seerChecks
const {
    validTargets, swappedPlayers, lastSwap, existingRoles,
    dayCount, nightContext, seerChecks,  // ← 死解构
    knownGods, suspectedWolves, hasRevealed,
} = params;
```
- `seerChecks` 出现在解构列表，但函数体 115 行内无任何引用
- `knownGods` 和 `suspectedWolves` 是 aiPrompts.js 调用端从 `seerChecks` 计算得出后分别传入的，函数内部直接使用 `knownGods`/`suspectedWolves` 即可，无需再持有原始 `seerChecks`

#### getMagicianDaySpeechPrompt（第 231 行）
```js
// 修改前
const { swappedPlayers, lastSwap, seerChecks, deathHistory } = params;
// 修改后
const { swappedPlayers, lastSwap } = params;
```
- `seerChecks`：daySpeech 函数体（~90 行）内无任何引用
- `deathHistory`：同样无任何引用，且从未在 roleParams 中传递（永远为 undefined）

### 2. 修复 aiPrompts.js NIGHT_MAGICIAN 调用端冗余传参

```js
// 修改前
return magicianModule.nightAction({
    ..., seerChecks: magicianSeerChecks || [], ...
});
// 修改后
return magicianModule.nightAction({
    ..., /* seerChecks 已删除 */ ...
});
```

`magicianSeerChecks`（从 params 解构的 `seerChecks`）在 NIGHT_MAGICIAN case 中已被用于计算 `knownGods`/`suspectedWolves`，两者被分别传入——`seerChecks` 本身的传入是冗余的。

### 3. Item 99 测试文件：dataFlowChain.test.js（9 个测试）

**文件**：`src/services/__tests__/dataFlowChain.test.js`

| 测试 | 目标 |
|------|------|
| T1 | aiPrompts.js 中所有 `gameState.X` 直接访问 key 均在 useAI.js gameState 对象中定义 |
| T2 | prepareGameContext 解构的所有 key 均在 useAI.js gameState 对象中 |
| T3 | aiPrompts.js 中所有 gameState key 均在 WerewolfModule.jsx useAI() 调用中传入 |
| T4 | useAI.js gameState 包含 14 个必须核心字段（防止字段意外删除） |
| T5 | getMagicianNightActionPrompt 解构无 seerChecks（R49 清理验证） |
| T6 | getMagicianNightActionPrompt 保留必要字段 hasRevealed/swappedPlayers/lastSwap |

**关键实现细节**：

`extractGameStateKeys(src)` 通过匹配 `const gameState = {` → 平衡括号配对找闭括号 → 提取属性名，正确处理 shorthand property（如 `dreamweaverHistory,`）和 alias（如 `speechHistory: enhancedSpeechHistory`）两种形式。

T3 使用直接文本包含检查（`werewolfModuleSrc.includes(k)`）而非结构化解析，因为 WerewolfModule.jsx 的 useAI() 调用跨多行，结构化提取复杂且容易漏掉 alias 形式（如 `gameSetup: selectedSetup` 中的 `gameSetup`）。
