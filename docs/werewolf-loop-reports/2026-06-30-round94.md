## 本轮优化内容：NIGHT_GUARD 两连平安夜 NIGHT 侧二阶推断（R94）

### 问题识别

R85 为 NIGHT_GUARD 添加了单夜平安夜推断（`guardNightPeaceStep`）：用上一夜守护目标的票压判断是否拦截了狼刀。但没有处理"连续两夜平安夜"的场景——在这种情况下，守卫有两夜守护记录可以交叉分析，精度远高于单夜。

DAY 侧 R91 已为 DAY_SPEECH 守卫添加了 `isConsecutivePeacefulGuard + consecutivePeaceHintGuard`。NIGHT 侧的对称扩展缺失。

### 设计选择：零间接推断

守卫 NIGHT 侧的核心优势：
- `guardHistory`（`gameState.guardHistory`）直接记录每夜守护目标
- `prevPrevNightGuardTarget = gameState.guardHistory?.find(g => g.night === guardNightPrevPrevDay)?.targetId`
- 这是**零间接推断**——不需要用票压猜测"刀口是谁"，直接知道守了谁

对比其他角色推断精度层级：
- 狼人（NIGHT）：知道刀口，用票压推断守卫/女巫来源
- 守卫（NIGHT）：知道守了谁，用票压推断是否命中（R85），两连时直接比对两夜记录（R94）
- 好人（DAY）：不知道刀口，需先推断再判断守护来源

### 修改内容

**文件**：`src/services/aiPrompts.js`，`NIGHT_GUARD` case 内

**新增变量**（在 `guardNightPrevDay` 之后、`guardNightPeaceStep` 之前）：

```javascript
// R94: NIGHT 侧两连平安夜二阶推断
const isConsecutivePeacefulNightGuard = ctx.dayCount >= 3 && isNightPeacefulGuard &&
    ctx.fullGameTimeline?.includes(`N${ctx.dayCount - 2}:平安夜`);
const guardNightPrevPrevDay = ctx.dayCount >= 3 ? ctx.dayCount - 2 : 0;
const prevPrevNightGuardTarget = isConsecutivePeacefulNightGuard
    ? (gameState.guardHistory?.find(g => g.night === guardNightPrevPrevDay)?.targetId ?? null)
    : null;
const consecutivePeaceNightHintGuard = (isConsecutivePeacefulNightGuard && cannotGuard !== null)
    ? `⭕【守卫两连平安夜 NIGHT 二阶推断...路径A/B...】`
    : '';
```

**修改** `guardNightPeaceStep` 正值分支前缀注入 `${consecutivePeaceNightHintGuard}`。

### 内容设计（路径A/B）

- **路径A**（连守同一目标）：N-2 和 N-1 均守 `cannotGuard` 号 → 两次平安夜同目标=拦截可信度极高 → confidence 升 25-35 → 今晚换守打破规律
- **路径B**（两夜目标不同）：N-2 守 X，N-1 守 Y → 分别用各自当日票压评估命中概率（高票存活者恰是守护目标→该夜命中，confidence 升 15-20）→ 综合两夜分析选今晚守护

### 激活条件

- `dayCount >= 3`：需要至少 3 夜才有两连可能
- `isNightPeacefulGuard`（上一夜平安）AND `fullGameTimeline 含 N${dayCount-2}:平安夜`（再前一夜也平安）
- `cannotGuard !== null`：守卫必须有明确的上夜守护目标才能做路径A/B分析

## 博弈论依据（Wang 2025，arxiv:2408.17177）

守卫的两连平安夜推断核心价值：
- **路径A（连守命中）**：连续两夜守同一人都平安，说明守卫成功拦截了两次狼刀（或狼刀在别处）。按 confidence 最大化原则，守卫应把该目标 confidence 升至 85-95，同时换守打破规律防被预判。
- **路径B（轮守分析）**：轮守时，每次平安夜仅对应该夜守护目标的命中分析，不能交叉。两夜独立分析后，选 confidence 最高的未守过候选——避免重复守护降低覆盖率。
- **信息利用率原则**：守卫拥有两夜历史的组合能力远高于单夜——单夜只能做一次票压推断（准确率~60%），两夜组合后，路径A命中概率接近90%（两次平安夜同目标的随机概率极低）。
