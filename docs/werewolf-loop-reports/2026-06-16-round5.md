## 2. 本轮优化详情

### LAST_WORDS 提示词全角色化扩展

**文件**：`src/services/aiPrompts.js`（`PROMPT_ACTIONS.LAST_WORDS` case）

**问题**：原版 LAST_WORDS 只有 3 个分支：
- 狼人：正向框架尚可，但不够具体（缺少"职能好人优先"策略）
- 预言家：框架对，但 12 人局未提示警徽流建议
- 通用好人（含女巫/猎人/守卫/村民）：一段泛化提示，完全没有利用这些角色在遗言时的专属信息优势

**信息浪费分析**：
| 角色 | 遗言时的独有信息 | 原版是否提取 |
|------|------|------|
| 预言家 | 全部查验结果 + 12人局警徽流 | ✅ 部分（无警徽流） |
| 女巫 | 药品状态（双药/单药/全用完） | ❌ 未提取 |
| 猎人 | 最高怀疑目标 / 即将开枪目标 | ❌ 未提取 |
| 守卫 | 历史守护记录（帮助识别狼人谎言） | ❌ 未提取 |
| 村民 | 发言分析 + 投票阵营线索 | ❌ 过于泛化 |

**修复方案（6分支）**：

```javascript
// 狼人：全正向框架，避免负向激活（LEARNINGS 2）
'以普通好人的视角留遗言，基于之前建立的立场…发言自然坚定'

// 预言家：结构化3步 + 12人局第4步
① 按夜次列出全部查验结果 ② 怀疑排序 ③ 高嫌疑未验证玩家 [④ 12人局警徽流建议]

// 女巫：动态药品状态（currentPlayer?.hasWitchSave/hasWitchPoison）
'双药已用完' | '解药已用，毒药尚存' | '解药尚存，毒药已用' | '双药均未使用'

// 猎人：开枪目标宣告导向
'即使此刻无法开枪，留下线索能让好人接棒追查；若即将触发开枪，直接说明目标和理由'

// 守卫：守护记录公开
'公开完整守护记录…帮助好人识别可能的谎言（如有人自称被守护但实际没有）'

// 通用好人（村民/骑士/摄梦人/魔术师）：投票阵营线索导向
'以最高信息密度传递你的分析——谁的发言逻辑最可疑、投票中的阵营线索、你最信任的玩家'
```

**技术关键**：
- `currentPlayer` 在 `generateUserPrompt` 的 switch 作用域全段可用（line 1143 const 定义）
- 女巫药品状态直接读 `currentPlayer?.hasWitchSave/hasWitchPoison`，无需额外参数传递
- `hasPoliceFlow = isLargeGame(gameState?.gameSetup)` 已在 SHERIFF_SPEECH 中验证安全

**依据**：
- 遗言是信息传递的最后机会，信息密度最大化原则
- 守卫历史守护公开是识别"假守卫自证"的关键反制手段
- 猎人遗言应以目标宣告为核心，让好人继续施压
- Wang 2025 (arxiv:2408.17177): 高质量角色特化信息比泛化指导对 AI 决策影响更大

在对局中遗言出现的频率约为：每局 1-4 次（被投票出局者 + 首夜死者）。之前的通用提示词会让所有非神职角色生成大同小异的"我怀疑 X 是狼"式告别。

新版后：
- **女巫遗言**："解药已用完，毒药尚存。我最怀疑 3 号是狼，他的发言在第 2 天明显护 7 号。毒药建议用在 3 号上。"→ 明确传递行动指令
