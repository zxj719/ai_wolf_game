## 本轮优化内容

### 问题发现：特殊角色 DAY Step 0 缺失

R59 建议检查骑士/摄梦人/魔术师写指导是否有"追加不覆盖历史"规则和"【追加示例】"。执行此检查时发现更关键问题：

| 角色 | 文件 | DAY Step 0 | NIGHT Step 0 | 【追加示例】 | **追加** 格式 |
|------|------|------------|--------------|-------------|---------------|
| 骑士 | knight.js | ✅ R44 `knightHistoryStep` | N/A | ✅ R44 | ⚠ 平文本 → ✅ R60 粗体 |
| 摄梦人 | dreamweaver.js | **❌ 缺失** → ✅ R60 | ✅ aiPrompts.js `dreamweaverHistoryStep` | ✅ 已有 | ⚠ 平文本 → ✅ R60 粗体 |
| 魔术师 | magician.js | **❌ 缺失** → ✅ R60 | ✅ R43 `magicianHistoryStep` | ✅ 已有 | ⚠ 平文本 → ✅ R60 粗体 |

**根因分析**：
- 主路径 6 角色的 DAY Step 0 在 R54-R58 通过 aiPrompts.js 补全（wolf/hunter/seer/witch/guard/villager）
- 特殊角色 3 个在独立 rolePrompts 文件中实现，R54-R58 的 aiPrompts.js 改动未覆盖这些文件
- 骑士在 R44 已手工完成 DAY Step 0，但摄梦人/魔术师被遗漏到 R60（16 轮差距）

**博弈论依据**：
- 摄梦人的核心策略循环是"白天分析谁是高威胁狼/谁需保护 → 夜间入梦执行"，DAY Step 0 让白天分析从历史"连梦候选"出发，而非每轮归零评估
- 魔术师的换刀策略依赖连续追踪"谁是狼"+"谁是关键神职"，DAY Step 0 让白天发言从历史"换刀候选"出发，避免每轮丢失前轮积累的嫌疑证据

### 实施内容

**文件 1：dreamweaver.js**

改动 1 — 新增 `dreamweaverDayHistoryStep` 变量（在 `return` 前）：
```javascript
// R60 DAY→DAY 读写闭环：历史连梦候选读取步骤（与 NIGHT_DREAMWEAVER Step 0 共用"连梦候选"关键词）
const dreamweaverDayHistoryStep = ctx.dayCount > 1
  ? 'Step0: 【读取历史连梦候选与防御目标（D2+适用）】查看系统提示中【你之前的身份推理表】：哪些玩家的 reason 含"连梦候选"标注？将其作为今晚进攻策略的评估起点（结合今日新发言判断是否升级/降级威胁；已死亡目标跳过）。含"防御入梦候选"的玩家是今晚优先入梦的保护对象。'
  : 'Step0: 【首日无历史候选记录】直接从 Step1 开始——今天是第一天，尚无跨轮积累的连梦/防御候选。';
```

改动 2 — 在思维链前注入 `${dreamweaverDayHistoryStep}`（Step0 → Step1 → Step2…）

改动 3 — write guide 添加 `**追加不覆盖历史**：每轮在上轮 reason 基础上追加本轮新观察（用分号拼接），不覆盖历史积累`
