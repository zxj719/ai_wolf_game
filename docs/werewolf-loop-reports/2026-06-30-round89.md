## 问题背景

### 感知-执行分裂（R80-A 模式第 N+1 次出现）

R86 + R87 分别修复了骑士"续战搜索框架"和"personalityLens 双阶段"，但决斗成功后的骑士领袖期思维链起点仍然是 Step1 "场上局势分析"——这是适合 **pre-duel** 阶段的通用分析框架，放在 **post-duel** 阶段是错配：

| 阶段 | 当前状态 | 问题 |
|------|----------|------|
| Pre-duel | knightHistoryStep (Step0) 读决斗候选 → Step1-5 通用分析 | ✅ 正常 |
| Post-duel | **同一** knightHistoryStep (Step0) → Step1-5 通用分析 | ❌ 无领袖期专属框架 |

骑士决斗成功 = 确认了一个狼人。这是好人阵营在整局中最强的单点实证。但当前 Step1 只做"局势分析"，完全没有利用这个实证锚点的推断框架：
- 谁与被决斗狼人发言立场一致？（应该下调 confidence）
- 被决斗狼人是否给过"金水"？（金水玩家双向验证，应大幅下调）
- 今天以"领袖"身份做什么？（集火 / 调查 / 保护三选一）

### 白熊效应安全性

Step0.5 三步全部使用正向描述：
- "查 identity_table 中含'已决斗出局'的玩家" ✅（读取指令，正向）
- "confidence 下调 15-25 / 40-50" ✅（量化行动，正向）
- "战略A（集火型）... 战略B（调查型）... 战略C（保护型）" ✅（三选一框架，正向）
- 无"不要""禁止""绝不能"等负向禁词 ✅（T19/T20 验证）

## 实施内容

### 新增 `knightLeaderStep` 变量（knight.js）

```javascript
// R89：骑士领袖期专属 Step0.5（post-duel 全场引导规划——感知-执行分裂修复）
const knightLeaderStep = hasUsedDuel
  ? `Step0.5: 【领袖期战略规划（thought 中完成）】
   ① 查 identity_table 中含"已决斗出局"的玩家，确认决斗验证信息：
      - 被决斗者角色（狼人确认）→ 好人阵营获得该轮第一块实证锚点
      - 查看与被决斗狼人发言立场一致的玩家（D${ctx.dayCount}前），其 confidence 下调 15-25
      - 若被决斗狼人给出过"金水"，该金水玩家 confidence 立即下调 40-50（两人成组验证）
   ② 规划今日引导战略（三选一）：
      战略A（集火型）：场上有高嫌疑狼人目标 → 利用领袖公信力带全场集票
      战略B（调查型）：狼人团队构成不清晰 → 引导好人发言交叉验证，收集更多信息
      战略C（保护型）：关键好人或真预言家处于危险 → 优先确保神职安全
   ③ identity_table 在本轮 Step4 推断完成后追加"D${ctx.dayCount}领袖指令：[战略A/B/C]，[本轮核心目标玩家号]"\n`
  : '';
```

### 注入位置

```
【思维链（必须完成）】
${knightHistoryStep}
${knightLeaderStep}Step1: 场上局势分析   ← knightLeaderStep 注入在此
...
```

当 `hasUsedDuel=false` 时，`knightLeaderStep=''`，输出完全退化为原始状态（向下兼容）。

### identity_table 写指导新增条目

```

  "D${ctx.dayCount}领袖指令：[战略A/B/C]，[说明]"（供下轮阅读领袖行动历史）
```

## 测试设计关键点

### T11 的区分技巧（R89 新教训）

`identity_table 写指导`中有静态文本 "Step0.5 执行后"，所以无论 `hasUsedDuel` 为何值，输出中都会含 "Step0.5"。T11 原本检查 `not.toContain('Step0.5')` 就失败了。

解决：检查 `'Step0.5:'`（带冒号），这是 `knightLeaderStep` 块头 `` `Step0.5: 【领袖期战略规划` `` 的独特形式，identity_table 中的 "Step0.5 执行后" 没有冒号。

**通用规则**：当某个语义片段（如 "Step0.5"）在函数体中多处出现时，区分"激活态"和"静态说明"的最可靠方法是找到语法上唯一的 suffix（如冒号、双引号包裹、特定前缀），而非整个字符串。
