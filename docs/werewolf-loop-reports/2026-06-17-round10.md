## 2. 本轮优化详情

### 2a. 消除 getBaseContext 函数分叉

**文件**：`src/services/aiPrompts.js`

**背景（LEARNINGS.md 第 11 条）**：

自 P1 重构以来，`getBaseContext` 在代码库中以两种形式存在：
1. **`baseRules.js` 导出版**（`export const getBaseContext`）：被骑士/摄梦人/魔术师的 `rolePrompts/*.js` 模块使用
2. **`aiPrompts.js` 本地版**（`const getBaseContext`，lines 1121-1140）：被 6 个主要角色的 `ROLE_DAY_SPEECH_PROMPTS` 使用

Round 9 证明了两者的维护成本：当 `currentPlayerTraits` 字段需要同步时，必须在两处都修改，且极易遗漏。

**本轮改动（3 个精准 Edit）**：

1. **Import 注入**（1 行）：
   ```javascript
   // Before
   import { detectExistingRoles, getRoleModule } from './rolePrompts';
   // After
   import { detectExistingRoles, getRoleModule, getBaseContext } from './rolePrompts';
   ```

2. **删除本地定义**（20 行 = JSDoc注释 4 行 + 函数体 16 行）

3. **删除 `baseContext` 死代码**（7 行 = 注释 1 行 + 变量定义 6 行）
   - `const baseContext = ...` 定义于 `generateUserPrompt` 顶部，但在整个 switch 语句中从未被任何 case 使用
   - 这是历史重构遗留的死变量，之前未发现是因为 JS 不报"defined but never used"错误

**改动后的依赖链**：

```
aiPrompts.js  ─(import)→  rolePrompts/index.js  ─(re-export)→  baseRules.js
                                                                    ↑
knight.js ─────────────────────────────────────────────────────────┘
dreamweaver.js ─────────────────────────────────────────────────────┘
magician.js ─────────────────────────────────────────────────────────┘
```

所有路径现在收敛于同一个真理源，未来任何修改只需修改 `baseRules.js` 一处。

**为什么这样做而不是合并两个 import 路径**：
- `aiPrompts.js` 已从 `./rolePrompts/baseRules` 单独 import `CLAIMS_SCHEMA_SUFFIX`
- 新增 `getBaseContext` 加到已有的 `./rolePrompts` import（通过 index.js re-export）是更简洁的做法
- 不需要改变 `knight.js` 等模块的 import（他们直接从 `baseRules` import 是正确的，更明确）
