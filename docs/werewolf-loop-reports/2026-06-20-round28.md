## 2. 本轮优化详情

### 问题发现

`isSheriff` 字段从第 1 天警长竞选结束后就设置在 `player.isSheriff`，并在 `useDayFlow.js` 中用于计算投票权重（`getVoteWeight` 函数），但在整个 `aiPrompts.js` 中从未被任何提示词消费：

```bash
grep -n "isSheriff" src/services/aiPrompts.js
# → 0 结果（修复前）
```

导致的问题：
1. **DAY_SPEECH**：警长 AI 发言时不知道自己是警长，不会在发言结尾"警长指路"——竞技狼人杀中警长的最核心职责被静默跳过
2. **DAY_VOTE**：警长 AI 投票时不知道自己的票值 1.5，可能在接近平票时轻易弃票（好人阵营损失 1.5 票）

### 修复方案（"后处理注入"模式）

**修复原则**：避免修改 9 个角色的 `ROLE_DAY_SPEECH_PROMPTS` 函数（遵循开闭原则），改为在 DAY_SPEECH case 末尾插入警长任务块：

```javascript
// roleParams 中新增
isSheriff: currentPlayer?.isSheriff || false,

// 后处理：在角色提示词之后、CLAIMS_SCHEMA_SUFFIX 之前插入
const sheriffHint = roleParams.isSheriff
    ? `\n\n【⚖️ 警长任务（本轮额外职责）】...警长指路...
${playerRole === '狼人' ? '狼人警长：...' : playerRole === '预言家' ? '预言家警长：...' : '好人警长：...'}`
    : '';
return rolePromptGenerator(ctx, roleParams) + sheriffHint + CLAIMS_SCHEMA_SUFFIX;
```

**DAY_VOTE 修复**：

```javascript
const isVoterSheriff = currentPlayer?.isSheriff || false;
const sheriffVoteHint = isVoterSheriff
    ? `\n【🎖️ 警长投票】你的投票权重为1.5票...切勿弃票（警长弃票=放弃1.5票，好人阵营损失最重）。`
    : '';
```

### 警长指路三分支设计（博弈依据）

| 身份 | 指路策略 | 依据 |
|------|---------|------|
| 好人警长（通用） | 优先指向：查杀目标 > 发言混乱者 > 历史多轮被投者 | 警长公开指向是好人票型形成的最强引力，必须方向正确 |
| 预言家警长 | 直接指向查杀目标/悍跳者，用查验链背书 | 预言家+警长=信息权威²，对悍跳者的指向有双重来源 |
| 狼人警长 | 分析语气指向好人，绝不指向队友，不要过于强硬 | Wang 2025: 权威滥用（过于明确的命令式指向）是身份暴露信号 |
