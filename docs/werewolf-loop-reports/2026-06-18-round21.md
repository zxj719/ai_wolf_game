### 修复：2 文件同步

#### 1. `src/services/rolePrompts/dreamweaver.js`

**新增以下动态内容**：

```javascript
// 解构 hasRevealed
const { dreamHistory, lastDreamTarget, nightDeaths, seerChecks, hasRevealed } = params;

// 身份状态头部
const revealedStatus = hasRevealed
  ? '【⚠ 身份已公开】你已宣告摄梦人身份，当前处于阶段2/3。不要重复宣告身份...'
  : '【身份未公开】目前伪装成平民，根据局势决定是否本轮跳身份...';

// let + if 块（R15 教训：多分支不用三元）
let stageContent;
if (hasRevealed) {
  stageContent = `（阶段1 已完成，当前处于阶段2/3）\n\n阶段2 - 博弈期...阶段3 - 威慑期...`;
} else {
  stageContent = `阶段1 - 潜伏期（身份未暴露，当前阶段）：...\n阶段2/3（设置 shouldReveal=true）...`;
}

// shouldReveal 指导（仅 !hasRevealed 时显示）
let revealGuide;
if (!hasRevealed) {
  revealGuide = `【是否跳身份判断（shouldReveal）】...`;
} else {
  revealGuide = '';
}

// step4 也条件化
let step4Content;
if (hasRevealed) { step4Content = `...利用入梦名单指挥好人...`; }
else             { step4Content = `...shouldReveal 决策...`; }
```

**输出 JSON schema 新增** `shouldReveal:true/false`：
```
输出JSON:{"thought":"...","speech":"...","shouldReveal":true/false,"confidence":0-100,"voteIntention":...,
"identity_table":{"玩家号":{"suspect":"角色","confidence":0-100,"reason":"依据（追加入梦/死讯线索，示例：上轮reason=X → 本轮更新为X；Y）"}}}
```

#### 2. `src/hooks/useSpeechFlow.js`

在魔术师消费块之后新增：
```javascript
// 摄梦人跳身份处理：更新 hasRevealed 使后续发言进入已公开阶段
if (currentSpeaker.role === ROLE_DEFINITIONS.DREAMWEAVER && res.shouldReveal && !currentSpeaker.hasRevealed) {
  logger.debug(`[摄梦人暴露] ${currentSpeaker.id}号摄梦人决定跳身份`);
  setPlayers(prev => prev.map(p =>
    p.id === currentSpeaker.id ? { ...p, hasRevealed: true } : p
  ));
}
```

注意：与骑士不同，这里**不 `return`**—摄梦人跳身份不结束当天，只是状态更新（与魔术师一致）。
