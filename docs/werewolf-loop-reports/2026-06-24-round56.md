## 本轮优化内容

### 问题分析

猎人 DAY_SPEECH 思维链（`ROLE_DAY_SPEECH_PROMPTS['猎人']`）的 identity_table 写指导区有如下内容（旧版）：
```
- 开枪前回顾上轮 identity_table：若某人 reason 已记"开枪优先级：高"，优先锁定
  【追加示例】3号 N1 reason="开枪备选：发言可疑" → N2 末尾追加为...
```

**问题 1（位置错误）**：这条"读取"指令在写指导区末尾，不在思维链 Step 中。AI 在执行 Step3（"如果我死了带走谁"）时已经走完思维链，写指导是事后填表指导，不会倒回头去影响 Step3 的决策过程。等价于备忘录只在结算时翻，而不在做决策前看。

**问题 2（感知-执行分裂）**：HUNTER_SHOOT 的 `hunterHistoryStep` 已经建立了 Step 0 读取（R41），但 DAY_SPEECH 侧没有对称的读取步骤，形成非对称闭环。

**博弈论依据**：
- Wang 2025 信息链理论：跨轮行为模式积累比单轮印象更可靠
- 猎人在残局的开枪是好人方最高价值战略动作，跨轮积累的候选比临场判断更精准
- 保持白天分析中开枪目标锁定的一致性，让猎人在整局游戏中扮演"暗中锁定"的战略角色

### 实施内容

**文件**：`src/services/aiPrompts.js`，`ROLE_DAY_SPEECH_PROMPTS['猎人']`

**改动 1 — 函数体化（R3 教训）**：
```javascript
// 旧版（箭头函数，无法前置变量）
'猎人': (ctx, params) => `${getBaseContext(ctx)}...`

// 新版（函数体，前置计算 hunterDayHistoryStep）
'猎人': (ctx, params) => {
    const hunterDayHistoryStep = ctx.dayCount > 1
        ? 'Step0: 【读取历史开枪候选（D2+适用）】...'
        : 'Step0: 【首日无历史可跳过】直接从 Step1 开始。';
    return `${getBaseContext(ctx)}...`;
},
```

**改动 2 — 思维链插入 Step0**：
```
【思维链】
${hunterDayHistoryStep}    ← 新增 Step0（D2+读取历史/D1跳过）
Step1: 我现在需要暴露猎人身份吗？...
Step2: 场上局势分析...
Step3: 如果我死了，最应该带走谁？（结合 Step0 历史候选评估；...）  ← 更新引用
Step4: 投票投谁？...
```

**改动 3 — 写指导格式更新**：
```
// 旧版末尾条目（读取指令在错误位置）
- 开枪前回顾上轮 identity_table：若某人 reason 已记"开枪优先级：高"，优先锁定

// 新版（移除读取指令，改为追加规范 + 前向引用）
- 高威胁候选（开枪首选）：...，当前开枪优先级：高"（下轮 Step 0 将直接从此读取）  ← 前向引用
- **追加不覆盖历史**：在上轮 reason 基础上追加...（用分号拼接），不覆盖历史
  【追加示例】D1 reason="开枪备选：发言可疑" → D2 末尾追加为"...升为开枪优先级：高"
```

### 关键词对齐验证（DAY→DAY→SHOOT 三环）
