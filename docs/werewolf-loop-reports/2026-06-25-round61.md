### 2. 狼人 DAY_VOTE 感知-执行分裂修复

**问题**：`DAY_VOTE` case 狼人策略仅 1 行：
```
"2. 【狼人投票博弈】投票本身是身份信号——考虑：投谁最大化狼队利益？这票会暴露你吗？能否影响最终结果形成多数？"
```

好人侧同一位置有 24 行（热力图 Step A/B + 场景提示 + 警长权重 + 预言家对跳优先）。

**根因**：与 R26/R33/R37 识别的"感知-执行分裂"相同模式——狼人在 DAY_SPEECH 的 `identity_table` 里写了"高优先刀口"标注，但 DAY_VOTE 没有任何框架指引如何将这个积累的知识转化为投票行动。

**修复**（`src/services/aiPrompts.js` lines 1830-1841）：

```javascript
// 3 点框架替换原有 1 行
`2. 【狼人投票博弈框架】
   a) 刀口对齐：先查身份推理表 reason 含"高优先刀口"的存活好人 → 首选投票目标
   b) 场景选择（正常/防守/残局）
   c) 掩护一致性：维持发言 voteIntention；改票须台词铺垫`
```

**读写闭环延伸**：复用"高优先刀口"关键词形成 DAY_SPEECH写→DAY_VOTE读 新闭环。
