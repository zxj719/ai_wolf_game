## 本轮优化目标

**骑士 DAY_SPEECH 平安夜推断三层体系（单夜→两连→三连）**

骑士是游戏中唯一没有平安夜推断框架的非村民角色。所有其他角色（村民/预言家/守卫/女巫）均已在 R80-R100 期间完成三层体系。骑士的独特视角：无私有夜间信息，但可将守护概率转化为**决斗候选 confidence 降幅**（被守护→守卫认为是好人→骑士决斗价值下降）。

### 三层激活条件（超集原则）

| 层级 | 条件 | confidence 降幅 |
|------|------|----------------|
| 单夜 (`isPeacefulNightKnight`) | `dayCount > 1 && lastNightInfo 含平安夜` | 10-20 |
| 两连 (`isConsecutivePeacefulKnight`) | `dayCount >= 3 && isPeacefulNightKnight && fullGameTimeline 含 N(dayCount-2):平安夜` | 25-35 |
| 三连 (`isTripleConsecutivePeacefulKnight`) | `dayCount >= 4 && isConsecutivePeacefulKnight && fullGameTimeline 含 N(dayCount-3):平安夜` | 35-45 |

### 注入结构（Prepend Injection 第 16 次应用）

```javascript
const consecutivePeaceHintKnight = isConsecutivePeacefulKnight
    ? `${tripleConsecutivePeaceHintKnight}⭕【两连平安夜二阶推断...`
    : '';

knightPeaceNightStep = `${consecutivePeaceHintKnight}⭕【平安夜守护来源推断...`;
```

Return 模板中在 `${knightHistoryStep}` 之后注入 `${knightPeaceNightStep}`。

### 骑士特有推断角度

普通角色（村民/预言家）的平安夜推断结论是"该目标可能是好人"。骑士的结论不同：
- "该目标可能被守护 → 守卫认为其是好人 → **决斗候选 confidence 下调**（杀错了帮狼人）"
- `identity_table` 追加：`"N${knightPrevDay}平安夜：可能被守护，决斗候选优先级下调"`

这与骑士的角色定位紧密结合：平安夜信息直接影响决斗收益计算（Step3）。

## 本轮特殊事件：Git 分叉问题

### 背景

本会话开始时发现代码库分叉：
- Local main（fa236cb）= R86（骑士延迟决斗策略）
- Remote origin/main（f8b9dad）= R87-R100 所有改进（30 commits ahead）

### 根因

在某次 detached HEAD 状态下完成的 R87-R100 工作最终通过 push 进入了 origin/main，但 local main 指针未更新，导致本轮工作时 local 和 remote 分叉。

### 解决策略（可复用）

```bash
# 1. 保存 R101 工作文件到 /tmp
cp src/services/rolePrompts/knight.js /tmp/knight_r101.js
cp src/services/__tests__/round101KnightPeaceNightInference.test.js /tmp/knight_test_r101.js

# 2. 硬性丢弃本地过期状态，快进到远程
git stash push -u -m "R101"
git pull origin main   # fast-forward: fa236cb → f8b9dad

# 3. 从 /tmp 恢复 R101 文件（避免 stash pop 的 diff 基点冲突）
git stash drop
cp /tmp/knight_r101.js src/services/rolePrompts/knight.js
cp /tmp/knight_test_r101.js src/services/__tests__/round101KnightPeaceNightInference.test.js

# 4. 正常 add/commit/push
```

关键洞察：`git stash pop` 计算 diff 相对于 **stash 创建时的 HEAD**（fa236cb, R86），应用到远程 HEAD（f8b9dad, R87-R100）时产生冲突。改用 `/tmp` 备份 + `stash drop` + 直接复制文件，绕过 diff 基点冲突。
