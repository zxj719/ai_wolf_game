## 本轮优化：摄梦人 SHERIFF_VOTE 专属提示词（R115）

### 问题根因

SHERIFF_VOTE 中摄梦人走通用空字符串 fallback，未利用角色私有信息：
- `currentDreamTarget`/`lastDreamTarget`（入梦目标）
- 若入梦目标当选警长后被刀/投出局 → 摄梦人连带死亡 + 警徽 1.5 票权重同时作废
- "同生共死"连带风险是唯一需要排除特定候选人的约束

### 修复内容（src/services/aiPrompts.js）

**新增变量块（SHERIFF_VOTE case 内，svHunterHint 之后）**：

```javascript
// R115: 摄梦人 SHERIFF_VOTE 专属提示词
const svDWHistory = gameState.dreamweaverHistory || {};
const svDWCurrentTarget = svDWHistory.currentDreamTarget ?? svDWHistory.lastDreamTarget ?? null;
const svDWTargetIsCandidate = svDWCurrentTarget !== null && svCandidateSet.has(Number(svDWCurrentTarget));
let svDreamweaverHint = '';
if (playerRole === '摄梦人') {
    const svDWTargetNote = svDWTargetIsCandidate
        ? `⚠️【连带风险】当前入梦目标 ${svDWCurrentTarget}号 正在竞选 → 若 ${svDWCurrentTarget}号 当选警长后遭狼刀/被投出局，你同时死亡，警徽 1.5 票权重随之作废`
        : '（当前入梦目标不在候选名单中，无直接连带风险）';
    svDreamweaverHint = `\n【摄梦人投票策略】你与入梦目标"同生共死"——入梦目标死亡你也死亡。投票排序：⚡金水候选人（最高可信度） > 发言最稳健的存活好人 > 弃票（-1）。${svDWTargetNote}`;
}
```

**svRoleHint 从 5-path 升级为 6-path**（在猎人分支后追加摄梦人）：

```javascript
: playerRole === '摄梦人'
? svDreamweaverHint
: '';
```

### 关键设计决策

1. **currentDreamTarget ?? lastDreamTarget** — 双路回退确保摄梦人刚开始时（无 current）仍能获取历史目标
2. **Number(svDWCurrentTarget)** — 类型安全转换，防止 Set.has() 字符串/数字不匹配（同 R96 教训）
3. **svDWTargetIsCandidate 两路径** — 有风险时展示警告，无风险时展示"无直接连带风险"（正向描述，无白熊效应）
4. **初始化为空字符串** — 非摄梦人时 svDreamweaverHint = ''，不影响其他角色

## 平衡性评估（Monte Carlo，N=600/配置）

| 配置 | 狼人胜率 | 判断 |
|------|---------|------|
| 8人标准 (2狼) | ~65% | 🟡 略偏狼人 |
| 9人标准 (2狼) | ~60% | ✅ 较为平衡 |
| 9人骑士 (2狼) | ~65% | 🟡 略偏狼人 |
| 10人魔术 (3狼) | ~87% | ⚠️ 狼人过强 |
| 10人摄梦 (3狼) | ~90% | ⚠️ 狼人过强 |
| 10人全能 (3狼) | ~84% | ⚠️ 狼人过强 |

**核心发现**：3狼配置（10人局）狼人胜率显著偏高，主要原因：
- 好人投票引入随机噪声（模拟 AI 发言不完美），狼人协调投票无噪声
- 多神职联动（守护+解药）在 3 狼压力下覆盖率不足
- **不建议在本轮实现平衡调整**（属于大改，写入 pending decisions 跟踪）
