## 2. 本轮优化详情

### 2a. baseRules.js getBaseContext 发言风格注入修复

**文件**：`src/services/rolePrompts/baseRules.js`（`getBaseContext` 函数末尾）

**问题根因**：

代码库中存在两个 `getBaseContext` 函数：
1. **`aiPrompts.js` 本地版**（被 6 个主要角色的 `ROLE_DAY_SPEECH_PROMPTS` 使用）
2. **`baseRules.js` 导出版**（被 3 个特殊角色的 rolePrompts/*.js 模块使用）

在 Round 5/6/7 迭代中，`aiPrompts.js` 的本地版本追加了：
```javascript
${ctx.currentPlayerTraits ? `\n【你的发言风格】${ctx.currentPlayerTraits} 用这个风格说话，不要和其他人发言风格一样。` : ''}
```

但 `baseRules.js` 的导出版未同步，形成了"函数分叉漂移"。

**影响范围**：
- 骑士的 `getKnightDaySpeechPrompt` 调用 `baseRules.getBaseContext` → 无个性
- 摄梦人的 `getDreamweaverDaySpeechPrompt` 调用 `baseRules.getBaseContext` → 无个性
- 魔术师的 `getMagicianDaySpeechPrompt` 调用 `baseRules.getBaseContext` → 无个性
- 6 个主要角色（狼人/预言家/女巫/猎人/守卫/村民）不受影响

**修复**：在 `baseRules.js` 的 `getBaseContext` 末尾追加：
```javascript
${ctx.currentPlayerTraits ? `\n【你的发言风格】${ctx.currentPlayerTraits} 用这个风格说话，不要和其他人发言风格一样。` : ''}
```

**为什么这样修而不是合并两个函数**：
- 合并会改变调用路径（`aiPrompts.js` 的本地版调用者接近 6 个角色的所有 DAY_SPEECH 路由），风险较高
- 追加一行是最小改动，且两个版本现在内容完全一致
