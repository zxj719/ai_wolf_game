## 本轮优化内容：女巫两连推断路径A细化（R95）

### 问题识别

R92 为女巫添加了 `consecutivePeaceHintWitch`，其中 `witchAntidoteHint`（路径A 药水联动段）的 `hasWitchSave=true` 分支（解药未动场景）仍为模糊语义：

```
'解药未动，两连平安夜均为守卫所为；守卫正积极连守，你的解药应保留到守卫无法覆盖的关键时机'
```

"关键时机"是抽象说法，AI 无法从中推导出具体的触发节点——这是感知有脚手架、执行无脚手架的问题。

### 设计决策

**双信号框架**：
- 信号①：后续出现非平安夜且连守目标死亡（守卫换守失效）
- 信号②：票压最高存活者换人（连守节奏被打破）

这两个信号分别对应守卫 coverage 的两种中断方式：
- 信号①：守卫主动换守后，原目标曝露在狼刀之下
- 信号②：连守节奏因其他因素打破（多目标高票、守卫换策略）

**identity_table 追加**：新增"守卫D${prevPrevDay}-D${prevDay}双轮连守，候选保护期，待①②信号出手"，使女巫在后续轮次能从 identity_table 回读连守状态并监测信号。

### 修改内容

**文件**：`src/services/aiPrompts.js`，`witchAntidoteHint` 的 `hasWitchSave=true` 分支

**从**（静态字符串）：
```
'解药未动，两连平安夜均为守卫所为；守卫正积极连守，你的解药应保留到守卫无法覆盖的关键时机'
```

**到**（模板字面量 + 双信号框架）：
```javascript
`解药未动，两连平安夜=守卫极可能连守同一目标（D${prevPrevDay}+D${prevDay}，confidence ≥ 95%）——今晚继续储药。出手触发信号：① 后续出现非平安夜且连守目标死亡（守卫换守失效）；② 票压最高存活者换人（连守节奏被打破）。identity_table 将 D${prevDay} 高票存活者 reason 追加"守卫D${prevPrevDay}-D${prevDay}双轮连守，候选保护期，待①②信号出手"`
```

**关键改进**：
1. `'...'` → `` `...` ``（静态字符串→模板字面量，允许 prevPrevDay/prevDay 插值）
2. "守卫正积极连守" → "守卫极可能连守同一目标（confidence ≥ 95%）"（量化置信度）
3. "保留到守卫无法覆盖的关键时机" → "出手触发信号：①②"（可操作触发条件）
4. 新增 identity_table 追加指导（read-write 闭环，下轮可从 reason 读到"待①②信号出手"）

## 博弈论依据（Wang 2025，arxiv:2408.17177）

女巫的解药是一次性资源，使用时机决定了其边际价值最大化点：
- **连守期**：守卫已在保护连守目标，解药使用价值低（好人已被守卫覆盖）
- **守卫换守后**：守卫不再覆盖原目标，若狼人立即切刀，解药在此时有最高价值
- **信号①的数学依据**：若两连平安夜后出现非平安夜且连守目标死亡，说明守卫换守后的第一晚狼人成功切刀——这是双信号中优先级最高的响应节点
