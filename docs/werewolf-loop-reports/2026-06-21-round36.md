## 2. 问题诊断

### 根因：通用 pkHint 的"感知-执行分裂"在女巫和猎人上的延伸

Round 35 修复了预言家和守卫的 pkHint 感知-执行分裂。Item 77 明确指出评估女巫/猎人是否需要专属框架。本轮评估结论：**是**。

**通用 pkHint（女巫/猎人的不足）：**
```
- 明确陈述你存活的价值（不要泛说"我是好人"，说明你有什么未完成的信息/判断/保护……）
```

**对女巫的问题：**
- 女巫的"存活价值"不是抽象的——是**可量化的具体资产**（药量）
- 不同药量状态对应完全不同的说服策略：
  - 双药均可用 → 透露"还有两张牌未打出"
  - 仅解药 → 报出救人记录（可核实的信任锚点）
  - 仅毒药 → 透露"出局等于阵营失去定向消灭机会"（最强论据）
  - 无药 → 纯逻辑辩护（无药量可用，只能靠分析）
- 通用框架的"说明你有什么未完成的保护"对女巫是**四条完全不同的路**，而不是一个点

**对猎人的问题：**
- 猎人在 PK 中面临**二元路线选择**：
  - 隐性路线（默认更优）：维持"倒下了好人也有合理结果"的不确定性威慑
  - 明示路线：直接透露猎人身份（"我是猎人，出局时开枪"），让好人有明确动机保留
- 路线选择取决于**对手性质**：对手是嫌疑玩家 → 隐性维持压力；对手是明确好人 → 明示更有说服力
- 通用框架不知道猎人有"死亡也不是纯损失"这个独特竞技优势，更不知道这个优势的两种利用路线

## 3. 技术实现

### 3.1 新增女巫专属 pkHint 块

在 `aiPrompts.js` 的 pkHint 分支链中，于 `守卫` 之后、通用 `else` 之前插入：

```javascript
} else if (playerRole === '女巫') {
    const hasSave = roleParams.hasWitchSave;
    const hasPoison = roleParams.hasWitchPoison;
    let witchMedLine;
    if (hasSave && hasPoison) {
        witchMedLine = '- 药水资产：解药和毒药均未使用——可透露双药状态作为存活价值的实质论据（"阵营还有两张未打出的牌"）';
    } else if (hasSave) {
        witchMedLine = '- 药水资产：解药尚存——可报出救人记录（哪夜救了谁），这是可核实的信任锚点';
    } else if (hasPoison) {
        witchMedLine = '- 药水资产：毒药尚存——这是最强的存活价值论据：你出局等于好人失去一次定向消灭的机会；在 thought 中评估是否透露';
    } else {
        witchMedLine = '- 药水资产：双药已用——聚焦逻辑辩护：找出对手发言中的具体矛盾，比泛说"好人"更有说服力';
    }
    pkHint = `\n\n【⚔️ PK 辩护模式（平票决赛）—— 女巫专属框架】
${witchMedLine}
- 直接回应对手：找出对方发言中的矛盾或逻辑漏洞，主动对抗可信度远高于被动辩解
- 必须提供新论点——重复之前说过的内容等于说服力归零`;
```

**设计要点：**
- `hasSave` = `roleParams.hasWitchSave`，`hasPoison` = `roleParams.hasWitchPoison`（已在 roleParams 中，无需改调用端）
- `witchMedLine` 预先计算（R18 规范），`let + if` 块（R15 规范）
- 四分支互斥，覆盖所有药量状态

### 3.2 新增猎人专属 pkHint 块

```javascript
} else if (playerRole === '猎人') {
    pkHint = `\n\n【⚔️ PK 辩护模式（平票决赛）—— 猎人专属框架】
你的存活价值有两个维度，在 thought 中选择侧重哪个：
- 维度A（隐性路线，默认）：展示判断力和逻辑分析，用"即使最坏情况好人也有合理结果"建立投票动机——维持对手的不确定性，对嫌疑玩家最优
- 维度B（明示路线）：若对手是明确好人，可透露身份（"我是猎人，出局时开枪带走可疑者"）——让好人有明确保留你的理由，但需权衡身份暴露的代价
- 无论选哪个维度：必须提供之前未说过的新论点——重复旧话等于说服力归零`;
```

**设计要点：**
- 不预设路线，在 thought 中自主选择（符合渐进式披露+信息感知原则）
- 隐性路线用正向描述（"即使最坏情况好人也有合理结果"），不含技能词汇（避免白熊效应激活 speech 输出"开枪"）
- 明示路线明确包含"开枪"——这与 R30（sheriff election）不同：PK 场景中透露身份是合法策略，不需要抑制
- 对手性质引导（嫌疑玩家 → A；明确好人 → B）是决策的关键信息
