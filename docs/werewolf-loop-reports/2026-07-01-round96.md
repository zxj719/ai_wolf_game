## 本轮优化内容：猎人延迟开枪票压参考（R96）

### 问题识别（感知-执行分裂）

猎人在 HUNTER_SHOOT 时，`hunterContext` 已包含：
- 【查杀】预言家查出的狼人（最高优先级）
- 【金水】已确认好人（排除目标）
- 【今日发言】当天发言摘要

但 **缺失前轮投票票压**：`voteHistory` 中的投票数据是公开信息，显示了好人阵营最近一轮集体怀疑的目标分布。AI 能看到 `gameState` 但 `hunterHistoryStep` 没有引导它使用票压——典型感知-执行分裂。

### 博弈论依据（Wang 2025）

票压高的存活玩家有双重含义：
1. **好人视角**：高票目标是好人阵营集体认定的嫌疑者，与"开枪优先级：高"重叠时置信度极高
2. **狼人视角**：狼人通常不会投票给自己的队员（除非故意掩护），高票存活者中出现狼人概率较高

与 identity_table 的"开枪优先级：高"交叉验证，能将猎人决策的错误率从"纯猜测"降低到"有多维度实证支撑的判断"。

### 修改内容

**文件 1：`src/hooks/useDayFlow.js`**

在 `handleAIHunterShoot` 的 `hunterContext` 构建区，发言摘要之后、BT 决策之前，新增：

```javascript
// R96: 前轮投票票压摘要（供猎人交叉参考存活高危目标，与 identity_table "开枪优先级：高" 重叠者优先级提升）
if (voteHistory && voteHistory.length > 0) {
  const latestVote = voteHistory[voteHistory.length - 1];
  if (latestVote.votes && latestVote.votes.length > 0) {
    const voteCountMap = {};
    latestVote.votes.forEach(v => {
      if (v.to !== ABSTAIN_TARGET && aliveTargets.includes(v.to)) {
        voteCountMap[v.to] = (voteCountMap[v.to] || 0) + 1;
      }
    });
    const sortedVotePressure = Object.entries(voteCountMap)
      .sort((a, b) => Number(b[1]) - Number(a[1]))
      .slice(0, 3);
    if (sortedVotePressure.length > 0) {
      hunterContext += `【D${latestVote.day}票压】${sortedVotePressure.map(([id, n]) => `${id}号(${n}票)`).join(' > ')}\n`;
    }
  }
}
```

**关键设计点**：
- 取 `voteHistory` 最后一条（最新一轮），覆盖"狩猎者死于投票"（D${n}已记录）和"死于夜间"（D${n-1}最新）两种场景
- `ABSTAIN_TARGET` 过滤弃票，`aliveTargets.includes()` 过滤已死亡玩家
- 最多显示 3 个（`.slice(0, 3)`），避免提示词过长
- 降序排列，最高票在前
