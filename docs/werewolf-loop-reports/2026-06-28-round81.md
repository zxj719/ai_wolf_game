## 实施内容

### 预言家（seerPeaceNightStep）

**角色专属推断优势**：预言家已知验证历史（金水/查杀），在平安夜时能与高票存活玩家信息结合，做出村民无法做到的三路径精确推断。

**新增代码**（在 `seerSpeechLen` 赋值后、`return` 之前）：
```javascript
// R81：平安夜推断框架（预言家专属：结合查验记录精确推断守护对象/刀口目标）
const isPeacefulNightSeer = ctx.dayCount > 1 && ctx.lastNightInfo?.includes('平安夜');
let seerPeaceNightStep = '';
if (isPeacefulNightSeer) {
    const prevDay = ctx.dayCount - 1;
    seerPeaceNightStep = `⭕【预言家平安夜推断（thought 中完成；speech 正常报验即可）】
- 查 D${prevDay} 投票记录中票压最高的存活玩家——昨夜最可能被狼刀但未死
- 路径A：该玩家已被我验为金水 → 狼明知他是好人仍刀他=核心好人；守卫/女巫保住了他；identity_table confidence 升至 90-100；今夜查验转向未知可疑目标
- 路径B：该玩家尚未被我查验过 → 狼选刀他=他对狼有威胁=更可能是关键好人角色；identity_table confidence 升 15-20，追加"排队查验优先级：①（今夜优先验）"
- 路径C：D${prevDay} 票型分散（无明显高票目标）→ 平安夜信息量有限，维持当前查验队列\n`;
}
```

**模板注入**：`${seerDayHistoryStep}\n${seerPeaceNightStep}Step1:`

**三路径设计依据**：
| 路径 | 条件 | 推断逻辑 | 行动 |
|------|------|---------|------|
| A | 高票存活者=已验金水 | 狼明知是好人仍刀=核心目标 | confidence→90-100，今夜改查其他目标 |
| B | 高票存活者=未验证 | 狼选刀=该玩家对狼有威胁 | confidence +15-20，加入查验队列①位 |
| C | 票型分散 | 信息量不足 | 维持当前队列不变 |

### 守卫（guardPeaceNightStep）

**角色专属推断优势**：守卫知道昨夜守护了谁（`lastGuardTarget`），平安夜时可推断守护是否命中狼刀目标——这是预言家和村民都无法做到的精确推断。

**新增代码**（在 `guardSpeechLen` 赋值后、`return` 之前）：
```javascript
// R81：平安夜推断框架（守卫专属：已知昨夜守护对象，推断是否命中狼刀目标）
const isPeacefulNightGuard = ctx.dayCount > 1 && ctx.lastNightInfo?.includes('平安夜');
let guardPeaceNightStep = '';
if (isPeacefulNightGuard) {
    const prevDay = ctx.dayCount - 1;
    if (lastGuardTarget !== null) {
        guardPeaceNightStep = `⭕【守卫平安夜推断（thought 中完成；speech 按普通村民发言，不提守护细节）】
- 昨夜我守了${lastGuardTarget}号，平安夜表明无人死亡
- 命中推断：若狼刀了${lastGuardTarget}号 → 守护成功，${lastGuardTarget}号 identity_table confidence 升 15-25（狼重点针对=高价值好人）
- 未中推断：若狼刀了其他人被女巫救活 → 维持${lastGuardTarget}号当前 confidence 判断
- 区分方法：D${prevDay} 高票存活玩家若恰是${lastGuardTarget}号，命中推断更可能成立\n`;
    } else {
        guardPeaceNightStep = `⭕【守卫平安夜推断（昨夜未守护）】
- 查 D${prevDay} 投票高票存活玩家——可能是女巫救或狼刀其他目标；平安夜信息量有限，维持当前守护候选判断\n`;
    }
}
```

**模板注入**：`${guardDayHistoryStep}\n${guardPeaceNightStep}Step1:`

**两分支设计**：
- `lastGuardTarget !== null`（守护已知）：精确推断命中/未中，confidence 量化更新
- `lastGuardTarget === null`（无守护记录）：回退到通用高票推断，与村民 R80 同构
