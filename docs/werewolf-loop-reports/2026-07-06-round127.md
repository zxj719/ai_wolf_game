## 优化目标（本轮：R127）

**猎人 SHERIFF_BADGE_PASS 专属分支 — 枪击×传徽协同框架**

### 问题识别

SHERIFF_BADGE_PASS 全角色覆盖状态（R127 前）：

| 角色 | 完成轮次 | 核心私有信息锚点 |
|------|---------|----------------|
| 狼人 | R64 | 传队友/混淆策略 |
| 女巫 | R113 | witchHistory.savedIds（银水） |
| 守卫 | R113 | guardHistory（守护频次） |
| 骑士 | R117 | hasUsedDuel（能力状态） |
| 魔术师 | R117 | hasRevealed + magicianHistory |
| 预言家 | R122 | seerChecks（一手验证） |
| 摄梦人 | R123 | dreamedPlayers（入梦频次） |
| **猎人** | **R127** | **博弈约束（枪击×传徽协同）** |
| 村民 | 通用 fallback（有意设计）| — |

猎人是唯一仍走通用"好人警长" fallback 的神职，且与其他角色不同：猎人无结构化私有历史（无 `hunterHistory` 存储枪击目标），但有独特博弈约束——**死亡时同时触发两个输出（开枪 + 传徽），两张牌应互补而非叠加**。

### 根因分析

猎人 BADGE_PASS 之所以被推迟到 R127：
1. R123 报告：「猎人无特殊私有历史记录，分支价值较低，暂推后」
2. 但实际上：猎人的角色本身决定了一个独特约束（两终端输出协同），即使没有历史数据也是值得编码的策略差异

### 博弈论依据

猎人死亡时：
- 输出 A（枪）：开枪带走嫌疑最高目标（确定性最大打击）
- 输出 B（徽）：传给可信好人延续 1.5 票权重

**关键约束**：输出 A 和输出 B 不应指向同一玩家。传徽给枪击目标 = 双重消耗，失去协同价值（那个玩家同时被枪和徽"花费"）。最优策略：让两张牌分别最大化各自效用。
