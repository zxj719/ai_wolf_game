## 本轮优化：4 角色发言字数差异化

### 问题

R69 已为猎人（`hunterSpeechLen`）和守卫（`guardSpeechLen`）完成字数差异化，但其余 4 个 DAY_SPEECH 角色仍使用固定字数：
- 狼人：`40-100字`（固定）
- 预言家：`40-100字`（固定）
- 女巫：`40-80字`（固定）
- 村民：`40-100字`（固定）

不同个性的 AI 在字数分布上没有任何差异，`aggressive` 预言家和 `cautious` 预言家的发言长度完全相同——这与玩家预期不符（进攻型玩家应该短促直接，谨慎型应该分析详尽）。

### 修复内容（`src/services/aiPrompts.js`）

**设计原则（R70 新建议）**：字数差异化应反映该性格在"信息处理深度"上的自然差异：
- `aggressive`：结论先行，省略铺垫，短促有力（字数下界降低 5-15 字）
- `cautious`：全面分析再表态，解释推理过程（字数上界提高 0-20 字）
- 其他类型保持默认范围

1. **狼人 `wolfSpeechLen`**（R70 新增）
   - `wolfPersonalityType` 变量新增（从 `params.personalityType` 读取）
   - aggressive → `35-55字`（短促攻击型伪装）
   - cautious → `60-100字`（谨慎建立可信形象）
   - 默认 → `40-100字`（不变）
   - 注入位置：输出JSON `"完全像好人的公开发言(${wolfSpeechLen})"`

2. **预言家 `seerSpeechLen`**（R70 新增）
   - aggressive → `40-60字`（强推不绕弯，直接结论）
   - cautious → `60-100字`（详细说明验证决策推理）
   - 默认 → `40-100字`（不变）
   - 注入位置：输出JSON `"报验人+坚定带票(${seerSpeechLen})"`

3. **女巫 `witchSpeechLen`**（R70 新增）
   - aggressive → `35-55字`（简短有力推票）
   - cautious → `50-80字`（低调精炼，稳健保守）
   - 默认 → `40-80字`（不变）
   - 注入位置：输出JSON `"发言内容(${witchSpeechLen})"`

4. **村民 `villagerSpeechLen`**（R70 新增）
   - aggressive → `35-55字`（直觉驱动，果断指名不绕弯）
   - cautious → `60-100字`（综合全框架再表态）
   - 默认 → `40-100字`（不变）
   - 注入位置：输出JSON `"有观点有分析的发言(${villagerSpeechLen}，必须包含...)"`

### 白熊效应合规

speechLen 块均为纯变量赋值（`let x = '...'; if (...) x = '...';`），无任何禁止词。T21-T24 覆盖验证 ✅。

## 下轮建议

1. **DAY_VOTE 语气风格一致性**：`personalityType` 已影响 DAY_SPEECH 风格，但 DAY_VOTE 中各角色的投票理由表述无个性差异。`aggressive` 应该一句话直接投票，`cautious` 应该先分析再投票。
2. **Night 阶段个性化（守卫）**：守卫守护决策风格差异化——`aggressive` 风格倾向多换目标、`cautious` 倾向连守同一对象。
3. **窗口截断预防性检查**：每轮修改 aiPrompts.js 大函数（尤其是 '女巫'/'狼人'/'预言家'）后，运行 `node -e "const fs=require('fs'); const src=fs.readFileSync('src/services/aiPrompts.js','utf8'); ['女巫','狼人','预言家','猎人','守卫','村民'].forEach(r => { const i=src.lastIndexOf(\"'\" + r + \"': (ctx, params) =>\"); if(i>-1) { const ret=src.indexOf('return \`', i); console.log(r+'函数至return:',ret-i,'bytes'); }})"` 检查各函数体积变化，提前预测窗口风险。
