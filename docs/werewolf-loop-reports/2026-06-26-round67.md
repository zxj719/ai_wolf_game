## 根因分析

个性类型（logical/aggressive/steady 等）已在**系统提示词**（buildPersonaPrompt）中注入 `traits` 描述，但用户提示词的【分析框架】4步骤对所有村民完全相同，导致：
- 逻辑怪和感性派使用同样的"先验证预言家 → 再分析发言"路径
- 发言内容无法体现"数据驱动 vs 直觉驱动 vs 引导驱动"的性格差异
- 系统提示词的性格描述被用户提示词的统一框架"压制"

## 修改内容

### 1. `src/services/aiPrompts.js`

**villager DAY_SPEECH 函数体转换**（R3 教训：需要预计算变量时必须用函数体）：
```js
// 旧（R66）
'村民': (ctx, params) => `${getBaseContext(ctx)}...`

// 新（R67）
'村民': (ctx, params) => {
    const personalityType = params.personalityType || '';
    let personalityLens = '';
    if (personalityType === 'logical' || personalityType === 'analytical') {
        personalityLens = `\n【你的分析视角】数据驱动型：优先用分析框架第3项...`;
    } else if (personalityType === 'aggressive') {
        personalityLens = `\n【你的分析视角】直觉驱动型：重点用分析框架第2项...`;
    }
    // ... 7 种分支，无类型 fallback = ''
    return `${getBaseContext(ctx)}
【村民发言要求】...
${personalityLens}
【分析框架】...`;
}
```

**注入位置设计**（遵循 R30 白熊效应规则）：
- 位于【村民发言要求】之后、【分析框架】之前
- 引导"优先用哪个步骤"，而非"禁止用其他步骤"
- 7 种 lens 均以"XX驱动型：优先/重点/以..."开头

**roleParams 新增字段**：
```js
personalityType: currentPlayer?.personality?.type || '',
```
