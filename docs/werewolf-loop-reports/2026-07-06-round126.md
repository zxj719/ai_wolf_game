## 优化目标（本轮：R126）

**摄梦人 DAY_VOTE 读写闭环 — dreamweaverHistory 入梦频次作投票锚点**

### 问题识别

DAY_VOTE 各角色私有信息注入状态（R126 前）：

| 角色 | 读写闭环方式 | 实现轮次 |
|------|------------|---------|
| 预言家 | seerChecks 结构体直接注入 | R119 |
| 守卫 | guardHistory 直接注入 | R120 |
| 女巫 | witchHistory 直接注入 | R121 |
| 摄梦人 | **NLP 解析 identity_table（gap）** | R72（旧） |
| 魔术师 | magicianHistory / hasRevealed | R118 |

摄梦人是唯一仍依赖 NLP 解析 `identity_table` 的神职 DAY_VOTE 策略 — 与守卫/预言家/女巫的直接结构注入不一致。

### 根因

`dreamweaverHistory.dreamedPlayers`（`useWerewolfGame.js` 初始化）是历史入梦目标 ID 数组（允许重复）：
```javascript
dreamweaverHistory: {
    dreamedPlayers: [],    // 历史入梦ID数组（可重复）
    lastDreamTarget: null,
    currentDreamTarget: null
}
```

入梦次数最多的存活玩家 = 摄梦人多轮主动判定的最高嫌疑 — 这是零歧义的私有信息，远优于 NLP 解析推断。

## 实现细节

### 新增变量（`aiPrompts.js` DAY_VOTE block）

```javascript
// R126: 摄梦人 DAY_VOTE 读写闭环
const dvDWHistory = playerRole === '摄梦人' ? (gameState.dreamweaverHistory || {}) : {};
const dvDWDreamedRaw = playerRole === '摄梦人' ? (dvDWHistory.dreamedPlayers || []) : [];
const dvDWAliveIdSet = new Set(alivePlayers.map(p => p.id));
const dvDWFreqMap = {};
dvDWDreamedRaw.forEach(id => { if (dvDWAliveIdSet.has(id)) dvDWFreqMap[id] = (dvDWFreqMap[id] || 0) + 1; });
const dvDWTopDreamed = Object.entries(dvDWFreqMap).sort((a, b) => b[1] - a[1]).slice(0, 3).map(([id, cnt]) => `${id}号(入梦${cnt}次)`);
const dvDWDreamedSummary = dvDWTopDreamed.length > 0 ? `入梦频次最高：${dvDWTopDreamed.join('、')}` : '';
```

### `dreamweaverVoteStrategy` 改为两分支条件表达式

- **Branch A（有入梦历史）**：直接注入频次锚点，梦票对齐
- **Branch B（无历史，冷启动）**：保留原 NLP 连梦候选框架作回退

```javascript
const dreamweaverVoteStrategy = dvDWDreamedSummary
    ? `2. 【摄梦人投票—入梦频次锚点框架（R126）】...a) 🌙入梦频次锚点：${dvDWDreamedSummary}...`
    : `2. 【摄梦人投票—梦票对齐框架】...（原 R72 NLP 框架）`;
```

### 设计亮点

- **角色门控**：`playerRole === '摄梦人'` 确保非摄梦人角色不注入（dvDWDreamedRaw 为空数组）
- **死亡过滤**：`dvDWAliveIdSet` 排除已出局玩家（避免投已死玩家）
- **频次排序**：重复入梦同一玩家自动累计，前3名排序展示
- **冷启动安全**：第一天无历史时回退 NLP 框架，无副作用
