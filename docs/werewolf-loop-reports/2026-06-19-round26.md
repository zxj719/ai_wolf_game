## 2. 本轮优化详情

### 优化 A：DAY_SPEECH 预言家悍跳应对策略（item 47）

**问题根因**：第 25 轮 NIGHT_SEER 增加了"悍跳响应"查验优先级（夜间），但 `ROLE_DAY_SPEECH_PROMPTS['预言家']` 中面对对跳局面的白天发言策略仍停留在"拆解对方逻辑漏洞，用你的查验记录建立公信力"这一句泛化描述，无具体执行框架。结果：AI 知道"夜间应优先验证对跳报告目标"，但白天不知道"如何用昨晚验证结果戳穿悍跳"——策略链条在日/夜转换点断裂。

**博弈论依据（Wang 2025 arxiv:2408.17177）**：
- 对跳局面的核心不是"证明自己是真的"，而是"让对方的报告无法自圆其说"——两者的认知负担完全不同，后者更可操作
- 真预言家有完整的心路历程（为什么验这个人），悍跳狼只有结果（没有决策过程），这是最可靠的鉴别维度
- Step B（矛盾点）依赖夜间策略（NIGHT_SEER 的"悍跳响应优先"），形成跨天策略协调

**改动位置（3 处）**：

| 文件 | 改动 | 说明 |
|------|------|------|
| `aiPrompts.js` DAY_SPEECH case `roleParams` | 添加 `counterClaimants` 字段 | 预言家白天可以感知对跳局面 |
| `aiPrompts.js` `ROLE_DAY_SPEECH_PROMPTS['预言家']` | 添加动态 counterClaimSection、endgameHint；重构思维链 Step2 | 主路径 |
| `seer.js` `getSeerThinkingDimensions` | 第二维度从"防守逻辑"改为"戳穿悍跳三步法" | 系统提示词同步 |
| `aiPrompts.js` `ROLE_PERSONAS['预言家']` | `thinkingDimensions[2]` 同步更新 | 降级路径同步（R4 教训） |

**counterClaimants 注入逻辑**：

```javascript
// DAY_SPEECH case roleParams（仅预言家，只过滤他人的 jump_seer 声明）
counterClaimants: playerRole === '预言家'
    ? (gameState.claimHistory || [])
        .filter(c => c.type === 'jump_seer' && c.playerId !== currentPlayer?.id)
        .map(c => c.playerId)
    : [],
```

**对跳局面专属策略（动态注入，无对跳时为空）**：

```
Step A - 主动公开查验记录：逐夜清晰报出"N[X]夜我查了[目标号]，结果是[狼人/好人]"
Step B - 找对方报告的矛盾点：若对方报某人是金水而你验过是查杀（或反之），直接指出
Step C - 用心路历程收口：解释你"为什么验这个人"——悍跳狼只有结果，没有合理的查验心路历程
```

**残局策略（存活≤5人，动态注入）**：

```
信息战让位于集票战：不需要长篇心路历程，直接用查验结果推全员集票；
若本夜验证了对跳者的报告目标，今天一锤定音公布结果
```

**格式安全（遵循 R18 教训）**：
- `counterClaimSection` 字符串中只有 `${ccIds}` 是合法插值，指导文本全部使用方括号占位符 `[目标号]`/`[狼人/好人]`
- Step A/B/C 三行均验证无 `${` 插值（T20-T22）
