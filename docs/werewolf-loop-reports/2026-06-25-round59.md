## 本轮优化内容

### 问题发现：横向一致性审计

执行 R58 建议的"横向一致性审计（三项检查：关键词四环对齐 / D1跳过逻辑 / 写指导格式统一）"：

**审计结果汇总：**

| 维度 | 审计发现 | 状态 |
|------|----------|------|
| 关键词四环对齐 | 所有角色四环（DAY写→DAY读→NIGHT读→NIGHT写）关键词精确对齐 | ✅ |
| D1跳过逻辑 | 大角色均有条件分支；wolf/villager 使用内联说明"首日无历史可跳过" | ✅ 可接受 |
| 写指导"追加不覆盖历史"格式 | **狼人 DAY 写指导和 NIGHT_WOLF 写指导均缺少此约束** | ❌ 修复 |

**缺口具体内容：**

- 已有"追加不覆盖历史"的角色：预言家（R57 line 1084）、女巫（R58 line 1127）、猎人（R56 line 1160）、守卫 DAY（R58 line 1198）、守卫 NIGHT（R40 line 1453）、村民（R18/R24 line 1232）
- 缺失此约束的角色：**狼人**（DAY 写指导 line 1007-1010 无"追加"；NIGHT 写指导 line 1545-1549 仅有"若本轮刀了某目标：追加'已行刀'"特定案例，无通用追加规则）

**博弈论依据：**
- 缺少"追加"约束时，AI 在更新"高优先刀口"标注时会倾向于重写 reason 字段（LLM 默认行为）
- 重写会导致 N3 的 identity_table 中某玩家的 reason 从"高优先刀口：N1带节奏攻预；N2推假金水"变成"高优先刀口：继续攻预"——丢失了 N1 的原始威胁记录
- 狼人的"高优先刀口"标注是跨轮刀法决策的核心依据（NIGHT_WOLF Step 0 直接读取），信息丢失直接影响刀法质量

### 实施内容

**文件**：`src/services/aiPrompts.js`

**改动 1 — 狼人 DAY_SPEECH 写指导**（添加在"对已知队友"条目之后、"suspect 字段统一用好人视角"之前）：
```
- **追加不覆盖历史**：在上轮 reason 基础上追加本轮新威胁评估（用分号拼接），不覆盖历史积累——"高优先刀口"标注应随每轮新证据叠加，而非替换
  【追加示例】D2 reason="高优先刀口：带节奏攻真预" → D3 末尾追加为"高优先刀口：带节奏攻真预；D3继续强推查杀（威胁升级）"
```

**改动 2 — NIGHT_WOLF 写指导**（在"高威胁好人"条目内，添加在"下轮 Step 0 将直接从此读取"之后、"若本轮刀了某目标"之前）：
```
  **追加不覆盖历史**：在上轮 reason 基础上追加本轮新威胁证据，不覆盖历史——每轮叠加证据，升级标注
  【追加示例】N2 reason="高优先刀口：N1带节奏攻预" → N3 末尾追加为"高优先刀口：N1带节奏攻预；N2推假金水（威胁升级）"
```
