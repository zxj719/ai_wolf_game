## 设计背景：平安夜 DAY→NIGHT 响应闭环

### R83 之前的缺口

| 阶段 | 平安夜响应 | 问题 |
|------|---------|------|
| DAY_SPEECH (R83) | wolfPeaceNightStep：两路径推断守卫/女巫来源，规划 speech | ✅ 已完成 |
| NIGHT_WOLF (R84前) | `· 若平安夜 → 守卫可能守住/女巫救了，重新评估优先目标` | ❌ 单行通用，无结构化决策框架 |

### R84 完成后的闭环

```
昨夜平安夜
├── DAY（R83 wolfPeaceNightStep）
│   ├── 用票压推断守卫/女巫来源（白天不知道刀口是谁）
│   └── 规划当天 speech 应对（次日叙事预案）
└── NIGHT（R84 wolfNightPeaceStep）
    ├── ① 直接从 identity_table 查"→已NX夜行刀"条目，确认刀口ID（不需要代理推断）
    ├── ② 用刀口目标昨日票压判断守卫 vs 女巫来源
    │   ├── 路径A：票压高 → 守卫保住（连守概率高）→ 今晚换刀
    │   └── 路径B：票压低 → 女巫消耗救药 → 维持刀口高优先
    └── ③ 追记 identity_table：追加"NX平安夜：[A换刀/B维持]"
```

### 狼人视角的独特优势（R83教训延伸到NIGHT侧）

好人侧（村民/预言家/守卫/女巫）在 DAY 侧用票压推断"昨夜刀口是谁"——这是间接推断。
狼人在 NIGHT 侧**直接知道刀口是谁**（identity_table `→已NX夜行刀`），只需用票压推断"守卫守的还是女巫救的"。
这使狼人的平安夜推断精度高于好人侧所有角色，两步合一，直接输出换刀/维持决策。

## 实施内容

### 新增变量（在 wolfLastNightBlock 之后、wolfHistoryStep 之前）

```javascript
// R84: NIGHT_WOLF 平安夜换刀决策框架 — 狼人直接知晓刀口目标，两路径推断守卫/女巫来源，优于好人侧票压代理推断
const isNightPeacefulWolf = ctx.dayCount > 1 && ctx.lastNightInfo?.includes('平安夜');
const wolfNightPrevDay = ctx.dayCount > 1 ? ctx.dayCount - 1 : 0;
const wolfNightPeaceStep = isNightPeacefulWolf
    ? `· 若平安夜 → ⭕【换刀决策框架（thought 中完成）】
      ① 查 identity_table 中含"→已N${wolfNightPrevDay}夜行刀"的条目，确认昨夜刀口目标ID
      ② 查系统提示游戏时间线 D${wolfNightPrevDay} 投票记录，分析刀口目标昨日票压：
         路径A：刀口目标昨日票压高（≥2票指向TA）→ 守卫与狼判断对齐，连守概率高 → 今晚换刀（次优目标，连守目标命中率极低）
         路径B：刀口目标昨日票压低或无票压 → 守卫守了第三方，女巫消耗了救药 → 维持今晚高优先，女巫无救药覆盖
      ③ 更新 identity_table：在刀口目标 reason 末尾追加"N${wolfNightPrevDay}平安夜：[A换刀/B维持]"`
    : '· 若平安夜 → 守卫可能守住/女巫救了，重新评估优先目标';
```

### wolfHistoryStep 改动（第 3 个分支改为 `${wolfNightPeaceStep}` 插值）

**Before**（硬编码单行）：
```
· 若平安夜 → 守卫可能守住/女巫救了，重新评估优先目标`
```

**After**（条件化变量插值）：
```
${wolfNightPeaceStep}`
```

非平安夜时，`wolfNightPeaceStep` 返回原始单行文本，向下兼容 ✅

## 两路径决策依据

| 路径 | 条件 | 推断逻辑 | 决策 |
|------|------|---------|------|
| A（守卫守住）| 刀口目标昨日票压高（≥2票）| 守卫与狼判断对齐，连守概率高 | 今晚换刀（次优目标） |
| B（女巫救了）| 刀口目标昨日票压低/无 | 守卫守了第三方，女巫消耗救药 | 维持高优先（无守护覆盖） |

**竞技依据（Wang 2025，arxiv:2408.17177）**：LLM 狼人弱点之一是跨轮资源推断——无法区分"守卫守住"和"女巫救了"两种平安夜来源，导致换刀/维持决策系统性偏差（连守目标不换刀 = 效率损失；女巫场景维持 = 已正确）。wolfNightPeaceStep 提供结构化推断框架，直接修复此弱点。
