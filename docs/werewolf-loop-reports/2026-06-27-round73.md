## 本轮优化：守卫夜间守护决策个性化

### 问题诊断

R69 补完了守卫 DAY_SPEECH personalityLens（伪装发言风格差异化），R71 补完了 DAY_VOTE personalityType（投票表述风格差异化）。但 NIGHT_GUARD case 从未有任何个性化注入——所有个性类型的守卫在夜间守护决策时使用完全相同的思维框架，"换守 vs 连守"的博弈权重对所有个性一视同仁。

这是一个隐性的个性化一致性漏洞：DAY 侧完整个性化 → NIGHT 侧零个性化。R72 的"下轮建议"明确将此列为优先级 #1。

### 博弈论依据

守卫夜间最核心的决策是：**换守 vs 连守**。Wang 2025 (arxiv:2408.17177) 和竞技 meta 共识均支持"守护行为的不可预测性"是守卫最大价值之一——狼人无法判断守卫的守护规律时，需要更高的"突破成本"才能清场。不同个性类型对这个权衡的博弈考量本质不同：

| 个性类型 | 核心特质 | 守护博弈倾向 |
|---------|---------|------------|
| aggressive | 主动进攻 | 频繁换守，不让狼人摸清规律，每次换守作为信息探测 |
| cautious | 谨慎稳妥 | 连守2-3晚，宁可被空手也保护最关键神职 |
| logical/analytical | 数据驱动 | 把守护结果当信息来源，每次决策后更新刀口推断 |
| cunning | 博弈欺骗 | 刻意制造假象再突然切换，出乎意料是核心目标 |
| emotional | 感知驱动 | 依赖当晚局势感知，感受谁最危险再交叉验证 |
| contrarian | 反直觉 | 若狼人可能预判了守护目标，选择意外目标 |
| steady | 平衡渐进 | 有据连守/有信息换守，避免过于随机或过于固定 |

### 修复内容（`src/services/aiPrompts.js`，NIGHT_GUARD case）

**注入位置**：在 Step 0（`guardHistoryStep`）之后、Step 1（守护优先级）之前，自然形成"读历史候选 → 应用风格策略 → 按优先级守护"的三阶段思维链。

```javascript
// R73：守卫夜间守护策略个性化——不同个性类型对"换守 vs 连守"博弈权重不同
const guardNightPersonalityType = currentPlayer?.personality?.type || '';
let guardNightStyle = '';
if (guardNightPersonalityType === 'aggressive') {
    guardNightStyle = '\n【你的守护风格】主动探索型：...';
} else if (guardNightPersonalityType === 'cautious') {
    guardNightStyle = '\n【你的守护风格】稳健连守型：...';
} // ... 7 种类型，无 else（向下兼容）
```

**模板插入**：
```
${guardHistoryStep}
${guardNightStyle}       ← 新增
1. 【守护优先级】...
```

**调用端无需修改**：`currentPlayer` 在 `generateUserPrompt` 闭包中直接可用（R5 教训）。

**白熊效应合规**：所有 7 种风格均为正向指令（"主动探索型"、"稳健连守型"等），无"不要频繁换守"等负向禁词 ✅。

**向下兼容**：无个性类型时 `guardNightStyle = ''`，prompt 退化到 R72 状态 ✅。
