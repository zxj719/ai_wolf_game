## 本轮优化：DAY_VOTE 语气风格一致性

### 问题

R67-R70 已为 6 个主角色的 DAY_SPEECH 添加 `personalityType` 驱动的 `personalityLens`（影响发言分析框架）和 `speechLen`（影响发言字数）。但 `DAY_VOTE` case 从未使用 `personalityType`：

- 投票时，所有角色的 `reasoning` 字段描述相同："一句话分析(言行一致投X/听了Y发言改投Y/信息不足弃票)"
- `aggressive` 预言家和 `cautious` 预言家的投票理由表述风格完全相同
- 与 DAY_SPEECH 已建立的个性化体系产生断层

### 修复内容（`src/services/aiPrompts.js`，DAY_VOTE case）

**设计原则**：`voteStyleHint` 调整"如何表达决策"，不影响"选谁"的策略框架（策略已由角色专属块覆盖）。

新增变量：
```javascript
const votePersonalityType = currentPlayer?.personality?.type || '';
let voteStyleHint = '';
if (votePersonalityType === 'aggressive') {
    voteStyleHint = `\n【投票风格—结论先行】直接说"投X号，因为Y"——一句话，不铺垫，不迟疑。`;
} else if (votePersonalityType === 'cautious') {
    voteStyleHint = `\n【投票风格—分析铺垫】先说推理过程（综合了A和B之后），再表态——reasoning 完整呈现推理链条。`;
} else if (votePersonalityType === 'emotional') {
    voteStyleHint = `\n【投票风格—感知表达】reasoning 带一丝直觉色彩（"感觉X号今天很不对劲"），仍基于发言证据但语气更感性。`;
} else if (votePersonalityType === 'cunning') {
    voteStyleHint = `\n【投票风格—策略表达】reasoning 给出表面合理的理由（观察到的行为证据），thought 中记录真实策略考量。`;
} else if (votePersonalityType === 'contrarian') {
    voteStyleHint = `\n【投票风格—差异化表达】若主流追投X，有理由时，reasoning 说明"为什么关注Y而不跟主流"。`;
}
```

注入位置：
1. `${voteStyleHint}` 注入在 `⚠️ 已验证预言家...` 行之后、`输出JSON格式:` 之前
2. 输出JSON `reasoning` 字段描述对 aggressive/cautious 差异化（ternary 表达式）

### 调用端无需修改的原因

`generateUserPrompt` 函数中 `currentPlayer` 已在所有 case 块作用域内可用（通过闭包），直接读取 `currentPlayer?.personality?.type` 即可，与 DAY_SPEECH 采用相同模式（`personalityType: currentPlayer?.personality?.type || ''` 在 roleParams 中）。

### 白熊效应合规

所有 5 种风格 hint 均为正向描述（"用X方式表达"），无"不要"/"禁止"/"绝不能"禁止语句 ✅（T11 验证）
