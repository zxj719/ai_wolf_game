## 2. 问题诊断

### 根因：感知-执行分裂（Perception-Execution Split）

**之前的状态（R15 Round 15 实现后）：**
```
wolf DAY_SPEECH 提示词：
"【2狼角色分化】先开口的狼担任【主动方】...后开口的担任【低调方】..."
```

**问题**：这个指导是给"知道自己发言顺序的人"写的，但 AI 不知道。`askAI(currentSpeaker, PROMPT_ACTIONS.DAY_SPEECH)` 不传任何参数，AI 只能靠猜：
- 如果 AI 默认当自己是"先开口"的，两只狼都会是主动方 → 两只狼同时质疑同一目标 → 协作关系曝光
- 如果 AI 默认当自己是"后开口"的，两只狼都会装低调 → 失去主动攻击能力
- 实际上 AI 可能直接忽略这个指导（无法执行的规则 = 垃圾收集）

**博弈论依据（Wang 2025 §4.3）**：多代理欺骗策略需要"全局知识 + 局部感知"两层：
- 全局知识（策略框架）：主动方/低调方分工 → 已有（R15 完成）
- 局部感知（当前状态）：我是主动方还是低调方？→ 本轮补全

## 3. 技术实现

### 3.1 `useSpeechFlow.js`（调用端）

在 `triggerAISpeech` 内、`askAI` 调用前，添加发言顺序计算：

```javascript
// 狼人发言顺序感知：判断本轮是否为首个说话的狼，用于主动方/低调方分工
const wolfSpeakParams = {};
if (currentSpeaker.role === '狼人') {
    const wolfTeammateIds = players
        .filter(p => p.isAlive && p.role === '狼人' && p.id !== currentSpeaker.id)
        .map(p => p.id);
    if (wolfTeammateIds.length > 0) {
        const alreadySpokenWolves = wolfTeammateIds.filter(id =>
            speechHistory.some(s => s.day === dayCount && s.playerId === id)
        );
        wolfSpeakParams.isFirstWolfToSpeak = alreadySpokenWolves.length === 0;
    }
}
const res = await askAI(currentSpeaker, PROMPT_ACTIONS.DAY_SPEECH, wolfSpeakParams);
```

**设计要点：**
- 只在 `wolfTeammateIds.length > 0`（有存活队友）时才设 `isFirstWolfToSpeak`，solo wolf 保持 `{}` 不传（避免无意义的单狼分支）
- 用 `speechHistory.some(s => s.day === dayCount && s.playerId === id)` 而非 `speakerIndex` 比较，确保跨天记录不干扰当天判断
- 纯函数计算，无新状态，无副作用

### 3.2 `aiPrompts.js`（提示端）

在 `wolfTeammatesHint` 构建块内，`roleDiv` 和 `lateHint` 之间插入：

```javascript
// 当知道发言顺序时，给出明确的本轮分工角色（而非模糊的"先开口者"指导）
let wolfRoleAssignment = '';
if (params.isFirstWolfToSpeak !== undefined) {
    wolfRoleAssignment = params.isFirstWolfToSpeak
        ? `\n⭐【本轮你是：主动方】队友尚未发言，你率先开口——主动点名怀疑一名好人，带动讨论方向，为队友创造"中立回应你"的叙事空间`
        : `\n⭐【本轮你是：低调方】队友已先发言——以中立评委口吻回应，在某细节上与先发言队友轻微分歧（如"我感觉X方向更值得关注"），绝不追杀与队友完全相同的目标`;
}
```

将 `${wolfRoleAssignment}` 插入 `wolfTeammatesHint` 模板（在 `roleDiv` 和 `lateHint` 之间）。

**设计要点：**
- `⭐【本轮你是：...】` 用视觉突出标记（帮助 AI 在长提示词中优先定位本轮任务）
- `isFirstWolfToSpeak === undefined` 时 `wolfRoleAssignment = ''`（兜底：历史/PK 场景无此数据）
- 措辞避开"主动方/低调方"标签时透漏队伍关系的具体词汇（遵循 R1 白熊效应原则）
