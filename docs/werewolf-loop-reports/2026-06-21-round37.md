## 2. 问题诊断

### 根因：voteMomentumHint 的"感知-执行分裂"（item 79）

**修改前**（Round 24 实现，R24-R36 沿用）：
```
【跨轮投票热力】历史投票中：X号被投N次…。热力仅作参考——警惕刷票陷阱：
  ① 热力高但本轮发言逻辑清晰/有实质分析 → 可能是狼刷票的靶子，不要盲跟
  ② 多个狼嫌疑玩家一致力推同一目标 → 高度警惕合谋，热力降权
  ③ 热力高的人总是踩明显狼嫌疑 → 可能是好人在正确分析（非靶子，可跟投）
  决策原则：先独立评估本轮发言质量，再参考热力——热力仅作辅助，切勿"高热力就跟投"。
```

**问题分析**：
- "先独立评估本轮发言质量"是目标（感知层），但没有告诉 AI 什么叫"评估发言质量"（执行层）
- ①③ 是"热力高但发言好 → 降权"和"热力高且发言对 → 可跟"两个互逆的方向，但没有 HOW（如何判断发言好或对）
- AI 收到的实际指令等价于"你先评估一下，然后参考热力"——没有执行框架的"先评估"等于随机

**依据**：Wang 2025 §4.3（多代理决策需要全局策略 + 局部感知同时具备）；R33 总结（感知-执行分裂）；R26 教训（"参数注入解决感知，策略框架才解决执行"）。

## 3. 技术实现

### 3.1 `aiPrompts.js` — voteMomentumHint 升级

**改后（Round 37）：**
```javascript
voteMomentumHint = `\n【跨轮投票热力】历史投票中：${hotTargets.map(([id, cnt]) => `${id}号被投${cnt}次`).join('、')}。
热力不能直接决定投票——必须先独立评估，再用热力校正：
  【Step A：独立评估（在 thought 中先完成此步）】对热力高的目标做三维打分：
    (a) 逻辑自洽：本轮发言有清晰分析/证据链 → 高；空话/划水/不正面回答 → 低
    (b) 信息价值：提出新判断/合理质疑 → 高；重复旧话/无实质内容 → 低
    (c) 行为连贯：与历史发言/投票行为一致 → 可信；突然改口/自相矛盾 → 低
  【Step B：热力校正逻辑（根据 Step A 结果）】
  ① 热力高 + 独立评分低（空话/矛盾）→ 热力有效，可参考跟投
  ② 热力高 + 独立评分高（逻辑清晰）→ 警惕刷票靶子，热力降权，以本轮发言质量为准
  ③ 多个狼嫌疑玩家一致力推同一目标 → 高度警惕合谋，热力直接降权
  ④ 热力高的人本轮踩明显狼嫌疑 → 好人正确分析特征，热力有效可跟投`;
```

**设计要点：**
- **Step A 强制性**：`在 thought 中先完成此步` 明确要求 AI 在输出前先走三维打分，而非事后合理化
- **Step A → Step B 绑定**：Step B 的"独立评分低/高"直接引用 Step A 的结果（①"评分低→可跟"，②"评分高→降权"），执行路径闭合
- **旧版 ③ → 新版 ④**：原版③"总是踩明显狼嫌疑"改为更精确的"本轮踩明显狼嫌疑"（移除"总是"，降低判断门槛）
- **无 JS 变量插值风险**：Step A/B 内容均为纯静态文本，动态部分（hotTargets）只在 header 中（R18 规范）

### 3.2 数据依赖（无需改动调用端）

改动仅在 template literal 的静态文本内容，不涉及任何 JS 变量读取逻辑或调用端传参。
