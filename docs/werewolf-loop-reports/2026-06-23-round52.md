## 本轮优化内容

### 1. 问题发现过程

R51 报告在"路径审计"部分明确列出 5 个角色模块的 14 个死函数，并注明"下轮可做批量清理"。确认依据：

```bash
grep -rn "getSeerNightActionPrompt|getWerewolfDaySpeechPrompt|..." src/ --include="*.js" | grep -v "rolePrompts/"
# → (no output) 确认无外部引用
```

根本原因：`promptFactory.js` 中的 `getProgressiveActionPrompt` 调用了这些函数，但 `promptFactory.js` 本身从未被任何文件 import（R2 教训明确记录，本轮再次验证）。

### 2. 删除的内容

| 文件 | 删除的函数 | 删前行数 | 删后行数 |
|------|-----------|----------|----------|
| `seer.js` | `getSeerNightActionPrompt`, `getSeerDaySpeechPrompt`, `getSeerVotePrompt` | 169 | 73 |
| `werewolf.js` | `getWerewolfNightActionPrompt`, `getWerewolfDaySpeechPrompt`, `getWerewolfVotePrompt` | 181 | 93 |
| `hunter.js` | `getHunterShootPrompt`, `getHunterDaySpeechPrompt`, `getHunterVotePrompt` | 156 | 72 |
| `guard.js` | `getGuardNightActionPrompt`, `getGuardDaySpeechPrompt`, `getGuardVotePrompt` | 184 | 86 |
| `villager.js` | `getVillagerDaySpeechPrompt`, `getVillagerVotePrompt` | 150 | 79 |
| **合计** | **14 个死函数** | **840** | **403 (-437)** |

同时清理的无用导入：
- `seer.js`：`getBaseContext`（只在 `getSeerDaySpeechPrompt` 中使用）
- `werewolf.js`：`getBaseContext`、`isMiniGame`（均只在死函数中使用）
- `hunter.js`：`getBaseContext`（只在 `getHunterDaySpeechPrompt` 中使用）
- `guard.js`：`getBaseContext`（只在 `getGuardDaySpeechPrompt` 中使用）
- `villager.js`：`getBaseContext`（只在 `getVillagerDaySpeechPrompt` 中使用）

每个 PROMPTS 导出对象只保留活跃键：`persona`、`getThinkingDimensions`、`getPriorities`、`buildPersonaPrompt`。

### 3. 保留的活跃函数（未动）

对每个角色，以下函数均通过 `aiPrompts.js → buildPersonaPrompt() → getRoleModule(role).buildPersonaPrompt()` 调用链保持活跃：

- `getXxxPriorities`（优先级，由 `buildPersonaPrompt` 内部调用）
- `XXX_PERSONA`（人格定义常量）

### 4. knight aliveCount 参数合同扩展（R51 遗留建议）

在 `round52DeadCodeCleanup.test.js` 的 T26-T28 中补充了 delegateParams 检查：

- T27: `aiPrompts.js` roleParams 块中存在 `aliveCount`
