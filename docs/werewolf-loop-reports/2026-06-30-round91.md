## 本轮优化内容：守卫两连平安夜二阶推断（R91）

### 问题识别

R81 已为守卫添加了单夜平安夜推断（`isPeacefulNightGuard` + `guardPeaceNightStep`），但 D3+ 连续两夜平安夜场景下，守卫 AI 能看到 `fullGameTimeline` 中的两次平安夜记录，却没有推断脚手架来利用。守卫拥有 `guardHistory`——比任何好人侧角色更直接的历史信息——但存在典型的"感知-执行分裂"。

### 修改内容

**文件**：`src/services/aiPrompts.js`，守卫 DAY_SPEECH 函数

**新增外层变量**（在 `isPeacefulNightGuard` 声明之后、`if` 块之前）：
```javascript
const isConsecutivePeacefulGuard = ctx.dayCount >= 3 && isPeacefulNightGuard &&
    ctx.fullGameTimeline?.includes(`N${ctx.dayCount - 2}:平安夜`);
```

**新增 if 块内变量**（三路径：prevPrevDay + prevPrevNightGuardTarget + consecutivePeaceHintGuard）：
- `prevPrevDay = ctx.dayCount - 2`
- `prevPrevNightGuardTarget = guardHistory?.find(g => g.night === ctx.dayCount - 2)?.targetId ?? null`（守卫独有：直接从历史读取 N-2 守护目标）
- `consecutivePeaceHintGuard`（三元表达式：两连时注入差异化推断，否则为空字符串）

**注入内容**（`consecutivePeaceHintGuard` 为真时）：
- 查守护记录对比（N-2 vs N-1 守护目标）
- 路径A：连守同一目标 → 命中可信度极高，confidence 升 25-35，今晚应换守
- 路径B：守护目标不同 → 按 D-2 和 D-1 投票记录分别评估两夜命中可能
- `identity_table` 追加：命中目标 reason 加两连平安夜分析标记

**前置注入**：在原 `guardPeaceNightStep = \`` 赋值前以 `${consecutivePeaceHintGuard}` 前置拼接，保留原 R81 内容完整。

### 守卫两连推断的独特设计维度（R91-A）

守卫在好人侧两连推断中精度最高：
- **村民**（R88）：用 D-2 和 D-1 票压推断刀口是谁（两层间接推断）
- **预言家**（R90）：用查验记录做交叉验证（一层间接 + 一层直接验证）
- **守卫**（R91）：直接从 `guardHistory` 读取 N-2 守护目标，与 N-1 对比，精确判断连守/换守（零间接推断）

这使守卫的两连推断路径判断完全确定性：A=连守同一目标 vs B=不同目标，没有"可能是"的模糊性。
