## 2. 问题诊断

### 问题一：NIGHT_SEER identity_table 的读写闭环缺失

**发现方式**：人工评估——R40 已知"常驻诊断命令只能发现旧关键词类缺口"；R25 在 NIGHT_SEER 写指导中加入了"排队查验优先级：[①②③④⑤ 哪级]"标记，与 R38 NIGHT_WOLF "高优先刀口"完全同构，但 NIGHT_SEER 思维链从无 Step 0 读取该历史队列。

**信息流断裂（同 R38/R39/R40 模式）**：
```
N1 夜写：reason = "行为可疑：[发言带节奏]，排队查验优先级：①"  ← 写入 identity_table（R25 加入）
N1 夜后：identity_table 存入 identityTablesRef                  ← 正常存储
N2 夜始：系统提示注入 previousIdentityTable                      ← 正常传递
N2 夜思维链：直接进入 seerNightStrategy 五级框架（从零重评）      ← 从不提及"检查历史候选队列"
                                                                 ← 感知-执行分裂
```

**预言家特殊性**：与 WOLF/WITCH/GUARD 不同，预言家的"排队查验优先级"记录的是**尚未验证的嫌疑候选**，而非执行历史。Step 0 的说明因此加了"结合新信息重新排序；已死亡目标跳过"——历史候选是起点，不是硬约束（不同于守卫的"守护优先级"约束）。

**修复后**：
```
N2 夜 Step 0："查看系统提示中【你之前的身份推理表】：哪些玩家 reason 含'排队查验优先级'？
              将其作为今晚查验候选起点（结合新信息重新排序；已死亡目标跳过）"
                                                                 ← 显式读取指令，形成闭环
```

### 问题二：HUNTER_SHOOT identity_table 的读写闭环缺失

**发现方式**：人工评估——DAY_SPEECH 猎人写指导（lines 1130-1134）已说"开枪前回顾上轮 identity_table：若某人 reason 已记'开枪优先级：高'，优先锁定"，但 HUNTER_SHOOT case 本身（lines 1725-1763）无对应 Step 0 读取步骤。

**信息流断裂**：
```
多轮 DAY_SPEECH 写：reason 积累"开枪优先级：高：[多轮可疑描述]"  ← 写入 identity_table（R18 加入）
HUNTER_SHOOT 触发：思维链直接进入 hunterStrategies（查杀优先等）  ← 从不读取历史积累
                                                                    ← 16 轮写指导说"开枪前回顾"，
                                                                       但 HUNTER_SHOOT 从未真正读
```

**猎人特殊性**：① HUNTER_SHOOT 不是 NIGHT_* action，不需要 `hunterNightLabel`（R18 规范不适用）② 查杀信息（seer kills）优先于历史候选——hunterHistoryStep 明确说明"预言家查杀信息优先于此"③ 开枪是一次性行为，历史候选应作为"排查起点"而非硬约束。

**修复后**：
```
HUNTER_SHOOT Step 0："先查看系统提示中【你之前的身份推理表】：哪些玩家 reason 含'开枪优先级：高'？
                      这是你日间积累的高嫌疑候选（注：预言家查杀信息优先于此；已死亡目标跳过）"
                                                                    ← 显式读取指令 + 优先级澄清，形成闭环
```

## 3. 技术实现

### 3.1 NIGHT_SEER 修改（`src/services/aiPrompts.js`）

**变更 1：新增两个 const（R18 规范 + 读写闭环）**
```javascript
// R18 规范：避免在 return 模板字符串内直接使用 ctx.dayCount
const seerNightLabel = `N${ctx.dayCount}`;
// 读写闭环（同 R38/R39/R40 NIGHT_WOLF/WITCH/GUARD 模式）
const seerHistoryStep = ctx.dayCount > 1
    ? '0. 【读取历史查验候选】查看系统提示中【你之前的身份推理表】：哪些玩家的 reason 含"排队查验优先级"？将其作为今晚查验候选起点（结合新信息重新排序；已死亡目标跳过）'
    : '0. 【首夜】无历史查验队列——直接按下方策略选择查验目标';
```

**变更 2：return 模板更新**
```diff
- 【查验历史】本局已查验 ${mySeerChecksCount} 人，当前第 N${ctx.dayCount} 夜
+ 【查验历史】本局已查验 ${mySeerChecksCount} 人，当前第 ${seerNightLabel} 夜
  ${nightCot}
+ ${seerHistoryStep}       ← Step 0：读取历史查验队列
  ${seerNightStrategy}     ← ①-⑤ 优先级框架
```

**变更 3：identity_table 写指导升级**
```diff
- reason 写"N[X]夜查验确认：..."
+ reason 写"${seerNightLabel}夜查验确认：..."   ← 动态夜次标签

- 排队查验优先级：[①②③④⑤ 哪级]
+ 排队查验优先级：[①②③④⑤ 哪级]（下轮 Step 0 将直接从此读取）  ← 前向引用
```

### 3.2 HUNTER_SHOOT 修改（`src/services/aiPrompts.js`）

**变更 1：新增 hunterHistoryStep**
```javascript
// 读写闭环：读取 DAY_SPEECH 历史中积累的开枪优先候选（R38/R39/R40 同构模式）
const hunterHistoryStep = '0. 【读取历史开枪候选】先查看系统提示中【你之前的身份推理表】：哪些玩家 reason 含"开枪优先级：高"？这是你日间积累的高嫌疑候选（注：预言家查杀信息优先于此；已死亡目标跳过）';
```

**变更 2：return 模板更新**
```diff
+ 【开枪决策（思维链）】
+ ${hunterHistoryStep}       ← Step 0：读取历史积累
  【开枪策略——按优先级从高到低（Step 0 历史候选可提升以下优先级）】
  ${hunterStrategies.map(...).join('\n')}
```
