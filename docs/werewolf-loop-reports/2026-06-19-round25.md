## 2. 本轮优化详情

### 优化 A：NIGHT_SEER 信息增量策略（item 44）

**问题根因**：原版 NIGHT_SEER 的"后续夜策略"只有三条泛化建议（最可疑/排除法/信息增量），没有给 AI 具体的**优先级排序框架**。预言家在面对对跳局面、多路汇聚嫌疑、投票争议、信任链延伸等不同场景时，无法做出区分。

**博弈论依据（Wang 2025 arxiv:2408.17177 信息链策略）**：

信息增量最大化原则认为：每次查验应选择**能解开最多未知变量**的目标，而不仅仅是"最可疑"的目标。具体体现：
1. **悍跳响应优先级最高**：验证对跳报告中某个"金水"或"查杀"，一旦揭穿假报，整个对跳局面立即解决（信息增量 = 全局对跳迷雾 × 公信博弈）
2. **多路汇聚**：多条独立推理链指向同一人时，查验该人能同时验证多条链，信息效率是普通目标的倍数
3. **残局翻转**：当存活≤5人时，不再需要"最大化覆盖率"，而应切换到"精准确认当前投票焦点"——因为信息不再稀缺，而机会变得极为有限

**改动路径（2 文件 3 处）**：

| 位置 | 改动 | 说明 |
|------|------|------|
| `aiPrompts.js` NIGHT_SEER case | 添加 `{}` 花括号、动态悍跳警报（claimHistory.jump_seer）、查验历史计数、五级优先级框架、残局策略 | 主路径 |
| `seer.js` getSeerThinkingDimensions | 第一维度从"查谁能提供最大信息量"改为五级框架摘要 | 系统提示词同步 |
| `aiPrompts.js` ROLE_PERSONAS 预言家 | thinkingDimensions[0] 与 seer.js 保持一致 | 降级路径同步（R4 教训） |

**新增的三层动态上下文**：

```javascript
// 1. 悍跳警报：动态检测 claimHistory 中 jump_seer 类型的声明者
const counterClaimants = (gameState.claimHistory || [])
  .filter(c => c.type === 'jump_seer' && c.playerId !== currentPlayer?.id)
  .map(c => c.playerId);
const counterClaimText = counterClaimants.length > 0
  ? `⚠️ 【悍跳警报】${counterClaimants.join(',')}号已自称预言家（对跳）...`
  : '';

// 2. 查验历史计数
const mySeerChecksCount = (gameState.seerChecks || []).filter(c => c.seerId === currentPlayer?.id).length;

// 3. 残局判断阈值（≤5人）
const isEndgame = players.filter(p => p.isAlive).length <= 5;
```

**五级优先级框架（后续夜，非残局）**：
```
① 悍跳响应（最高优先）：验证对跳报告中的"金水"/"查杀"——揭穿假报赢公信权
② 多路汇聚：2+ 独立玩家同时指向同一人——查验验证多条推理链，信息倍增
③ 投票关键位：明天将被集中投票但仍有争议的玩家——提前查验引导集票
④ 信任链延伸：已确认 A 是好人，A 强推 B，B 未验证——沿信任链扩展确定性
⑤ 行为异常兜底：突然转向/沉默/强带节奏的未验证玩家
```

**残局策略（≤5人）**：
```
信息采集让位于精准打击：直接查验投票争议焦点目标——查杀立即带全员集票；查出好人也能防内讧
```
