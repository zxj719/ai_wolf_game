## 本轮优化内容

### 问题分析

预言家 DAY_SPEECH（`ROLE_DAY_SPEECH_PROMPTS['预言家']`）的 identity_table 写指导已有如下内容（旧版）：
```
- 未查验可疑玩家：reason 写"发言分析：[可疑点]，排队查验优先级：②③④⑤之一（下夜 Step 0 读取并结合新信息重排序）"
```

**问题 1（读写分裂）**：写指导的 `排队查验优先级` 关键词指向"下夜 Step 0 读取"（NIGHT_SEER Step 0 已建立于 R41）。但 DAY_SPEECH 思维链没有对称的 Step 0——每天白天发言时 AI 不会系统读取历史积累的候选队列，而是从头评估，导致跨天验证计划缺乏连贯性。

**问题 2（DAY→DAY 断链）**：现有闭环为 DAY_SPEECH 写→NIGHT_SEER Step0 读，缺少 NIGHT_SEER 写→DAY_SPEECH Step0 读的对称方向。NIGHT_SEER 写指导也有 `排队查验优先级` 标注，但 DAY_SPEECH 无法在白天发言时读取上一夜的积累。

**问题 3（追加格式缺失）**：写指导未包含"追加不覆盖历史"规范和【追加示例】（对比 R18/R19/R24 已为其他角色建立的格式）。

**博弈论依据**：
- Wang 2025 信息链理论：跨轮行为模式积累比单轮临场判断更可靠
- 预言家的验证计划应跨夜保持连贯性（D1 白天分析→N1 验证→D2 白天延续候选评估→N2 验证），而非每天重置
- 白天发言先读历史候选，才能在 Step3 投票推荐中对高优先候选保持公开一致性

### 实施内容

**文件**：`src/services/aiPrompts.js`，`ROLE_DAY_SPEECH_PROMPTS['预言家']`

**改动 1 — 新增 seerDayHistoryStep 变量（return 前）**：
```javascript
// 读写闭环：DAY_SPEECH Step 0（R57 — 与 NIGHT_SEER Step 0 共用"排队查验优先级"关键词）
const seerDayHistoryStep = ctx.dayCount > 1
    ? 'Step0: 【读取历史验证候选队列（D2+适用）】查看系统提示中【你之前的身份推理表】：哪些玩家的 reason 含"排队查验优先级"标注？将优先级①②③之一作为今天发言分析和投票引导的重点起点（结合今日新发言信息重排序；已死亡/已查验目标跳过）。今晚查验优先从此队列中选取。'
    : 'Step0: 【首日无历史候选队列】直接从 Step1 开始——今天是第一天，尚无跨轮积累的查验候选。';
```

**改动 2 — 思维链插入 Step 0 + 更新 Step3**：
```
【思维链】
${seerDayHistoryStep}    ← 新增（D1跳过/D2+读取历史候选）
Step1: 我有什么查验信息必须报？（全部报出，不要藏）
Step2: 场上有对跳预言家？→ 有：执行【对跳局面专属策略】Step A-B-C；无：跳过
Step3: 今天必须投谁出局？（查杀 > 对跳悍跳玩家 > Step0 历史候选 > 最可疑者）  ← 更新引用
Step4: 绝不能投金水！voteIntention 必须指向狼人或最可疑的人
```

**改动 3 — 写指导格式更新**：
```
// 旧版（仅有下夜引用，无 DAY→DAY 前向引用，无追加格式）
- 未查验可疑玩家：reason 写"...，排队查验优先级：②③④⑤之一（下夜 Step 0 读取并结合新信息重排序）"

// 新版（补全 DAY→DAY 前向引用 + 追加不覆盖历史 + 追加示例）
- 未查验可疑玩家：reason 写"...，排队查验优先级：②③④⑤之一（下夜 Step 0 读取；下轮 DAY Step 0 同样从此起点读取）"
- **追加不覆盖历史**：在上轮 reason 基础上追加本轮新观察（用分号拼接），不覆盖历史积累
  【追加示例】D2 reason="发言分析：N1 带节奏可疑，排队查验优先级：②" → D3 末尾追加为"发言分析：N1 带节奏可疑，排队查验优先级：②；D3 投票压票异常，优先级升为①"
```
