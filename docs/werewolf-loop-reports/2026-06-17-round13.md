## 2. 本轮优化详情

### 问题发现

在多狼场景（2狼及以上）中，`useNightFlow.js` 对每一只 AI 狼独立调用 `askAI(actor, PROMPT_ACTIONS.NIGHT_WOLF)`，且每次调用都执行 `mergeNightDecisions({ wolfTarget: res.targetId })`。这导致：

1. **目标覆盖**：后置狼的决策会覆盖前置狼的 `wolfTarget`，最终目标是"最后一只运行的 AI 狼的选择"
2. **API 浪费**：多只狼独立调用 LLM，但只有最后一只的结果被使用
3. **决策不一致**：各狼独立思考，可能选择不同目标（如 A 狼刀预言家，B 狼刀守卫），导致不协调

原有的 `multiWolfHint` 只泛化说"统一刀口避免资源浪费"，但没有解决根本问题——后置狼仍会独立决策并覆盖结果。

### 修复方案

**双层修复**：

**层 1 — 代码修复（useNightFlow.js lines 562-638）**：
```javascript
// 多狼协作：首狼决策后，后续狼直接确认，不重复 LLM 调用（防止后置狼覆盖首狼目标）
if (nightDecisions.wolfTarget !== null) {
    logger.debug(`[狼人AI] 队友已选定 ${nightDecisions.wolfTarget}号，${actor.id}号确认统一刀口`);
    if (gameMode === 'ai-only') {
        addLog(`[${actor.id}号] 狼人确认袭击 ${nightDecisions.wolfTarget}号（与队友统一刀口）`, 'system');
    }
    recordNightAction({ playerId: actor.id, type: '确认', target: nightDecisions.wolfTarget, ... });
    // 不调用 askAI，不重写 wolfTarget
} else {
    // 原有 AI 决策逻辑（首狼运行）
}
```

**层 2 — 提示词增强（aiPrompts.js NIGHT_WOLF 的 multiWolfHint）**：
```
⚡ 你是本晚的刀口决策人——你选定的目标即为全队行动目标，队友将直接执行你的决定。
综合评估：① 目标期望价值（角色威胁 × 存活优势） ② 被守护概率（连守规律） ③ 换人成本（猎人/骑士反杀）
```

**博弈论依据**：

Wang 2025 (arxiv:2408.17177) 指出多方合作欺骗（cooperative deception）中，团队内部协调一致是关键成功要素。狼队的最优策略是：首晚建立内部优先级体系，后续夜晚基于信息更新调整目标——而不是各自独立决策产生冲突。将首狼设为"决策人"模拟了真实游戏中狼队的内部协商机制。

### 关键设计考量

- **为何不让所有狼都参与投票？** 实现复杂，且真实游戏中狼队通常是"有主见的那个狼定调，其他狼协作"——首狼位置（先行动的狼）自然成为决策人，符合游戏逻辑。
- **后续狼 recordNightAction 是否必要？** 是的——为了保持夜间行动日志的完整性（每只狼的行动都被记录），便于回放和调试。
