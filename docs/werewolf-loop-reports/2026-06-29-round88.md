## 问题背景

### R80-R85 平安夜框架的局限

R80（村民）、R82（预言家）、R83（狼人）、R84（守卫 NIGHT）、R85（守卫/女巫/狼人 NIGHT）建立了完整的单次平安夜推断框架：

- **一阶信息**：单次平安夜 → 查高票存活者 → 推断守护来源 → 调整 confidence

但 D3+ 的**两连平安夜**提供了更强的二阶约束，完全缺乏对应支持：

| 场景 | 一阶推断（已有） | 二阶推断（R88 新增） |
|------|--------------|------------------|
| 单次平安夜 | 推断守护来源（守卫 or 女巫） | — |
| 两连平安夜（两夜高票同人） | 无 | 守卫连守同目标概率极高，该目标 confidence 大幅下调（核心好人） |
| 两连平安夜（两夜高票不同人） | 无 | 守卫轮换目标，两名玩家均下调，女巫资源仍可能完整 |
| 狼人两连平安夜 | 无 | 两夜刀口分析 → 换刀必要性 + 女巫解药状态评估 |

### `fullGameTimeline` 格式作为检测工具

`ctx.fullGameTimeline` 已在 `prepareGameContext()` 中构建，格式为：
```
N1:平安夜 → N2:5号死亡 → D2:3号出局 → N3:平安夜
```

通过模板字符串 `\`N${ctx.dayCount - 2}:平安夜\`` 可以精确检测前一夜是否也是平安夜，不需要解析完整时间线：

```javascript
const isConsecutivePeacefulVillager = ctx.dayCount >= 3 && isPeacefulNight &&
    ctx.fullGameTimeline?.includes(`N${ctx.dayCount - 2}:平安夜`);
```

## 关键设计决策

### "前置注入"模式（不是 else 分支）

选择 `${consecutivePeaceHintX}⭕【原始内容...】` 而非 `if (isConsecutive) { ... } else { 原始 }` 模式。

**原因**：R80 T17 测试在 `if (isPeacefulNight)` 之后 500 字符内检查无 `} else {`。使用三元表达式赋值（ternary），在两连情况下提供额外上下文，在单夜情况下为空字符串 — 两种情况下原始内容都完整保留。
