## 本轮优化内容

### 1. 发现 witch.js 主路径与降级路径的 taboos 分叉（49 轮未修复）

**背景**：LEARNINGS R1（2026-06-15）记录了女巫 taboos 的修复：`'首夜不救人'` → `'同一晚又救又毒'`。但实际执行时只修了 `aiPrompts.js` 的 `ROLE_PERSONAS['女巫'].taboos`（降级路径），未同步修 `witch.js` 的 `WITCH_PERSONA.taboos`（主路径）。

**两条路径的实际状态（修复前）**：
```js
// witch.js 主路径（buildWitchPersonaPrompt → WITCH_PERSONA.taboos）
taboos: ['首夜不救人', '盲毒好人', '过早暴露身份', '浪费解药救狼'],
//        ↑ 错误：含义为"必须首夜救人"，与有守卫时的策略矛盾

// aiPrompts.js 降级路径（ROLE_PERSONAS['女巫'].taboos）
taboos: ['同一晚又救又毒', '盲毒（逻辑不充分时用毒）', '过早暴露身份', '浪费解药救狼'],
//        ↑ 正确（R1 已修）
```

**`buildPersonaPrompt` 的调用链**：
```js
// aiPrompts.js line 225-227（主路径，优先级高于降级路径）
const roleModule = getRoleModule(player.role);
if (roleModule && roleModule.buildPersonaPrompt) {
    return roleModule.buildPersonaPrompt(player, existingRoles, gameSetup);  // ← 调用 witch.js 的版本
}
// 以下才是降级路径（只有 roleModule 不存在时才走）
```

**修复**：

```js
// witch.js WITCH_PERSONA.taboos（修复后）
taboos: ['同一晚又救又毒', '盲毒（逻辑不充分时用毒）', '过早暴露身份', '浪费解药救狼'],
```

### 2. NIGHT_WITCH firstNight hint 静态字符串未感知守卫存在

**背景**：NIGHT_WITCH case 的首夜提示词（`witchHint`）是静态字符串，不区分"有守卫"和"无守卫"：

```js
// 修复前（静态）
const witchHint = ctx.dayCount === 1 
    ? '【首夜策略】通常使用解药救人。⚠️ 重要：第一晚女巫可以自救（如果自己被刀）！'
    : '';
```

当局中有守卫时，"通常使用解药救人"与同守同救规则冲突（女巫和守卫同时保护同一目标会导致目标死亡）。dead function `getWitchNightActionPrompt`（witch.js 第 66 行）早在 R1 就已实现守卫感知，但 live path 从未同步。

**修复**：

```js
// 修复后（守卫感知）
const witchExistingRoles = detectExistingRoles(players);
const witchHint = ctx.dayCount === 1
    ? (witchExistingRoles.hasGuard
        ? '【首夜警告】守卫可能也守了被刀目标，同守同救会导致目标死亡！除非目标是关键神职，否则首夜建议不救。⚠️ 例外：若被刀者是你自己，必须自救！'
        : '【首夜策略】没有守卫，无同守同救风险，可直接救关键目标。⚠️ 重要：第一晚可以自救（若被刀）！')
    : '';
```
