## 2. 问题诊断

### 根因：PK 发言是「发言目的反转」场景

**正常白天发言**（`pkMode` 未设）：
- 狼人目标：**进攻**——分散好人注意力、推好人出局、与队友协调
- `wolfTeammatesHint` 提供：角色分化（主动方/低调方）、立场分散、投票错位

**PK 辩护发言**（`pkMode: true`）：
- 狼人目标：**自保**——说服场上玩家不投自己、推 PK 对手出局
- 但 `wolfTeammatesHint` 仍提供：进攻协作框架（主动方/低调方分工）

这是方向性错误，不仅无用，还会让 AI 感到困惑（"我应该质疑好人带节奏还是辩护自己？"）。

### 最危险场景：双狼 PK

当两只狼都进入 PK（平票局），如果其中一只收到「帮队友营造形象」的提示，好人立刻可以识别关系链。正确策略是反直觉的：**此时应该攻击队友（逻辑上），建立「各自为战」的假象**。

## 3. 技术实现

### 3.1 `useDayFlow.js`（调用端）

在 `handlePKRound` 的 DAY_SPEECH 调用中透传 `pkCandidates`：

```javascript
// 之前
const res = await askAI(candidate, PROMPT_ACTIONS.DAY_SPEECH, { pkMode: true });

// 之后
const res = await askAI(candidate, PROMPT_ACTIONS.DAY_SPEECH, { pkMode: true, pkCandidates: pkCandidateIds });
```

### 3.2 `aiPrompts.js` — 狼人 pkMode 覆盖逻辑

在 `wolfTeammatesHint` 构建之后、`return` 之前，添加 pkMode 覆盖：

```javascript
if (params.pkMode) {
    const teammatesInPk = (params.wolfTeammates || []).filter(id =>
        (params.pkCandidates || []).includes(id)
    );
    if (teammatesInPk.length > 0) {
        // 双狼 PK：反协调指令（攻击队友逻辑漏洞，建立各自为战叙事）
        wolfTeammatesHint = `\n🚨【PK 危机：队友${teammatesInPk.join(',')}号也在 PK！】...`;
    } else {
        // 有场外队友：中性自保提示
        const outsideHint = ...; // 场外队友正在观看的轻提示
        wolfTeammatesHint = outsideHint;
    }
}
```

**设计要点：**
- `if/else` 块（遵循 R15：多分支用 let + if，不用三元）
- 双狼 PK 时 `wolfTeammatesHint` 被完全替换，不保留任何进攻协作内容
- 场外队友场景：保留轻量的"他们在观看你"提示，不发送协作指令

### 3.3 `aiPrompts.js` — 全局 pkHint 后处理

在 DAY_SPEECH case 中，与 `sheriffHint` 并排添加 `pkHint`：

```javascript
const pkHint = roleParams.pkMode
    ? `\n\n【⚔️ PK 辩护模式（平票决赛）】你与其他候选人进入最终对决！...`
    : '';

return rolePromptGenerator(ctx, roleParams) + pkHint + sheriffHint + CLAIMS_SCHEMA_SUFFIX;
```

**pkHint 三要点：**
1. 提供新论点（不重复之前内容）
2. 直接质疑对方（主动对抗 > 被动辩解）
3. 陈述存活价值（具体信息/技能，不泛说"我是好人"）
