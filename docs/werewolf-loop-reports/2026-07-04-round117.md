## 本轮优化：骑士 + 魔术师 SHERIFF_BADGE_PASS 专属提示词（R117）

### 问题根因

SHERIFF_BADGE_PASS 中骑士和魔术师均走通用 fallback（"你是好人警长：把警徽传给你最确信的好人"），未利用角色私有信息：

- **骑士**：`hasUsedDuel`（能力状态）— 未用时能力将随离场作废（从未打出的战略资产），传徽更应优先最强好人；已用时能力结果是最高可信度判断依据
- **魔术师**：`hasRevealed`（身份暴露）+ `magicianHistory.swappedPlayers`（交换记录）— 身份公开时交换知识是一手信息（优于所有人的 identity_table 推断）；身份隐藏时传徽选择不应暴露私有信息

### 修复内容（src/services/aiPrompts.js）

**骑士 bpKnightHint 新增（R117 标注）**：

```javascript
// R117: 骑士 SHERIFF_BADGE_PASS 专属提示词
const bpKnightDueled = currentPlayer?.hasUsedDuel ?? false;
let bpKnightHint = '';
if (playerRole === '骑士') {
    if (bpKnightDueled) {
        bpKnightHint = '\n🗡️【骑士传徽私有信息】你已使用能力，能力结果揭示了场上身份真相——以此为最高优先信任锚点...';
    } else {
        bpKnightHint = '\n🗡️【骑士传徽私有信息】你的能力尚未使用，将随你离场作废...确保 1.5 票落在最强好人手中；无充分依据时撕掉（-1）。';
    }
}
```

**魔术师 bpMagicianHint 新增（R117 标注）**：

```javascript
// R117: 魔术师 SHERIFF_BADGE_PASS 专属提示词
const bpMagHistory = gameState.magicianHistory || { swappedPlayers: [], lastSwap: null };
const bpMagSwappedCount = (bpMagHistory.swappedPlayers || []).length;

let bpMagicianHint = '';
if (playerRole === '魔术师') {
    if (bpMagIsRevealed) {
        // 已公开：交换知识作一手信息
    } else {
        // 隐藏：按通用优先级，不暴露私有信息
    }
}
```

**bpIdentityStep 从 4-path 扩展至 6-path**：

新增骑士分支：`能力结果确认的好人（若已使用）`
新增魔术师分支：`交换知识确认的好人（若已公开）`

**bpHint 从 4-path 扩展至 6-path**：

新增骑士：`'你是骑士警长：参考下方能力信息（见私有信息块）和预言家金水，选出最可信的传徽对象；无充分依据时撕掉（-1）。'`
新增魔术师：`'你是魔术师警长：参考下方交换知识（见私有信息块）和预言家金水，选出最可信的传徽对象；无充分依据时撕掉（-1）。'`

**return 模板**：`${bpKnightHint}${bpMagicianHint}` 追加至末尾

### 关键设计决策

1. **骑士两路径基于能力使用状态**：未用 = 能力作废，强调未打出资产，传徽优先最强好人；已用 = 能力结果是可信依据，以能力结论为最高锚点
2. **魔术师两路径基于身份暴露状态**：未公开 = 隐蔽传徽，不通过选择暴露交换知识；已公开 = 交换知识作一手信息，优于 identity_table
3. **白熊效应（第 39 次验证）**：骑士块无"决斗"技能词（R45 铁律），两个 hint 块均无"自曝/千万别/绝不能"
4. **`bpKnightDueled` / `bpMagIsRevealed` 读取方式**：与 R116 SHERIFF_VOTE `svKnightDueled`/`svMagIsRevealed` 设计完全一致
