## 优化目标（本轮：R125）

**骑士悍跳骑士 Priority 0 决斗检测**

### 问题识别

骑士 `getKnightDaySpeechPrompt` 有三级决斗优先级：
- 优先级A（≥70%）：对跳预言家场景，决斗假跳者
- 优先级B（≥60%）：可疑假金水，一石二鸟
- 优先级C（紧急救场）：真预即将被投出

**缺失**：没有「优先级0」——当其他玩家声称自己是骑士时（悍跳骑士）。  
骑士身份全局唯一，任何声称骑士的他人必然撒谎，命中率 100%（身份独占规则）。  
数据层 `claimHistory` 已有 `jump_knight` 类型（`publicFacts.js` line 121 确认），只缺决策逻辑。

### 实现路径

**aiPrompts.js**（roleParams 参数注入）：
```javascript
knightCounterClaimants: playerRole === '骑士'
    ? (gameState.claimHistory || [])
        .filter(c => c.type === 'jump_knight' && c.playerId !== currentPlayer?.id)
        .map(c => c.playerId)
    : [],
```

**knight.js**（四处变更）：
1. 解构参数：`const { hasUsedDuel, aliveCount: aliveNow = 8, knightCounterClaimants = [] } = params;`
2. 新增 `knightP0Hint` 变量（条件化，无悍跳时为空字符串）
3. 决斗决策系统标题：`三级优先级` → `四级优先级`，模板中插入 `${knightP0Hint}`
4. 续战搜索优先级排序：`A > B > C` → `0 > A > B > C`，追加候选标签格式 `优先级0/A/B/C`
5. Step1 思维链：新增「首先检查优先级0」提示行

### 代码设计亮点

- **条件化零副作用**：`knightCounterClaimants` 空数组时 `knightP0Hint = ''`，对无骑士或无悍跳局面零影响
- **角色门控**：`playerRole === '骑士'` 确保非骑士角色不注入（与 `counterClaimants` 的 `'预言家'` 门控完全对称）
- **正向描述**：「立即决斗该玩家：命中率100%，无需积累额外证据」（非「不要等待」）

### 博弈论依据

Wang 2025 (arxiv:2408.17177)：角色私有信息注入显著提升 AI 决策质量。骑士对悍跳骑士的身份独占推断是确定性 100% 的私有信息——是所有决斗场景中最高价值目标，高于概率性优先级 A/B/C。
