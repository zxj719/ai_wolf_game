## 优化目标

### LAST_WORDS 警长传徽声明提示补全（5 角色）

**背景**：LAST_WORDS 中"警长传徽声明"提示的覆盖状态：
- 预言家：R54 ④警徽流建议（`const sheriffHint = hasPoliceFlow ? '...' : ''`）✅
- 猎人：R128 🎖️hunterBadgeHint（`hasPoliceFlow && isSheriff`）✅
- 守卫/女巫/骑士/摄梦人/魔术师：❌ 缺失 → 本轮修复

**设计原则**（Wang 2025 arxiv:2408.17177）：

警长临死时在遗言中公开传徽意向与 BADGE_PASS 互补：
- **LAST_WORDS** = 自然语言信息展示（其他玩家能听到，形成即时信息）
- **BADGE_PASS** = 结构化 JSON 决策（实际传徽 targetId）

当警长说出"我把徽传给X因为Y"，场上好人立即知道谁接棒票权，策略协调效率提升。各角色利用私有信息差异化：

| 角色 | 传徽依据 | 私有信息锚点 |
|------|---------|-------------|
| 女巫 | 银水好人优先 | savedIds |
| 守卫 | 守护次数最多者 | guardHistory 频次 |
| 骑士 | 金水候选，行动背书 | hasUsedDuel + seerChecks |
| 摄梦人 | 入梦次数最多存活者 | dreamedPlayers |
| 魔术师 | 交换记录确认的可信好人 | swappedPlayers |

**实现细节**（5处 Prepend Injection，均追加在各角色 `lwRoleHint` 末尾）：

```javascript
// 示例：守卫
const guardBadgeHint = hasPoliceFlow && (currentPlayer?.isSheriff ?? false)
    ? '\n🎖️【你也是警长】遗言中同时声明传徽意向——守护次数最多的存活好人是最可信候选（守护记录是你最私有的信任信号，优先于发言印象）。'
    : '';
lwRoleHint = `你是守卫：公开你的完整守护记录...${guardBadgeHint}`;
```
