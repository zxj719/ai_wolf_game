## 本轮优化内容：预言家两连平安夜二阶推断框架（R90）

### 问题识别

R81 已为预言家添加了单夜平安夜推断框架（`seerPeaceNightStep`），但 D3+ 连续两夜平安夜场景下，AI 仅能使用单夜框架，无法利用跨夜的票压模式与查验记录交叉分析。

相比之下：
- **村民侧**（R88）：已有两连推断（两夜高票存活者相同/不同分支）
- **狼人侧**（R88）：已有两连推断（换刀建议 + 资源损耗评估）
- **预言家侧**：仅有单夜推断，两连场景存在"感知-执行分裂"——预言家 AI 能看到 fullGameTimeline 中有两个连续平安夜，但思维链没有脚手架来利用这一信息

### 修改内容

**文件**：`src/services/aiPrompts.js`，预言家 DAY_SPEECH 函数变量声明区

**新增变量**：

```javascript
// R90：两连平安夜二阶推断（预言家专属：结合查验历史精确定位双夜刀口目标）
const isConsecutivePeacefulSeer = ctx.dayCount >= 3 && isPeacefulNightSeer && 
    ctx.fullGameTimeline?.includes(`N${ctx.dayCount - 2}:平安夜`);
```

在 `if (isPeacefulNightSeer)` 块内新增：

```javascript
const prevPrevDay = ctx.dayCount - 2;
const consecutivePeaceHintSeer = isConsecutivePeacefulSeer
    ? `⭕【预言家两连平安夜二阶推断（N${prevPrevDay}+N${prevDay}均无死亡；thought 中完成；speech 正常报验即可）】
- 连续两夜平安夜 → 好人防御资源两次有效拦截，预言家视角信息价值放大
- 查 D${prevPrevDay} 和 D${prevDay} 投票记录中票压最高的存活玩家（两夜各一人）：
  · 两夜高票存活者相同 → 该玩家被狼两次针对，防御两次救护；
    若已验为金水：confidence 升至 95-100，今天重点警告保护+切换查验方向；
    若未验：列为今夜查验最高优先级（追加"排队查验优先级：①（紧急，两夜被针对）"）
  · 两夜高票存活者不同 → 狼队两轮各刀不同目标；
    对两人分别分析：已验者优先保护，未验者均升至查验优先级②；守卫可能在轮换守护
- 查验记录交叉验证（预言家独有）：
  · 已验金水目标出现在两夜高票存活者 → 强化金水保护意愿
    （狼持续追杀确认好人=高度忌惮该目标）
  · 已验查杀目标出现在高票存活者名单 → 矛盾预警：
    重新评估查验结论或该目标是陷阱（狼队不会针对自己人）
- identity_table 更新：守卫/女巫玩家 reason 追加
  "N${prevPrevDay}+N${prevDay}两连平安夜：预言家视角防御有效（信任度上调）"\n`
    : '';
seerPeaceNightStep = `${consecutivePeaceHintSeer}⭕【预言家平安夜推断...（原R81内容）`;
```

**预言家独有特性**：查验记录交叉验证（金水玩家/查杀目标与两夜高票存活者比对），是村民/狼人无法实现的功能，体现了神职信息的更高精度。
