## 本轮优化：女巫 NIGHT_WITCH 夜间用药策略个性化

### 问题诊断

R73 补完了守卫 NIGHT_GUARD 个性化，守卫成为首个三阶段全覆盖角色（DAY_SPEECH R69 + DAY_VOTE R71 + NIGHT R73）。R73 的"下轮建议"将女巫 NIGHT_WITCH 个性化列为优先级 #1（高优先）。

NIGHT_WITCH case 的当前状态：历史毒药候选读取（witchHistoryStep）+ 临界推理（criticalGuidance）均完备，但"激进用药 vs 保守持药"博弈权重对所有个性类型一视同仁。女巫最核心的策略差异恰恰在于"何时出手"——aggressive 玩家早出手建立确定性，cautious 玩家保药等待临界，cunning 玩家刻意延迟制造假象。

### 博弈论依据

Wang 2025 (arxiv:2408.17177) 指出不同认知风格的玩家在稀缺资源（女巫的两瓶药）管理上有本质不同的风险偏好：

| 个性类型 | 核心特质 | 用药决策倾向 |
|---------|---------|------------|
| aggressive | 主动进攻 | 激进出手型：有价值目标即救/发现高嫌疑即毒，早出手=信息确定性 |
| cautious | 谨慎稳妥 | 保守持药型：双药保留到临界，高阈值（confidence≥85）才开毒 |
| logical/analytical | 数据驱动 | 推理验证型：identity_table≥75 + 交叉验证再行动，量化推理链 |
| cunning | 博弈欺骗 | 博弈伪装型：刻意不用毒制造"女巫保守"假象，关键轮次精准出手 |
| emotional | 感知驱动 | 直觉感知型：当晚威胁感优先于数据，直觉信号先行再检查临界 |
| contrarian | 反直觉 | 反预判型：预判狼人在预判用药节奏，反向操作 |
| steady | 平衡渐进 | 平衡节药型：中等阈值（identity_table≥75），避免过早/过晚消耗 |

### 修复内容（`src/services/aiPrompts.js`，NIGHT_WITCH case）

**注入位置**：在 `witchHistoryStep` 之后、Step 1（解药考量）之前，与 R73 守卫模式完全对称：

```
${witchHistoryStep}
${witchNightStyle}       ← 新增
1. 解药考量：...
2. 毒药考量：...
```

**代码结构**：
```javascript
const witchNightPersonalityType = currentPlayer?.personality?.type || '';
let witchNightStyle = '';
if (witchNightPersonalityType === 'aggressive') {
    witchNightStyle = '\n【你的用药风格】激进出手型：...';
} else if (witchNightPersonalityType === 'cautious') {
    witchNightStyle = '\n【你的用药风格】保守持药型：...';
} // ... 7 种类型，无 else（向下兼容）
```

**调用端无需修改**：`currentPlayer` 在 `generateUserPrompt` 闭包中直接可用（R5 教训）。

**白熊效应合规**：所有 7 种风格均为正向指令（"激进出手型"、"保守持药型"等），无"不要用毒"等负向禁词 ✅。

**向下兼容**：无个性类型时 `witchNightStyle = ''`，prompt 退化到 R73 状态 ✅。
