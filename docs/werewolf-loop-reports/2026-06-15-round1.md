## 2. 本轮优化详情

### 2a. 女巫人格禁忌矛盾修复（Critical）

**文件**：`src/services/rolePrompts/witch.js` + `src/services/aiPrompts.js`（降级路径）

**问题**：`WITCH_PERSONA.taboos` 包含 `'首夜不救人'`（含义：不在首夜救人是禁忌 = 必须救）。但 `getWitchNightActionPrompt` 中的 `firstNightHint` 在有守卫时说"首夜建议不救"。LLM 同时接收到两条正好相反的指令。

**依据**：提示词矛盾导致 LLM 输出不稳定（会随机选择遵从哪条指令），这在有守卫局中会频繁触发"同守同救"死亡（女巫听了 taboo 选择救人，守卫也在守同一目标）。

**修复**：
- 删除 `'首夜不救人'`，改为 `'同一晚又救又毒'`（实际上禁止的行为）
- 首夜策略完全由 `firstNightHint`（条件化，有守卫时"建议不救"，无守卫时"放心救"）决定
- 在人格注释中明确说明这是有意设计

### 2b. 狼人发言负向词汇列表（Medium）

**文件**：`src/services/rolePrompts/werewolf.js`

**问题**：发言禁令列出了具体词汇 `狼人、队友、刀、狼队、保狼、抗推`。根据 LLM 激活机制，明确列出"禁止词"会让模型对这些词汇的关联更强（"不要想白熊"效应）。

**依据**：LEARNINGS.md 既有教训（狼人自曝陷阱类别），Wang 2025 (arxiv:2408.17177) 的分析表明正向行为描述比负向禁止更有效。

**修复**：删除词汇列表，改为正向描述："以普通好人视角写就：只谈分析、怀疑、投票逻辑，不暴露私有信息"。

### 2c. 女巫 JSON Schema 冗余字段（Bug Fix）

**文件**：`src/services/rolePrompts/witch.js`

**问题**：输出 JSON schema 定义了 `usePoison: 数字或null` 和 `poisonTarget: 数字或null` 两个字段，代码只读 `usePoison`。AI 若只填 `poisonTarget` 则毒药静默失效，无任何错误提示。

**修复**：删除 `poisonTarget` 字段，只保留 `usePoison`，并在注释中明确语义（数字=目标，null=不毒）。同时补充"usePoison 和 useSave 不能同时为真"的约束说明。
