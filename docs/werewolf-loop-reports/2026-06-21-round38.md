## 2. 问题诊断

### 根因：NIGHT_WOLF identity_table 的读写闭环缺失

**历史背景**：
- Round 16：NIGHT_WOLF 添加了 identity_table 战略用途指导：`reason 写"高优先刀口：[具体威胁原因]"，供下轮夜间决策时优先锁定`
- 这条指导有明确的"写"目标，但对"读"端只说了"供下轮...优先锁定"——这是愿景陈述，不是可执行指令

**感知-执行分裂（Perception-Execution Split）**：
- **感知已有**：系统提示（`buildPersonaHint`）通过 `previousIdentityTable` 将上轮 identity_table 以 JSON 格式注入，狼人 AI 可看到所有历史标记
- **执行框架缺失**：NIGHT_WOLF 用户提示的思维链从未说"先检查 identity_table 中的高优先刀口"，AI 每晚从头评估所有目标，等于没有跨轮记忆

**信息流断裂**：
```
N1 夜写：reason = "高优先刀口：预言家发言锋利"   ← 写入 identity_table
N1 夜后：identity_table 存入 identityTablesRef                 ← 正常存储
N2 夜始：系统提示注入 previousIdentityTable                    ← 正常传递
N2 夜思维链："根据白天发言，谁最可能是什么角色？"              ← 从不提及"检查高优先刀口"
                                                              ← 感知-执行分裂
```

**修复后**：
```
N2 夜 Step 0："查看系统提示中【你之前的身份推理表】：哪些玩家 reason 含'高优先刀口'？将其列为今晚刀口候选起点"
                                                              ← 显式读取指令
```

**依据**：
- Wang 2025 §4.3：多代理欺骗需要"全局策略 + 局部感知"同时具备
- R26 教训：参数注入解决感知，策略框架才解决执行，两者缺一不可
- R37 教训："先 A 再 B"决策原则必须同时给出 A 步骤的具体执行方法
- 本轮是该模式的夜间行动版本：write 端指导 ≠ read 端执行框架

**变更 2：新增两个 const（R18 规范）**
```javascript
// wolfNightLabel：R18 规范，不在 template 字面量内直接用 ctx.dayCount
const wolfNightLabel = `N${ctx.dayCount}`;

// wolfHistoryStep：首夜/N2+夜条件分支
const wolfHistoryStep = ctx.dayCount > 1
    ? '0. 【读取历史刀口】查看系统提示中【你之前的身份推理表】：哪些玩家的 reason 含"高优先刀口"？将其列为今晚刀口候选起点（若该目标已死或不在可袭击列表中，重新评估其他高优先项）'
    : '0. 【首夜】无历史刀口记录——直接进行角色推断';
```

**变更 3：思维链升级（原 3 步 → Step 0 + 3 步）**

改前：
```
【思维链】
1. 根据白天发言，谁最可能是什么角色？
2. 消灭谁对狼队的期望收益最大？（信息链、投票权、技能威胁）
3. 对方可能被保护吗？
```

改后：
```
【思维链】
${wolfHistoryStep}
1. 【角色推断】白天发言中谁的行为印证或更新了上轮 identity_table 的判断？是否出现新的神职暴露信号？
2. 【期望价值评估】高优先目标是否仍存活？综合：角色威胁 × 被守护概率（连守规律） × 猎人/骑士反杀代价
3. 【最终决策】确定今晚目标——若切换了历史"高优先刀口"目标，在 thought 中说明切换原因（避免随机换刀）
```

**变更 4：identity_table 写指导升级（标题 + 读写前向引用 + 执行状态追踪）**

改前：
```
【identity_table 战略用途（你已知所有人真实身份）】
- 对高威胁好人（神职/...）：...reason 写"高优先刀口：[具体威胁原因]"，供下轮夜间决策时优先锁定
- 对已知狼队友：...
- 对低威胁村民：...
```

改后：
```
【identity_table 战略更新（读写闭环）】
- 高威胁好人（神职/...）：...reason 写"高优先刀口：[具体威胁原因]"（下轮 Step 0 将直接从此读取）
- 若本轮刀了某目标：在TA的 reason 末尾追加"→已${wolfNightLabel}夜行刀"，标记执行状态，防止下轮重复评估
- 已知狼队友：...（维持公开发言一致性）
- 低威胁村民：...
```

### 3.2 无调用端改动

`previousIdentityTable` 已由 `useAI.js` 注入系统提示，本轮改动仅修改用户提示内容，无需改动任何调用端或数据流。
