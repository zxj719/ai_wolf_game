## 2. 本轮优化详情

### 2a. NIGHT_DREAMWEAVER 补全 identity_table

**文件**：`src/services/aiPrompts.js`（`PROMPT_ACTIONS.NIGHT_DREAMWEAVER` case 输出行，line 1388）

**问题**：摄梦人的输出 schema 是:
```json
{"dreamTarget":数字,"dreamMode":"defense/offense/sacrifice","dreamReason":"...","isConsecutiveDream":true/false,"confidence":0-100,"thought":"..."}
```
缺少 `identity_table`，而这恰恰是摄梦人最需要的字段——摄梦人的核心进攻能力"连梦击杀"需要对狼人有高置信度判断（≥75%），没有结构化的 identity_table，AI 无法在多轮夜间行动中累积更新对身份的推断。

**修复**：在输出 schema 末尾追加标准 identity_table 字段：
```json
{..., "identity_table":{"玩家号":{"suspect":"角色","confidence":0-100,"reason":"依据"}}}
```

**意义**：NIGHT_* 统一审计在 Round 7 完结，不再有"遗漏的夜间行动"。

### 2b. HUNTER_SHOOT 临界局势推理框架

**文件**：`src/services/aiPrompts.js`（`PROMPT_ACTIONS.HUNTER_SHOOT` case，line 1418-1456）

**问题**：原版猎人开枪提示提供了策略优先级列表，但没有引导 AI 进行"临界场面推断"——即：带走某个目标后，局面是否会达到"好人数 ≤ 狼人数"的即时失败条件？这在游戏尾盘（2-3人局面）是决定胜负的关键。

**修复**：增加 `hunterCriticalGuidance` 变量，注入到 return 模板中：

```javascript
const hunterAliveCount = (aliveTargets || []).length;  // 猎人自身不计
const hunterCriticalGuidance = `\n\n⚠️【临界局势推理】
你死亡开枪 = 本轮同时出局两人（你 + 你带走的人）。当前存活${hunterAliveCount}人（不含你）。
请自行推断：
1. 结合已知死亡记录和查杀信息，场上大约剩几个狼人？
2. 如果你带走了好人：好人数会减少1，是否触发"好人数 ≤ 狼人数"直接输？
3. 如果你带走了已验狼人：狼人减少1，好人重获优势——这是最优解
4. 无查验信息时：选发言逻辑崩塌/倒钩明显的，避开沉默/模糊中间型

【决策框架】
- 有查杀信息 → 必须带走查杀目标
- 无查杀但有可疑 → 带走最高怀疑度，宁错不弃
- 猎人开枪是好人阵营最后一次主动出击机会，"稳"不是选项`;
```

**同时更新** `thought` 字段说明：`"开枪决策思考过程（必须包含推断：场上约剩X狼，带错是否直接输）"` — 强制要求 AI 在 thought 中写出存活狼人数量推断，提升可观战性。

**设计原则**：
- **强制行动** vs **可选行动** 的框架差异：女巫的 criticalGuidance 问"要不要动"（可选），猎人的问"带谁能赢"（必须动）。这是关键区别，不能套用相同模板。
- 不提供答案，引导推断：`hunterAliveCount` 告知存活数量，但"场上剩几个狼"由 AI 自行推断（符合渐进式披露原则，避免信息泄露）。

**依据**：Wang 2025 (arxiv:2408.17177)：在强约束决策场景（如猎人必须开枪），提供明确的"后果推断"框架比"直觉优先级"策略显著降低次优决策率。
