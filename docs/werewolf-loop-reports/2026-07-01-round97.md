## 本轮优化内容

### 1. 主优化：守卫 NIGHT 三连平安夜三阶推断（R97）

#### 问题识别

R94 建立了 NIGHT 侧两连平安夜推断（dayCount >= 3），但缺少三连推断框架（dayCount >= 4）。三连平安夜在游戏中是极低概率事件（好人存活≥6时 < 5% 自然发生率），是守卫策略极度成功的信号，值得专属三阶推断框架。

#### 博弈论依据（Wang 2025）

三夜守护模式识别能解决一个关键决策问题：**在第四夜（被迫切换连守目标时），应优先哪个候选？**

- **路径A**（三夜锁守同目标）：被迫切换，选次高 confidence 候选，同时警惕狼人趁切换补刀
- **路径B**（有两夜守同目标）：高频目标 confidence 最高，今晚回归高频目标  
- **路径C**（三夜全不同）：轮守策略有效，按票压重新评估

#### 实现内容

**文件：`src/services/aiPrompts.js`（NIGHT_GUARD case）**

新增四个变量（紧接 `prevPrevNightGuardTarget` 声明之后、`consecutivePeaceNightHintGuard` 之前）：

```javascript
// R97: NIGHT 侧三连平安夜三阶推断
const isTripleConsecutivePeacefulNightGuard = ctx.dayCount >= 4 && isConsecutivePeacefulNightGuard &&
    ctx.fullGameTimeline?.includes(`N${ctx.dayCount - 3}:平安夜`);
const guardNightThreePrevDay = ctx.dayCount >= 4 ? ctx.dayCount - 3 : 0;
const threeNightGuardTarget = isTripleConsecutivePeacefulNightGuard
    ? (gameState.guardHistory?.find(g => g.night === guardNightThreePrevDay)?.targetId ?? null)
    : null;
const tripleConsecutivePeaceNightHintGuard = (isTripleConsecutivePeacefulNightGuard && cannotGuard !== null)
    ? `⭕【守卫三连平安夜 NIGHT 三阶推断（...）】
- 路径A - 三夜锁守同目标: 近确定性拦截，今晚因禁连守切换次高候选
- 路径B - 三夜中有两夜守同目标: 高频目标 confidence 升 30-40
```
