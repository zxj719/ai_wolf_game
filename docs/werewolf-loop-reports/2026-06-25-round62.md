## 问题识别与修复

### 猎人 DAY_VOTE 读写闭环缺口（R61 建议，本轮执行）

**问题**：猎人在 `DAY_SPEECH`（D2+）写入"开枪优先级：高"标注到 identity_table，`HUNTER_SHOOT` 读取该标注作为开枪候选。但 `DAY_VOTE` 走通用分支：`"有查杀 → 跟投查杀！无查杀 → 投发言逻辑最混乱..."`——通用分支不读取 identity_table，猎人多轮积累的开枪候选在投票阶段被完全忽略。

**读写链路对比**：
```
修复前：
  DAY_SPEECH写"开枪优先级：高" → DAY_VOTE（通用，不读）→ HUNTER_SHOOT读✅
                                          ↑
                                        断链点

修复后：
  DAY_SPEECH写"开枪优先级：高" → DAY_VOTE（猎人专属框架，读✅）→ HUNTER_SHOOT读✅
                                  完整三段闭环 ✅
```

**博弈论依据**：
- 猎人的投票目标和开枪目标应对齐（刀票对齐原则）
- 若今天能通过投票出局该目标 → 节省开枪作为后期保险（更高价值时刻使用）
- 若今天无法翻转票型 → 仍然投向同一目标，积累多轮压力，帮助接棒好人识别威胁
- 这与狼人"高优先刀口对齐投票"（R61）是同构模式，在好人阵营侧的镜像实现

**修复**（`src/services/aiPrompts.js`）：

1. 在 `seerVoteStrategy` 后新增 `hunterVoteStrategy` 常量（3 点框架）：
```javascript
// 猎人投票策略：DAY_SPEECH积累的"开枪优先级：高"候选与投票方向对齐（刀票对齐原则，R62）
const hunterVoteStrategy = `2. 【猎人投票—刀票对齐框架】
   a) 从【身份推理表】(identity_table) 读取 reason 含"开枪优先级：高"的存活玩家——这是你多轮积累的最高嫌疑候选，投票首选应与开枪方向一致（刀票对齐：投他 + 死后射他）
   b) 场景评估：今天能推票出局该目标？→ 全力推票（出局即清除，开枪留作后期保险）；今天无法翻转票型？→ 仍然投向他，积累多轮压力，帮助接棒的好人识别威胁
   c) 无"开枪优先级：高"候选时：跟投预言家查杀 > 发言逻辑最崩塌的玩家 > 历史热力最高的目标`;
```

2. 在 DAY_VOTE 模板的角色分支中插入猎人分支（狼人→预言家→猎人→通用）：
```javascript
: playerRole === '猎人'
? hunterVoteStrategy
```
