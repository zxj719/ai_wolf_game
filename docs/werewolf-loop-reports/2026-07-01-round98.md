# Round 98 — 预言家 NIGHT 侧平安夜三级推断框架

**日期**：2026-07-01  
**轮次目标**：实现 NIGHT_SEER case 的平安夜三级推断（单夜/两连/三连），与 NIGHT_GUARD 三层体系对齐

---

## 背景与问题

R97 完成了守卫 NIGHT 三连平安夜推断（NIGHT_GUARD 三层体系：R85→R94→R97），并将预言家 NIGHT 侧三连平安夜推断列为 R98 首要目标。

经确认：NIGHT_SEER case 完全没有平安夜推断逻辑，而 NIGHT_GUARD 已有完整的三层梯度体系。两者均能"感知"平安夜（通过 `ctx.lastNightInfo`），但预言家毫无脚手架指导如何调整查验策略。

**核心不对称性**：
- 守卫：平安夜 → 读 `guardHistory` → 零间接推断（知道自己保护了谁）
- 预言家：平安夜 → 无处理 ← **R98 修复点**
- 村民：平安夜 → 纯票压 → 二阶间接推断

预言家的私有信息是 `identity_table`（每个玩家的嫌疑度 confidence），因此推断路径是：「confidence≥65 的未验证存活玩家」是狼刀可能目标（一阶间接推断）。

## 实现内容

### 新增变量（NIGHT_SEER case，`aiPrompts.js`）

```javascript
// R98: NIGHT_SEER 平安夜查验优先级调整框架
const isNightPeacefulSeer = ctx.dayCount > 1 && ctx.lastNightInfo?.includes('平安夜');
const seerNightPrevDay = ctx.dayCount > 1 ? ctx.dayCount - 1 : 0;
const isConsecutivePeacefulNightSeer = ctx.dayCount >= 3 && isNightPeacefulSeer &&
    ctx.fullGameTimeline?.includes(`N${ctx.dayCount - 2}:平安夜`);
const seerNightPrevPrevDay = ctx.dayCount >= 3 ? ctx.dayCount - 2 : 0;
const isTripleConsecutivePeacefulNightSeer = ctx.dayCount >= 4 && isConsecutivePeacefulNightSeer &&
    ctx.fullGameTimeline?.includes(`N${ctx.dayCount - 3}:平安夜`);
const seerNightThreePrevDay = ctx.dayCount >= 4 ? ctx.dayCount - 3 : 0;
```

### 三级条件化 hint 变量（前置注入模式，第 13 次应用）

| 变量 | 激活条件 | 内容 |
|------|---------|------|
| `seerNightPeaceStep` | `isNightPeacefulSeer` | 单夜推断：confidence≥65 筛选，路径A/B |
| `consecutivePeaceNightHintSeer` | `isConsecutivePeacefulNightSeer` | 两连推断：两天票压高票存活者交集 |
| `tripleConsecutivePeaceNightHintSeer` | `isTripleConsecutivePeacefulNightSeer` | 三连推断：三天票压综合评估，路径A/B/C |

**嵌套结构**（前置注入）：
```
seerNightPeaceStep = `${consecutivePeaceNightHintSeer}⭕【单夜推断...】`
consecutivePeaceNightHintSeer = `${tripleConsecutivePeaceNightHintSeer}⭕【两连推断...】`
```

三连激活时，AI 收到全三层叠加内容（精度最高）。

### 注入位置（return 模板）

```javascript
return `预言家查验决策。
...
${seerNightStyle}
${seerNightPeaceStep}${seerNightStrategy}
...`;
```

`${seerNightPeaceStep}` 位于 `${seerNightStyle}`（个性化查验风格）之后、`${seerNightStrategy}`（三阶段策略框架）之前——与 NIGHT_GUARD 的注入位置对称。

## 精度层级设计

| 角色 | 推断精度 | 信息来源 | confidence 调整幅度 |
|------|---------|---------|-------------------|
| 守卫 NIGHT | 零间接 | `guardHistory` 直读 | ±30-40（三连）|
| **预言家 NIGHT** | **一阶间接** | `identity_table` confidence≥65 | **±10-15** |
| 村民 DAY | 二阶间接 | 票压观察 | ±5-10 |

预言家的调整幅度比守卫低，因为"嫌疑高"≠"被狼刀"，这是间接推断的天然误差。
