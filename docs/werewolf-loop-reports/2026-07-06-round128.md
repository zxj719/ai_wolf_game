## 优化目标（本轮：R128）

**猎人 LAST_WORDS 专属分支 — identity_table 枪击锚点 + 警长协同提示**

### 问题识别（R127 下轮建议"优先 B"）

R127 完成 BADGE_PASS 全角色覆盖后，猎人 LAST_WORDS 仍是通用文本：
```
'你是猎人：明确宣告你最高度怀疑的狼人目标（即使此刻无法开枪，留下线索能让好人接棒追查）；若你即将触发开枪，直接说明目标和理由。'
```

与其他神职 LAST_WORDS 对比，缺少：
1. **identity_table 阅读指引**：所有其他神职（预言家/女巫/守卫/骑士/摄梦人/魔术师）均有角色专属数据注入，猎人没有
2. **警长协同提示**：当猎人兼任警长时（`hasPoliceFlow && isSheriff`），遗言应同时声明传徽意向；枪击目标与传徽目标应分离（R127 双输出协同原则延伸）

### 博弈论依据

- **identity_table 锚点**：Wang 2025 (arxiv:2408.17177) — 结构化历史上下文显著提升 AI 决策精准度；猎人通过公共发言积累的身份推理表中，`suspect="狼人"` 且 `confidence` 最高的玩家是枪击优先目标
- **枪×徽协同**：R127 原则延伸 — 猎人兼任警长时，遗言中应提前声明传徽意向（实际 BADGE_PASS 决策在单独 prompt 中完成），遗言的声明有利于好人接棒时理解猎人的意图分配

### 设计决策

猎人 LAST_WORDS 与 BADGE_PASS 的差异：
- **BADGE_PASS**：猎人作为警长死亡，决定徽传给谁（输出 JSON targetId）
- **LAST_WORDS**：猎人死亡，给出最后发言内容（输出 JSON speech + thought）

因此 hunterBadgeHint 是**遗言内容的提示**（告诉 AI 在 speech 中提及传徽意向），不是传徽决策本身。

**位置**：LAST_WORDS case 内，`playerRole === '猎人'` 分支

**Old（~100 chars）**：

} else if (playerRole === '猎人') {
    lwRoleHint = '你是猎人：明确宣告你最高度怀疑的狼人目标（即使此刻无法开枪，留下线索能让好人接棒追查）；若你即将触发开枪，直接说明目标和理由。';

**New（~517 chars）**：
```javascript
} else if (playerRole === '猎人') {
    // R128: 猎人 LAST_WORDS 专属分支 — identity_table 枪击锚点 + 警长协同提示
    const hunterIsSheriff = hasPoliceFlow && (currentPlayer?.isSheriff ?? false);
    const hunterBadgeHint = hunterIsSheriff
        ? '\n🎖️【你也是警长】遗言中同时声明传徽意向（传给哪位可信好人）——枪与徽各自指向不同目标，两张牌发挥最大协同价值。'
        : '';
    lwRoleHint = `你是猎人：先查看【你之前的身份推理表】——confidence 最高且 suspect 为"狼人"的玩家是最有依据的枪击目标；明确宣告你最高度怀疑的狼人及你的逻辑依据（即使此刻无法开枪，清晰线索让好人接棒追查）；若你即将触发开枪，明确说明目标。${hunterBadgeHint}`;
```
