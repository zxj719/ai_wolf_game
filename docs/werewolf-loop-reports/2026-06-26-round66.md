## 2. 本轮改进

### 问题：DAY_SPEECH 与 DAY_VOTE 防守链断裂

**R65 修复了什么**：
- DAY_VOTE 阶段：当 ≥ ceil(N/2) 已表态者指向狼人队友时，注入 `wolfDefenseTrigger` 信号，AI 知道"现在要执行防守局面"
- DAY_VOTE 中的防守操作是："将票转向第三方好人"

**R65 遗留的问题**：
- 防守局面的执行依赖一个前提："须在发言阶段已预铺'更可疑'的理由才能说服他人"（aiPrompts.js 第1902行已注明此前提）
- 但 DAY_SPEECH 提示词**没有任何机制**指导狼人在票压出现时主动引入第三嫌疑目标
- 结果：投票时收到防守信号，但没有发言叙事基础 → 临时"改票转向Z"但Z没有任何被质疑的背景 → 好人看到毫无逻辑的突然转票 → 关系链暴露概率上升

这是经典的**跨阶段感知-执行分裂**：执行指令（DAY_VOTE）在第二阶段，感知能力（发言阶段的票型信息）在第一阶段，但两阶段之间没有信息传递机制。

### 修复方案

#### A. `useSpeechFlow.js` — 发言阶段票压感知计算（R66）

在 wolf `roleParams` 构建块中，新增压力图计算：

```javascript
// R66：检测当天已有≥2名玩家将票意向指向同一队友
const pressureMap = {};
speechHistory.forEach(s => {
  if (s.day === dayCount &&
      s.voteIntention !== undefined && s.voteIntention !== null &&
      s.voteIntention !== -1 &&
      wolfTeammateIds.includes(Number(s.voteIntention))) {
    const t = Number(s.voteIntention);
    pressureMap[t] = (pressureMap[t] || 0) + 1;
  }
});
const pressureEntries = Object.entries(pressureMap)
  .filter(([, cnt]) => cnt >= 2)
  .sort(([, a], [, b]) => b - a);
if (pressureEntries.length > 0) {
  wolfSpeakParams.pressuredTeammate = Number(pressureEntries[0][0]);
  wolfSpeakParams.pressuredCount = pressureEntries[0][1];
}
```

**阈值选择 = 2**（对比 R65 DAY_VOTE 的 ceil(N/2)）：
- DAY_SPEECH 阶段需要更早反应（尚未到投票），2票压力即值得采取预防措施
- 1票可能是随机波动，不够稳定；2票意味着有初步共识形成
- 与 R65 的"DAY_VOTE 阶段严格多数触发"形成梯度：早发现(2票)→早预铺(DAY_SPEECH)→晚执行(DAY_VOTE严格多数)

#### B. `aiPrompts.js` — 狼人 DAY_SPEECH `wolfSpeechPressureHint`

当 `pressuredTeammate` 参数存在时，注入：

```
⚡【发言时机：队友X号承压（已有N票意向指向）】
发言策略：在 speech 中主动引入一名发言有逻辑漏洞的好人作为"更值得关注的焦点"
（如"Z号今天的发言有几处值得注意"），让好人注意力转移——为后续投票阶段执行"转移票型"提前构建叙事支撑。
叙事要点：用分析语气提出替代目标，不对当前票型发表评论（避免暴露你在意队友的风向）。
```

**白熊效应检查**：
- ✅ 纯正向指令（"引入Z作为焦点"），没有负向禁止词
- ✅ 不提"防守队友"或"保护"词汇
- ✅ PK 模式下自动跳过（避免与 PK 专属逻辑冲突）

**与 R65 的配合**：
| 阶段 | R65 提供 | R66 提供 |
|------|---------|---------|
| DAY_SPEECH | （无） | 票压感知 → 引入第三嫌疑人叙事 |
| DAY_VOTE | wolfDefenseTrigger → 执行防守 | （依赖 DAY_SPEECH 叙事铺垫） |

两轮形成完整防守链：发言阶段建立叙事基础 → 投票阶段执行数据驱动的防守。

## 4. 关键决策备忘

### 为什么 DAY_SPEECH 的压力阈值选 2 而不是 ceil(N/2)？

DAY_SPEECH 是防守准备阶段，需要在票压形成早期即采取行动（因为后面没有发言机会了）。`ceil(N/2)` 是"投票已经不可逆"的临界点，在这个点上再预铺已经太晚。2票是"有初步共识方向"的信号，值得提前布局。

### 为什么 speech 中不直接替换 wolfTeammatesHint？

`wolfSpeechPressureHint` 是**叠加逻辑**，不是覆盖逻辑。多狼协作规则（立场分散/投票错位/制造分歧感）依然适用——压力提示只是在协作框架上增加一个"当前局势"的具体行动指令。
