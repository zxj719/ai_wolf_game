## 2. 本轮优化详情

### 2a. NIGHT_WITCH identity_table 补全

**文件**：`src/services/aiPrompts.js`（`PROMPT_ACTIONS.NIGHT_WITCH` case 末尾 output schema）

**问题**：原版输出：
```json
{"useSave": true/false, "usePoison": 数字或null, "reasoning": "...", "thought": "..."}
```
缺少 `identity_table`。而 NIGHT_SEER、NIGHT_GUARD、NIGHT_WOLF 均有 `identity_table`，导致女巫夜间行动阶段无法更新身份推理表——这是信息最密集的时刻（女巫得知了被刀目标，可以推断该玩家对狼队有威胁）。

**修复**：在输出 schema 末尾追加 `identity_table` 字段，与其他夜间行动格式完全一致：
```json
{
  "useSave": true/false,
  "usePoison": 数字或null,
  "reasoning": "决策理由（含推断：当前大概还剩X好人Y狼人）",
  "thought": "用药思考过程",
  "identity_table": {"玩家号": {"suspect": "角色", "confidence": 0-100, "reason": "依据"}}
}
```

**依据**：
- 女巫在夜间得到最高价值私有信息（被刀目标），这是识别狼人行动偏好的关键
- 未来轮次的 `previousIdentityTable` 机制依赖夜间 identity_table 的持久化（Round 4 建立的推理链）
- 统一所有 NIGHT_* schema 格式是「减少信息孤岛」的系统性要求（LEARNINGS Round 4 教训）

### 2b. SHERIFF_BADGE_PASS 接通 seerChecks 分级提示

**文件 1**：`src/hooks/useDayFlow.js`（`handleSheriffBadgePass` 函数，line 276）  
**文件 2**：`src/services/aiPrompts.js`（`PROMPT_ACTIONS.SHERIFF_BADGE_PASS` case）

**问题**：原版 SHERIFF_BADGE_PASS 只知道「有哪些候选人」，但不知道「哪些候选人已被预言家验证」。AI 只能靠记忆推断，容易因 context 压缩丢失关键查验信息，导致警徽误传给已被验证的狼人（查杀目标）——这是竞技最致命的失误之一。

**修复方案**：

*useDayFlow.js*（调用端）：
```javascript
// 原来
const res = await askAI(deadSheriff, PROMPT_ACTIONS.SHERIFF_BADGE_PASS, { validTargets: aliveTargets });
// 修复后
const res = await askAI(deadSheriff, PROMPT_ACTIONS.SHERIFF_BADGE_PASS, { validTargets: aliveTargets, seerChecks });
```

*aiPrompts.js*（提示端）：
```javascript
case PROMPT_ACTIONS.SHERIFF_BADGE_PASS: {
  const { validTargets: badgeTargets, seerChecks: bpSeerChecks } = params || {};
  const badgeableSet = new Set(badgeTargets || []);

  // 只过滤候选人中的金水/查杀
  const goldWaterTargets = (bpSeerChecks || [])
    .filter(c => !c.isWolf && badgeableSet.has(c.targetId));
  const killedTargets = (bpSeerChecks || [])
    .filter(c => c.isWolf && badgeableSet.has(c.targetId));

  let seerHint = '';
  if (goldWaterTargets.length > 0)
    seerHint += `\n⚡【预言家金水（已验好人）】${goldWaterTargets.map(c => `${c.targetId}号`).join('、')} → 优先移交对象`;
  if (killedTargets.length > 0)
    seerHint += `\n⛔【预言家查杀（已验狼人）】${killedTargets.map(c => `${c.targetId}号`).join('、')} → 绝对不能移交（传给狼等于送1.5票，直接输）`;
  // ...
```

**动态行为**：
- **有金水候选**：提示明确列出金水玩家号，引导优先传徽
- **有查杀候选**：明确禁止，并解释后果（1.5票优势直接输）
- **无查验数据**：`seerHint` 为空，回退到原有通用提示，不增加干扰

**依据**：
- 竞技狼人杀中，已验证信息 > 发言印象 > 直觉判断。警徽流是 12 人局的重要机制，错误传徽是一票差距决定胜负的关键节点
- Wang 2025 (arxiv:2408.17177)：具有明确约束（黑名单/白名单）的提示词比"综合判断"型提示词显著降低 AI 的决策误差率
