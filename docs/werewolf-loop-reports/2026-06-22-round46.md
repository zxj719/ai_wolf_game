## 二、本轮修复的4个 Bug

### Bug 1 — dreamweaverHistory / magicianHistory 三层传递链断裂

**症状**：`aiPrompts.js` 的 `DAY_SPEECH`（摄梦人发言）和 `LAST_WORDS`（魔术师遗言）使用 `gameState?.dreamweaverHistory` / `gameState?.magicianHistory`，但这两个字段从未加入 `useAI.js` 的参数列表和 `gameState` 构建对象，`WerewolfModule.jsx` 的 `useAI()` 调用也未传参。

**影响**：摄梦人/魔术师的"历史换位记录"对发言 AI 和遗言 AI 永远不可见——这两个角色的跨轮记忆功能完全失效（但静默失效，无报错）。

**修复文件**：
- `src/hooks/useAI.js`：参数列表加 `dreamweaverHistory = null, magicianHistory = null`；gameState 对象中加对应字段
- `src/modules/werewolf/WerewolfModule.jsx`：`useAI()` 调用传递 `dreamweaverHistory, magicianHistory`

### Bug 2 — NIGHT_DREAMWEAVER hasRevealed 从未传入

**症状**：`aiPrompts.js` NIGHT_DREAMWEAVER case 试图从 `params.hasRevealed` 读取身份是否已暴露，但 `useNightFlow.js` 的调用只传了 `dreamHistory`、`lastDreamTarget`、`aliveTargets`，hasRevealed 从未传入，故牺牲模式（`dwRevealedAlert`）永远不激活。

**修复**：
```javascript
// R46 Bug 修复：调用端未传 hasRevealed，从 currentPlayer 读取（R12 gameState 访问模式）
const dwHasRevealed = params.hasRevealed ?? params.currentPlayer?.hasRevealed ?? false;
```
调用端无需改动，prompt 端 fallback 读取。

### Bug 3 — NIGHT_MAGICIAN 传了 hasRevealed 但 magician.js 未消费

**症状**：`aiPrompts.js` NIGHT_MAGICIAN case 中 `getMagicianNightActionPrompt({ ..., hasRevealed: currentPlayer?.hasRevealed })` 确实传了 hasRevealed，但 `magician.js` 的 `getMagicianNightActionPrompt` 解构列表缺少 `hasRevealed`，导致"身份已公开时自保跃升最高优先"特性静默失效。

**修复**：`magician.js` 解构加入 `hasRevealed`，重写策略提示块：
- `hasRevealed === true`：优先级 C（自保）升为最高优先，提示"换自己和最高嫌疑狼人"
- `hasRevealed === false`（默认）：保持原有 A→B→C 顺序，C 仍可选

### Bug 4 — SHERIFF_BADGE_PASS seerChecks 依赖 params 传参（永远为空）

**症状**：SHERIFF_BADGE_PASS 从 `params` 解构 `seerChecks`，而 `useDayFlow.js` 调用 `await askAI(deadSheriff, PROMPT_ACTIONS.SHERIFF_BADGE_PASS, { validTargets: aliveTargets })` 从未传 `seerChecks`，导致金水/杀手提示（`seerHint`）永远是空字符串。

**修复**：改用 R12 gameState 直读模式：
```javascript
const bpSeerChecks = gameState.seerChecks || [];  // 不依赖调用端传参
```
与 SHERIFF_VOTE 的处理方式统一。

## 四、LEARNINGS.md 新增条目

1. **三层传递链检查**（R46）：每次新增 `gameState.X` 访问点，必须同步检查 WerewolfModule → useAI param → useAI gameState 三处。
2. **委托函数两侧成对**（R46）：传了就要用，用了就要传——`case` 中加参数必须立即更新目标模块解构列表。
3. **公共数据直接从 gameState 读**（R46）：seerChecks / deathHistory 等高频数据使用 R12 模式，彻底规避调用端遗漏导致的静默 bug。
