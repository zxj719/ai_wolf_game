## 2. 本轮优化详情

### 问题发现

LEARNINGS.md item 62（Round 30 下轮建议）：
> "SHERIFF_SPEECH 摄梦人/魔术师专属分支评估：两者目前走好人 fallback；若在实局出现上警场景，可补充角色专属竞选发言框架（摄梦人有"宣告"策略价值，魔术师有"信息不对称"优势）"

同时，SHERIFF_RUN 中摄梦人/魔术师也走 `好人 fallback`（"可以上警为阵营争取空间..."），未能给出角色差异化的上警决策建议。

### 博弈论依据

**摄梦人竞选框架 — "死亡成本最高的警长"：**
- 同生共死机制：刀警长摄梦人 = 1.5 票权重消失 + 被入梦者随之出局（额外代价）
- 不确定性威慑：若未宣告身份，狼方无法确认入梦目标，刀的代价无法精确估算
- 战略含义（Wang 2025 信息博弈视角）：不确定性本身是战略资产，与 R30 猎人框架同理
- **已宣告**：可明说"我已连好了目标，刀我他们也得走"（但绝不透露具体目标，保持目标不确定）
- **未宣告**：用模糊底气话暗示"倒了好人不亏"，措辞应比猎人更模糊（猎人/摄梦人都可能是这话的说者，保持最大不确定性）

**魔术师竞选框架 — "信息修正权威 × 1.5 票杠杆"：**
- 魔术师掌握每次交换后的真实技能指向，这是场上任何人都不具备的信息优势
- **已宣告**：可明确承诺"我的1.5票是本局最精准的票"；鼓励展示修正后结论（不需要逐次交换细节）
- **未宣告**：以普通好人身份竞选，可暗示"我对场上信息有些独到角度"（不说来源），绝不泄露任何交换信息

### 技术实现

**SHERIFF_SPEECH 改动（aiPrompts.js）：**

在 `badgeFlowLine` 之后、`ssHint` 之前，新增三个变量：
```javascript
const hasRevealedIdentity = currentPlayer?.hasRevealed || false;
const dreamweaverSsHint = hasRevealedIdentity ? `revealed text` : `unrevealed text`;
const magicianSsHint = hasRevealedIdentity ? `revealed text` : `unrevealed text`;
```

在 `ssHint` 三元链中，`猎人` 分支之后、`好人 fallback` 之前插入：
```javascript
: playerRole === '摄梦人'
? dreamweaverSsHint
: playerRole === '魔术师'
? magicianSsHint
: `你是好人（${playerRole}）：...`
```

**SHERIFF_RUN 改动（aiPrompts.js）：**

在 `守卫` 分支之后、`好人 fallback` 之前插入两个 `else if` 块（各含 `const revealed` + ternary）

### 编辑过程踩坑

**Python `\$` 字符问题**：在 Python 字符串中写 `\${badgeFlowLine}` 时，`\$` 在 Python 中是两个字符（反斜杠 + 美元符号）——不是 JS 意义上的转义，而是字面量 `\$`。写入 JS 文件后，JS 模板字符串收到 `\${badgeFlowLine}` 会把 `$` 转义，导致输出字面量字符串 `${badgeFlowLine}` 而非变量值。修复：`content.replace('\\${badgeFlowLine}', '${badgeFlowLine}')` 清除4处错误转义，并在 T15 中加入负向断言防止回归。
