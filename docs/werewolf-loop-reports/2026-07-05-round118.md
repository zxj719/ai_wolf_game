## 改动细节

### 骑士 post-duel 验证锚（aiPrompts.js DAY_VOTE）

**原文（单行通用）**：
```
决斗已使用，你是场上公信力锚点：优先投预言家查杀目标；无查杀时投你本轮综合分析最高嫌疑的玩家
```

**新文（R118 多步结构）**：
```
a) 🗡️决斗验证锚点（R118）：从【身份推理表】找到 reason 含"已决斗出局"的玩家 → 该狼人确认；
   查其生前"保护/力挺/金水"过的存活玩家，confidence 下调 20-30（狼人同组互保信号）→ 列为本轮优先投票目标
b) 投票优先排序：① 预言家查杀（外部验证） → ② 决斗狼人生前力挺的存活玩家（连带嫌疑） → ③ 本轮发言逻辑最崩塌者
c) 票权（若为警长 1.5 票）应锁定有验证依据的目标，服务于信息优势而非直觉猜测
```

**设计依据**：`hasUsedDuel=true` 在 DAY_VOTE 阶段必然代表成功决斗（决斗失败→骑士死亡→无法进入 DAY_VOTE），因此 `knightHasDueledForVote=true` 时 identity_table 中必有"已决斗出局"的确认狼人。

### 魔术师双路径（R118）

新增 3 个变量：
```javascript
const dvMagIsRevealed = currentPlayer?.hasRevealed ?? false;
const dvMagHistory = gameState.magicianHistory || { swappedPlayers: [], lastSwap: null };
const dvMagSwappedCount = (dvMagHistory.swappedPlayers || []).length;
```

新增 `dvMagRevealedVoteStrategy`（公开路径）：
- 交换知识锚：根据历史交换记录，订正 identity_table 中的好人/狼人置信度至 90+
- 投票排序：① 交换验证确认狼人 → ② 经交换修正的预言家查杀 → ③ 发言逻辑崩塌者
- 领袖义务：率先带票引导好人阵营对齐正确信息流

返回链修改（隐藏 vs 公开路由）：
```javascript
// Before:
: playerRole === '魔术师'
? magicianVoteStrategy

// After:
: playerRole === '魔术师'
? (dvMagIsRevealed ? dvMagRevealedVoteStrategy : magicianVoteStrategy)
```

## 本轮教训（→ LEARNINGS.md）

1. **hasUsedDuel 在 DAY_VOTE 的语义**：后决斗路径（`knightHasDueledForVote=true`）必然代表成功决斗，因为决斗失败会导致骑士死亡且无法投票。故 post-duel 路径可无条件读取"已决斗出局"锚点，无需 fallback。
2. **内嵌三元保护原有测试**：为避免破坏 R72 的 `getMagicianStrategyContent()` 提取器（依赖 `const magicianVoteStrategy = \`` 关键字），将隐藏路径保持为独立变量，新增公开路径变量，通过内嵌三元路由。这是"最小干扰"测试兼容原则的实例。
3. **窗口参数版本控制**：每次在 return 链中插入额外三元时，后续变量名在窗口内的偏移会增加约 5-50 chars。已修复 T13（80→120），未来如再插入三元需再次检查窗口。
