## 2. 本轮优化详情

### 优化背景

Round 16 完成了狼人 identity_table 战略化（高优先刀口 + 队友公开叙事一致性）。好人方（预言家/女巫/猎人/守卫/村民）的 identity_table 仍使用泛化 `"reason":"依据"`。

LEARNINGS.md item 21 指出应逐角色评估"该角色有什么独有信息"。本轮聚焦收益最高的两个角色：

1. **预言家**：有**确定性知识**（查验结果是二元事实，不是推断）
2. **守卫**：有**守护历史状态**（知道自己守了谁，可据此调整夜间策略）

---

### 优化 1：NIGHT_SEER identity_table 夜间查验指导

**位置**：`src/services/aiPrompts.js`，`case PROMPT_ACTIONS.NIGHT_SEER:` 输出行之前

**新增内容**：
```
【identity_table 填写指导（夜间查验：记录确认知识与候选优先级）】
- 已查验玩家：confidence 填 95-100，reason 写"N[X]夜查验确认：[狼人/好人角色名]（已公开/待明日报）"——标"待报"提醒自己次日发言必须优先公开
- 今晚查验目标：confidence 填 50-70，reason 写"本轮查验候选：[策略原因，如'发言可疑'或'位置策略']"
- 高度可疑但未查者：confidence 填 60-80，reason 写"行为可疑：[具体原因]，排队查验"
```

**博弈论依据**：Wang 2025 研究表明预言家的信息公开时机直接影响好人胜率。"待明日报"标记让 AI 在 N1 晚查完后，将查验信息持久化写入 identity_table，次日 LLM 调用时能立即识别"此条需要在今天发言中公开"，防止跨轮遗忘。

---

### 优化 2：DAY_SPEECH 预言家 identity_table 差异化指导

**位置**：`ROLE_DAY_SPEECH_PROMPTS['预言家']` 输出 JSON 行之前

**新增内容**：
```
【identity_table 填写指导（预言家有确定性知识，应差异化填写）】
- 已查验玩家：confidence 填 95-100，reason 写"N[X]夜查验确认：[狼人/好人角色名]"（例："N1夜查验确认：村民"、"N2夜查验确认：狼人"）
- 正在质疑你（悍跳嫌疑）的未验证玩家：reason 写"悍跳攻预嫌疑—优先查验候选，当前不能被其带节奏"
- 未查验普通玩家：reason 写"发言分析：[可信/可疑点]，下次查验候选"
```

**效果**：当 `previousIdentityTable` 在次轮注入时，AI 能立即区分"N1夜查验确认：村民（confidence 98）"和"发言分析：略显主动（confidence 55）"——前者是已知事实，后者是推断，不会混淆。

---

### 优化 3：DAY_SPEECH 守卫 identity_table 守护历史指导

**位置**：`ROLE_DAY_SPEECH_PROMPTS['守卫']` 输出 JSON 行之前

**新增内容**：
```
【identity_table 填写指导（记录守护判断以提升夜间决策连贯性）】
- 守护过的玩家：reason 写"N[X]夜守护，结果：[平安夜/存活/死亡]；今晚换守判断：[是否换、为什么]"
- 疑似关键神职（值得守护）：reason 写"白天行为疑似[预言家/女巫]，守护优先级高"
- 高度可疑狼人：reason 写"发言[带节奏/攻真预/逻辑矛盾]，投票怀疑对象"
```

**效果**：守卫的 NIGHT_GUARD 提示词会注入 `previousIdentityTable`，此时守卫能看到"N2夜守护，结果：存活；今晚换守判断：昨晚狼刀别处，改守预言家"，从而做出有历史依据的守护决策，而非每轮从零开始分析。
