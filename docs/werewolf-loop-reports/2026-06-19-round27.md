## 3. 本轮优化详情

### 优化：DAY_VOTE 预言家对跳投票优先级（item 4 from R26 recommendations）

**问题根因**：R25 NIGHT_SEER 建立了"夜间悍跳响应最高优先"，R26 DAY_SPEECH 建立了"白天三步戳穿法"，但 DAY_VOTE 的预言家策略仍是静态的"率先投查杀目标"——对跳局面中悍跳者就在投票目标里，预言家却没有收到"投悍跳者等同投查杀"的明确引导。

**博弈论依据**：
- 悍跳行为本身就是公开谎言——预言家已知对方身份为假，其欺骗性等同于已验查杀
- PK 场景尤其关键：若 PK 是"真预 vs 悍跳"，弃票等于放过已知狼人

**改动位置（2 处）**：

| 文件 | 改动 | 说明 |
|------|------|------|
| `aiPrompts.js` DAY_VOTE case | 新增 `seerCounterClaimantsInVote` 检测 | 仅预言家，过滤 claimHistory jump_seer 在 voteTargets 中的玩家 |
| `aiPrompts.js` DAY_VOTE case | 新增 `seerVoteStrategy` 变量 + 更新模板引用 | 三路分支：对跳+PK / 对跳+普通投票 / 无对跳回退 |

**关键代码**：
```javascript
// 预言家对跳场景：识别场上悍跳者且在本轮投票目标中（PK时 voteTargets 只有2人）
const seerCounterClaimantsInVote = playerRole === '预言家'
    ? (gameState.claimHistory || [])
        .filter(c => c.type === 'jump_seer' && c.playerId !== currentPlayer?.id
               && voteTargets.includes(c.playerId))
        .map(c => c.playerId)
    : [];

// 预言家投票策略：对跳局面时悍跳者优先于其他目标（悍跳 = 公开谎言 = 已知狼人）
const seerVoteStrategy = seerCounterClaimantsInVote.length > 0
    ? `2. 【预言家投票策略—对跳优先】${seerCounterClaimantsInVote.join(',')}号是在场悍跳者...`
    : `2. 【预言家投票策略】率先投查杀目标！...`;
```

**完整策略闭环（R25+R26+R27）**：

| 阶段 | 策略 | 实现位置 |
|------|------|---------|
| **夜间**（NIGHT_SEER） | 五级优先级，悍跳响应最高：优先验证对方报告 | NIGHT_SEER case |
| **白天发言**（DAY_SPEECH） | 三步戳穿框架 Step A-B-C | ROLE_DAY_SPEECH_PROMPTS['预言家'] |
| **投票**（DAY_VOTE）🆕 | 对跳者优先于其他目标；PK 必须投对跳者 | DAY_VOTE case |
